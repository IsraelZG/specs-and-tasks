> Subtask de T-313 (decomposta 2026-06-29, arquiteto opção B). Implementa a máquina de
> pack/unpack/verify de Archive Cargo. T-313b (assignCustodian) e T-313c (GC+migration)

## 1. Objetivo

Implementar a serialização self-certifying de Archive Cargo no pacote `@plataforma/core`:
empacotar lote de nós + arestas via `zstd + AES-256-GCM`, derivar `archiveId =
blake2s256(ciphertext)`, e expor roundtrip íntegro + verificação. Archive é **cego**:
custodiante jamais vê `pub_key`/autoria — esta task foca só o envelope crypto + pack/unpack
verificado. A atribuição a custodiante é T-313b; a storage/GC é T-313c.

*(derivado de `tasks/T-313.md` §1; `docs/plano-de-implementacao.md` §6 T-313 — "RFC-005 §A.11";
`tasks/T-101.md` — wrappers AES/blake2s256; `tasks/T-102.md` — EntityId/ULID)*

### Contratos exatos (assinaturas TS fixadas)

```ts
// --- packages/core/src/archive/archiveCargo.ts 
---
import type { AesKey, Sha256Digest } from '@plataforma/crypto';   // T-101
import { EntityId, ULID } from '@plataforma/protocol';            // T-102
import type { Fingerprint } from '@plataforma/protocol';           // T-301 (B-Tree de fingerprints)

/** Identificador cego do escopo do archive (SHA-256 do entity_id real). */
export type BlindScopeId = Uint8Array;

/** Digest self-certifying do archive: blake2s256(ciphertext). */
export type ArchiveId = Uint8Array;

/** Payload opaco do archive (lote de nós+arestas cifrados). */
export interface ArchiveCargoPayload {
  /** Scope cego (SHA-256 do entity_id real) — T-313b usa pra consistent hashing. */
  blindScopeId: BlindScopeId;
  /** Época lógica (key derivation). */
  epochIndex: number;
  /** Range HLC coberto por este archive. */
  hlcRange: { start: number; end: number };
  /** Tamanho do ciphertext em bytes. */
  size: number;
  /** Timestamp Unix ms de expiração. */
  expiresAt: number;
}

/** Manifesto público do archive (sem revelar pub_key/autoria). */
export interface ArchiveManifest {
  archiveId: ArchiveId;            // blake2s256(ciphertext)
  blindScopeId: BlindScopeId;
  epochIndex: number;
  hlcRange: { start: number; end: number };
  size: number;
  expiresAt: number;
}

/** Archive Cargo: ciphertext + manifesto. */
export interface ArchiveCargo {
  ciphertext: Uint8Array;         // zstd(AES-256-GCM(plaintext))
  manifest: ArchiveManifest;
}

/** Configuração de empacotamento — chave de época. */
export interface ArchiveConfig {
  /** Chave AES-256 da época do escopo. */
  readonly epochKey: AesKey;
  /** Nível zstd (default 3, range -7..22). */
  readonly compressionLevel?: number;
}

/**
 * Empacota lote de nós + arestas em Archive Cargo.
 * - Concatena `nodes` e `edges` bytewise, aplica zstd, depois AES-256-GCM.
 * - `archiveId = blake2s256(ciphertext)` (self-certifying).
 * - Manifest é derivado do payload + do archiveId computado.
 * - Tag GCM é embutida no fim do ciphertext (tamanho 16 bytes).
 */
export function packArchive(
  nodes: Uint8Array[],
  edges: Uint8Array[],
  epochIndex: number,
  entityId: EntityId,
  config: ArchiveConfig,
): Promise<ArchiveCargo>;

/**
 * Desempacota Archive Cargo: decifra AES-256-GCM, descomprime zstd, retorna
 * arrays originais. Lança `Error` se a tag GCM não bater (arquivo adulterado)
 * ou se o `archiveId` (manifesto) não bater com `blake2s256(ciphertext)`.
 */
export function unpackArchive(
  cargo: ArchiveCargo,
  config: ArchiveConfig,
): Promise<{ nodes: Uint8Array[]; edges: Uint8Array[] }>;

/**
 * Verifica integridade self-certifying sem decifrar:
 * - `blake2s256(ciphertext) === manifest.archiveId`
 * - tag GCM do ciphertext é válida (requer `config.epochKey`)
 * Retorna `true` se íntegro, `false` caso contrário. NUNCA lança.
 */
export function verifyArchiveIntegrity(
  cargo: ArchiveCargo,
  config: ArchiveConfig,
): Promise<boolean>;
```

Casos de teste (numerados — 1:1 em `archiveCargo.test.ts`):
1. **roundtrip pack/unpack íntegro:** `packArchive(nodes, edges, 0, entityId, { epochKey })` → `unpackArchive(cargo, { epochKey })` devolve `nodes` e `edges` bytewise idênticos.
2. **verifyArchiveIntegrity com archive íntegro → true:** após `packArchive`, `verifyArchiveIntegrity(cargo, config)` retorna `true`.
3. **verifyArchiveIntegrity com ciphertext adulterado (1 byte) → false:** flip de 1 byte em `cargo.ciphertext` (sem mexer no manifesto) → `false` (tag GCM não confere) **e** `archiveId` (blake2s256) não confere.
4. **verifyArchiveIntegrity com archiveId errado (manifest mentiroso) → false:** alterar `manifest.archiveId` sem mexer no ciphertext → `false` (blake2s256 não confere).
5. **packArchive com nós e arestas vazias:** inputs `[]`/`[]` → archive válido (zstd de buffer vazio é OK; ciphertext não é zero-bytes devido ao GCM nonce+tag).

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** decifre antes de verificar — a ordem é: `archiveId` (blake2s256) → GCM tag → retornar. Decifrar primeiro anula a fronteira de confiança.
> - **NÃO** exponha `pub_key` ou autoria no manifesto — o archive é cego; só `blindScopeId` (hash do entity_id), `archiveId`, `epochIndex`, `hlcRange`, `size`, `expiresAt`.
> - **NÃO** implemente consistent hashing aqui (T-313b) nem GC (T-313c).
> - **NÃO** use `setTimeout`/`setInterval` nativos — não há timers aqui; só crypto síncrono + zstd sync/async (wrapper Node).

### Pegadinhas conhecidas
- **zstd no Node 20+:** `node:zlib` (≥ v22) tem `zstdCompressSync`/`zstdDecompressSync`. Em v20, use `@bokuweb/zstd-wasm` ou `fzstd` (puro JS). Verificar o que T-101/T-301 já usam para não introduzir dep nova — se nada, usar `node:zlib` se disponível e cair de volta para `fzstd` puro JS.
- **AES-256-GCM nonce:** 12 bytes aleatórios (`crypto.getRandomValues`). Concatenar `nonce ‖ ciphertext ‖ tag` (formato T-101) ou armazenar nonce separado. Seguir o que T-101 padronizou.
- **`blake2s256` (32 bytes):** diferente de `sha256` (também 32 bytes). O T-101 expõe ambos. `archiveId = blake2s256(ciphertext)` é o requisito do spec; **NÃO** trocar por sha256.
- **AAD no GCM:** usar `manifest` (sem `archiveId`, pra evitar circular) como AAD — bind manifest ao ciphertext. Documentar escolha.
