# T-1045 · Formalizar camada de projecao sobre o query-model

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo (`pnpm build`, `pnpm test`, `pnpm lint` na raiz afetam todos os pacotes)
- **Test Runner:** `vitest` (pacotes core/protocol)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Formalizar a interface e o mecanismo de atualização da camada de projeção e índices locais (por exemplo, views e tabelas de heads de entidade) que residem na camada de leitura/índice (associada com TinyBase), desvinculando-as inteiramente do armazenamento central de grafos.

## 2. Contexto RAG (Spec-Driven Development)
- [[adr-005-storage-engine-agnosticism]]
- `packages/core/src/schema.ts`

## 3. Escopo de Arquivos (Inputs e Outputs)
- **[UPDATE]** `packages/core/src/schema.ts` (Ajustar definições de tabelas locais/projeções para ficarem desacopladas do DDL de nodes/edges)
- **[CREATE]** `packages/core/src/projection.ts` (Formalizar o indexador local que consome eventos do grafo e escreve nos índices locais/TinyBase)

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- **Framework:** Vitest (Node puro)
- **Ambiente do Teste:** Node puro
- **Metodologia:** Criar testes em `packages/core/tests/projection.test.ts` para verificar que as projeções (como `entity_heads`) são corretamente geradas a partir de um log de escrita do grafo sem depender de queries SQL cruas vindas de fora da camada do indexador.

## 5. Instruções de Execução (Step-by-Step)
1. Estabeleça em `packages/core/src/projection.ts` a interface `ProjectionManager` encarregada de assinar updates do grafo e re-materializar as projeções locais.
2. Certifique-se de que, ao usar o SQLite, as projeções sejam geradas no mesmo banco via triggers ou indexação local transparente, enquanto em KV elas rodem puramente na camada TinyBase ou em memória.
3. Garanta que o core (especialmente o `Index Worker` e a sincronização) utilize essa camada formalizada para atualizar as visualizações locais consumidas pela interface reativa do TinyBase.
