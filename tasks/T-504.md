# T-504 · Revogação + revogação por cortesia

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)

## 1. Objetivo
Implementar revogação e revogação por cortesia no pacote `@plataforma/core`: lápide de revogação de UCAN; sinal de expurgo (`retention_state=expunged`) propagado a peers cooperativos; teste do limite (peer não-cooperativo retém)
*(derivado de `docs/plano-de-implementacao.md` §8 T-504 e `docs/caderno-2-protocol/02-cryptographic-lineage-and-auth.md` §5.2)*.

### Contratos exatos
```ts
// --- packages/core/src/auth/revocation.ts ---
import { EntityId, ULID } from '@plataforma/protocol'; // T-102
import { StoragePort } from '@plataforma/protocol'; // T-004
import { SignedUcan } from './ucan-codec.js'; // T-501a
import { SwarmRegistry } from '@plataforma/transport'; // T-205 (packages/transport/src/SwarmRegistry.ts)

export enum RetentionState {
  ACTIVE = 0,
  TOMBSTONED = 1,      // lápide soft-delete
  REPLICATED = 2,       // replicado para peers
  EXPUNGED = 3,         // sinal de expurgo propagado
}

export interface RevocationRecord {
  ucanId: string;                // ID do UCAN revogado
  revokedAt: number;             // timestamp
  revokedBy: EntityId;           // quem revogou
  reason: string;
  retentionState: RetentionState;
  courtesy: boolean;             // true se revogação por cortesia (peer cooperativo)
}

export interface RevocationResult {
  success: boolean;
  existingAccessRevoked: boolean;  // UCAN deixa de servir dados imediatamente
  expungePropagated: boolean;      // sinal enviado a peers cooperativos
}

/** Revoga UCAN. */
export function revokeUcan(
  storage: StoragePort,
  ucan: SignedUcan,
  revokerEntityId: EntityId,
  reason: string,
): Promise<RevocationResult>;

/** Verifica se UCAN está revogado. */
export function isRevoked(storage: StoragePort, ucanId: string): Promise<boolean>;

/** Propaga sinal de expurgo para peers no SwarmRegistry. */
export function propagateExpunge(
  storage: StoragePort,
  ucanId: string,
  swarm: SwarmRegistry,
): Promise<number>; // número de peers que receberam o sinal

/** Revogação por cortesia: peer cooperativo executa revogação local. */
export function courtesyRevoke(
  storage: StoragePort,
  revocationRecord: RevocationRecord,
): Promise<void>;

/** Documenta limite: peer não-cooperativo pode reter dados. */
export function assertNonCooperativeLimit(): string;
```
- `revokeUcan`: cria lápide do UCAN no storage; `retention_state = EXPUNGED`; UCAN deixa de servir dados na próxima requisição.
- `propagateExpunge`: envia sinal de expurgo a peers cooperativos no swarm (ver decisão aberta sobre transporte).
- `courtesyRevoke`: peer cooperativo recebe sinal e aplica revogação local.
- Limite documentado: peer não-cooperativo pode reter dados — `assertNonCooperativeLimit()` retorna string explicativa.

## 6. Feedback de Especificação

> **⚠️ DECISÃO EM ABERTO: Mecanismo de transporte/envio no `propagateExpunge`**
> A função `propagateExpunge` recebe apenas `StoragePort`, `ucanId` e `SwarmRegistry`, mas nenhum adaptador de rede (`NetworkAdapterPort`) ou objeto de conexão. Não está claro como a propagação do sinal de expurgo deve ocorrer em produção.
>
> **Opções recomendadas para o Arquiteto:**
> - **Opção (a):** Alterar a assinatura para receber também um adaptador `NetworkAdapterPort` ou um client/sync manager para realizar o envio das mensagens de transporte.
> - **Opção (b):** Fazer com que `propagateExpunge` apenas enfileire a mensagem de expurgo no `StoragePort` (como um registro local a ser replicado pelo Sync Worker/Gossip).
> - **Opção (c):** Para fins do teste de unidade, simular/mockar o canal de transporte associado a cada peer do `SwarmRegistry`.
