## 1. Objetivo
Implementar `encodeFrame` e tipos `FrameType`/`WireFrame` no pacote `@plataforma/protocol`: serialização MessagePack + Length-Prefixed Framing (LENGTH/VERSION/FRAME_TYPE/PAYLOAD)
*(extraído de T-201 §1; derivado de `docs/caderno-2-protocol/05-wire-protocol.md` §1)*.

### Contratos exatos
```ts
// --- packages/protocol/src/wireFormat.ts ---
export const WIRE_VERSION = 0x01;
export const MAX_FRAME_LENGTH = 1_048_576; // 1 MiB

export enum FrameType {
  RBSR_ROOT=0x01, RBSR_RANGE=0x02, RBSR_REQUEST_NODES=0x03, RBSR_NODES=0x04,
  EPHEMERAL=0x05, HEARTBEAT_PING=0x06, HEARTBEAT_PONG=0x07, IDENTITY_CATCHUP=0x08,
  KEY_REQUEST=0x09, KEY_RESPONSE=0x0A, CONTROL=0x0B,
}

export interface WireFrame {
  version: number;
  frameType: FrameType;
  payload: unknown;
}

/** Codifica payload MessagePack em frame binário.
 *  [LENGTH: uint32 BE] [VERSION: uint8] [FRAME_TYPE: uint8] [PAYLOAD: MessagePack] */
export function encodeFrame(frameType: FrameType, payload: unknown): Uint8Array;
```

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).

Casos de teste:
1. `encodeFrame(RBSR_ROOT, {key:'val'})` → frame binário começa com 4 bytes de LENGTH.
2. LENGTH no header = tamanho do resto (VERSION+TYPE+PAYLOAD).
3. FrameType `RBSR_NODES` (0x04) aparece como byte 5.
4. Payload MessagePack decodificado corretamente (roundtrip com msgpackr.decode).
5. Payload `null` → encode sem erro.
6. Payload com Uint8Array aninhado → preservado.
