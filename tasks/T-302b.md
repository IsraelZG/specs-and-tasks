# T-302b · RBSR exchange: applyNodes (validação Layer 1 + inserção)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** haiku

## 1. Objetivo
Implementar `applyNodes` no pacote `@plataforma/protocol`: recebe nós binários do peer remoto, valida assinatura Layer 1 (T-107) e insere no storage via `insertNode` (T-108a). Rejeita nós com assinatura inválida sem contaminar o grafo local
*(extraído de T-302 §1)*.

### Contratos exatos
```ts

  nodes: Uint8Array[],
  storage: StoragePort,
): Promise<{ imported: number; rejected: number }>;
```

## 4. Estratégia de Testes
- [x] **Framework:** Vitest + SQLite `:memory:`.

Casos de teste:
1. 3 nós válidos → `imported: 3, rejected: 0`.
2. 1 nó com assinatura adulterada → `imported: 2, rejected: 1`.
3. Array vazio → `imported: 0, rejected: 0`.
4. Nó duplicado (mesmo id) → ignorado, não duplica.
