## 1. Objetivo
Implementar `EpochRotation` no pacote `@plataforma/core`: rotação de chave AES de época com forward secrecy — novos nós cifram com chave nova, ex-membros não conseguem decifrar conteúdo pós-rotação, mas mantêm acesso ao histórico antigo
*(derivado de `docs/visao-arquitetural.md` §7 — "chaves de época efêmeras, TTL 4h"; T-501 — UCAN governa quem recebe chave; T-110 — KeyVault armazena chaves em RAM)*.

### Contratos exatos
```ts
// --- packages/core/src/vault/EpochRotation.ts ---
import { KeyVault } from '../keyVault.js'; // T-110
import { AesKey } from '@plataforma/crypto'; // T-101
import { SignedUcan } from '../auth/ucan.js'; // T-501

export interface EpochConfig {
  epochIndex: number;
  key: AesKey;
  issuedAt: number;
  expiresAt: number;    // issuedAt + TTL (4h)
  issuerUcan: SignedUcan; // UCAN que autorizou esta época
}

export class EpochRotation {
  constructor(vault: KeyVault, ttlMs?: number);

  /** Gera nova época (nova chave AES-256) e registra no vault. */
  rotate(authorization: SignedUcan): Promise<EpochConfig>;

  /** Retorna chave da época vigente. */
  getCurrentEpoch(): EpochConfig | null;

  /** Retorna chave de época específica (se ainda no vault). */
  getEpoch(index: number): EpochConfig | null;

  /** Lista épocas ativas no vault. */
  listEpochs(): EpochConfig[];

  /** Força expiração de época (revogação de acesso). */
  revokeEpoch(index: number): void;

  /** Época corrente. */
  get currentIndex(): number;
}
```
- `rotate`: gera `AesKey` nova, registra no vault com TTL, incrementa `epochIndex`.
- Forward secrecy: nós novos usam `currentIndex`; ex-membros sem a chave da época atual não decifram.
- Imutabilidade: histórico NÃO é re-cifrado — membros antigos mantêm acesso ao que já tinham.
- Integração: UCAN (T-501) autoriza a rotação; KeyVault (T-110) armazena chaves em RAM.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/core test`. Usar VirtualClock (T-003) para TTL.

Casos de teste:
1. `rotate` → `currentIndex` incrementa, chave armazenada no vault.
2. `rotate` 2× → `listEpochs` retorna 2 épocas.
3. Época expirada (VirtualClock avança TTL) → `getCurrentEpoch` retorna null.
4. `getEpoch(0)` retorna chave da primeira época.
5. `revokeEpoch` → chave removida do vault, `getEpoch` retorna null.
6. UCAN inválido → `rotate` rejeitado.
