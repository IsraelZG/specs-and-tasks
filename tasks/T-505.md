# T-505 · Rotação de Épocas (Forward Secrecy)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (pacotes core/protocol) e `playwright` (E2E/Frontend)

## 1. Objetivo
Rotação de chave AES do escopo com consolidação offline. Novos nós cifram com a chave nova, garantindo corte de acesso a ex-membros.

## 2. Contexto RAG (Spec-Driven Development)
- [rfc-005](../docs/rfcs/rfc-005.md) (A.12 Key Vault)

## 3. Escopo de Arquivos (Inputs e Outputs)
- **[UPDATE]** `packages/core/src/vault/KeyVault.ts`
- **[CREATE]** `packages/core/src/vault/EpochRotation.ts`

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [ ] **Framework:** Vitest
- [ ] **Métricas:** Membro revogado consegue ler nós antigos em cache, mas falha em decifrar nós novos gerados na época atual.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** crie uma chave AES por dispositivo. A chave de época é compartilhada no escopo.
> - **NÃO** reescreva (re-cifre) o histórico. A criptografia é imutável.

1. **[TDD]** Crie os testes garantindo que falhem.
2. Implemente a funcionalidade no pacote indicado.
3. Garanta que o linter aprove o código.
