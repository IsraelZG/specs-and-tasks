# T-018d · Testes Tier C batch 1 — 8 componentes display/estrutural

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo

## 1. Objetivo
Escrever testes unitários para **8 componentes display/estruturais** do `@plataforma/design-system`, cobrindo os critérios do Tier C (smoke, a11y role/atributo, ref se aplicável).
*(Derivado de T-018 — subdivisão para execução por modelos haiku)*.

1. `Alert`
2. `AspectRatio`
3. `Avatar`
4. `Badge`
5. `Card`
6. `Label`
7. `Message`
8. `NavGroup`

### Critérios de aceite (Tier C — display/estrutural)
1. **Smoke:** renderiza sem erro com props mínimas obrigatórias.
2. **A11y:** role/atributo esperado presente (ex.: `Avatar` com `alt` ou `aria-label`, `Alert` com `role="alert"`, `Badge` com `aria-label` — verificar metadata de cada um).
3. **Ref forwarding:** somente se `React.forwardRef` — maioria deste tier é função simples sem ref.
4. **Não é necessário** teste de interação — são componentes sem estado interativo próprio.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - NÃO invente testes de interação — Tier C não tem estado interativo próprio.
> - NÃO corrija bugs de produto — documente na Seção 6.
> - NÃO copie o mesmo `it()` para todos sem ler o componente real.

### Pegadinhas conhecidas
- **Alert:** verificar `role="alert"` + testar variantes (`success`, `warning`, `danger`, `info`).
- **AspectRatio:** wrapper simples — verificar se children renderiza e se o atributo de aspect ratio (via `style` ou wrapper) está presente.
- **Avatar:** testar fallback (iniciais) quando `src` não provido e acessibilidade (`alt`).
- **Badge:** testar variantes de intent (neutral, success, warning, danger, info).
- **Card:** testar modos (estático vs `interactive`), ref se `as="button"` ou `as="a"`.
- **Message:** testar variantes (sent/received) e densidade se houver.
- **Label/NavGroup:** renderização simples + `htmlFor` (Label) / children (NavGroup).

1. Para cada componente: leia `<Nome>.tsx` + `<Nome>.metadata.ts`.
2. Escreva `tests/components/<Nome>.test.tsx` cobrindo os critérios do Tier C.
3. Rode `pnpm --filter @plataforma/design-system test` após cada 2-3 componentes.
4. Ao final dos 8, rode build + lint (Seção 7).
