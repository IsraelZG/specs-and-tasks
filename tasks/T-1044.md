# T-1044 · Segundo adapter KV de prova para GraphStorePort

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo (`pnpm build`, `pnpm test`, `pnpm lint` na raiz afetam todos os pacotes)
- **Test Runner:** `vitest` (pacotes core/protocol)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar um segundo adapter de armazenamento baseado em chave-valor ordenado (ex: LMDB, RocksDB ou um mock em memória ordenado por árvore) implementando a interface `GraphStorePort`, para comprovar na prática o agnosticismo de engine de armazenamento da plataforma.

## 2. Contexto RAG (Spec-Driven Development)

- `packages/protocol/src/ports.ts`

## 3. Escopo de Arquivos (Inputs e Outputs)
- **[CREATE]** `packages/core/src/kvStorage.ts` (Implementação do adapter KV para `GraphStorePort`)
- **[UPDATE]** `packages/core/package.json` (Adicionar dependências necessárias para a engine KV de prova, se houver)

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- **Framework:** Vitest (Node puro)
- **Ambiente do Teste:** Node puro
- **Metodologia:** Criar um conjunto de testes unitários em `packages/core/tests/kvStorage.test.ts` rodando a mesma suíte de conformidade de persistência do grafo usada para o SQLite, garantindo paridade de comportamento.
- **Remediação T-1045 (adicionada 2026-07-14, arquiteto claude-fable):** T-1045 (`ProjectionManager`
  em `packages/core/src/projection.ts`) foi concluída e mergeada ANTES desta task — a alegação de
  agnosticismo de engine da camada de projeção está **provada só contra SQLite**. Esta task fecha o
  gap: rodar também `packages/core/tests/projection.test.ts` (ou suíte equivalente de conformidade)
  contra `KvGraphStorage`, provando que as projeções (`entity_heads` etc.) materializam idêntico nos
  dois motores. Sem esse caso, o "segundo adapter de prova" não prova o que T-1045 deixou em aberto.

## 5. Instruções de Execução (Step-by-Step)
1. Crie o arquivo `packages/core/src/kvStorage.ts` contendo a classe `KvGraphStorage implements GraphStorePort`.
2. Implemente os métodos tipados de escrita e leitura de nós/arestas.
3. Para `rangeScan`, certifique-se de que a leitura de chaves/ULIDs ordenadas use as primitivas de scan ordenadas nativas do motor de KV (garantindo eficiência O(log N) ou similar).
4. Assegure a conformidade das transações do grafo.
