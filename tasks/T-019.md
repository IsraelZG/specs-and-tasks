# T-019 · Matriz de CI multiplataforma (linux x64 + linux arm64)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` + `playwright`
- **Capacidade-alvo:** sonnet *(edição de YAML de matriz é simples, mas exige entender corretamente o que cada runner hospedado oferece)*

## 1. Objetivo
O CI atual (`.github/workflows/ci.yml`) roda um único job em `runs-on: ubuntu-latest` (linux x64). Esta task adiciona **um segundo job rodando em `ubuntu-24.04-arm`** (runner GitHub-hosted, linux arm64), garantindo que dependências nativas (ex.: `better-sqlite3` usado pelo `StoragePort`/schema SQLite da T-106) compilem e passem em arm64.

**Não adiciona Windows ao CI** — não existe runner Windows-ARM64 hospedado no GitHub Actions; o ambiente Windows ARM64 do dev local é uma característica da máquina, não alvo de deploy.

### Layout YAML final (referência — derivado do ci.yml atual + matriz)

```yaml
jobs:
  ci:
    name: Build, Lint and Test
    runs-on: ${{ matrix.os }}
    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, ubuntu-24.04-arm]
    steps:
      # ... steps existentes (checkout, pnpm setup, node setup, cache turbo, install, lint, build, test) ...
      # Mudança na chave de cache:
      - name: Cache Turborepo
        uses: actions/cache@v4
        with:
          path: |
            .turbo
            packages/*/.turbo
            apps/*/.turbo
          key: ${{ runner.os }}-${{ runner.arch }}-turbo-${{ github.sha }}
          restore-keys: |
            ${{ runner.os }}-${{ runner.arch }}-turbo-
```

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - Não remover o job x64.

> - Não trocar trigger/branches monitorados.
> - Não usar QEMU para emular arm64 em x64 — testaria falso positivo.

### Pegadinhas conhecidas
- `runner.os` retorna `Linux` tanto para x64 quanto para arm64 → sem `runner.arch` na chave, os dois jobs competem pelo mesmo cache. **Use `${{ runner.os }}-${{ runner.arch }}-turbo-...`**.
- `better-sqlite3` e outras deps nativas podem precisar de rebuild via `node-gyp` em arm64. Se `pnpm install` falhar nesse step, é exatamente o tipo de regressão que esta task existe para capturar — documente na Seção 6, **não** adicione `--ignore-scripts` para "fazer passar".

1. Edite `.github/workflows/ci.yml`: troque `runs-on: ubuntu-latest` fixo por `strategy.matrix` com `os: [ubuntu-latest, ubuntu-24.04-arm]` e `runs-on: ${{ matrix.os }}`.
2. Ajuste a chave de cache para incluir `runner.arch`.
3. Abra PR; confirme que ambos os jobs da matriz completam com sucesso.
4. Cole o link do run + saída dos jobs na Seção 8.
