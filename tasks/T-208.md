## 1. Objetivo
Implementar máquina de estados FPP (First Peer Protocol) no pacote `@plataforma/core`: cold start → descoberta de swarm → conexão ou gênese da rede
*(derivado de `docs/conceitos/swarm-registry.md` — FPP gerencia fases `JOINING → WAITING_FOR_SWARM → CONNECTED | GENESIS | OFFLINE_RETRY`; `docs/plano-de-implementacao.md` §4 T-208)*.

### Contratos exatos
```ts
// --- packages/core/src/fpp.ts ---
import { SwarmRegistry } from './swarmRegistry.js'; // T-205

export enum FppState {
  JOINING            = 'joining',
  WAITING_FOR_SWARM  = 'waiting_for_swarm',
  CONNECTED          = 'connected',
  GENESIS            = 'genesis',
  OFFLINE_RETRY      = 'offline_retry',
}

export interface GenesisBlock {
  networkId: string;
  createdAt: number;
  founderPubKey: Uint8Array;
}

export interface FppOptions {
  swarm: SwarmRegistry;
  genesisTimerMs?: number; // default 8000 (8s timeout para swarm)
  isFounder?: boolean;     // detentor da chave de fundação
}

export class FirstPeerProtocol {
  constructor(opts: FppOptions);
  get state(): FppState;

  /** Inicia FPP. Se `isFounder` e nenhum peer em `swarm.listConnected()` após timer → GENESIS. */
  start(): Promise<void>;

  /** Callback quando estado muda. */
  onStateChange(cb: (state: FppState) => void): void;

  /** Em GENESIS, cria o bloco SPECIFICATION:NETWORK_BIRTH. */
  createGenesisBlock(): GenesisBlock;
}
```
- Transições: `JOINING` → `WAITING_FOR_SWARM` → se swarm vazio após 8s + founder → `GENESIS`; se peer encontrado → `CONNECTED`; se erro → `OFFLINE_RETRY`.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/core test`.

Casos de teste:
1. FPP com swarm vazio + `isFounder=true` → transita para `GENESIS` após timer.
2. FPP com swarm vazio + `isFounder=false` → transita para `OFFLINE_RETRY`.
3. FPP com peer conectado no swarm → transita para `CONNECTED` imediatamente.
4. `createGenesisBlock` retorna bloco com `networkId` único + `founderPubKey`.
5. `onStateChange` callback dispara em cada transição.
6. FPP com `genesisTimerMs=100` → transição rápida (não espera 8s reais).
