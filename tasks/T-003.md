## 1. Objetivo
Implementar `VirtualClock` e `SeededRandom` no pacote `@plataforma/testkit`, implementando `ClockPort` e `RandomPort` definidos em T-004
*(derivado de `docs/plano-de-implementacao.md` §2.1 Anel 1 — Simulação Determinística; contratos de porta em `tasks/T-004.md` §1)*.

### Contratos exatos (assinaturas TS fixadas)

```ts
// --- packages/testkit/src/clock.ts ---
import { ClockPort } from '@plataforma/protocol';

/** Relógio virtual determinístico. Tempo inicial configurável; avança via advance(). */
export class VirtualClock implements ClockPort {
  constructor(initialTime?: number);
  now(): number;
  /** Avança o relógio em `ms` milissegundos. */
  advance(ms: number): void;
  /** Agenda callback para daqui a `ms` milissegundos (tempo virtual). Retorna timerId. */
  setTimeout(callback: () => void, ms: number): number;
  /** Cancela timer agendado. */
  clearTimeout(timerId: number): void;
}
```

```ts
// --- packages/testkit/src/random.ts ---
import { RandomPort } from '@plataforma/protocol';

/** PRNG determinístico com semente (xoshiro128**). */
export class SeededRandom implements RandomPort {
  constructor(seed: string);
  bytes(length: number): Uint8Array;
  /** Próximo double em [0, 1). Método auxiliar, não está na porta. */
  nextDouble(): number;
}
```

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente do Teste:** Node puro, `pnpm --filter @plataforma/testkit test`.
- [x] **Fora de Escopo:** Timer de sistema real, `Math.random()`.

Casos de teste (numerados):
1. `VirtualClock`: `new VirtualClock(1000).now()` retorna 1000.
2. `VirtualClock`: `clock.advance(50)` → `clock.now()` retorna 1050.
3. `VirtualClock`: `clock.advance(0)` é no-op.
4. `VirtualClock`: `setTimeout(cb, 100)` + `advance(50)` → callback NÃO chamado ainda.
5. `VirtualClock`: `setTimeout(cb, 100)` + `advance(100)` → callback chamado exatamente 1×.
6. `VirtualClock`: `clearTimeout` cancela callback pendente.
7. `SeededRandom`: mesma seed → mesma sequência de `bytes(16)` (2 chamadas).
8. `SeededRandom`: seeds diferentes → sequências diferentes.
9. `SeededRandom`: `bytes(0)` retorna `Uint8Array` vazio.
10. `SeededRandom`: `nextDouble()` retorna valor em [0, 1).

## 5. Instruções de Execução (Step-by-Step)

6. Re-exporte em `packages/testkit/src/index.ts`.
