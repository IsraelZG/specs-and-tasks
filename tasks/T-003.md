# T-003 · Pacote testkit: Relógio e Random

## 1. Objetivo
Implementar utilitários determinísticos (`VirtualClock` e `SeededRandom`) no pacote `packages/testkit` que implementem os contratos estabelecidos em `packages/protocol` (`ClockPort` e `RandomPort`).

## 2. Contexto RAG (Spec-Driven Development)
- [plano-de-implementacao.md](../docs/plano-de-implementacao.md) (Seção 2.1 Anel 1 - Simulação Determinística, e Seção 3, T-003)

## 3. Estratégia de Testes (Test-Driven Development)
- [ ] **Abordagem**: Testes unitários puros com Vitest. Deve-se provar que `VirtualClock.advance(100)` de fato avança o tempo interno. Provar que `SeededRandom` com a mesma *seed* gera exatamente a mesma sequência de bytes.

## 4. Referências de Código
- Arquivos Alvo: `packages/testkit/src/VirtualClock.ts`, `packages/testkit/src/SeededRandom.ts`, `packages/testkit/tests/*.test.ts`

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** redefina as interfaces `ClockPort` ou `RandomPort` aqui. Elas devem ser importadas do pacote `@plataforma/protocol`.
> - **NÃO** use `Math.random()` nativo dentro do `SeededRandom`. Use um PRNG determinístico (ex: LCG simples, xorshift ou importar `seedrandom`).
> - **NÃO** polua o escopo global. Não substitua `global.Date` ou `global.setTimeout` a menos que seja via uma API de injeção explícita.

1. **[TDD]** Crie `tests/VirtualClock.test.ts`. Escreva testes afirmando que o tempo inicial é X, e após chamar `clock.advance(50)`, o tempo é X+50.
2. Crie a classe `VirtualClock` implementando a porta importada do `@plataforma/protocol`. A classe deve gerenciar o tempo em uma variável de estado isolada. Adicione métodos estáticos ou internos para avançar o tempo e lidar com chamadas agendadas (`setTimeout` vitualizado).
3. **[TDD]** Crie `tests/SeededRandom.test.ts`. Teste instanciando duas vezes o objeto com a mesma seed ("test-seed") e verifique se as invocações sequenciais resultam na mesma ordem exata.
4. Implemente `SeededRandom` baseada na porta de randomização importada, retornando bytes Uint8Array de forma determinística usando a seed.
5. Garanta que o Vitest de `testkit` passa perfeitamente.
