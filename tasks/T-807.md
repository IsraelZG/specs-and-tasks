## 1. Objetivo
Implementar aba "Mídia" no app Bancada (T-008): upload de arquivo de teste, escolha de modo de cifra (convergent/unique), visualização de seeders/fontes, player, botão "derrubar WebSeed"
*(derivado de `docs/plano-de-implementacao.md` §11 T-807)*.

### Contratos
```tsx
// --- apps/bancada/src/tabs/MidiaTab.tsx ---

export type InfoHash = Uint8Array; // Ou string hex? Ver Decisão Aberta 1
export type ChunkingMode = 'convergent' | 'unique';

export interface FileUploadState {
  infoHash: InfoHash | null;
  fileName: string;
  size: number;
  cipherMode: ChunkingMode;
  progress: number;              // 0–100
  seeders: number;
  leechers: number;
  webSeedAvailable: boolean;
}

export interface MidiaTabProps {
  onUpload: (file: File, mode: ChunkingMode) => Promise<InfoHash>;
  onPlay: (infoHash: InfoHash) => void;
  onKillWebSeed: (infoHash: InfoHash) => void;
}

export function MidiaTab(props: MidiaTabProps): JSX.Element;
```
- Upload: input file + seletor de modo de cifra (`convergent` | `unique`).
- Progresso de upload com barra.
- Visualização: tabela de seeders (peerId, progress, speed) e fontes (WebTorrent, WebSeed).
- Player: `<video>` ou `<audio>` para o arquivo cifrado.
- Botão "Derrubar WebSeed" → testa fallback P2P.

## 6. Feedback de Especificação

> [!WARNING]
> **Decisões abertas para o Arquiteto:**
>
> 1. **Definição e localização de `ChunkingMode` e `InfoHash`**:
>    - O contrato importava esses tipos de `@plataforma/protocol` (referenciando `T-801a` que foi cancelada). No pacote `@plataforma/media` (T-801), o `InfoHash` é a raiz da MerkleTree e é do tipo `Sha256Digest` (`Uint8Array`). `ChunkingMode` não está definido.
>    - **Opções**:
>      - (A) Definir `ChunkingMode = 'convergent' | 'unique'` e `InfoHash = Uint8Array` localmente em `MidiaTab.tsx`.
>      - (B) Exportar estes tipos de um pacote de contrato comum (como `@plataforma/media` ou `@plataforma/protocol`).
>
> 2. **Comportamento de `onKillWebSeed`**:
>    - Como deve ser simulado o comportamento de "derrubar o WebSeed" na Bancada? A ação apenas oculta o WebSeed das fontes ativas na UI para verificação visual ou ela dispara uma API HTTP para o `system-peer` (T-804)?
>    - **Opções**:
>      - (A) Simular apenas na UI alterando o estado local `webSeedAvailable: false`.
>      - (B) Fazer uma chamada HTTP real para o endpoint do `system-peer` e refletir na tabela.
>
> 3. **Origem dos Dados de Seeders e Leechers**:
>    - Como o frontend obtém a lista de seeders (peerId, progress, speed) e leechers em tempo real? Existe um hook ou serviço de monitoramento P2P exposto globalmente?
>    - **Opções**:
>      - (A) Mockar o estado de seeders/leechers com valores dinâmicos fictícios para fins de demonstração da aba.
>      - (B) Criar uma interface `P2PMonitorPort` a ser injetada no componente.
