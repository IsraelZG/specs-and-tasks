## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (React Testing Library) / `playwright` (E2E)

## 1. Objetivo
Implementar aba "Mídia" no app Bancada (T-008): upload de arquivo de teste, escolha de modo de cifra (convergent/unique), visualização de seeders/fontes, player, botão "derrubar WebSeed"
*(derivado de `docs/plano-de-implementacao.md` §11 T-807)*.

### Contratos
```tsx
// --- apps/bancada/src/tabs/MidiaTab.tsx ---

export type InfoHash = Uint8Array; // Ou string hex? Ver Decisão Aberta 1
export type ChunkingMode = 'convergent' | 'unique';

export interface FileUploadState {
  infoHash: InfoHash | null;
  fileName: string;
  size: number;
  cipherMode: ChunkingMode;
  progress: number;              // 0–100
  seeders: number;
  leechers: number;
  webSeedAvailable: boolean;
}

export interface MidiaTabProps {
  onUpload: (file: File, mode: ChunkingMode) => Promise<InfoHash>;
  onPlay: (infoHash: InfoHash) => void;
  onKillWebSeed: (infoHash: InfoHash) => void;
}

export function MidiaTab(props: MidiaTabProps): JSX.Element;
```
- Upload: input file + seletor de modo de cifra (`convergent` | `unique`).
- Progresso de upload com barra.
- Visualização: tabela de seeders (peerId, progress, speed) e fontes (WebTorrent, WebSeed).
- Player: `<video>` ou `<audio>` para o arquivo cifrado.
- Botão "Derrubar WebSeed" → testa fallback P2P.

## 6. Feedback de Especificação

### Decisões de Arquiteto (2026-07-01, resolvidas via `/arquiteto-decisoes`)

**Decisão #1 — `ChunkingMode`/`InfoHash`: DECIDIDO opção (B).** Exportados de `@plataforma/media`
(mesma decisão de T-804 para `InfoHash` — fonte única). `ChunkingMode = 'convergent' | 'unique'`
também passa a viver em `@plataforma/media`, junto de `InfoHash`, em vez de definição local em
`MidiaTab.tsx`.

**Decisão #2 — `onKillWebSeed`: DECIDIDO opção (B) — chamada HTTP real ao `system-peer`.**
Dispara uma chamada HTTP real ao endpoint do `system-peer` (T-804) e reflete o resultado na
tabela — não é simulação apenas de UI. **Nota de consistência:** como a ação passa a ser real,
a Decisão #3 (seeders/leechers) segue a mesma linha — dados reais via `P2PMonitorPort`, não
mockados, para não misturar uma ação real com uma lista fake na mesma aba.

**Decisão #3 — dados de seeders/leechers: DECIDIDO opção (B).** Interface `P2PMonitorPort`
injetada no componente, consultando o `system-peer` de verdade — consistente com a Decisão #2.
