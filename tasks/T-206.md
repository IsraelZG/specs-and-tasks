# T-206 · Heartbeat implícito/explícito + evicção

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro, VirtualClock)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Estender `SwarmRegistry` (T-205) com heartbeat implícito (tráfego reseta timer), explícito (PING 8B após 15s ocioso), evicção (3 falhas/45s → cool-off 5min), modo economia (bateria <15% desativa PING explícito). Tudo sob VirtualClock
*(derivado de `docs/plano-de-implementacao.md` M2 — T-206; `docs/conceitos/swarm-registry.md`)*.

### Contratos exatos
```ts
// --- packages/core/src/swarmRegistry.ts (extensão) ---

export interface HeartbeatOptions {
  pingIntervalMs?: number;      // default 15000
  evictionThresholdMs?: number; // default 45000 (3× ping)
  coolOffMs?: number;           // default 300000 (5min)
  batteryPort?: { level: number }; // <15% → desativa ping explícito
}

export class SwarmRegistry {
  // ... API existente (T-205)
  /** Registra tráfego recebido (heartbeat implícito). */
  recordTraffic(peerId: DevicePeerId): void;

  /** Dispara heartbeat explícito se ocioso > pingInterval. */
  checkHeartbeats(): PeerId[]; // retorna peers que precisam de PING

  /** Processa timeout: retorna peers para evicção. */
  evictStale(): PeerId[];
}
```

## 4. Estratégia de Testes
- [x] **Framework:** Vitest + VirtualClock (T-003).

Casos de teste:
1. Tráfego recebido → `lastSeenAt` atualizado, sem PING.
2. 15s ocioso → `checkHeartbeats` retorna peer para PING.
3. 3 PINGs sem resposta (45s) → `evictStale` retorna peer.
4. Cool-off 5min → peer não reconecta antes do tempo.
5. Bateria <15% → `checkHeartbeats` não retorna PING.
6. Evicção remove peer de candidatos a líder de sync.

## 5. Instruções de Execução

2. Implemente heartbeat implícito/explícito.
3. Implemente evicção + cool-off + modo economia.
4. Rode build + test.
