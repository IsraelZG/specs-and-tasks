## 1. Objetivo

### Contratos exatos
```ts
// --- packages/transport/src/SwarmRegistry.ts (extensão) ---
import { PeerId, ClockPort } from '@plataforma/protocol';

export interface HeartbeatOptions {
  pingIntervalMs?: number;      // default 15000 (15s)
  evictionThresholdMs?: number; // default 45000 (45s, ou seja, 3× ping)
  coolOffMs?: number;           // default 300000 (5min)
  batteryPort?: { level: number }; // <15% → desativa ping explícito
}

export class SwarmRegistry {
  // Construtor estendido para aceitar clock e opções de heartbeat
  constructor(clock: ClockPort, opts?: HeartbeatOptions);

  // ... API existente (T-205)

  /** Registra tráfego recebido (heartbeat implícito). */
  recordTraffic(peerId: PeerId): void;

  /** Dispara heartbeat explícito se ocioso > pingInterval. Retorna peers que precisam de PING. */
  checkHeartbeats(): PeerId[];

  /** Processa timeout: remove e retorna peers para evicção. Coloca-os em cool-off. */
  evictStale(): PeerId[];
}
```
- **Cool-off Logic:** Ao evitar um peer em `evictStale()`, a data de término de cool-off é calculada como `clock.now() + coolOffMs`. Durante esse período, qualquer chamada para `upsert()` tentando definir o estado do peer como `'connected'` ou `'connecting'` é ignorada, mantendo o estado como `'disconnected'`.
- **Battery Port Mode:** Se `batteryPort` for informado e `batteryPort.level < 15`, `checkHeartbeats()` não emitirá solicitações de PING explícitas (sendo mantido o comportamento de evicção se o peer ultrapassar o timeout de qualquer forma).
