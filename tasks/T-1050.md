# T-1050 · Project references + tsc -b (item 3)

## 1. Objetivo
Migrar os pacotes TS do workspace para **project references** (`composite: true` + `references`) e trocar a compilação por **`tsc -b`**, para:
- **build topológico determinístico** — fim da corrida do turbo ao paralelizar deps (o que hoje alterna TS5055/TS2307);
- **detecção explícita de ciclo de referência** — o próprio `tsc -b` recusa referências cíclicas na origem.

Reforço estrutural para quando o monorepo crescer; só faz sentido com os ciclos já removidos (EST-34 + T-1048).

## 2. Contexto RAG
- Handoff TS5055 do EST-33 (turbo paraleliza ciclo → corrida).
- CLAUDE.md — turbo na raiz (`pnpm build`).
- `tsconfig` base do workspace (T-1049 já normaliza `rootDir`/`outDir`/`exclude`).

## 3. Escopo (Inputs/Outputs)
- **[UPDATE]** tsconfig de cada pacote: `composite: true`, `declaration`, `declarationMap`, e `references` para os pacotes dos quais depende.
- **[CREATE]** `tsconfig.json` de **solução** na raiz com `references` para todos os pacotes.
- **[UPDATE]** scripts de build para `tsc -b` (por pacote via turbo continua ok; a raiz pode usar `tsc -b`).
- **[UPDATE]** `turbo.json` se necessário (`dependsOn: ["^build"]`, outputs `dist/**`).

## 4. Testes
- `tsc -b` da raiz compila em ordem topológica, **sem TS5055**.
- Introduzir uma referência cíclica **falha explicitamente** no `tsc -b`.
- `pnpm -r build` e `pnpm -r test` verdes, sem regressão.

## 5. DoD
Build por project references verde e determinístico; ciclo de referência vira erro de compilação; sem regressão de build/test.

## 6. Feedback de Especificação
- As `references` devem formar um **DAG** — depende de EST-34/T-1048 terem removido os ciclos. Se algum pacote ainda cicla, o `tsc -b` recusa: **PARE** e escale, não contorne.
- Não misturar com a migração de bundler/emit — escopo é só a topologia de compilação TS.

## 7. Verificação (comandos exatos)
```bash
tsc -b                # (ou pnpm build) — ordem topológica, sem TS5055
pnpm -r test          # sem regressão
```
