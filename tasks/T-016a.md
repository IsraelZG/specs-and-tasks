# T-016a · Tema Multi-Nível: Style Dictionary (outputReferences)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest`
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Habilitar `outputReferences: true` no Style Dictionary do `packages/design-system` e recompilar tokens. O CSS gerado deve preservar cadeias `var()` entre camadas (tema → semântico → componente), permitindo que um override na camada de tema propague automaticamente para todos os componentes
*(extraído de T-016 §1; `docs/rfcs/design-system-proposal.md` D2)*.

### Contratos
- `style-dictionary.config.js`: `options.outputReferences: true` nos builds de tema e semântico.
- Resultado esperado no CSS:
  ```css
  --ds-component-button-primary-bg: var(--ds-theme-intent-primary-fill);
  --ds-theme-intent-primary-fill: var(--ds-color-ocean-500);
  ```
  Em vez de valores literais achatados.
- Tokens compilados: `build/web/theme-{light,dark}.css` atualizados.
- `pnpm --filter @plataforma/design-system build:tokens` exit 0.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest + inspeção de CSS gerado.

Casos de teste:
1. `pnpm build:tokens` → exit 0.
2. CSS gerado contém `var(--ds-` em vez de valores literais.
3. Token de componente referencia token de tema (cadeia não achatada).
4. Token de tema referencia token global (primitivo).

## 5. Instruções de Execução
1. Alterar `outputReferences: true` nos builds relevantes.
2. Recompilar tokens.
3. Inspecionar CSS gerado.
4. Rode build + validate.
