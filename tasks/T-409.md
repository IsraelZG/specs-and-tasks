## 1. Objetivo
Implementar aba "Topologia" no app Bancada (T-008): grafo visual de conexões (direta/relay/ws), estado de promoção por par, botão de tentar/cancelar promoção e indicador de quem é âncora
*(derivado de `docs/plano-de-implementacao.md` §7 T-409)*.

### Contratos
```tsx
// --- apps/bancada/src/tabs/TopologiaTab.tsx 
---
import type { PeerId } from '@plataforma/protocol';
import { ConnectionPromotionEngine, PromotionState, SwarmRegistry, type PeerEntry } from '@plataforma/transport';

export interface TopologiaTabProps {
  swarm: SwarmRegistry;
  promotionEngine: Pick<ConnectionPromotionEngine, 'getState' | 'cancel' | 'directCount'>;
  /** A composição resolve NAT e chama a API real; a UI não importa testkit. */
  tryPromote(peerId: PeerId): Promise<PromotionState>;
}

export function TopologiaTab({ swarm, promotionEngine }: TopologiaTabProps): JSX.Element;
```
- Renderiza grafo visual de conexões: nós = peers, arestas = conexões.
- Cada aresta colorida por tipo: verde = direta (WebRTC), azul = relay, cinza = WS.
- Indicador de âncora (peer do sistema) com ícone distinto.
- Estado de promoção por par: "direta", "promovendo...", "relay (NAT simétrico)".
- Botão "cancelar tentativa" chama `promotionEngine.cancel(peerId)` somente enquanto `PROMOTING`.
- Botão "tentar promoção" chama `tryPromote(peerId)` fornecido pela composição.
- Atualização em tempo real via polling (5s) ou eventos do SwarmRegistry.

## 6. Decisão registrada

**DECIDIDO (2026-07-16) — adaptar à API existente.** `SwarmRegistry` e `PeerEntry` vêm de
`@plataforma/transport`; a UI consulta `getState`, usa `cancel` somente durante `PROMOTING` e não
inventa `promote()`/`demote()`. Como `NatType` é hoje insumo de teste e não há telemetria canônica
no runtime, `TopologiaTab` recebe `tryPromote(peerId)` da composição. Ela é responsável por chamar
`ConnectionPromotionEngine.tryPromote(peerId, natType)` ou reportar indisponibilidade; a UI nunca
importa `@plataforma/testkit` nem simula uma promoção.

**Trade-off aceito.** A task não cria ainda um provedor de NAT de produção; isso pertence ao
subsystem de rede. Em troca elimina uma ação impossível, usa a API real e mantém a fronteira
UI/transport explícita e testável.
