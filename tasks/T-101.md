## 1. Objetivo
Criar wrappers isomórficos (Node/Browser) para as primitivas criptográficas do pacote `@plataforma/crypto`
*(derivado de `docs/plano-de-implementacao.md` §1.1 — crypto: "Ed25519, AES-256-GCM, HKDF, BIP39, PBKDF2, SSS, blake2s256 — wrappers sobre WebCrypto/noble"; `docs/visao-arquitetural.md` §1 camada 2 "Cripto: assinaturas Ed25519, cifra AES-256-GCM")*.

### Contratos exatos (assinaturas TS fixadas)

```ts
// --- packages/crypto/src/wrappers.ts ---

// === Ed25519 (assinatura/verificação) ===

/** Chave privada Ed25519 — 32 bytes. Derivado de `docs/visao-arquitetural.md` §1 camada 2. */
export type Ed25519PrivateKey = Uint8Array;
/** Chave pública Ed25519 — 32 bytes. */
export type Ed25519PublicKey = Uint8Array;
/** Assinatura Ed25519 — 64 bytes. */
export type Ed25519Signature = Uint8Array;

/** Gera par de chaves Ed25519. Backend: @noble/curves ou WebCrypto conforme ambiente. */
export function ed25519GenerateKeyPair(): Promise<{ publicKey: Ed25519PublicKey; privateKey: Ed25519PrivateKey }>;

/** Assina `message` com chave privada. */
export function ed25519Sign(privateKey: Ed25519PrivateKey, message: Uint8Array): Promise<Ed25519Signature>;

/** Verifica assinatura. */
export function ed25519Verify(publicKey: Ed25519PublicKey, message: Uint8Array, signature: Ed25519Signature): Promise<boolean>;

// === AES-256-GCM (cifra simétrica autenticada) ===

/** Chave AES-256 — 32 bytes. */
export type AesKey = Uint8Array;
/** Nonce/IV AES-GCM — 12 bytes (recomendado NIST). */
export type AesNonce = Uint8Array;
/** Resultado de cifra AES-GCM: { ciphertext, authTag }. */
export interface AesGcmResult { ciphertext: Uint8Array; authTag: Uint8Array; }

/** Cifra `plaintext` com AES-256-GCM. Gera nonce internamente (12 bytes) e o retorna. */
export function aesGcmEncrypt(key: AesKey, plaintext: Uint8Array): Promise<{ nonce: AesNonce; ciphertext: Uint8Array; authTag: Uint8Array }>;

/** Decifra `ciphertext` com AES-256-GCM. Lança se authTag não confere. */
export function aesGcmDecrypt(key: AesKey, nonce: AesNonce, ciphertext: Uint8Array, authTag: Uint8Array): Promise<Uint8Array>;

// === SHA-256 ===

/** Digest SHA-256 — 32 bytes. */
export type Sha256Digest = Uint8Array;

export function sha256(data: Uint8Array): Promise<Sha256Digest>;

// === Blake2s-256 ===

export function blake2s256(data: Uint8Array): Promise<Uint8Array>;

// === HKDF (SHA-256) ===

export function hkdfSha256(ikm: Uint8Array, salt: Uint8Array, info: Uint8Array, length: number): Promise<Uint8Array>;
```

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente do Teste:** Node puro, `pnpm --filter @plataforma/crypto test`.
- [x] **Fora de Escopo:** Testes em browser real (testados no CI via playwright em tarefas posteriores).

Casos de teste (numerados):
1. `ed25519GenerateKeyPair` gera chaves de 32 bytes; `publicKey !== privateKey`.
2. `ed25519Sign` + `ed25519Verify` roundtrip com vetor RFC 8032 §7.1 (TEST 2).
3. `ed25519Verify` retorna `false` com assinatura corrompida (1 bit trocado).
4. `ed25519Verify` retorna `false` com chave pública errada.
5. `aesGcmEncrypt` + `aesGcmDecrypt` roundtrip com plaintext variável (vazio, 1 byte, 1 KB, 1 MB).
6. `aesGcmDecrypt` lança com authTag corrompido.
7. `aesGcmDecrypt` lança com ciphertext corrompido.
8. `sha256` sobre `Uint8Array` vazio retorna digest RFC 6234 (e3b0c442...).
9. `blake2s256` sobre vetor RFC 7693.
10. `hkdfSha256` roundtrip com vetor RFC 5869 §A.1 (Test Case 1).

- **Comentários de Revisão:**
  - **[B2] Desvio de contrato de API (Retorno síncrono em vez de Promise):** As funções `ed25519GenerateKeyPair`, `ed25519Sign`, `ed25519Verify`, `sha256`, `blake2s256` e `hkdfSha256` foram modificadas no Rework para retornar diretamente valores síncronos de forma a silenciar o lint. Contudo, a especificação na Seção 1 exige explicitamente que estas funções retornem `Promise` (ex. `Promise<Sha256Digest>`, `Promise<boolean>`, etc.). A sonda adversarial demonstrou que `sha256(new Uint8Array(0))` não retorna uma `Promise`, violando a especificação do contrato de tipo da API pública.
  - **Ação Corretiva:** O executor deve reestabelecer o tipo de retorno `Promise` em todas as funções afetadas. Para evitar o lint `@typescript-eslint/require-await`, evite usar a keyword `async` e em vez disso retorne explicitamente `Promise.resolve(...)` ou wrap semelhante.
  - **Nota de Ambiente (Reviewer):** A transição CLI de status para `request_changes` via `manage-task.mjs` não pôde ser persistida devido a timeout de permissão no terminal. O veredito de mérito foi avaliado como Requer Refatoração.
