# T-211 · Bancada: aba Rede

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (React Testing Library) / `playwright` (E2E)
- **Capacidade-alvo:** haiku

## 1. Objetivo
Implementar aba "Rede" no app Bancada (T-008): exibe `SwarmRegistry.listConnected()` com estado, latência e tier de cada peer, com botão simulado de desconexão
*(derivado de T-008 §1 — "Bancada: cockpit de diagnóstico com abas"; T-205 §1 — SwarmRegistry em RAM)*.

### Contratos
```tsx
// --- apps/bancada/src/tabs/RedeTab.tsx ---
import { SwarmRegistry, PeerRecord } from '@plataforma/core'; // T-205

export interface RedeTabProps {
  swarm: SwarmRegistry;
}

export function RedeTab({ swarm }: RedeTabProps): JSX.Element;
```
- Renderiza tabela/listagem de `swarm.listConnected()`.
- Cada linha: DevicePeerId (truncado), ConnectionState (ícone colorido), latência (ms), tier, personas count.
- Botão "Derrubar" por peer → `swarm.delete(peerId)`.
- Atualização: re-renderiza a cada 5s ou via evento do swarm (polling leve aceitável para ferramenta de diagnóstico).

## 4. Estratégia de Testes
- [x] **Framework:** Vitest + React Testing Library (JSDOM).
- [x] **Ambiente:** `pnpm --filter bancada test`.

Casos de teste:
1. `RedeTab` com swarm vazio → mostra mensagem "Nenhum peer conectado".
2. `RedeTab` com 2 peers conectados → renderiza 2 linhas.
3. Clique em "Derrubar" → `swarm.delete` é chamado com peerId correto.
4. Peer com estado `connecting` → ícone diferente de `connected`.
5. Latência formatada corretamente (ex: "42ms").
