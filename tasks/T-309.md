## 1. Objetivo
Implementar `GlobalThrottle` no pacote `@plataforma/core`: token bucket por swarm com cotas 70/20/10 por visibilidade, degradação mobile e reavaliação a cada 30s; executa no contexto dono do banco (SharedWorker singleton ou Líder por Web Locks)
*(derivado de `docs/plano-de-implementacao.md` §6 T-309)*.

### Contratos exatos
```ts
// --- packages/core/src/throttle/globalThrottle.ts 
---

export enum VisibilityBucket {
  ACTIVE = 'active',       // cota 70% — aba ativa/visível
  BACKGROUND = 'background', // cota 20% — aba em background (Page Visibility API)
  DORMANT = 'dormant',     // cota 10% — abas ocultas sem foco
}

export enum ThrottleMode {
  NORMAL = 'normal',
  MOBILE_BATTERY = 'mobile_battery',   // bateria <30%
  MOBILE_DATA = 'mobile_data',         // dados móveis
  DEGRADED = 'degraded',               // só aba ativa
}

export interface ThrottleConfig {
  maxTokens: number;           // tokens totais no bucket (default 1000)
  refillRatePerSec: number;    // tokens por segundo (default 10)
  visibilityQuotas: Record<VisibilityBucket, number>; // default {active: 0.7, background: 0.2, dormant: 0.1}
  reEvalIntervalMs: number;    // default 30000 (30s)
  batteryThreshold: number;    // default 0.3 (30%)
}

export interface ThrottleState {
  availableTokens: number;
  mode: ThrottleMode;
  bucketAllocations: Map<string, number>; // swarmId → tokens alocados
}

export interface VisibilityPort {
  getVisibility(): VisibilityBucket;
  getBatteryLevel(): number | null;  // 0–1 ou null se desktop
  isMeteredConnection(): boolean;
}

export class GlobalThrottle {
  constructor(visibility: VisibilityPort, config?: Partial<ThrottleConfig>);

  /** Solicita N tokens para um swarm. Retorna true se concedido. */
  requestTokens(swarmId: string, count: number): boolean;

  /** Devolve tokens não usados. */
  returnTokens(swarmId: string, count: number): void;

  /** Atualiza modo baseado em visibilidade/bateria/conexão. */
  evaluateMode(): ThrottleMode;

  /** Estado atual para telemetria. */
  getState(): ThrottleState;

  /** Reavalia modo e cotas a cada 30s (timer interno). */
  startReEvalLoop(): void;

  /** Para loop de reavaliação. */
  stopReEvalLoop(): void;
}
```
- Token bucket: tokens reabastecem a `refillRatePerSec` até `maxTokens`.
- Visibilidade: 70% tokens para aba ativa, 20% background, 10% dormant.
- Mobile: bateria <30% OU dados móveis → `ThrottleMode.MOBILE_BATTERY`/`MOBILE_DATA` → só aba ativa recebe tokens.
- Swarm sincronizado (sem divergência) consome 0 tokens.
- Reavaliação a cada 30s.
