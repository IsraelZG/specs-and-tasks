# P-02 · Telemetria JSONL embutida nos scripts + projetor relatorio.mjs

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+ · **Package Manager:** `pnpm`
- **Repo:** **controle** (`C:\Dev2026\Docs\tools\scripts\`). Tooling-do-controle.
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Substituir a telemetria **manual em prosa** (cara, inconsistente, `not_measured` no tempo de raciocínio) por telemetria **embutida nos scripts que os agentes já chamam** — **zero chamadas extras** (regra de ouro: o overhead está NAS chamadas, ~5–10 s cada; qualquer solução que adicione chamadas é anti-solução). Um projetor gera o esqueleto do relatório de execução a partir do JSONL, deixando ao agente só o julgamento. Relatório-por-task vira subproduto automático, como o `ledger.mjs`.

## 4. Estratégia de Testes Estrita
- [x] **Framework:** funcional (Node puro) + um `telemetry.test.mjs`.
- [x] **Casos:** (1) `emit()` appenda linha JSON válida com `{task, phase, cmd, startedAt, wallMs, exitCode, actor}`; nunca lança (telemetria falha silenciosa — nunca quebra o comando hospedeiro); (2) uma transição de `manage-task` emite 1 evento; (3) `relatorio.mjs <ID>` lê o JSONL e imprime tabela de fases + **gaps derivados** (fim de um evento → início do próximo = tempo agente+harness, "por subtração"; raciocínio NÃO é medido diretamente) + totais por camada (setup/gate/transição); (4) sem JSONL para o ID → mensagem clara, não erro.
- [x] **Fora de Escopo:** dashboard agregado multi-task (é o `saldo`/`painel`, separado); medir latência de API isolada (impossível — registrar como limite).

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **REGRA DE OURO:** zero chamadas novas para o agente. A telemetria é efeito colateral de chamadas que já existem. Nada de "script start/end por etapa".
> - **NÃO** deixar a telemetria quebrar o hospedeiro: `emit` em try/catch que engole tudo. Melhor perder um evento que falhar um `finish`.
> - **NÃO** medir/estimar tempo de raciocínio — só derivar gaps entre eventos; o resto é `not_measured` honesto.
> - **NÃO** commitar `.telemetry/` (gitignored).

Passos: (1) `lib/telemetry.mjs`: `emit({...})` resolve `<ID>` (arg ou env `MGTIA_TASK`), appenda uma linha JSON com `os.hostname()`/arch/node em campos fixos + timestamp; (2) instrumentar cada emissor com 1 chamada `emit` por operação (medir `wallMs` com `performance.now()`); (3) `relatorio.mjs <ID>`: parseia, ordena por `startedAt`, computa gaps, agrupa por fase→camada, imprime markdown pronto para `docs/handoffs/`.
