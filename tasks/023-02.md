# 023-02 · Wave-picking com Zen sobre bins: deposito como subtipo SPEC:PLACE

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo (`pnpm build`, `pnpm test`, `pnpm lint` na raiz afetam todos os pacotes)
- **Test Runner:** `vitest` (pacotes core/protocol)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar o fluxo de wave-picking utilizando a Zen Engine para determinar o roteamento de separação (picking) ótimo de itens localizados em posições físicas (bins) em um depósito. 
- O depósito deve ser modelado como um subtipo de localidade, representado pelo nó de conteúdo `SPEC:PLACE` com categoria `deposit`.
- A posição física (`bin`) não deve ser um tipo novo no grafo, mas sim um atributo associado ao nó de inventário (`ASSET:INVENTORY`).
- O workflow de agrupamento (onda/wave) deve orquestrar as reservas (`ASSET:LOCK`) dos itens e delegar a decisão de ordenação de rota para a Zen Engine.

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- **Framework:** `vitest` orquestrado pelo `turborepo`.
- **Ambiente:** Node puro (sem browser/DOM).
- **Fora de Escopo:** Banco de dados de rede ativo, interface do usuário (UI) e conexões com sistemas WMS legados de terceiros.
- **Casos de Teste (Mínimo Obrigatório):**
  1. **Alocação de bin a inventory:** Criar um nó `InventoryAsset` do tipo `INVENTORY` com o atributo `bin` setado como string (ex: `"A-12-3"`). Assertar que a propriedade é persistida e lida corretamente como atributo e não como um tipo de nó separado.
  2. **Modelar depósito como subtipo PLACE:** Criar um nó de conteúdo do tipo `PLACE` com `category: 'deposit'`. Assertar que o validador de tipos o aceita como depósito válido de destino/origem de inventário.
  3. **Criar wave de picking:** Inicializar um objeto `PickingWave` com múltiplos itens reservados (`ASSET:LOCK`). Assertar que a wave inicia no status `created` e mapeia corretamente cada item para a sua respectiva posição física (`bin`).
  4. **Zen decide rota ótima (Routing):** Passar uma lista não ordenada de itens e um mapa de sequenciamento de distância de bins para `evaluatePickingRoute`. Assertar que a rota resultante (`route`) lista os IDs dos itens ordenados pela sequência ótima de bins resolvida pela Zen Engine.
  5. **Completar wave:** Rotear e mudar o status da `PickingWave` de `picking` para `completed`. Assertar as transições de estado do workflow e o timestamp de finalização.

```typescript
/**
 * Localização física representada como Place.
 * Derivado de docs/caderno-3-sdk/23-mapa-reference-spec.md §1.
 */
export interface Place {
  id: string; // PLACE node ID (ULID)
  nodeType: 'PLACE';
  payload: {
    category: 'deposit' | string;
    name: string;
    latitude: number;
    longitude: number;
    address?: string;
    metadata?: Record<string, unknown>;
  };
}

/**
 * Item de inventário com atributo bin (posição física).
 * Derivado de docs/caderno-3-sdk/25-logistica-reference-spec.md §1.2.
 */
export interface InventoryAsset {
  id: string; // INVENTORY asset ID (ULID)
  nodeType: 'INVENTORY';
  payload: {
    sku: string;
    quantity: number;
    placeId: string; // Referência a PLACE.id (Depósito)
    bin: string;     // Atributo posicional do inventário (ex: "A-12-3")
    metadata?: Record<string, unknown>;
  };
}

export interface PickingItem {
  id: string; // ID único do item de picking
  sku: string;
  quantity: number;
  sourceBin: string; // Atributo bin de origem
  status: 'pending' | 'picked' | 'missing';
}

export interface PickingWave {
  id: string; // WORKFLOW instance ID
  depositId: string; // ID do depósito (Place) onde ocorre
  items: PickingItem[];
  status: 'created' | 'picking' | 'completed' | 'cancelled';
  startedAt?: number;
  completedAt?: number;
}

export interface ZenRoutingInput {
  items: Array<{
    id: string;
    sku: string;
    bin: string;
  }>;
  binSequences: Record<string, number>; // Mapeia o nome do bin para um índice de ordenação de rota
}

export interface ZenRoutingResult {
  route: string[]; // Lista ordenada de IDs de PickingItem
  estimatedTimeSeconds: number;
}
```

#### `packages/core/src/logistics/picking-zen.ts`
```typescript
import { ZenEngine } from "@gorules/zen-engine";
import type { ZenRoutingInput, ZenRoutingResult } from "@plataforma/protocol";

/**
 * Avalia o roteamento do wave-picking usando a Zen Engine.
 * Derivado de docs/caderno-3-sdk/24-workflow-reference-spec.md §2.
 */
export async function evaluatePickingRoute(
  input: ZenRoutingInput,
  workflowContent: string
): Promise<ZenRoutingResult> {
  const engine = new ZenEngine();
  const decision = engine.createDecision(JSON.parse(workflowContent) as object);
  const response = await decision.evaluate(input);
  const result = response.result as { route: string[]; estimatedTimeSeconds: number };
  
  return {
    route: result.route,
    estimatedTimeSeconds: result.estimatedTimeSeconds,
  };
}
```
