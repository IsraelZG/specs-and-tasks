# T-302a · RBSR exchange: máquina de estados reconcile

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo

## 1. Objetivo
Implementar máquina de estados `reconcile` no pacote `@plataforma/protocol`: root exchange → range compare → request nodes → done. Orquestra o protocolo RBSR usando NetworkAdapterPort para transporte e RBSRBTree para fingerprints
*(extraído de T-302 §1; T-301b — compareRange; T-204 — WebSocket)*.

### Contratos exatos
```ts
// --- packages/protocol/src/rbsr/exchange.ts ---
import { RBSRBTree, Fingerprint } from './BTree.js'; // T-301a
import { NetworkAdapterPort, PeerId } from '../ports.js'; // T-004

export enum RBSRState { IDLE, ROOT_EXCHANGE, RANGE_COMPARE, REQUEST_NODES, APPLY, DONE }

export interface RBSRSession {
  localTree: RBSRBTree;
  remotePeerId: PeerId;
  transport: NetworkAdapterPort;
  state: RBSRState;
}

/** Inicia reconciliação completa. Retorna qtd de nós importados. */
export async function reconcile(session: RBSRSession): Promise<number>;

/** Envia root fingerprint para peer remoto. */
export function sendRootFingerprint(transport: NetworkAdapterPort, peerId: PeerId, rootXor: Fingerprint): Promise<void>;

/** Requisita nós específicos do peer remoto. */
export function requestNodes(transport: NetworkAdapterPort, peerId: PeerId, ids: ULID[]): Promise<void>;
```
Fluxo: IDLE → ROOT_EXCHANGE (envia rootXor) → se igual → DONE; se ≠ → RANGE_COMPARE (split recursivo, envia diffs) → REQUEST_NODES (baixa ids divergentes) → APPLY (T-302b).

## 4. Estratégia de Testes
- [x] **Framework:** Vitest + SimNetwork (T-005).

Casos de teste:
1. 2 peers com árvores idênticas → DONE sem download.
2. Peer A tem 5 nós que B não tem → reconcile importa 5 em B.
3. Ambos têm divergências → reconcile bidirecional.
4. Timeout durante request → retorna partial (nós já recebidos).
5. `sendRootFingerprint` + recebimento do outro lado → root comparado.

- **Achados:**
  - **[M1] Lint com 3 erros em `packages/protocol/src/rbsr/exchange.ts`**
    (linhas 53, 133, 299). Viola DoD §7 ("`pnpm --filter @plataforma/protocol build` e `test` verdes")
    estendido a lint, e a Regra 6 do MGTIA (qualidade que causará bugs downstream — non-null assertion
    em `bytes[i]!` pode mascarar buffer underflow, template literal com `number` vira string silenciosa
    no erro, condition impossível mascara regressão futura no BTree.reconcile).
    **Ação:** corrigir as 3 linhas (ex.: `result += String.fromCharCode(bytes[i] ?? 0)`, ou refatorar
    para iteração tipada; `Timeout waiting for message type 0x${expectedType.toString(16)}`; remover
    o `if (step.kind === 'diverged')` morto em `collectDivergentLeafIds` ou tratá-lo de fato).
  - **[M2] Cobertura do caso 2 da Seção 4 não bate com a spec.**
    Spec: "Peer A tem 5 nós que B não tem → reconcile importa 5 em B". O teste cria `treeA` (5 nós)
    como local e `treeB` (vazio) como remote — direção invertida, o que retornaria `count = 0`.
    O autor do teste reconheceu o gap e relaxou para `expect(count).toBeGreaterThanOrEqual(0)`.
    **Ação:** montar o teste de forma que o peer remoto (A) tenha os 5 nós e o local (B) esteja
    vazio, configurar handler de `REQUEST_NODES` em A chamando `respondNodes` (já exportado),
    e asserir `expect(count).toBe(5)` (ou número exato de IDs divergentes).
- **Achados menores (não-bloqueantes):**

  - **[m2] `respondNodes` exportado sem estar no bloco de Contratos exatos da Seção 1.** Aparece
    apenas na implementação — adicionar à spec ou remover do `index.ts`.
  - **[m3] Spec declara `ids: ULID[]` em `requestNodes`; impl usa `string[]`.** ULID é
