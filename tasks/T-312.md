### Contratos exatos
```ts
// --- packages/core/src/sync/dbOwner.ts ---

export enum DbRole {
  LEADER = 'leader',
  FOLLOWER = 'follower',
}

export interface DbOwnerState {
  role: DbRole;
  lockAcquired: boolean;
  leaderTabId: string | null;
}

export interface DbOwnerConfig {
  lockName: string;              // default 'plataforma-db-owner'
  lockTimeoutMs: number;         // default 10000 (10s timeout para adquirir lock)
  broadcastChannelName: string;  // default 'plataforma-db-channel'
}

export interface DbOwnerEvents {
  onBecomeLeader(): Promise<void>;      // inicializa pilha, abre OPFS
  onBecomeFollower(leaderId: string): void;
  onCrashRecovery(): Promise<void>;     // abre banco, checkpoint WAL, retoma do Гәltimo checkpoint de range
}

export class DbOwner {
  constructor(config: DbOwnerConfig, events: DbOwnerEvents);

  /** Registra para competir pelo lock. */
  requestLock(): Promise<void>;

  /** Libera lock explicitamente (Гәtil para troca manual). */
  releaseLock(): void;

  /** Estado atual. */
  getState(): DbOwnerState;

  /** Envia mensagem para outros tabs no BroadcastChannel. */
  broadcast(message: unknown): void;

  /** Registra handler para mensagens do canal. */
  onBroadcastMessage(handler: (message: unknown) => void): void;
}
```
- Caminho feliz: aba 1 abre вҶ’ adquire lock вҶ’ `onBecomeLeader`; aba 2 abre вҶ’ nГЈo adquire вҶ’ `onBecomeFollower`.
- Crash-recovery: aba LГӯder fecha (nГЈo depende de `beforeunload`) вҶ’ lock liberado automaticamente pelo browser вҶ’ prГіxima aba adquire вҶ’ `onCrashRecovery` (abrir banco, checkpoint WAL, retomar).
- Seguidora nunca abre OPFS diretamente вҖ” recebe dados via BroadcastChannel.
- Teste crГӯtico: matar aba LГӯder вҶ’ posse transfere, sync retoma sem corrupГ§ГЈo.
