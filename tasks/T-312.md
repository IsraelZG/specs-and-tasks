## 1. Objetivo
Implementar eleição de dono do banco por Web Locks no pacote `@plataforma/core`: caminho B do multi-aba — lock exclusivo `navigator.locks.request('plataforma-db-owner')`, primeira aba = Líder, demais = Seguidoras (BroadcastChannel/MessageChannel); troca de posse sem `beforeunload` via liberação do lock → próxima aba assume com crash-recovery
*(derivado de `docs/plano-de-implementacao.md` §6 T-312)*.

### Contratos exatos
```ts
// --- packages/core/src/sync/dbOwner.ts ---

export enum DbRole {
  LEADER = 'leader',
  FOLLOWER = 'follower',
}

export interface DbOwnerState {
  role: DbRole;
  lockAcquired: boolean;
  leaderTabId: string | null;
}

export interface DbOwnerConfig {
  lockName: string;              // default 'plataforma-db-owner'
  lockTimeoutMs: number;         // default 10000 (10s timeout para adquirir lock)
  broadcastChannelName: string;  // default 'plataforma-db-channel'
}

export interface DbOwnerEvents {
  onBecomeLeader(): Promise<void>;      // inicializa pilha, abre OPFS
  onBecomeFollower(leaderId: string): void;
  onCrashRecovery(): Promise<void>;     // abre banco, checkpoint WAL, retoma do último checkpoint de range
}

export class DbOwner {
  constructor(config: DbOwnerConfig, events: DbOwnerEvents);

  /** Registra para competir pelo lock. */
  requestLock(): Promise<void>;

  /** Libera lock explicitamente (útil para troca manual). */
  releaseLock(): void;

  /** Estado atual. */
  getState(): DbOwnerState;

  /** Envia mensagem para outros tabs no BroadcastChannel. */
  broadcast(message: unknown): void;

  /** Registra handler para mensagens do canal. */
  onBroadcastMessage(handler: (message: unknown) => void): void;
}
```
- Caminho feliz: aba 1 abre → adquire lock → `onBecomeLeader`; aba 2 abre → não adquire → `onBecomeFollower`.
- Crash-recovery: aba Líder fecha (não depende de `beforeunload`) → lock liberado automaticamente pelo browser → próxima aba adquire → `onCrashRecovery` (abrir banco, checkpoint WAL, retomar).
- Seguidora nunca abre OPFS diretamente — recebe dados via BroadcastChannel.
- Teste crítico: matar aba Líder → posse transfere, sync retoma sem corrupção.

- **Achado bloqueante [B1] — addInitScript registrado APÓS `page.goto`, harness não é injetado na primeira navegação:**

  - **Root cause:** O helper `injectElectionHarness(page)` chama `page.addInitScript(...)`, que **só aplica em NAVEGAÇÕES FUTURAS**. Mas o `test.beforeEach` faz `page.goto("/")` ANTES de `injectElectionHarness(page)`, e nos testes 5-8 o padrão é `ctx.newPage() → page.goto("/") → injectElectionHarness(page) → elect(page)` — a primeira navegação de cada `page` acontece SEM o init script registrado.
  - **Fix (3 linhas):** chamar `injectElectionHarness(pageX)` **ANTES** de `pageX.goto("/")` em cada teste (5, 6, 7, 8) e no `beforeEach` (que aliás é inútil porque os testes não usam a `page` do fixture — remover o `beforeEach`).
