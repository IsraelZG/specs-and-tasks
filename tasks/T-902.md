# T-902 · Suíte adversarial consolidada (12 vetores §2.5 + mapa de cobertura)

> **Origem:** absorve a *enumeração canônica dos 12 vetores* que estava (erroneamente) na antiga
> `T-905` como `ADVERSARIAL_SCENARIOS`. O insight preservado do agente é a lista dos 12 vetores
> **citando `plano-de-implementacao.md` §2.5**. A correção essencial: cada vetor já é critério de
> aceite de uma task de protocolo (§2.5 — "cada item vira teste permanente na suíte"); esta task
> **não reimplementa** os testes — ela os **consolida** numa suíte nomeada e bloqueante de CI e
> produz o **mapa de cobertura** vetor→teste→seção da spec.

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (tag/projeto `adversarial`)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Criar a suíte adversarial nomeada `pnpm test:adversarial` que agrega os 12 testes-alvo do §2.5
(cada um implementado e mantido pela sua task-dono), e um relatório de cobertura de invariantes
que mapeia cada vetor ao seu teste e à seção da spec — falhando o CI se algum vetor ficar
descoberto *(derivado de `docs/plano-de-implementacao.md` §12 — "T-902"; e §2.5 — os 12 vetores)*.

### Contratos exatos
```ts
// --- packages/testkit/src/adversarial/index.ts 
---
export interface AdversarialVector {
  id: number;          // 1..12 (§2.5)
  name: string;
  ownerTask: string;   // ex.: 'T-507'
  specSection: string; // ex.: '§2.5 item 2'
  testRef: string;     // caminho do teste permanente que cobre o vetor
}

/** Os 12 vetores canônicos do §2.5 (fonte da verdade da cobertura). */
export const ADVERSARIAL_VECTORS: AdversarialVector[];

export interface CoverageRow { vector: number; covered: boolean; testRef: string; }

/** Cruza ADVERSARIAL_VECTORS com os testes encontrados; cobertura < 12 → falha. */
export function coverageReport(): CoverageRow[];
```
- A suíte é uma `vitest` *project*/tag `adversarial` que inclui os testes-alvo das tasks-dono.
- `pnpm test:adversarial` roda a suíte; o relatório de cobertura é emitido e **bloqueia** o CI se
  qualquer vetor estiver `covered:false`.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (tag `adversarial`).

Casos de teste:
1. `ADVERSARIAL_VECTORS.length === 12` e ids 1..12 sem buracos.
2. `coverageReport()` com todos os testes presentes → 12 linhas `covered:true`.
3. `coverageReport()` com um teste ausente → linha `covered:false` e a suíte falha.
4. Cada `testRef` aponta para um arquivo de teste existente.
