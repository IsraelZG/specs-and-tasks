# M-016 · Fix parentAutoClose: lê children: + backfill EST-02/03/04

> **Origem do gap:** 3 pais decompostos (`EST-02`, `EST-03`, `EST-04`) ficaram presos em `draft:decomposed`
> porque o `task.service.js` (T-1029 §parentAutoClose) lê só o campo `subtasks:` do frontmatter, mas
> esses pais usam `children:`. Como todas as filhas já estão `done` desde 2026-07-06, o auto-close
> nunca disparou (não há `approve` novo pra disparar o check, e a leitura do `subtasks:` retorna vazio
> → `allDone === false`). Em sessão anterior (2026-07-06), foi aplicado o **metadata fix**:
> `subtasks: [...]` (mirror de `children:`) foi adicionado aos 3 pais + `parent: "..."` foi adicionado
> às 10 filhas. Este M-016 fecha o **service fix** (lê `children:` como fallback) + **backfill**
> (fecha os 3 pais pré-existentes via script one-shot).

### Contrato do script de backfill (NOVO)

```ts
// --- tools/scripts/close-decomposed-parents.mjs
// Uso: node tools/scripts/close-decomposed-parents.mjs [--dry-run]
// Detecta tasks com status: draft:decomposed em tasks/ e meta-tasks/, verifica se todas as
// filhas (em `subtasks:` ou `children:`) estão `done`, e fecha via TaskService.autoTransition.

import { TaskService } from '../../apps/nexus-backend/dist/services/task.service.js';
const svc = new TaskService({ rootDir: process.cwd() });
const parents = svc.listTasks().filter(r => r.frontmatter.status === 'draft:decomposed');
for (const p of parents) {
  const fm = svc.parseFrontmatterNaive(svc.getTask(p.id).body /* via raw read */);
  const childIds = parseArray(fm.subtasks || fm.children);
  const allDone = childIds.length > 0 && childIds.every(id => {
    const c = svc.getTask(id);
    return c && c.frontmatter.status === 'done';
  });
  if (allDone) {
    if (dryRun) console.log(`[dry-run] would close ${p.id}`);
    else svc.autoTransition(p.id, 'done', '[Auto-encerrado retroativo]', 'M-016: todas as filhas done — backfill one-shot');
  }
}
```
