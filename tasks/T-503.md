# T-503 · Validação de consentimento single-pass

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar validação de consentimento single-pass no pacote `@plataforma/core`: traversal que alcança `ASSET:CONSENT`/`CONTENT:PERSONAL_DATA` rebaixa tier de TTL; emissão com TTL longo nesses escopos é barrada (Zen Engine); checagem de consentimento + edge_filter num único passe
*(derivado de `docs/plano-de-implementacao.md` §8 T-503)*.

### Contratos exatos
```ts
// --- packages/core/src/auth/consent.ts ---
import { EntityId } from '@plataforma/protocol'; // T-102
import { SignedUcan } from './ucan-codec.js'; // T-501a
import { EdgeFilter, ResolutionResult } from './permissions.js'; // T-502
import { StoragePort } from '@plataforma/protocol'; // T-004

export enum ConsentTier {
  STANDARD = 'standard',         // TTL máximo 4h
  SENSITIVE = 'sensitive',       // TTL máximo 30min (dados pessoais)
  RESTRICTED = 'restricted',     // TTL máximo 5min (consentimento explícito)
}

export interface ConsentResult {
  tier: ConsentTier;
  requiresExplicitConsent: boolean;
  maxTtlMs: number;
  consentNodesFound: EntityId[];
  personalDataNodesFound: EntityId[];
}

export interface EmissionValidation {
  allowed: boolean;
  maxTtlMs: number;
  reason?: string;
}

/** Analisa escopo do UCAN e determina tier de consentimento. */
export function analyzeConsentTier(
  storage: StoragePort,
  ucan: SignedUcan,
  edgeFilter?: EdgeFilter,
): Promise<ConsentResult>;

/** Valida se emissão de UCAN é permitida dado o tier. */
export function validateEmission(
  proposedTtlMs: number,
  tier: ConsentTier,
): EmissionValidation;

/** Single-pass: checa consentimento + edge_filter em um único traversal. */
export function singlePassConsentCheck(
  storage: StoragePort,
  scopeEntityIds: EntityId[],
  proposedTtlMs: number,
  edgeFilter?: EdgeFilter,
): Promise<EmissionValidation>;
```
- `analyzeConsentTier`: percorre o DAG do escopo; se encontra `ASSET:CONSENT` → tier `RESTRICTED`; se encontra `CONTENT:PERSONAL_DATA` → tier `SENSITIVE`; senão `STANDARD`.
- `validateEmission`: rejeita TTL proposto se > `maxTtlMs` do tier.
- `singlePassConsentCheck`: combina consentimento + edge_filter num único passe O(1 traversal).
- Custo O(1) em profundidade de traversal (limitado por depth do escopo).
