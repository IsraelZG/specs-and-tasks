# T-1041 · Fixar as assinaturas de GraphStorePort

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo (`pnpm build`, `pnpm test`, `pnpm lint` na raiz afetam todos os pacotes)
- **Test Runner:** `vitest` (pacotes core/protocol)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Fixar o contrato tipado e agnóstico de dialeto de banco de dados para o grafo replicado, definindo a `GraphStorePort` no pacote `protocol`.

## 2. Contexto RAG (Spec-Driven Development)
- [[adr-005-storage-engine-agnosticism]]
- `packages/protocol/src/ports.ts`

## 3. Escopo de Arquivos (Inputs e Outputs)
- **[UPDATE]** `packages/protocol/src/ports.ts` (Adicionar `GraphStorePort`, `GraphStoreTx` e tipos auxiliares do grafo; depreciar `StoragePort` se necessário)

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- **Framework:** Vitest (Node puro)
- **Ambiente do Teste:** Node puro
- **Metodologia:** Compilação do tipo de porta e teste unitário simples usando um mock ou `DummyGraphStore` para garantir o type wiring.

## 5. Instruções de Execução (Step-by-Step)
1. Declare a nova interface `GraphStorePort` and a interface de transação `GraphStoreTx` em `packages/protocol/src/ports.ts`.
2. As assinaturas devem cobrir operações do domínio do grafo append-only (imutável e replicado):
   ```ts
   export interface GraphStorePort {
     putNode(node: SignedNode): Promise<void>;
     getNode(id: EntityId): Promise<SignedNode | null>;
     putEdge(edge: SignedEdge): Promise<void>;
     getEdges(query: { source?: EntityId; target?: EntityId; type?: EdgeType }): Promise<SignedEdge[]>;
     rangeScan(lo: EntityId, hi: EntityId): AsyncIterable<SignedNode>;
     transaction<T>(fn: (tx: GraphStoreTx) => Promise<T>): Promise<T>;
   }
   ```
3. Assegure que todos os tipos necessários (`SignedNode`, `SignedEdge`, `EntityId`, `EdgeType`) estejam devidamente importados ou exportados no pacote `protocol`.
