## 1. Objetivo
Substituir o stub de autorização da API de rede do KeyVault por validação real de UCAN, e fechar o vazamento de referência de chave.

> **ACHADO (auditoria 2026-07-01) — [M2 · segurança] em `packages/core/src/keyVault.ts`:**
> - `requestEpochKey(scope, proof)` (`keyVault.ts:48`) é a porta que **entrega chave de época a peers remotos**, mas aceita **qualquer string não-vazia** como UCAN (`if (!proof.ucan || !proof.delegatedTo) return denied; return this.requestKey(scope)`). Marcado "Stub M1" — esta task cobra a validação real via `validateUcan` (T-501) + checagem de capability sobre o `scope`.
> - Menor, mesmo arquivo: `requestKey` (`keyVault.ts:40-45`) devolve `entry.key` por referência — `store` faz cópia defensiva na entrada (`keyVault.ts:34`), mas a saída não. Caller pode mutar a chave no vault. Retornar `new Uint8Array(entry.key)`.

**Meta:** `requestEpochKey` valida a prova de delegação (assinatura + cadeia + capability sobre o `scope` pedido) antes de servir; ambas as APIs retornam cópia da chave.

### Contratos exatos (assinaturas TS fixadas, derivadas de `packages/core/src/keyVault.ts:9-53` + `packages/core/src/auth/ucan.ts:30-117`)

```ts
// --- packages/core/src/keyVault.ts (atualização) 
---
import type { ClockPort } from '@plataforma/protocol';
import type { SignedUcan, UcanCapability } from './auth/ucan.js';
import { validateUcan } from './auth/ucan.js';

/** Entrada interna do vault — nunca serializada. */
interface VaultEntry {
  key: Uint8Array;
  expiresAt: number;
}

/** Resultado de requestKey / requestEpochKey. */
export type KeyResult =
  | { ok: true; key: Uint8Array }
  | { ok: false; reason: 'not_found' | 'expired' | 'denied' };

/**
 * Payload de prova de delegação. Estruturado (T-501):
 * `ucan` é `SignedUcan` (assinatura + payload canônico) — NÃO mais `string`.
 * `delegatedTo` é a chave pública Ed25519 hex (32 bytes) do destinatário
 * declarado no UCAN; o caller DEVE fornecê-la para que `validateUcan` possa
 * conferir `payload.aud` (regra "audience_mismatch" de T-501).
 */
export interface DelegationProof {
  readonly ucan: SignedUcan;
  readonly delegatedTo: string; // hex Ed25519PublicKey
}

export class KeyVault {
  static readonly DEFAULT_TTL_MS = 4 * 60 * 60 * 1000;

  private clock: ClockPort;
  private entries: Map<string, VaultEntry> = new Map();

  constructor(clock: ClockPort) { this.clock = clock; }

  store(scope: string, key: Uint8Array, ttlMs?: number): void {
    this.entries.set(scope, {
      key: new Uint8Array(key), // cópia defensiva na entrada
      expiresAt: this.clock.now() + (ttlMs ?? KeyVault.DEFAULT_TTL_MS),
    });
  }

  /** API LOCAL — consumo do Sync Worker. Retorna cópia defensiva. */
  requestKey(scope: string): KeyResult {
    const entry = this.entries.get(scope);
    if (!entry) return { ok: false, reason: 'not_found' };
    if (this.clock.now() >= entry.expiresAt) return { ok: false, reason: 'expired' };
    return { ok: true, key: new Uint8Array(entry.key) }; // cópia na saída
  }

  /**
   * API DE REDE — servida a peers remotos. Valida o UCAN estruturalmente
   * (assinatura + cadeia + audience) e confere capability sobre o `scope`
   * pedido antes de liberar a chave. Retorna cópia defensiva.
   *
   * Regras (T-501 `validateUcan` + caderno `sync-dirigido-por-ucan`):
   *   1. `validateUcan(proof.ucan, delegatedToPublicKey)` deve retornar `{ok: true}`.
   *      Qualquer `{ok: false, reason}` → `denied` (sem expor o motivo ao peer;
   *      log local detalhado).
   *   2. `proof.ucan.payload.capabilities` deve conter ao menos 1 capability
   *      com `resource === 'entity:' + scope` E `ability === 'SERVE_EPOCH_KEY'`.
   *   3. Expiração da entry do vault é checada por `requestKey` (após autorização).
   */
  requestEpochKey(scope: string, proof: DelegationProof): KeyResult {
    // (1) Validação estrutural do UCAN (assinatura + cadeia + audience)
    const validation = validateUcan(proof.ucan, hexToBytes(proof.delegatedTo));
    if (!validation.ok) {
      // log local detalhado (não exposto ao peer — peer só vê 'denied')
      console.warn('[KeyVault] UCAN rejected:', validation.reason, 'scope:', scope);
      return { ok: false, reason: 'denied' };
    }
    // (2) Capability sobre o scope pedido
    const hasCapability = proof.ucan.payload.capabilities.some(
      (cap: UcanCapability) =>
        cap.resource === 'entity:' + scope && cap.ability === 'SERVE_EPOCH_KEY',
    );
    if (!hasCapability) {
      console.warn('[KeyVault] UCAN without scope capability:', scope);
      return { ok: false, reason: 'denied' };
    }
    // (3) Já chama requestKey (que aplica TTL + cópia defensiva)
    return this.requestKey(scope);
  }

  // ... purgeExpired, size (inalterados)
}
```

> - `KeyVault` API existente — `packages/core/src/keyVault.ts:21-72` (T-110).
