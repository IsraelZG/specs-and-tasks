## 1. Objetivo
Substituir o stub de autorização da API de rede do KeyVault por validação real de UCAN, e fechar o vazamento de referência de chave.

> **ACHADO (auditoria 2026-07-01) — [M2 · segurança] em `packages/core/src/keyVault.ts`:**
> - `requestEpochKey(scope, proof)` (`keyVault.ts:48`) é a porta que **entrega chave de época a peers remotos**, mas aceita **qualquer string não-vazia** como UCAN (`if (!proof.ucan || !proof.delegatedTo) return denied; return this.requestKey(scope)`). Marcado "Stub M1" — esta task cobra a validação real via `validateUcan` (T-501) + checagem de capability sobre o `scope`.
> - Menor, mesmo arquivo: `requestKey` (`keyVault.ts:40-45`) devolve `entry.key` por referência — `store` faz cópia defensiva na entrada (`keyVault.ts:34`), mas a saída não. Caller pode mutar a chave no vault. Retornar `new Uint8Array(entry.key)`.

**Meta:** `requestEpochKey` valida a prova de delegação (assinatura + cadeia + capability sobre o `scope` pedido) antes de servir; ambas as APIs retornam cópia da chave.
