# T-012 · Incorporar apps/design-system-showcase

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo (`pnpm build`, `pnpm test`, `pnpm lint` na raiz afetam todos os pacotes)
- **Test Runner:** `playwright` (E2E/Frontend)

## 1. Objetivo
Trazer o app de showcase do design system (repo externo, pasta `showcase/`, Vite + React 19) para `apps/design-system-showcase`, consumindo a lib `@plataforma/design-system` por **workspace link** (não mais por path alias `@ds/*`). Ao fim, `pnpm --filter design-system-showcase dev` e `... build` rodam verdes dentro do monorepo.

## 2. Contexto RAG (Spec-Driven Development)
- [ ] [plano-de-implementacao.md §1.1 Monorepo](../docs/plano-de-implementacao.md) — `apps/*`.
- [ ] [T-011 · Incorporar a lib](./T-011.md) — provê `@plataforma/design-system`. **Pré-requisito.**
- [ ] Fonte da cópia: `c:\Dev2026\Design System\showcase\` (registrada em `design-system-stub.txt`).

## 3. Escopo de Arquivos (Inputs e Outputs)
- **[READ]** `c:\Dev2026\Design System\showcase\**` (fonte)
- **[READ]** `packages/design-system/package.json` (nome/exports do consumido)
- **[CREATE]** `apps/design-system-showcase/**` (cópia do showcase: `src/`, `index.html`, `vite.config.ts`, `tsconfig.json`)
- **[UPDATE]** `apps/design-system-showcase/package.json` — dependência `"@plataforma/design-system": "workspace:*"`
- **[DELETE]** `package-lock.json` próprio do showcase

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [ ] **Framework:** Playwright smoke (opcional nesta fase) + `tsc --noEmit` + `vite build`.
- [ ] **Métricas/Cobertura:** binário — `pnpm --filter design-system-showcase build` exit 0; o app sobe em `dev` e renderiza as seções.
- [ ] **Ambiente do Teste:** Headless browser (se Playwright) / build Node.
- [ ] **Fora de Escopo:** cobertura visual de cada seção de componente.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** mantenha o alias `@ds/*` apontando para o código-fonte da lib — consuma a lib pelo nome do workspace.
> - **NÃO** duplique dependências que já vêm da lib como peer (`react`, `react-dom` ficam no app; o resto vem transitivo).
> - **NÃO** crie lockfile próprio.

1. Copiar `showcase/` externo → `apps/design-system-showcase/` (sem `.git`/`node_modules`/`dist`).
2. `package.json`: `private: true`, `type: module`; trocar quaisquer refs a `@your-app/design-system`/`@ds/*` por `@plataforma/design-system` (`workspace:*`).
3. Ajustar `vite.config.ts`/`tsconfig.json`: remover alias `@ds`→fonte; resolver pelo pacote.
4. `pnpm install` na raiz; `pnpm --filter design-system-showcase build` e `dev`.
5. Conferir que `ThemeEditor`/presets e `compileThemeToCSS` (importados da lib) funcionam.
