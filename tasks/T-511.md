# T-511 · Modelo soberano

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)

## 1. Objetivo
Implementar modelo soberano de identidade no pacote `@plataforma/core`: fluxo BIP39 puro com avisos de perda definitiva; export/import seguro; round-trip de identidade entre devices sem servidor
*(derivado de `docs/plano-de-implementacao.md` §8 T-511)*.

### Contratos exatos
```ts
// --- packages/core/src/identity/sovereignModel.ts ---
import { Ed25519PrivateKey, Ed25519PublicKey } from '@plataforma/crypto'; // T-101a

export interface SovereignIdentity {
  mnemonic: string;              // BIP39 12 ou 24 palavras
  masterPrivateKey: Ed25519PrivateKey;
  masterPublicKey: Ed25519PublicKey;
  devicePrivateKey: Ed25519PrivateKey;
  devicePublicKey: Ed25519PublicKey;
  personaPrivateKey: Ed25519PrivateKey;
  personaPublicKey: Ed25519PublicKey;
}

export interface ExportPayload {
  mnemonic: string;
  exportedAt: number;
  warnings: string[];
}

export interface ImportResult {
  identity: SovereignIdentity;
  warnings: string[];
}

/** Cria identidade soberana (BIP39 puro). */
export function createSovereignIdentity(wordCount?: 12 | 24): SovereignIdentity;

/** Deriva identidade a partir de mnemônico existente. */
export function deriveFromMnemonic(mnemonic: string): SovereignIdentity;

/** Exporta identidade com avisos de perda definitiva. */
export function exportIdentity(identity: SovereignIdentity): ExportPayload;

/** Importa identidade com verificação de checksum BIP39. */
export function importIdentity(mnemonic: string): ImportResult;

/** Valida mnemônico (checksum BIP39). */
export function validateMnemonic(mnemonic: string): boolean;

/** Avisos de perda definitiva (display ao usuário). */
export function getLossWarnings(): string[];
```
- `createSovereignIdentity`: gera mnemônico BIP39 (12 ou 24 palavras) → deriva chave mestra (plataforma-master-key) → deriva chaves de dispositivo e persona (ver decisão aberta sobre caminhos na Seção 6).
- `exportIdentity`: exibe mnemônico + avisos explícitos de perda definitiva.
- `importIdentity`: valida checksum BIP39 → reconstroi identidade completa.
- Nenhum servidor envolvido: round-trip puramente local.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/core test`.

Casos de teste:
1. `createSovereignIdentity` gera identidade com todas as chaves.
2. Mesmo mnemônico → mesma identidade (determinístico).
3. Mnemônicos diferentes → identidades diferentes.
4. `exportIdentity` contém warnings de perda definitiva.
5. `importIdentity` roundtrip: export → import → mesma identidade.
6. `importIdentity` com mnemônico inválido → erro.
7. `validateMnemonic` com checksum correto → true.
8. `validateMnemonic` com checksum errado → false.
9. Round-trip entre dois "devices": Device A exporta → Device B importa → mesmas chaves.
10. `getLossWarnings` retorna avisos não vazios.

## 6. Feedback de Especificação

### Decisão de Arquiteto (2026-07-01, resolvida via `/arquiteto-decisoes`)

**DECIDIDO: opção (a).** HKDF-SHA256 a partir do seed BIP39 de 64 bytes (mesmo padrão de
`deriveMasterKeyPair`), com `info = "plataforma-device-key"` e `info = "plataforma-persona-key"`,
gerando chaves privadas Ed25519 de 32 bytes. Consistente com o restante do pacote — sem lib nova
(BIP32/SLIP-0010 descartado; caso de uso é só 2 chaves fixas, não sub-derivação em árvore).
