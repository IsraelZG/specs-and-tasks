## 1. Objetivo
Implementar `issueUcan` (emissão com assinatura Ed25519) e `verifyUcan` (validação de assinatura + expiração) no pacote `@plataforma/core`
*(extraído de T-501 §1)*.

### Contratos exatos
```ts
// --- packages/core/src/auth/ucan-issue.ts ---
import { Ed25519PrivateKey, Ed25519PublicKey } from '@plataforma/crypto'; // T-101a
import { SignedUcan, UcanPayload } from './ucan-codec.js'; // T-501a

/** Emite UCAN assinado. TTL default = 4h. */
export function issueUcan(
  issuerKey: Ed25519PrivateKey,
  audiencePubKey: Ed25519PublicKey,
  capabilities: UcanPayload['att'],
  ttlMs?: number,
): Promise<SignedUcan>;

/** Valida assinatura + expiração. Não valida atenuação (T-501c). */
export function verifyUcan(ucan: SignedUcan, rootIssuerPubKey: Ed25519PublicKey): Promise<boolean>;
```

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).

Casos de teste:
1. `issueUcan` + `verifyUcan` roundtrip.
2. `verifyUcan` com assinatura adulterada → false.
3. `verifyUcan` com UCAN expirado → false.
4. `verifyUcan` com chave pública errada → false.
5. `issueUcan` com TTL customizado → exp reflete corretamente.

## 5. Instruções de Execução
1. Escreva testes com pares de chaves Ed25519.
2. Implemente `issueUcan` (encode payload → canonicalize → sign).
3. Implemente `verifyUcan` (decode → verify signature → check exp).
4. Rode build + test.
