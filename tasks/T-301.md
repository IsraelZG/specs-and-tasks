# T-301 · B-Tree de fingerprints (RBSR)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)

## 1. Objetivo
Implementar B-Tree de fingerprints RBSR no pacote `@plataforma/protocol`: estrutura em RAM indexada por ULID com fingerprint individual (SHA-256) e XOR agregado por sub-range para reconciliação O(log n)
*(derivado de `docs/visao-arquitetural.md` §8 — RBSR; T-106 — nodes/edges; T-107 — assinatura + canonicalize)*.

### Contratos exatos
```ts
// --- packages/protocol/src/rbsr/BTree.ts ---
import { ULID } from '@plataforma/core'; // T-102

/** Fingerprint = SHA-256 do canonicalize(nó). 32 bytes. */
export type Fingerprint = Uint8Array;

/** Nó da B-Tree RBSR. */
export interface RBSRNode {
  id: ULID;
  fingerprint: Fingerprint;       // SHA-256 individual
  rangeXor: Fingerprint;          // XOR agregado da sub-árvore
  left?: RBSRNode;
  right?: RBSRNode;
}

/** Resultado da comparação de ranges. */
export interface RangeDiff {
  onlyLocal: ULID[];    // ids que só existem localmente
  onlyRemote: ULID[];   // ids que só existem remotamente
  conflict: ULID[];     // ids iguais com fingerprint diferente
}

export class RBSRBTree {
  constructor();

  /** Insere ou atualiza nó. Recalcula XOR da sub-árvore. */
  insert(id: ULID, fingerprint: Fingerprint): void;

  /** Remove nó. */
  delete(id: ULID): void;

  /** Busca fingerprint por id. */
  get(id: ULID): Fingerprint | undefined;

  /** XOR agregado de toda a árvore. */
  get rootXor(): Fingerprint;

  /** Compara ranges com árvore remota e retorna diff. */
  compareRange(rangeStart: ULID, rangeEnd: ULID, remoteXor: Fingerprint, remoteIds: ULID[]): RangeDiff;

  /** Reconcilia: recebe ids do diff e decide o que sincronizar. */
  size: number;
}

/** XOR de dois fingerprints (byte a byte). */
export function xorFingerprints(a: Fingerprint, b: Fingerprint): Fingerprint;
```

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/protocol test`.

Casos de teste:
1. `insert` + `get` roundtrip.
2. `rootXor` = XOR de todos os fingerprints inseridos.
3. `rootXor` se atualiza corretamente após `delete`.
4. `compareRange`: ranges idênticos → `onlyLocal` e `onlyRemote` vazios.
5. `compareRange`: id extra no local → `onlyLocal` contém o id.
6. `compareRange`: fingerprint diferente para mesmo id → `conflict`.
7. Árvore com 1000 nós: `insert` escala O(log n), `rootXor` correto.
8. `xorFingerprints` idempotente: a XOR a XOR b = b.
