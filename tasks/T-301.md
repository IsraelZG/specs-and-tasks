## 1. Objetivo
Implementar a B-Tree em memória que suporta o protocolo RBSR (Range-Based Set Reconciliation): armazena IDs ordenáveis (ULIDs) e seus fingerprints individuais (SHA-256 de `id ‖ signature`), agregando fingerprints de sub-ranges via XOR para reconciliação O(1) no caso de match e recursão em sub-ranges no caso de mismatch
*(derivado de `docs/caderno-2-protocol/03-set-reconciliation-protocol.md` §1.1 e §1.2)*.

### Contratos exatos (assinaturas TS fixadas)

```ts
// --- packages/protocol/src/rbsr/BTree.ts ---

/** Fingerprint de 256 bits (SHA-256) de um elemento `(id, signature)`.
 *  Derivado de caderno §1.1: F(x) = SHA-256(id_x ‖ signature_x). */
export type Fingerprint = Uint8Array; // 32 bytes

/** Range léxico de IDs (intervalo fechado [lo, hi]). */
export interface IdRange {
  readonly lo: string;  // ULID inicial (inclusive)
  readonly hi: string;  // ULID final (inclusive)
}

/** Entrada da B-Tree: ID ordenável + fingerprint individual. */
export interface BTreeNode {
  readonly id: string;            // ULID ordenável
  readonly fingerprint: Fingerprint; // SHA-256(id ‖ signature)
}

/** RangeHeader: metadados de um sub-range + XOR agregado dos fingerprints contidos. */
export interface RangeHeader {
  readonly range: IdRange;
  readonly count: number;          // quantidade de elementos no range
  readonly xorAggregate: Fingerprint; // XOR cumulativo de todos os fingerprints do range
}

/** Resultado da troca de fingerprints em uma sub-divisão recursiva. */
export type ReconciliationStep =
  | { kind: 'equal'; range: IdRange }                                // F1 = F2 → range sincronizado
  | { kind: 'diverged'; left: IdRange; right: IdRange }              // F1 ≠ F2 → subdividir
  | { kind: 'leaf'; range: IdRange; missingFromA: string[]; missingFromB: string[] }; // IDs divergentes

/** Lançado por `rangeHeader` quando o range `[lo, hi]` não contém elementos. */
export class RangeEmptyError extends Error {
  constructor(message: string) {
    super(message);
    this.name = 'RangeEmptyError';
  }
}

/**
 * B-Tree em memória ordenada por `id` (ULID, ordem lexicográfica).
 * Mantém fingerprints individuais e agrega o XOR de ranges.
 * Operações (7 públicas):
 *  - insert(node): adiciona/atualiza (id, fingerprint)
 *  - remove(id): remove
 *  - get(id): busca exata, retorna fingerprint ou null
 *  - size(): quantidade total de elementos
 *  - rangeHeader(range): computa count + XOR agregado do range
 *  - rangeFooter(range): computa count + SHA-256(id₁ ‖ id₂ ‖ … ‖ idₙ) para o range
 *  - reconcile(range, remoteXor): executa um passo de reconciliação contra o range do peer
 */
export class BTree {
  /** Cria B-Tree vazia. */
  constructor();

  /** Insere (ou atualiza) o nó com `id` e `fingerprint`. Idempotente. */
  insert(node: BTreeNode): void;

  /** Remove o nó com `id`. Retorna `true` se removeu, `false` se não existia. */
  remove(id: string): boolean;

  /** Busca exata por `id`. Retorna o fingerprint ou `null`. */
  get(id: string): Fingerprint | null;

  /** Quantidade total de elementos. */
  size(): number;

  /**
   * RangeHeader do range `[lo, hi]`. Conta elementos e XOR-agrega os fingerprints.
   * XOR(range) deve ser exatamente igual a XOR dos filhos.
   * Lança `RangeEmptyError` se o range não tem elementos.
   */
  rangeHeader(range: IdRange): RangeHeader;

  /**
   * RangeFooter do range `[lo, hi]`: count + SHA-256 dos IDs em ordem.
   * Derivado de caderno §1.2 item 6: count e checksum (SHA-256(id₁ ‖ … ‖ idₙ)).
   */
  rangeFooter(range: IdRange): { count: number; checksum: Fingerprint };

  /**
   * Reconciliação O(1) com o peer:
   *  - F(remote) === F(local) → `{kind:'equal', range}` em O(1) sem leitura adicional.
   *  - F(remote) !== F(local) e o range contém > 1 elemento → subdivide ao meio em `left` e `right`
   *    (`{kind:'diverged'}`); o caller itera recursivamente.
   *  - F(remote) !== F(local) e o range contém 1 elemento → `{kind:'leaf', missingFromA/B}`.
   *
   * `remoteXor` deve ser o XOR agregado do peer remoto para o mesmo `range`.
   */
  reconcile(range: IdRange, remoteXor: Fingerprint): ReconciliationStep;
}
```

### Derivados formais (caderno §1.1)
- `F(x) = SHA-256(id_x ‖ signature_x)` — fingerprint individual de um elemento.
- `F([A, B]) = ⊕_{x ∈ [A, B]} F(x)` — fingerprint do range (XOR cumulativo).
- B-Tree mantém IDs ordenados; rangeHeader computa count + XOR do range em O(|range|) (linear, aceitável: a árvore é cacheada e reconstruída por queries; caderno §1.1).

### Casos de teste (numerados)
1. B-Tree vazia: `size() === 0`, `get` retorna `null`, `rangeHeader` lança `RangeEmptyError`.
2. `insert` + `get` roundtrip: id recuperado, fingerprint byte-a-byte.
3. `insert` idempotente: re-inserir mesmo (id, fingerprint) não muda `size`.
4. `remove` existente: retorna `true`; `size` decrementa; `get` retorna `null`.
5. `remove` inexistente: retorna `false`.
6. **XOR(range) === XOR(filhos):** com N=4 fingerprints `f1, f2, f3, f4`, `xor([min(f), max(f)]) === f1 ⊕ f2 ⊕ f3 ⊕ f4`.
7. **Propriedade sub-divisão:** split ao meio do range → XOR(left) ⊕ XOR(right) === XOR(full).
8. `rangeFooter`: count correto + SHA-256 dos IDs concatenados em ordem lexicográfica.
9. **Reconciliação O(1) — equal:** `remoteXor` igual ao local → `{kind:'equal'}`, sem iterar.
10. **Reconciliação — diverged com > 1 elemento:** `remoteXor` diferente → `{kind:'diverged', left, right}` onde left+right = range particionado.
11. **Reconciliação — leaf com 1 elemento:** range unitário + `remoteXor` diferente → `{kind:'leaf', missingFromA/B}`.
12. **Falsificação por colisão de range XOR (básico):** com 1 elemento local e 1 elemento remoto (IDs diferentes), o `reconcile` retorna `leaf` com IDs diferentes em `missingFromA` e `missingFromB`.
13. **Inserção mantém ordem:** inserir IDs em ordem aleatória, `get` em varredura retorna IDs em ordem lexicográfica.

## 6. Feedback de Especificação (Spec Feedback Loop)

> **✅ DECISÕES RESOLVIDAS (2026-06-23 — arquiteto):** (1) **`rangeHeader` O(|range|)** — varredura linear somando XOR; sem agregados pré-computados nos nós internos por ora (árvore reconstruída por query; otimizar só sob profiling). (2) **Estrutura interna: array ordenado com inserção binária** (não BST de ponteiros) — mais simples, in-memory, sem dep externa.
- **Decisão em aberto — performance do `rangeHeader`:** o caderno §1.1 não fixa se rangeHeader é O(|range|) ou O(log n) com agregados pré-computados. Implementação simples é O(|range|) (varre o range somando XOR). Se o autor quiser O(log n), a B-Tree precisa de nós internos com XOR pré-computado. **Sugestão: começar com O(|range|); otimizar depois se necessário** — confirma com arquiteto.
- **Decisão em aberto — estrutura interna:** árvore binária balanceada via array (heap-like) ou BST com ponteiros? Implementação simples pode usar `Array<BTreeNode>` ordenada por inserção binária. Sem dependência externa.
