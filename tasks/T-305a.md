# T-305a · Sync UCAN: scopeRBSRTree (filtro por UCAN)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar `scopeRBSRTree` no pacote `@plataforma/protocol`: reconstrói RBSRBTree apenas com nós autorizados pelo UCAN, impedindo vazamento de subgrafos não permitidos
*(extraído de T-305 §1; T-301a — RBSRBTree; T-501a — UCAN codec)*.

### Contratos exatos
```ts
// --- packages/protocol/src/auth/ucanScope.ts ---
import { SignedUcan } from '@plataforma/core'; // T-501a
import { RBSRBTree } from '../rbsr/BTree.js'; // T-301a

/** Reconstrói B-Tree apenas com entity_ids no escopo do UCAN. */
export function scopeRBSRTree(tree: RBSRBTree, ucan: SignedUcan): RBSRBTree;

/** Extrai entity_ids autorizados do UCAN (prefixo 'entity:' em att[].with). */
export function extractEntityIds(ucan: SignedUcan): string[];
```

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).

Casos de teste:
1. UCAN com escopo `entity:A` → B-Tree filtrada só contém A.
2. UCAN com escopo `entity:A` + `entity:B` → contém A e B.
3. UCAN sem escopo (vazio) → B-Tree vazia.
4. `extractEntityIds` extrai prefixos corretos.
