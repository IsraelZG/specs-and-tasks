# T-1005 · Interface do Frontend (MGTIA-v2 + Editor de Docs)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v22+
- **Package Manager:** `pnpm`
- **Test Runner:** `playwright`

## 1. Objetivo
Criar o dashboard web do Nexus. A aplicação React deverá ter uma tela de quadro Kanban (que lê a pasta `/tasks` e renderiza os cards usando MGTIA-v2 visualmente) e um visualizador/editor de arquivos Markdown focado na pasta `/docs`.

## 2. Contexto RAG (Spec-Driven Development)
- [ ] [Plano de Implementação Nexus](../plano-de-implementacao.md)

## 3. Escopo de Arquivos (Inputs e Outputs)
- **[CREATE]** `apps/nexus-frontend/src/pages/TaskBoard.tsx`
- **[CREATE]** `apps/nexus-frontend/src/pages/DocsEditor.tsx`

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [ ] **Framework:** Playwright (Component Testing)
- [ ] **Ambiente do Teste:** Headless Browser. Garantir que as colunas do Kanban processam os arquivos corretamente.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** utilize TailwindCSS puro sem componentes estilizados ou bibliotecas prontas como Radix/Shadcn para as tabelas e cards. Mantenha UI de classe enterprise.

1. Configure rotas da aplicação React (`react-router-dom`).
2. Crie a página TaskBoard, consumindo a API interna do backend para listar e atualizar `tasks/T-*.md`.
3. Crie a página de documentação, permitindo navegação pelos arquivos `.md`.
