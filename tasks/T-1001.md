# T-1001 · Configuração do Monorepo Nexus (Backend e Frontend)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v22+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo
- **Test Runner:** `vitest`

## 1. Objetivo
Inicializar o workspace do projeto Nexus usando Turborepo, contendo duas aplicações principais: um backend Node.js (Express/Fastify) e um frontend React (Vite).

## 2. Contexto RAG (Spec-Driven Development)

- **[CREATE]** `apps/nexus-frontend/package.json`
- **[CREATE]** `apps/nexus-frontend/src/main.tsx`

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [ ] **Framework:** Vitest (para utilitários core)
- [ ] **Ambiente do Teste:** Node puro

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** instale bibliotecas de UI complexas ainda. Apenas inicie o scaffolding.

1. Inicialize o package.json na raiz do projeto com o workspace pnpm configurado.
2. Crie a pasta `apps/` e inicialize os dois projetos base.
3. Configure o `turbo.json` com os scripts de `build` e `dev`.
