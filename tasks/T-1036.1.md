# T-1036.1 · Integration: wire requestEpochKey to a production caller

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest`
- **Capacidade-alvo:** haiku

## 1. Objetivo
Ligar a primitiva `KeyVault.requestEpochKey(scope, proof)` a um caller de produção real.
A primitiva foi implementada em T-1036 e validada com 17 testes unitários, mas nenhum
código de produção no monorepo a chama atualmente — está morta.

**Meta:** conectar a primitiva a pelo menos 1 endpoint de produção ou handler RPC que sirva
chaves de época a peers remotos autenticados por UCAN.

## 2. Contexto RAG
- [T-1036](./T-1036.md) — `KeyVault.requestEpochKey` com validação UCAN real
- [T-501](./T-501.md) — `SignedUcan`, `validateUcan`, estrutura de capability
- [T-110](./T-110.md) — spec original do KeyVault

## 3. Opções de integração (a decidir durante endurecimento)

Opções possíveis:
- **(A)** Adicionar endpoint `POST /api/keyvault/epoch/:scope` no `system-peer` que recebe
  `DelegationProof`, chama `requestEpochKey` e retorna a chave.
- **(B)** Wire via handler RPC em `packages/transport` — o peer recebe uma requisição de chave
  via protocolo de rede e chama `requestEpochKey` internamente.
- **(C)** Wire via endpoint já existente que precise de autenticação de chave de época.

**Escolha a ser feita durante endurecimento (`/endurecer-task`).**

## 4. DoD
