### Contratos exatos (DDL fixado)

```sql
-- Migration v1: tabelas do grafo append-only (categoria 1 do §4)

CREATE TABLE IF NOT EXISTS nodes (
  id          TEXT    NOT NULL PRIMARY KEY,   -- ULID (T-102)
  node_type   TEXT    NOT NULL,               -- 'PROFILE' | 'ASSET' | 'SPECIFICATION' | 'CONTENT'
  payload     BLOB    NOT NULL,               -- MessagePack serializado (T-201)
  hlc         TEXT    NOT NULL,               -- HLC como string (T-103)
  public_key  BLOB    NOT NULL,               -- Ed25519PublicKey (32 bytes, T-101)
  signature   BLOB    NOT NULL,               -- Ed25519Signature (64 bytes, T-101)
  created_at  INTEGER NOT NULL DEFAULT (unixepoch()) -- wall-clock for debugging only
);

CREATE TABLE IF NOT EXISTS edges (
  id          TEXT    NOT NULL PRIMARY KEY,   -- ULID (T-102)
  edge_type   TEXT    NOT NULL,               -- 'MUTATES' | 'DELEGATED_TO' | ...
  source_id   TEXT    NOT NULL REFERENCES nodes(id),
  target_id   TEXT    NOT NULL REFERENCES nodes(id),
  payload     BLOB,                           -- MessagePack opcional
  hlc         TEXT    NOT NULL,
  public_key  BLOB    NOT NULL,
  signature   BLOB    NOT NULL,
  created_at  INTEGER NOT NULL DEFAULT (unixepoch())
);

-- Índices essenciais
CREATE INDEX IF NOT EXISTS idx_nodes_type    ON nodes(node_type);
CREATE INDEX IF NOT EXISTS idx_edges_source  ON edges(source_id);
CREATE INDEX IF NOT EXISTS idx_edges_target  ON edges(target_id);
CREATE INDEX IF NOT EXISTS idx_edges_type    ON edges(edge_type);
```

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente do Teste:** Node puro, `pnpm --filter @plataforma/core test`. Usar `better-sqlite3` nos testes (já é isomórfico na interface StoragePort).
- [x] **Fora de Escopo:** WebCrypto/assinaturas, UCAN, linhagem.

Casos de teste (numerados):
1. `migrateSchema(mockStorage)` chama `storage.migrate()` com o array MIGRATIONS.
2. Rodar DDL contra SQLite real (`better-sqlite3`): tabelas `nodes` e `edges` criadas.
3. INSERT em `nodes` com campos obrigatórios funciona, SELECT retorna a linha.
4. INSERT em `edges` com FK para `nodes` existente funciona.
5. INSERT em `edges` com FK inexistente → erro de constraint.
6. `migrateSchema` é idempotente (rodar 2× não quebra).
7. Índices (`idx_nodes_type`, `idx_edges_source`, `idx_edges_target`, `idx_edges_type`) existem.
