## 1. Objetivo
Implementar `RelayTrustModel` no pacote `@plataforma/transport`: score local não-transitivo por relay (uptime/latência/descartes), janela deslizante com histerese e shadowban silencioso
*(derivado de `docs/plano-de-implementacao.md` §5 T-207)*.

### Contratos exatos
```ts
// --- packages/transport/src/relayTrustModel.ts ---
import { PeerId, ClockPort } from '@plataforma/protocol';

export interface RelayScore {
  uptimeFraction: number;    // 0–1
  avgLatencyMs: number;      // EWMA
  discardRate: number;       // descartes/total na janela
  lastBumpMs: number;        // timestamp da última violação
}

export interface TrustModelOptions {
  windowSizeMs?: number;      // default 600000 (10 min)
  shadowbanThreshold?: number; // default 0.3 (30% discard rate)
  hysteresisMargin?: number;   // default 0.05 (5%)
  shadowbanDurationMs?: number; // default 86400000 (24h)
  ewmaAlpha?: number;          // default 0.2
}

export class RelayTrustModel {
  constructor(clock: ClockPort, opts?: TrustModelOptions);

  /** Registra relay de frame bem-sucedido. */
  recordRelay(peerId: PeerId, latencyMs: number): void;

  /** Registra descarte (frame inválido/malformado). */
  recordDiscard(peerId: PeerId): void;

  /** Retorna score atual. */
  getScore(peerId: PeerId): RelayScore | undefined;

  /** Verifica se peer está em shadowban. */
  isShadowBanned(peerId: PeerId): boolean;

  /** Remove peer do shadowban (após expiração ou manualmente). */
  liftShadowBan(peerId: PeerId): void;

  /** Remove peer do modelo. */
  forget(peerId: PeerId): void;
}
```
- Janela deslizante: registros expiram após `windowSizeMs`.
- Shadowban: entra se `discardRate` ≥ `shadowbanThreshold`; sai só após `shadowbanDurationMs` ou se `discardRate` < `shadowbanThreshold - hysteresisMargin`.
- Anti-falso-positivo: congestionamento transitório não bane.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/transport test`.

Casos de teste:
1. `recordRelay` + `getScore`: uptime e latência EWMA corretos.
2. `recordDiscard` com taxa baixa → não shadowban.
3. `recordDiscard` acima do threshold → shadowban ativado.
4. `recordDiscard` cai abaixo de `threshold - hysteresis` → shadowban mantido (histerese).
5. Após `shadowbanDurationMs` → shadowban expira (usar `VirtualClock`).
6. `forget` remove peer do modelo.
7. Janela deslizante: registros antigos expiram e não afetam score.
8. Congestionamento transitório (burst de descartes curto) → não bane se abaixo do threshold de tempo.
