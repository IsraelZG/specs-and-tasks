# T-701a · LinkCipher: encryptFrame/decryptFrame (AES-256-GCM)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar `encryptFrame`/`decryptFrame` no pacote `@plataforma/transport`: cifra/decifra frames com AES-256-GCM usando chaves de sessão. Nonce único por frame, authTag validado na decifra
*(extraído de T-701 §1; T-101b — AES-GCM)*.

### Contratos exatos
```ts
// --- packages/transport/src/crypto/LinkCipher.ts ---
import { AesKey } from '@plataforma/crypto'; // T-101b

/** Cifra frame: [nonce:12B] [ciphertext] [authTag:16B]. Nonce aleatório por chamada. */
export function encryptFrame(sessionKey: AesKey, plaintext: Uint8Array): Promise<Uint8Array>;

/** Decifra frame. Lança se authTag inválido. */
export function decryptFrame(sessionKey: AesKey, ciphertext: Uint8Array): Promise<Uint8Array>;
```

Casos de teste:
1. `encryptFrame` + `decryptFrame` roundtrip.
2. `decryptFrame` com ciphertext adulterado → erro.
3. `decryptFrame` com chave errada → erro.
4. 2 frames consecutivos usam nonces diferentes.
5. Payload vazio → roundtrip ok.
