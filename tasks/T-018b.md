# T-018b · Testes Tier A batch 2 — 8 componentes interativos

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo

## 1. Objetivo
Escrever testes unitários para **8 componentes interativos** do `@plataforma/design-system`, cobrindo os critérios do Tier A (smoke, interação real, disabled, a11y role, ref forwarding).
*(Derivado de T-018 — subdivisão para execução por modelos haiku)*.

### Componentes cobertos por esta task
1. `Pagination`
2. `RadioGroup`
3. `Select`
4. `Slider`
5. `Table`
6. `Textarea`
7. `Toggle`
8. `ToggleGroup`

### Critérios de aceite (Tier A — interação)
1. **Smoke:** renderiza sem erro com props mínimas obrigatórias.
2. **Interação real:** a ação primária dispara o callback público esperado com o valor correto.
3. **Estado `disabled`:** interação não dispara o callback.
4. **A11y:** role ARIA resolvível via `getByRole`; nome acessível presente.
5. **Ref forwarding:** se `React.forwardRef`, ref chega ao nó DOM esperado.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - NÃO copie o mesmo `it()` para todos sem ler o componente real.
> - NÃO corrija bugs de produto — documente na Seção 6.
> - NÃO crie `Sidebar.test.tsx` (T-017).
> - **Select:** tem portal — usar `screen.getByRole` para encontrar opções fora do container.
> - **Slider:** interação é via `userEvent.keyboard` ou `userEvent.click` no track — testar callback `onValueChange`.
> - **Table:** componente estrutural composto — verificar roles `table`, `rowgroup`, `columnheader`, `row`, `cell`.

### Pegadinhas conhecidas
- `Checkbox.test.tsx`: padrão para testar `indeterminate` e callback ref + `useRef`.
- `Modal.test.tsx`: `flushEnterAnimation` — `Select` pode precisar (portal + animação).
- `Toggle` e `ToggleGroup`: testar `aria-pressed` no estado pressed/unpressed.

1. Para cada componente: leia `<Nome>.tsx` + `<Nome>.metadata.ts`.
2. Escreva `tests/components/<Nome>.test.tsx` cobrindo os 5 critérios do Tier A.
3. Rode `pnpm --filter @plataforma/design-system test` após cada 2-3 componentes.
