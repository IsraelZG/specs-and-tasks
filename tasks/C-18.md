**i2 (INFO — não-bloqueante) · i5 do EST-03d é fix parcial, não total:**

alinhamento total com i5 da spec, a refactor para metadado é uma mudança de design (provavelmente
≥ complexity 2, escopo para uma task dedicada). O Handover diz `fixed` mas é **parcial** — não há
mentira mas a literal "fixed" sugere completude que não houve. Track: nota para o arquiteto (não
bloqueia merge, é i5 não-m1).

**i3 (INFO — não-bloqueante) · Duplicação `VALID_STATUSES` em `service.ts:11-17` vs `stateMachine.ts:12-25`:**

union `TaskStatus` no schema (EST-03a re-endurecimento com `demote`?), `service.ts` precisa ser
atualizado manualmente também. Refactor natural: derivar `VALID_STATUSES` de `Object.keys(TRANSITIONS)`
— 1 linha. Worker não fez; a duplicação é inofensiva no estado atual. Track: pode ser absorvido
no re-endurecimento de EST-loader (próxima task lógica).
