## 1. Objetivo
Implementar integração do Automerge Repo no pacote `@plataforma/transport`: estritamente para rendezvous + ephemeral messages (RAM, sem histórico); ponte com o adapter WebRTC; sem introduzir Y.js
*(derivado de `docs/plano-de-implementacao.md` §7 T-403)*.

### Contratos exatos *(revisão arquiteto 2026-07-12: integração REAL do automerge-repo — o contrato anterior era satisfazível por um pub/sub em RAM sem nenhuma API do Automerge)*
```ts
// --- packages/transport/src/automerge/automergeShell.ts ---
// Dependência NOVA em packages/transport/package.json (única fonte de CRDT, ADR-001):
//   "@automerge/automerge-repo": "^2.5.6"   // embute @automerge/automerge 2.2.8–3; verificado npm 2026-07-12
import { Repo, NetworkAdapter, stringifyAutomergeUrl } from '@automerge/automerge-repo';
import type {
  DocHandle, DocHandleEphemeralMessagePayload,
  PeerId as AmPeerId, PeerMetadata, Message,
} from '@automerge/automerge-repo';
import type { ClockPort } from '@plataforma/protocol'; // T-003

/**
 * PONTE (revisão F2): o adapter NÃO acopla no NetworkAdapterPort cru — acopla em LANES
 * AUTENTICADOS pós-handshake Noise_XX (T-202). Superfícies reais existentes:
 *   - initiateNoiseXX(adapter, localKey, ...) → NoiseHandshakeResult   (noiseHandshake.ts:370)
 *   - acceptNoiseXX(adapter, opts) → NoiseServer.accept()              (noiseServer.ts:209)
 *   - NoiseHandshakeResult = { remoteDeviceId, send(data), receive(): AsyncIterable<Uint8Array> }
 *     — send cifra com ChaCha20-Poly1305 por cima do adapter (noiseHandshake.ts:354).
 * O payload da casca passa SEMPRE dentro do canal Noise. SimNetwork em teste implementa a
 * mesma interface de lane em claro (fake inline no teste — sem API test-only em produção).
 */
import type { NoiseHandshakeResult } from '../noiseHandshake.js'; // T-202
import { encodeFrame, decodeFrame, FrameType, MAX_FRAME_LENGTH } from '@plataforma/protocol'; // T-201
import type { WireFrame } from '@plataforma/protocol';

/** Lane autenticado: par (peer, canal seguro) já estabelecido pelo Noise. */
export interface NoiseLane {
  peerId: string;   // = NoiseHandshakeResult.remoteDeviceId (identidade autenticada, NUNCA claim do payload)
  channel: Pick<NoiseHandshakeResult, 'send' | 'receive'>;
}

/**
 * FRAMING (dentro do lane, lado claro do Noise) — wire format v1 (T-201, wireFormat.ts):
 *   encodeFrame(FrameType.EPHEMERAL /* 0x05, classe volátil *\/, { ch: 'am-repo/v1', msg })
 *   - `msg` é o Message do automerge-repo serializado direto em MessagePack (codec T-201;
 *     campos verificados em messages.ts: type, senderId, targetId, documentId?, data?,
 *     e no ephemeral: count, sessionId). SEM CBOR duplo — a serialização é do adapter.
 *   - Demux no receive: decodeFrame (nunca lança); {ok:false} → drop+contador;
 *     EPHEMERAL com ch !== 'am-repo/v1' (ex.: 'private-swarm/v1' do T-702) e demais
 *     FrameTypes → entregues a `onOtherFrame` (coexistência; futuro roteador RBSR pluga aí).
 *   - Teto: MAX_FRAME_LENGTH (1 MiB, T-201). Mensagem maior → erro no send, drop no receive
 *     (reason 'too_large'). Sem fragmentação nesta task (teto documentado).
 */
export class PortNetworkAdapter extends NetworkAdapter {
  constructor(localPeerId: AmPeerId, opts?: { onOtherFrame?: (from: string, frame: WireFrame) => void });
  /** Anexa lane autenticado; emite 'peer-candidate' {peerId}. Retorno destaca o lane
   *  (emite 'peer-disconnected'). Fim do AsyncIterable receive ⇒ detach automático. */
  attachLane(lane: NoiseLane): () => void;
  // Contrato abstrato do automerge-repo (NetworkAdapter.ts, verificado):
  isReady(): boolean;
  whenReady(): Promise<void>;
  connect(peerId: AmPeerId, peerMetadata?: PeerMetadata): void;
  send(message: Message): void;  // roteia por message.targetId → lane correspondente; sem lane → drop silencioso (peer caiu)
  disconnect(): void;
}

export interface AutomergeDocId {
  /** AutomergeUrl válida. Derivação: BinaryDocumentId = 16 primeiros bytes do
   *  RendezvousId (= SHA-256(rendezvous_secret ‖ ASSET:PERMISSION_ID), caderno-2 §2)
   *  → stringifyAutomergeUrl(binaryDocumentId). Determinística e não-enumerável. */
  docId: string;
  scope: 'rendezvous' | 'ephemeral';
}

export interface AutomergeShellConfig {
  clock: ClockPort;             // TTL testável com VirtualClock (T-003)
  localPeerId: string;
  maxPeersPerDoc?: number;      // default 50
  ttlMs?: number;               // TTL de OCIOSIDADE (default 300000 = 5min) — semântica abaixo
  onOtherFrame?: (from: string, frame: WireFrame) => void; // repassado ao PortNetworkAdapter
}
// A casca NÃO recebe NetworkAdapterPort: recebe lanes autenticados via attachLane
// (o runtime — sync-worker, task futura — faz adapter → Noise → shell.attachLane).
// SEGURANÇA POR CONSTRUÇÃO: só bytes vindos de lane anexado alcançam o Repo; a identidade
// do remetente é o peerId autenticado do lane. `Message.senderId` divergente do peerId do
// lane ⇒ frame descartado (spoof), teste 13.

export interface EphemeralMessage {
  docId: string;
  senderPeerId: string;         // = payload.senderId do evento 'ephemeral-message' do DocHandle
  payload: Uint8Array;
  timestamp: number;
}

export class AutomergeShell {
  /** Instancia UM Repo interno: new Repo({ network: [new PortNetworkAdapter(...)], peerId,
   *  sharePolicy: async () => true }) — SEM storage adapter: docs vivem só em RAM
   *  (é isso que garante "sem histórico persistido", não uma reimplementação). */
  constructor(config: AutomergeShellConfig);

  /** Anexa lane autenticado ao PortNetworkAdapter interno. Retorna detach. */
  attachLane(lane: NoiseLane): () => void;

  /** Cria/abre doc casca via repo (create/find no DocumentId derivado).
   *  REFCOUNT: cada openDoc incrementa; closeDoc decrementa. */
  openDoc(docId: AutomergeDocId): Promise<AutomergeDocId>;

  /** handle.broadcast(msg) do DocHandle do doc casca — canal ephemeral REAL do
   *  automerge-repo (CBOR + dedup de gossip), não um envelope próprio. */
  broadcastEphemeral(msg: EphemeralMessage): Promise<void>;

  /** Assina o evento 'ephemeral-message' ({handle, senderId, message}) de todos os
   *  docs abertos. Retorna unsubscribe idempotente (mesma convenção de NetworkAdapterPort). */
  onEphemeralMessage(handler: (msg: EphemeralMessage) => void): () => void;

  /** Decrementa refcount; em zero, agenda evicção por TTL. */
  closeDoc(docId: string): void;

  listOpenDocs(): AutomergeDocId[];
  peerCount(docId: string): number;
}
```
- **Semântica do TTL (lifecycle):** `ttlMs` é ociosidade — o timer zera a cada ephemeral
  enviado/recebido no doc. Doc **nunca** é evictado com refcount > 0: quem comita (T-602)
  mantém `openDoc` durante o ciclo inteiro, tornando "expiração durante commit" impossível
  por construção. Evicção = `handle.delete()` + remoção dos maps internos.
- **Pilha (fronteira de camadas):** `NetworkAdapterPort` (WebRTC T-402 / WS T-204 / SimNetwork)
  → handshake Noise_XX (T-202: `initiateNoiseXX` / `NoiseServer`) → **lane autenticado** →
  frame v1 `EPHEMERAL`+`ch:'am-repo/v1'` (T-201) → `PortNetworkAdapter` → `Repo`. A casca só
  enxerga lanes; nunca fala com o port cru, nunca inventa protocolo paralelo.
- Rendezvous docs: descoberta de peers (T-406) e sala de signaling.
- Ephemeral messages: broadcast que NÃO persiste no grafo (só RAM).
- Nenhuma dependência de Y.js. Automerge Repo como trilha única (ADR-001).
- **O doc casca nunca sofre `change()`** — é sala de encontro. O doc VIVO de conteúdo
  (para `Automerge.save()` do T-602) NÃO mora aqui: pertence ao CommitCycle (ver T-602 §1).

## 4. Estratégia de Testes
- [x] **Framework:** Vitest + SimNetwork (T-005).
- [x] **Ambiente:** `pnpm --filter @plataforma/transport test`.

Casos de teste:
1. `openDoc` cria documento rendezvous → docId é AutomergeUrl válida (`isValidAutomergeUrl`) e determinística (mesmo RendezvousId → mesma URL nos dois peers).
2. `broadcastEphemeral` envia mensagem para 3 peers conectados.
3. `onEphemeralMessage` handler recebe mensagem com payload correto e `senderPeerId` = senderId do evento `'ephemeral-message'` do automerge-repo.
4. `closeDoc` libera recursos; `listOpenDocs` não contém o doc fechado.
5. `peerCount` reflete número correto de peers no documento.
6. Ephemeral broadcast entre 3 peers: todas as mensagens entregues.
7. Nada persiste no grafo: verificar que `nodes`/`edges` não recebem dados de ephemeral.
8. TTL de ociosidade expira (VirtualClock) → doc SEM refcount removido da RAM; atividade ephemeral zera o timer.
9. **Anti-fake:** doc com refcount > 0 NUNCA expira, mesmo após N× ttlMs (VirtualClock) — pré-condição do ciclo de commit de T-602.
10. **Anti-fake:** o caminho de mensagem atravessa o automerge-repo de verdade — dois `AutomergeShell` com lanes fake (in-memory, mesma interface `NoiseLane`) trocam ephemeral e o teste assevera que o `PortNetworkAdapter` emitiu `'peer-candidate'`/`'message'` (spy no adapter). Um pub/sub próprio que não instancia `Repo` reprova este caso.

Casos adversariais (ponte/framing — F2):
11. **Frame malformado:** bytes aleatórios e frame truncado no lane → `decodeFrame` retorna `{ok:false}`, frame descartado, contador incrementa, `Repo` intocado, lane continua vivo.
12. **Canal errado:** frame `EPHEMERAL` com `ch: 'private-swarm/v1'` no mesmo lane → NÃO chega ao repo; entregue a `onOtherFrame`; tráfego `am-repo/v1` simultâneo segue normal (coexistência).
13. **Sender spoofado:** frame válido com `msg.senderId ≠ peerId` autenticado do lane → descartado (a identidade é a do Noise, nunca a do payload).
14. **Peer não autenticado:** bytes injetados por fora de qualquer lane anexado (ex.: direto no port cru do teste) → jamais alcançam o repo (segurança por construção — só `attachLane` liga fonte de bytes).
15. **Oversize:** mensagem > `MAX_FRAME_LENGTH` (1 MiB) → `send` rejeita com erro; no receive, `{ok:false, reason:'too_large'}` → drop sem crash.

## 6. Feedback de Especificação
- **[Recon arquiteto 2026-07-12, claude-fable]** O contrato original era satisfazível por um
  broadcast em RAM sem nenhuma API do Automerge (nenhum símbolo do pacote citado, nenhum acesso
  a documento, nenhuma dependência fixada; os 8 casos de teste passavam com `Map` + pub/sub).
  Contrato revisado para exigir integração real: `@automerge/automerge-repo@^2.5.6`,
  `PortNetworkAdapter extends NetworkAdapter`, ephemeral via `DocHandle.broadcast` /
  `'ephemeral-message'`, TTL de ociosidade com refcount, casos anti-fake 9–10. Todas as APIs
  citadas foram verificadas na fonte (npm registry + automerge-repo main) — nada inventado.
- **Extensão deferida (gatilho, não YAGNI-esquecida):** `getDocHandle(docId): DocHandle<T>` NÃO
  entra nesta task — o cluster T-602/T-603 não precisa (o doc vivo mora no CommitCycle). Ativar
  quando `T-MOD-03` ou `T-OFF-02` forem endurecidas (sessão/perfil como doc Automerge real).
- **S-05 (plano §565) nunca virou task:** o spike de memória em sessões longas segue pendente —
  ver pergunta humana no recon (Docs, sessão 2026-07-12).
- **[F2 2026-07-12, claude-fable — revisão de arquiteto sobre a ponte.]** A F1 dizia "bytes CBOR
  do repo → port.send" sem codec, demux, segurança nem limites. F2 fixa: acoplagem em **lanes
  Noise autenticados** (T-202 — `NoiseHandshakeResult`/`NoiseServer`, únicos artefatos de canal
  seguro existentes), framing **wire v1** (T-201 — `FrameType.EPHEMERAL` já reservado, MessagePack
  direto do `Message`, sem CBOR duplo), demux por `ch:'am-repo/v1'` com escape `onOtherFrame`
  (coexistência com `private-swarm/v1` do T-702), teto 1 MiB sem fragmentação, e 5 casos
  adversariais (11–15). Nenhuma porta nova em protocol; nenhum protocolo paralelo.
