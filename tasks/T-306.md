# T-306 · Ondas 0–2

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar o protocolo de ondas de sincronização no pacote `@plataforma/protocol`: Onda 0 (anti-entropy O(1) + catch-up de identidades), Onda 1 (cabeçalhos críticos/tela ativa com priorização por escopo), Onda 2 (histórico podado em background), com máscara de ranges conhecidos-divergentes-mas-adiados
*(derivado de `docs/plano-de-implementacao.md` §6 T-306)*.

### Contratos exatos
```ts
// --- packages/protocol/src/rbsr/waves.ts ---
import { RBSRBTree, Fingerprint } from './BTree.js'; // T-301a
import { RBSRSession, RBSRState } from './exchange.js'; // T-302a
import { NetworkAdapterPort } from '../ports.js'; // T-004

export enum WaveKind {
  WAVE_0 = 0,   // anti-entropy O(1): root exchange + catch-up de identidades (T-203)
  WAVE_1 = 1,   // cabeçalhos críticos / tela ativa (priorização por escopo)
  WAVE_2 = 2,   // histórico podado em background
}

export interface WaveConfig {
  kind: WaveKind;
  priorityScope?: string[];     // entity_ids prioritários (WAVE_1)
  maxEntities?: number;         // limite de entidades por onda (WAVE_2 default=100)
  deferOnDivergence?: boolean;  // adiar ranges divergentes para onda futura
}

export interface DeferredRange {
  rangeStartId: string;
  rangeEndId: string;
  deferReason: string;
  deferredAtWave: WaveKind;
}

export interface WaveResult {
  wave: WaveKind;
  entitiesSynced: number;
  deferredRanges: DeferredRange[];
  completed: boolean;
}

export interface WaveCoordinator {
  /** Executa Onda 0: troca de root fingerprint. */
  executeWave0(session: RBSRSession): Promise<WaveResult>;

  /** Executa Onda 1: reconciliação de cabeçalhos prioritários. */
  executeWave1(session: RBSRSession, config: WaveConfig): Promise<WaveResult>;

  /** Executa Onda 2: histórico podado em background. */
  executeWave2(session: RBSRSession, config: WaveConfig): Promise<WaveResult>;

  /** Verifica se estado é idêntico (encerra em O(1)). */
  isIdempotent(session: RBSRSession): Promise<boolean>;

  /** Retorna ranges adiados pendentes. */
  getDeferredRanges(): DeferredRange[];

  /** Limpa ranges adiados quando escopo é aberto. */
  clearDeferredForScope(scopeEntityIds: string[]): void;
}
```
- Onda 0: envia root fingerprint; se igual, DONE (anti-entropy O(1)). Inclui catch-up de identidades (`identity_epoch_index` divergente → pipeline de catch-up do T-203).
- Onda 1: reconciliação de cabeçalhos críticos e da tela ativa, priorizando escopo declarado.
- Onda 2: reconciliação de histórico podado, em background, com limite de entidades.
- Máscara de defer: ranges divergentes são marcados como "adiados" e não re-disparados em resumes subsequentes até abertura do escopo.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest + SimNetwork (T-005).
- [x] **Ambiente:** `pnpm --filter @plataforma/protocol test`.

Casos de teste:
1. Onda 0: peers com estado idêntico → `isIdempotent` true, encerra em O(1).
2. Onda 0: root fingerprints divergem → avança para Onda 1.
3. Onda 1: reconcilia cabeçalhos prioritários do escopo declarado.
4. Onda 2: reconcilia histórico em background sem bloquear UI/novas operações.
5. Peers divergentes: após sync completo, novo resume → Onda 0 retorna idempotente.
6. Ranges divergentes adiados → não re-processados no resume seguinte.
7. Abertura de escopo → limpa ranges adiados daquele escopo.
8. Três ondas em sequência com divergência plantada → convergência completa.
