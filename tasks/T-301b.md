# T-301b · RBSR B-Tree: compareRange + diff

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** haiku

## 1. Objetivo
Implementar `compareRange` no pacote `@plataforma/protocol`: compara ranges entre duas RBSRBTrees usando XOR agregado para detectar divergências (onlyLocal, onlyRemote, conflict) sem transferir todos os dados. O(n log n) no pior caso, O(1) no caso feliz
*(extraído de T-301 §1)*.

### Contratos exatos
```ts
// --- packages/protocol/src/rbsr/compare.ts ---
import { RBSRBTree, Fingerprint } from './BTree.js'; // T-301a
import { ULID } from '@plataforma/core'; // T-102

export interface RangeDiff {
  onlyLocal: ULID[];
  onlyRemote: ULID[];
  conflict: ULID[]; // mesmo id, fingerprint diferente
}

/** Compara ranges de duas árvores e retorna diff. */
export function compareRange(
  local: RBSRBTree,
  remote: RBSRBTree,
): RangeDiff;

/** Compara fingerprint de um range específico. */
export function compareFingerprint(
  local: RBSRBTree,
  remoteFingerprint: Fingerprint,
  startId: ULID,
  endId: ULID,
): 'equal' | 'divergent';
```

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).

Casos de teste:
1. Duas árvores idênticas → `onlyLocal` e `onlyRemote` vazios.
2. Local tem 3 ids extras → `onlyLocal` contém os 3 ids.
3. Remote tem ids que local não tem → `onlyRemote` preenchido.
4. Mesmo id com fingerprint diferente → `conflict` preenchido.
5. Árvore vazia vs árvore populada → todos os ids no diff.
6. `compareFingerprint`: range idêntico → `'equal'`; divergente → `'divergent'`.

## 5. Instruções de Execução
1. Escreva testes com duas RBSRBTrees populadas.
2. Implemente `compareRange` (walk recursivo, compara XOR por sub-árvore, desce onde diverge).
3. Implemente `compareFingerprint`.
4. Rode build + test.
