# T-305b · Sync UCAN: canAccess (verificação pontual)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** haiku

## 1. Objetivo
Implementar `canAccess` no pacote `@plataforma/protocol`: verifica se um UCAN autoriza acesso a um entity_id + ação específica, considerando expiração
*(extraído de T-305 §1; T-501b — verifyUcan)*.

### Contratos exatos
```ts
// --- packages/protocol/src/auth/ucanScope.ts (extensão) ---
import { SignedUcan } from '@plataforma/core'; // T-501a

/** Verifica se UCAN autoriza ação sobre entity_id. */
export function canAccess(ucan: SignedUcan, entityId: string, action: string): boolean;
```
- Verifica se `ucan.payload.att` contém `{with: 'entity:<entityId>', can: action}` ou padrão mais amplo.
- UCAN expirado → false.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).

Casos de teste:
1. UCAN com `entity:A` + `can:READ` → `canAccess(ucan, 'A', 'READ')` → true.
2. UCAN com `entity:A` + `can:READ` → `canAccess(ucan, 'B', 'READ')` → false.
3. UCAN com `can:READ` → `canAccess(ucan, 'A', 'WRITE')` → false.
4. UCAN expirado → false (via T-501b).
