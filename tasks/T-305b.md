## 1. Objetivo
Implementar `canAccess` no pacote `@plataforma/protocol`: verifica se um UCAN autoriza acesso a um `entity_id` + `ability` específica, considerando expiração
*(extraído de T-305 §1; T-305a — `extractEntityIds`; T-501 — `SignedUcan`, `validateUcan`)*.

### Contratos exatos (assinaturas TS fixadas, derivadas de T-305a e T-501)

```ts
// --- packages/protocol/src/auth/ucanScope.ts (extensão de T-305a) ---
// Reutiliza o tipo local SignedUcan (subset de @plataforma/core) definido em T-305a
// — importar de @plataforma/core causaria dep circular (core → protocol).

/** Resultado detalhado de canAccess — informa o motivo da negação para diagnóstico. */
export type CanAccessResult =
  | { ok: true }
  | { ok: false; reason: 'no_matching_capability' | 'expired' };

/**
 * Verifica se `ucan` autoriza `ability` sobre `entityId`.
 * - Match: existe capability com `resource: 'entity:<entityId>'` E `ability === ability`.
 * - Expiração: `ucan.payload.exp > BigInt(Date.now())` (exp em ms epoch).
 *   ATENÇÃO: valida só o `exp` — NÃO verifica assinatura/chains (responsabilidade
 *   de T-501 `validateUcan`); esta função assume UCAN já validado estruturalmente.
 *
 * @returns `CanAccessResult` (não boolean) para que o caller saiba distinguir
 *   "UCAN expirado" de "UCAN não tem essa capability" — semântica relevante para
 *   SyncWorker (retry vs buscar outro UCAN).
 */
export function canAccess(
  ucan: SignedUcan,
  entityId: string,
  ability: string,
  now?: bigint,
): CanAccessResult;
```

> **Por que `CanAccessResult` (não `boolean`):** spec T-305a §1 já usa
> `extractEntityIds: () => string[]` (retorna dados, não boolean). Mesma filosofia.
> Caller pode ignorar o `reason` se não precisar, ou mostrar "UCAN expirado" no
> diagnóstico sem novo `validateUcan`.

> **Por que `now?: bigint` opcional:** default = `BigInt(Date.now())` (produção);
> injetável para test determinístico (VirtualClock do `@plataforma/testkit`,
> padrão do pacote).

## 4. Estratégia de Testes Estrita
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/protocol test`.

Casos (numerados):
1. **Match exato:** UCAN com `entity:A` + `ability:READ` → `canAccess(ucan, 'A', 'READ')` → `{ok: true}`.
2. **Entity mismatch:** UCAN com `entity:A` + `ability:READ` → `canAccess(ucan, 'B', 'READ')` → `{ok: false, reason: 'no_matching_capability'}`.
3. **Ability mismatch:** UCAN com `ability:READ` → `canAccess(ucan, 'A', 'WRITE')` → `{ok: false, reason: 'no_matching_capability'}`.
4. **Expirado:** UCAN com `exp < now` → `canAccess(ucan, 'A', 'READ')` → `{ok: false, reason: 'expired'}`.
5. **Válido no limite:** UCAN com `exp === now` → `{ok: false, reason: 'expired'}` (estritamente `>`).
6. **Múltiplas capabilities:** UCAN com `entity:A:READ` + `entity:B:WRITE` → match em ambas via `canAccess`.
7. **`now` injetado:** `canAccess(ucan, 'A', 'READ', 5_000n)` quando `exp === 4_999n` → `expired`; quando `exp === 5_001n` → `ok`.
8. **Capability sem prefixo `entity:`:** UCAN com `resource: 'sync/global', ability: 'READ'` → `canAccess(ucan, 'A', 'READ')` → `no_matching_capability` (escopo A não está no escopo).
9. **Defensive (vide m1 de T-305a):** capability com `resource: 'entity:'` (ID vazio) → `canAccess(ucan, '', 'READ')` → `no_matching_capability` (não crasha).
