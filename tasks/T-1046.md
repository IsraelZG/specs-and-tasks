## 1. Objetivo
Completar a **transferência do payload real dos nós** no RBSR — hoje o protocolo só troca fingerprints (esqueleto de reconciliação) e nunca transfere `SignedNode`, deixando `applyNodes` desligado do caminho de exchange.

> **ORIGEM (decisão do arquiteto, 2026-07-02, T-1032).** Achado [m2]: `respondNodes` (`packages/protocol/src/rbsr/exchange.ts:204`) envia `{id, fingerprint}` mas `applyNodes` espera `readonly SignedNode[]`; `reconcile` insere fingerprints na árvore local, não dados. Ou seja, o RBSR compara o que difere mas **não transfere o conteúdo divergente**. T-1032 (enforcement UCAN da exposição de fingerprints) marcou explicitamente esta conclusão como **fora do seu escopo** — é funcionalidade, não privacidade. Esta task a completa.

**Meta:** após o `reconcile` identificar os ids divergentes, o responder envia os **`SignedNode` completos** (não fingerprints) desses ids, o requisitante os aplica via `applyNodes`, e **ambos os lados aplicam o mesmo padrão de autz servidor-side** que T-1032 estabeleceu (validar UCAN + `canAccess` pontual sobre o escopo antes de servir qualquer `SignedNode`). Um peer sem UCAN sobre o subgrafo não recebe nem os fingerprints (T-1032) nem os nós (esta task).

**Casos (numerados):**
1. **Transferência aplica nós divergentes.** Responder com `GraphStorePort` populado; requisitante pede N ids divergentes → recebe `SignedNode[]`, `applyNodes` retorna `{imported: N, rejected: 0}`, e o `localTree` do requisitante passa a conter os N fingerprints.
2. **Nó com assinatura inválida é rejeitado.** 1 dos N nós vem com `signature` adulterada → `applyNodes` retorna `{imported: N-1, rejected: 1}`; nó inválido não entra no storage.
3. **Peer sem UCAN sobre o escopo não recebe nós.** Requisição sem UCAN (ou UCAN sem capability sobre os ids) → responder recusa antes de servir (`canAccess` → `{ok:false}`), requisitante recebe resposta vazia. UCAN viaja por-sessão (D1 resolvida: estabelecido no `startSession`).
4. **UCAN expirado é recusado.** `canAccess(ucan, id, ability, now)` com `exp <= now` → `{ok:false, reason:'expired'}`, zero nós servidos.
5. **Lista vazia / nenhum id divergente.** `reconcile` com árvores iguais → nenhum request de nós, `applyNodes` não chamado.

> Não afeta UI (protocol/transport puro) — `ui: false`.
