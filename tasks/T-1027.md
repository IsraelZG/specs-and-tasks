## 1. Objetivo
Criar um comando único `pnpm nexus` (= `node tools/scripts/nexus-start.mjs`) que, de forma
**idempotente**, sobe no início do dia de trabalho:
1. **Uma** instância standing do **proxy Headroom no Windows** (porta 8787, bind `0.0.0.0`) — nunca
   uma segunda, e **nunca** no WSL. Desacopla o Headroom do ciclo de vida do opencode (acabou o
   `headroom wrap`: o opencode passa a só **apontar** pro proxy).
2. O **backend Nexus** no Windows (`tsx watch`, porta 3001), com `HEADROOM_URL=http://127.0.0.1:8787`.
3. O **frontend Nexus** no Windows (`vite`, porta 5173).

E um `pnpm nexus:stop` (= `node tools/scripts/nexus-stop.mjs`) que derruba backend+frontend (e o
Headroom só com a flag `--headroom`).

**Fora de escopo (vira task própria — Fase 1 do runner):** fazer o backend **spawnar opencode
autonomamente** pra rodar tasks. Isto aqui só prepara a infra standing que o runner vai usar.

## 3. Escopo de Arquivos (Inputs e Outputs) — contratos FIXOS

### **[CREATE]** `tools/scripts/nexus-start.mjs`
ES module Node. Sem dependências externas novas (só `node:child_process`, `node:fs`, `node:path`,
`node:http`/`fetch` global do Node 20). `repoRoot` = duas pastas acima de `tools/scripts/`.
Diretório de runtime: `<repoRoot>/.nexus/run/` (criar com `recursive:true`).

Comportamento, **nesta ordem**:

1. **Pré-requisito Headroom no Windows.** Resolva o `headroom.exe`: `where headroom` (PATH); se não
   achar, caia pro caminho conhecido `path.join(process.env.APPDATA, 'Python','Python313','Scripts','headroom.exe')`.
   Se nenhum existir → imprima a instrução e **saia com código 2** (NÃO auto-instale):
   ```
   headroom.exe não encontrado no Windows. Instale com:
       python -m pip install "headroom-ai[all]"
   ```
2. **Garanta o proxy Headroom no Windows (idempotente — instância única).**
   - Health check: `GET http://127.0.0.1:8787/stats` (fetch, timeout 2s).
   - Se **== 200**: logue `Headroom já rodando (Windows:8787)` e siga (NÃO suba outro).
   - Senão, suba **detached** no Windows: `spawn(headroomExe, ['proxy','--host','0.0.0.0','--port','8787','--openai-api-url','https://api.deepinfra.com/v1/openai','--no-subscription-tracking'], { detached:true, stdio:['ignore', fd, fd] })` com `fd` = append em `.nexus/run/headroom.log`. `child.unref()`. Grave o PID em `.nexus/run/headroom.pid`.
   - Depois **poll** o health check a cada 1s até 200 (timeout 25s; se estourar → erro + exit 1).
3. **`HEADROOM_URL` para o backend = `http://127.0.0.1:8787`** (backend e proxy no mesmo host Windows — trivial, sem resolução de IP).
4. **Suba o backend (detached).** `spawn('pnpm', ['--filter','nexus-backend','dev'], { cwd: repoRoot, detached:true, shell:true, env:{...process.env, HEADROOM_URL, PORT:'3001'}, stdio:['ignore', fd, fd] })` com `fd` = append em `.nexus/run/backend.log`. `child.unref()`. Grave o PID em `.nexus/run/backend.pid`.
5. **Suba o frontend (detached).** Igual, `['--filter','nexus-frontend','dev']`, log `.nexus/run/frontend.log`, PID `.nexus/run/frontend.pid`.
6. **Health-checks finais** (poll, timeout 30s cada): backend `GET http://127.0.0.1:3001/health` → 200; frontend `GET http://127.0.0.1:5173/` → 200 (ou qualquer 2xx/3xx do Vite).
7. **Imprima um resumo** e **retorne (exit 0)** — NÃO bloqueie o terminal (os 3 processos seguem detached):
   ```
   ✅ nexus up
      Headroom : http://127.0.0.1:8787  (Windows)
      Backend  : http://127.0.0.1:3001  (HEADROOM_URL=http://127.0.0.1:8787)
      Frontend : http://127.0.0.1:5173
      Logs     : .nexus/run/*.log    | Parar: pnpm nexus:stop
   ```
- **Idempotência total:** rodar `nexus start` 2× **não** sobe um 2º Headroom nem 2 backends. Para
  cada processo (headroom/backend/frontend), se houver `*.pid` com processo vivo (no Windows:
  `tasklist /FI "PID eq <pid>"`), **pule** o spawn e logue "já rodando".

### **[CREATE]** `tools/scripts/nexus-stop.mjs`
- Lê `.nexus/run/backend.pid` e `.nexus/run/frontend.pid`; para cada PID vivo, mata a **árvore** no
  Windows: `spawnSync('taskkill', ['/PID', pid, '/T', '/F'])`. Remove os `.pid`.
- Só se `--headroom` for passado: derruba o proxy (Windows) via `.nexus/run/headroom.pid`
  (`taskkill /PID <pid> /T /F`) e remove o `.pid`. Sem a flag, **deixa o Headroom de pé** (default).
- Imprime o que parou. Idempotente (no-op se nada estiver rodando).
