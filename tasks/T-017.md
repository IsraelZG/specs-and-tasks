# T-017 · Reset local do peer

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** haiku

## 1. Objetivo
Implementar função `resetPeer()` no pacote `@plataforma/core`: limpa todas as tabelas do SQLite (`nodes`, `edges`, `entity_heads`) e esvazia o KeyVault, restaurando o peer ao estado pós-bootstrap sem perder identidade (chave do dispositivo preservada)
*(derivado de `docs/plano-de-implementacao.md` M0/E0.3 — T-017; realocado de T-009 por colisão de ID — `tasks/_correlacao-plano.md` §1)*.

### Contratos exatos
```ts
// --- packages/core/src/reset.ts ---
import { StoragePort } from '@plataforma/protocol'; // T-004
import { KeyVault } from './keyVault.js'; // T-110

/** Limpa dados locais mantendo identidade do dispositivo. */
export async function resetPeer(storage: StoragePort, vault: KeyVault): Promise<void>;
```
- `storage`: `DELETE FROM edges; DELETE FROM nodes; DELETE FROM entity_heads;`
- `vault`: remove todas as entradas (`purge`).
- Chave do dispositivo (Ed25519) NÃO é afetada — vive em memória/secure storage, não no SQLite.
