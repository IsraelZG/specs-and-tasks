## 1. Objetivo
Implementar aba "Auth" no app Bancada (T-008): inspetor de UCANs (cadeia, escopo, TTL), épocas por escopo, lista BLOCKS, simular revogação/rotação, fluxos de recuperação acionáveis
*(derivado de `docs/plano-de-implementacao.md` §8 T-512)*.

### Contratos
```tsx
// --- apps/bancada/src/tabs/AuthTab.tsx ---
import { SignedUcan } from '@plataforma/core'; // T-501a (packages/core/src/index.ts)
import { KeyVault } from '@plataforma/core'; // T-110 (packages/core/src/index.ts)

export interface AuthTabProps {
  ucans: SignedUcan[];           // UCANs ativos
  keyVault: KeyVault;            // Key Vault para consulta de épocas
  onRevoke: (ucanId: string) => void;
  onRotateEpoch: (scopeId: string) => void;
  onRecover: (method: 'email' | 'shamir' | 'sovereign') => void;
}

export function AuthTab(props: AuthTabProps): JSX.Element;
```
- Inspetor de UCANs: lista cada UCAN com cadeia de delegação (expansível), escopo, TTL restante.
- Épocas por escopo: tabela de escopos com `current_epoch_index` e data da última rotação (ver decisão aberta sobre KeyVault).
- Lista BLOCKS: tabela de peers/entidades bloqueadas pelo usuário (ver decisão aberta sobre blocks).
- Botões: "Simular revogação" → chama `onRevoke`, "Simular rotação de época" → chama `onRotateEpoch`.
- Fluxos de recuperação: botões para iniciar recovery por email, Shamir ou soberano.

## 6. Feedback de Especificação

> **⚠️ DECISÃO EM ABERTO #1: Fonte de dados para a lista de BLOCKS na AuthTab**
> A especificação exige renderizar a "Lista BLOCKS (tabela de peers/entidades bloqueadas pelo usuário)", mas a interface `AuthTabProps` não recebe nenhuma propriedade com esses dados, nem há referências de onde no banco SQLite ou no core recuperá-los.
> **Opções recomendadas:**
> - **Opção (a):** Estender `AuthTabProps` com a propriedade `blockedPeers: string[]` ou similar.
> - **Opção (b):** Importar a porta de storage e consultar a tabela de bloqueios diretamente.
>
> **⚠️ DECISÃO EM ABERTO #2: Exposição de escopos/épocas a partir do KeyVault**
> O `KeyVault` in-memory em `@plataforma/core` não expõe métodos públicos para listar os escopos ativos ou ler seus índices de época (todas as entradas ficam no `private entries: Map<string, VaultEntry>`). Portanto, a `AuthTab` não consegue preencher a tabela de épocas recebendo apenas o `keyVault` como prop.
> **Opções recomendadas:**
> - **Opção (a):** Estender `KeyVault` com um método público `getEntries(): { scope: string; expiresAt: number }[]`.
> - **Opção (b):** Passar a lista de escopos/épocas pré-processada via prop `scopes: { id: string; currentEpochIndex: number }[]` na `AuthTabProps`.
