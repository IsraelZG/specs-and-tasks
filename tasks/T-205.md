## 1. Objetivo
Implementar `SwarmRegistry` — mapa em RAM de peers conhecidos com estado de conexão, latência (EWMA), tier e personas ativas, indexado por `DevicePeerId`, no pacote `@plataforma/core`
*(derivado de `docs/conceitos/swarm-registry.md` — "mapa em RAM mantido pelo sync-worker com estado atualizado dos peers ativos"; `docs/plano-de-implementacao.md` §4 T-205)*.

### Contratos exatos
```ts
// --- packages/core/src/swarmRegistry.ts ---
import { DevicePeerId, PersonaPeerId } from '@plataforma/protocol'; // T-105

export type ConnectionState = 'disconnected' | 'connecting' | 'connected';

export interface PeerRecord {
  devicePeerId: DevicePeerId;
  state: ConnectionState;
  latencyMs: number;          // EWMA
  lastSeenAt: number;         // timestamp ms
  personas: PersonaPeerId[];  // personas atestadas por este dispositivo
  tier: number;               // 0=unknown, 1=relay, 2=edge, 3=super
}

export interface SwarmRegistryOptions {
  heartbeatIntervalMs?: number; // default 15000 (15s)
  evictionThresholdMs?: number; // default 45000 (3 falhas × 15s)
}

export class SwarmRegistry {
  constructor(opts?: SwarmRegistryOptions);

  /** Registra ou atualiza peer. */
  upsert(devicePeerId: DevicePeerId, patch: Partial<Omit<PeerRecord, 'devicePeerId'>>): void;

  /** Retorna peer ou undefined. */
  get(devicePeerId: DevicePeerId): PeerRecord | undefined;

  /** Remove peer. */
  delete(devicePeerId: DevicePeerId): void;

  /** Lista todos os peers conectados. */
  listConnected(): PeerRecord[];

  /** Lista peers que não responderam heartbeat em `thresholdMs`. */
  listStale(thresholdMs?: number): PeerRecord[];

  /** Contagem de peers. */
  get size(): number;

  /** Atualiza latência via EWMA (α = 0.3). */
  recordLatency(devicePeerId: DevicePeerId, rttMs: number): void;
}
```
- 100% in-memory (RAM). Sem SQLite.
- `upsert` com `state: 'connected'` reseta `lastSeenAt`.
- `listStale` usa `lastSeenAt + evictionThresholdMs` para detectar peers inativos.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/core test`.

Casos de teste:
1. `upsert` + `get` → roundtrip.
2. `upsert` com patch parcial → mescla com dados existentes.
3. `listConnected` filtra apenas `state === 'connected'`.
4. `listStale` retorna peers com `lastSeenAt` além do threshold.
5. `recordLatency` atualiza EWMA (2 chamadas: `rtt=100` + `rtt=200` → `130`).
6. `delete` remove; `get` → undefined.
7. `size` reflete contagem correta após inserts/deletes.

## 5. Instruções de Execução
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** persista no SQLite. SwarmRegistry vive apenas em RAM.

1. **[TDD]** Escreva `swarmRegistry.test.ts` com casos 1–7.
2. Implemente `SwarmRegistry` com mapa interno `Map<DevicePeerId, PeerRecord>`.
3. Implemente EWMA: `newLatency = α * rtt + (1 - α) * oldLatency`, α = 0.3.
4. Re-exporte. Rode build + test.
