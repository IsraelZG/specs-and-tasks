## 1. Objetivo
Endurecer o manuseio do admin token do `system-peer`.

> **ACHADO (auditoria 2026-07-01) — [M3 · segurança].**
> - `packages/../apps/system-peer/src/adminToken.ts:9` imprime o token no console (`console.log('[system-peer] ADMIN_TOKEN:', token)`) — vaza em qualquer agregador de log.
> - `apps/system-peer/src/admin.ts:9` compara o token com `===` (`match[1] === adminToken`) — comparação não constante no tempo, sujeita a timing attack. Usar `crypto.timingSafeEqual` (com guarda de comprimento).

**Meta:** não logar o segredo (no máximo um fingerprint/hash curto para diagnóstico); comparar em tempo constante.

## 3. Escopo de Arquivos (Inputs e Outputs)
- **[UPDATE]** `apps/system-peer/src/adminToken.ts` — remover o `console.log` do token cru (logar só um hash curto ou nada)
- **[UPDATE]** `apps/system-peer/src/admin.ts` — `isAuthorized` compara via `crypto.timingSafeEqual` com guarda de comprimento (evitar throw em tamanhos diferentes)
- **[UPDATE]** testes de `admin` — cobrir token errado do mesmo comprimento e de comprimento diferente

> **Pegadinha:** `timingSafeEqual` lança se os buffers têm tamanhos diferentes — comparar comprimento antes (retornando `false`), sem short-circuit que reintroduza timing leak relevante.
