## 1. Objetivo
Endurecer o manuseio do admin token do `system-peer`.

> **ACHADO (auditoria 2026-07-01) — [M3 · segurança].**
> - `apps/system-peer/src/adminToken.ts:9` imprime o token no console (`console.log('[system-peer] ADMIN_TOKEN:', token)`) — vaza em qualquer agregador de log.
> - `apps/system-peer/src/admin.ts:9` compara o token com `===` (`match[1] === adminToken`) — comparação não constante no tempo, sujeita a timing attack. Usar `crypto.timingSafeEqual` (com guarda de comprimento).

**Meta:** não logar o segredo (no máximo um fingerprint/hash curto para diagnóstico); comparar em tempo constante.

### Contratos exatos (assinaturas TS fixadas, derivadas de `apps/system-peer/src/admin.ts:4-10` + `apps/system-peer/src/adminToken.ts:7-11`)

```ts
// --- apps/system-peer/src/adminToken.ts ---
import { createHash, randomBytes } from 'node:crypto';

/**
 * Retorna o ADMIN_TOKEN do ambiente ou gera um novo (32 bytes hex).
 * NUNCA loga o token cru. Loga apenas um fingerprint SHA-256 truncado (8 chars hex)
 * para correlação de logs entre instâncias.
 */
export function resolveAdminToken(): string {
  const token = process.env['ADMIN_TOKEN'] ?? randomBytes(32).toString('hex');
  // fingerprint: primeiros 8 hex do sha256(token) — não reversível, útil para log
  const fingerprint = createHash('sha256').update(token).digest('hex').slice(0, 8);
  console.log('[system-peer] ADMIN_TOKEN resolved, fp=' + fingerprint);
  return token;
}

// --- apps/system-peer/src/admin.ts ---
import { timingSafeEqual } from 'node:crypto';
import type { Request, Response, NextFunction } from 'express';
import type { Express } from 'express';

/**
 * Verifica se o header `Authorization: Bearer <token>` bate com `adminToken`.
 * Comparação em tempo constante via `timingSafeEqual`, com guarda de comprimento
 * (timingSafeEqual lança RangeError se buffers têm tamanhos diferentes).
 */
function isAuthorized(req: Request, adminToken: string): boolean {
  const header = req.header('authorization') ?? req.header('Authorization');
  if (!header) return false;
  const match = /^Bearer\s+(.+)$/.exec(header);
  if (!match || !match[1]) return false;
  const presented = Buffer.from(match[1], 'utf8');
  const expected = Buffer.from(adminToken, 'utf8');
  // Guarda de comprimento SEM short-circuit que reintroduza timing leak:
  // se tamanhos diferentes, ainda chamamos timingSafeEqual em buffers de mesmo
  // comprimento (padded) para preservar tempo constante.
  if (presented.length !== expected.length) {
    // Compara contra um buffer de mesmo tamanho (zeros) — o resultado é
    // sempre false, mas o tempo de execução é o mesmo que tokens de tamanho real.
    timingSafeEqual(presented, Buffer.alloc(presented.length));
    return false;
  }
  return timingSafeEqual(presented, expected);
}
```

> **Pegadinha coberta (achado da spec):** `timingSafeEqual` lança `RangeError` se
> os buffers têm tamanhos diferentes. A guarda de comprimento aqui preserva
> **tempo constante** mesmo no caminho de tamanhos diferentes (compara contra
> um buffer de zeros do mesmo tamanho), eliminando o leak que o `length` check
> ingênuo reintroduziria.
