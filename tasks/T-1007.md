# T-1007 · Integração do EpochDB para Agentic Memory (Lossless)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v22+
- **Package Manager:** `pnpm`
- **Test Runner:** `vitest`

## 1. Objetivo
Implementar o registro de conversas, queries MCP e handovers do MGTIA-v2 diretamente no EpochDB para permitir consultas "multi-hop" por parte dos agentes no futuro (filosofia "MeMo").

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** misture contexto de código vivo (TurboVec) com log de memória (EpochDB). São ferramentas distintas. O TurboVec guarda código e documentação. O EpochDB guarda o "raciocínio" dos agentes e as interações MCP.

1. Implemente o adaptador para inserir logs no formato esperado pelo EpochDB.
2. Adicione interceptadores no Servidor MCP (`T-1004`) para que toda chamada de agente ou log devolvido pelo Kanban caia no EpochDB como memória de curto/longo prazo.
