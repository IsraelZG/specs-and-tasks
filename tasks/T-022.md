## 1. Objetivo

Ativar regressão visual com **Playwright `toHaveScreenshot`** no `packages/design-system`,
rodando no CI. A ADR [0006](../docs/adr/0006-gate-regressao-visual.md) (T-020) decidiu esta
ferramenta e descartou o Lookout — esta task implementa essa decisão.

O gate visual é: screenshot do componente renderizado → diff contra baseline → Pass/Fail
automático no CI. Se o design muda legitimamente, aceitar nova baseline via `--update-snapshots`.

## 5. Instruções de Execução (Step-by-Step)

> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** criar `playwright.config.ts` dentro de `packages/design-system` — o config mora no
>   showcase e no preset (testkit). Design-system não é consumidor direto de Playwright.
> - **NÃO** instalar/configurar o Lookout — ADR-0006 descartou.
> - **NÃO** criar step de CI novo — o gate visual entra pelo mesmo `test:e2e` da T-021.
> - **NÃO** usar `expect(...).toMatchScreenshot()` (nome antigo/deprecado) — use
>   `expect(...).toHaveScreenshot()` (API atual do Playwright).
> - **NÃO** fotografar só o container do componente para componentes com **portal**
>   (Tier B: `Popover`, `Tooltip`, `DropdownMenu`) — o portal renderiza em `document.body`,
>   fora do container. Fotografar `document.body` inteiro ou `fullPage: true`.
> - **NÃO** aceitar baselines novas sem reviewer — `--update-snapshots` requer PR + review
>   dos diffs.

### Pegadinhas conhecidas *(preencher pelo Task Architect)*

- **Antialiasing cross-arch (x64 vs arm64).** O diff de pixel pode divergir entre
  `ubuntu-latest` (x64) e `ubuntu-24.04-arm` (arm64) por renderização diferente de fontes
  (subpixel rendering). **Mitigação:** usar `maxDiffPixelRatio` (tolerância) ou baselines
  **por arquitetura** (snapshotPathTemplate com `{project.name}` que inclua `runner.arch`).
  Decidir com PoC nos dois runners (rodar `test:e2e` em ambos e comparar).
- **Portais (Tier B) fora do frame.** `Popover`, `Tooltip`, `DropdownMenu` renderizam o overlay
  em `document.body`, não dentro do container do componente. `toHaveScreenshot()` no container
  corta o portal. **Fix:** fotografar `await page.goto('/component/popover')` e capturar o
  `page` inteiro (ou `document.body`) — não o container isolado.
- **Baselines desatualizadas = falso-positivo permanente.** Toda mudança legítima de design
  (ex.: troca de tokens pela T-013) invalida as baselines. Sem processo de re-baseline, o CI
  quebra para sempre. **Processo:** `npx playwright test --update-snapshots` para aceitar,
  commit das novas baselines, review dos diffs no PR.
- **`webServer` do preset da T-021.** O showcase sobe via `pnpm --filter design-system-showcase dev`
  (não `build`). O `webServer` do `createPlaywrightConfig` faz isso automaticamente. **Não**
  subir um segundo dev server ou build separado para o screenshot.
- **Tamanho do snapshotDir.** Os baselines são imagens PNG (centenas de KB cada). Versioná-las
  no git é correto (são a verdade do gate), mas o repo cresce. Não por em `.gitignore`.

   `playwright.config.ts` do showcase. Entenda o `webServer` e como os E2E funcionais já são
   executados.
3. **[DECISÃO DE ESTRATÉGIA CROSS-ARCH]** — antes de escrever o primeiro spec, rode um PoC
   rápido: adicione `toHaveScreenshot()` a **1 componente simples** (Tier A, sem portal),
   capture baselines no runner x64, e tente rodar no runner arm64.
   - Se o diff for **zero** (mesmo pixel) → `maxDiffPixelRatio` default basta; baselines únicos.
   - Se o diff for **não-zero** (antialiasing) → decida: (a) `maxDiffPixelRatio` com tolerância
     aceitável, **ou** (b) baselines por-arch via `snapshotPathTemplate` usando
     `{project.name}`.
   - Registre a decisão na Seção 8 (handover).
4. **[IMPLEMENT]** Adicione asserções `toHaveScreenshot(...)` aos specs E2E do showcase:
   - Comece com **1 componente de cada tier** (ex.: `Button` = A, `Popover` = B, `Icon` = C).
   - Para Tier B (portais): use `page` inteiro ou `document.body`, não o container isolado.
   - Nomes de snapshot descritivos (ex.: `button-primary-default-light.png`).
5. **[BASELINES]** Capture as baselines iniciais:
   ```bash
   npx playwright test --update-snapshots
   ```
   Versione as baselines geradas (commit + push junto com o spec).
6. **[VERIFY]** Execute o gate:
   ```bash
   pnpm --filter design-system-showcase build
   pnpm --filter design-system-showcase test:e2e
   ```
   Ambos devem passar sem erro. Cole a saída na Seção 8.
