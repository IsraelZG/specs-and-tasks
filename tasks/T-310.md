## 1. Objetivo
Implementar `evaluateTransportHints()` no pacote `@plataforma/transport`: união discriminada que consulta `SPECIFICATION.transport_hints` e roteia para `nodes/edges` (RBSR), `pending_changes` (efêmero), `device_state.db` (private swarm) ou RAM
*(derivado de `docs/plano-de-implementacao.md` §6 T-310)*.

### Contratos exatos
```ts
// --- packages/transport/src/transportMatrix.ts ---

export enum TransportTarget {
  NODES_EDGES = 'nodes_edges',           // RBSR — grafo durável
  PENDING_CHANGES = 'pending_changes',   // efêmero — canal Automerge/WebRTC
  DEVICE_STATE_DB = 'device_state_db',   // private swarm — device_state.db
  RAM = 'ram',                           // volátil — SwarmRegistry, cache de prefetch
}

export interface TransportHint {
  entity_type: string;          // tipo de entidade (ex: "CONTENT", "SPECIFICATION")
  target: TransportTarget;
  priority?: number;            // 0=mais alta
  condition?: string;           // expressão opcional de condição (ex: "scope=private")
}

export type TransportMatrix = TransportHint[];

/** Avalia as hints em ordem de prioridade e retorna o target para uma entidade. */
export function evaluateTransportHints(
  hints: TransportMatrix,
  entityType: string,
  scope?: string,
): TransportTarget;

/** Tabela de verdade: todas as combinações entity_type × condition → target. */
export function validateMatrix(hints: TransportMatrix): { valid: boolean; errors: string[] };

/** Matriz default do sistema. */
export const DEFAULT_TRANSPORT_MATRIX: TransportMatrix;
```
- `evaluateTransportHints`: percorre hints ordenadas por priority; retorna o primeiro match.
- `validateMatrix`: valida que toda combinação de entity_type + condition tem target único; detecta ambiguidades.
- `DEFAULT_TRANSPORT_MATRIX`: SPECIFICATION → RAM, CONTENT → NODES_EDGES, pending → PENDING_CHANGES, device_state → DEVICE_STATE_DB.
- Tipo errado não compila/roteia (type-safe via union discriminada).

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/transport test`.

Casos de teste:
1. `evaluateTransportHints` com `CONTENT` → `NODES_EDGES`.
2. `evaluateTransportHints` com `SPECIFICATION` → `RAM`.
3. `evaluateTransportHints` com tipo desconhecido → fallback `NODES_EDGES`.
4. `evaluateTransportHints` com scope=private → `DEVICE_STATE_DB`.
5. `validateMatrix` com hints bem formadas → valid.
6. `validateMatrix` com ambiguidade (dois targets mesma condição) → invalid + erro.
7. `DEFAULT_TRANSPORT_MATRIX` cobre todos os targets.
8. Tipo errado (não-`TransportTarget`) → erro de compilação TypeScript.
