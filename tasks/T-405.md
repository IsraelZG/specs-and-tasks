# T-405 · Relay de circuito

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+

## 1. Objetivo
Implementar relay de circuito no peer do sistema: retransmissão de frames entre dois peers sem decifrá-los (Noise é fim-a-fim); contabilização para o trust model; payloads opacos ao relay
*(derivado de `docs/plano-de-implementacao.md` §7 T-405)*.

### Contratos exatos
```ts
// --- apps/system-peer/src/relay.ts ---
import { DevicePeerId } from '@plataforma/protocol'; // T-105a
import { NetworkAdapterPort, WireData } from '@plataforma/protocol'; // T-004
import { RelayTrustModel } from '@plataforma/transport'; // T-207

export interface RelayCircuit {
  circuitId: string;            // ULID do circuito
  peerA: DevicePeerId;
  peerB: DevicePeerId;
  createdAt: number;
  bytesRelayed: number;
  frameCount: number;
}

export interface RelayConfig {
  trustModel: RelayTrustModel;
  maxCircuits?: number;         // default 1000
  maxBytesPerCircuit?: number;  // default 10 * 1024 * 1024 (10 MiB)
  idleTimeoutMs?: number;       // default 300000 (5min)
}

export class CircuitRelay {
  constructor(adapter: NetworkAdapterPort, config: RelayConfig);

  /** Estabelece circuito entre dois peers. */
  createCircuit(peerA: DevicePeerId, peerB: DevicePeerId): Promise<string>;

  /** Retransmite frame de peerA para peerB sem decifrar. */
  relayFrame(circuitId: string, fromPeer: DevicePeerId, frame: WireData): Promise<void>;

  /** Encerra circuito. */
  closeCircuit(circuitId: string): void;

  /** Lista circuitos ativos. */
  listCircuits(): RelayCircuit[];

  /** Callback quando frame é relayado (para métricas). */
  onFrameRelayed(handler: (circuitId: string, bytes: number) => void): void;
}
```
- `relayFrame`: recebe frame de um peer, encaminha para o outro sem decifrar — asserção de não-decifração.
- Payloads permanecem opacos ao relay: Noise_XX (T-202) é fim-a-fim.
- Cada frame relayado → `trustModel.recordRelay()` para atualizar score do peer.
- Limites: maxCircuits, maxBytes por circuito, idle timeout.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest + SimNetwork (T-005).
- [x] **Ambiente:** `pnpm --filter @plataforma/system-peer test`.

Casos de teste:
1. `createCircuit` entre peerA e peerB → circuitId válido.
2. `relayFrame`: peerA envia frame → peerB recebe exatamente o mesmo frame.
3. `relayFrame` com circuitId inexistente → erro.
4. Dois peers `symmetric` (SimNetwork) conversam via relay; payloads opacos ao relay.
5. Asserção de não-decifração: relay NÃO tem acesso ao conteúdo do frame.
6. `closeCircuit` remove circuito; `listCircuits` não contém.
7. `maxCircuits` atingido → novo circuito rejeitado.
8. Idle timeout: circuito fechado após 5min sem tráfego.

## 5. Instruções de Execução
