# T-009b · WebSocketControlClient (client-sdk → Hub QA)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro, `ws`)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar `WebSocketControlClient` no pacote `@plataforma/client-sdk`: adaptador que conecta ao Hub QA via WebSocket, faz handshake (client_id + token), despacha comandos recebidos para `ControlPort` e envia telemetria de volta
*(extraído de T-009 §1)*.

### Contratos exatos
```ts
// --- packages/client-sdk/src/WebSocketControlClient.ts ---
import { ControlPort, ControlEvent } from '@plataforma/core'; // T-009a

export class WebSocketControlClient {
  constructor(url: string, token: string);

  connect(): Promise<void>;
  bind(controlPort: ControlPort): void;
  sendTelemetry(event: ControlEvent): void;
  close(): void;
}
```
- `connect`: estabelece WebSocket, envia handshake `{client_id, token}`, aguarda ACK.
- `bind`: vincula `ControlPort` local — comandos recebidos do Hub são despachados via `controlPort.execute()`.
- `sendTelemetry`: push de eventos ao Hub (peer_connected, sync_complete, etc).

## 4. Estratégia de Testes
- [x] **Framework:** Vitest + `ws` server in-memory.

Casos de teste:
1. `connect` → handshake OK com servidor `ws` local.
2. Servidor envia `{corrId, cmd:'drop_connection', payload}` → `ControlPort.execute` chamado.
3. Resposta com mesmo `corrId` retornada ao servidor.
4. `sendTelemetry` → servidor recebe evento.
5. Token inválido → handshake rejeitado.
6. Reconexão automática (até 3 tentativas).

## 5. Instruções de Execução

2. Implemente handshake + despacho de comandos.
