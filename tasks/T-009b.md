## 1. Objetivo
Implementar `WebSocketControlClient` no pacote `@plataforma/client-sdk`: cliente que conecta ao **Hub de Controle real** (`apps/system-peer`, endpoint WS `/control` — T-010, `done`), autentica, recebe comandos `{corr_id, cmd, payload}`, despacha-os para o `ControlPort` local (T-009a) e devolve `{corr_id, result}`.

> **⚠️ CONFRONTAÇÃO (endurecimento 2026-07-08, claude-opus).** A prosa anterior desta task
> ("handshake `{client_id, token}` + aguarda ACK") veio do antigo T-009 master e **NÃO bate com o
> protocolo real do Hub, que já está `done` e merged** (`apps/system-peer/src/controlRelay.ts`,
> T-010 §1/§4). Protocolo real: autenticação por **`Authorization: Bearer <token>` no upgrade
> HTTP** (não mensagem in-band), **sem ACK**, **`clientId` gerado pelo servidor** (UUID; o cliente
> não envia client_id — em dev é inspecionável via `GET /control/registry`, `server.ts:101-125`).
> A spec abaixo segue o Hub done. Registrado na Seção 6 (reconciliação) + 3 decisões abertas D1–D3.

### Contratos exatos (derivados do Hub done + T-009a)
```ts
// --- packages/client-sdk/src/WebSocketControlClient.ts ---
import type { ControlPort, ControlEvent } from '@plataforma/core'; // T-009a (spec ready; código ainda não existe — ver Coordenação §3)

export class WebSocketControlClient {
  /**
   * @param url   URL completa do endpoint de controle, ex.: 'ws://localhost:8787/control'
   * @param token session token (M0: qualquer string non-empty — T-010 §6 OPEN-2)
   */
  constructor(url: string, token: string);

  /** Abre o WebSocket autenticando com Bearer token. Resolve no evento 'open'. Rejeita se o servidor fechar com 1008. */
  connect(): Promise<void>;
  /** Vincula o ControlPort local — cada mensagem {corr_id, cmd, payload} vira controlPort.execute({corrId: corr_id, cmd, payload}) e a resolução vira {corr_id, result} de volta ao Hub. */
  bind(controlPort: ControlPort): void;
  /** [PENDE D2] push de eventos ao Hub — o Hub atual NÃO tem ingestão de telemetria. */
  sendTelemetry(event: ControlEvent): void;
  close(): void;
}
```

**Wire format (derivado de `apps/system-peer/src/controlRelay.ts` + T-010 §4 casos 8–9 — código `done`):**
- Comando (Hub → cliente): JSON `{ corr_id: string, cmd: string, payload: unknown }` (`controlRelay.ts:98`). **Atenção: `corr_id` snake_case no wire ↔ `corrId` camelCase no `ControlCommand` de T-009a — o cliente faz o mapeamento.**
- Resposta (cliente → Hub): JSON `{ corr_id: string, result: unknown }` com o MESMO `corr_id` (`controlRelay.ts:106-115`); o Hub aguarda com timeout default de 5000ms (`controlRelay.ts:90`).
- Auth: header `Authorization: Bearer <token>` no upgrade; token ausente/vazio → servidor fecha com código **1008** e reason `missing_session_token` (`controlRelay.ts:46-51`). M0 aceita qualquer Bearer non-empty (T-010 §6 OPEN-2).
- Mensagens que não parseiam como JSON são ignoradas pelo Hub (`controlRelay.ts:117-119`).

Casos de teste (numerados):
1. `connect()` com token non-empty → servidor de teste recebe upgrade com header `Authorization: Bearer <token>`; promise resolve no 'open'.
2. Servidor envia `{corr_id:'abc', cmd:'drop_connection', payload:{peerId:'A'}}` → `controlPort.execute` chamado com `{corrId:'abc', cmd:'drop_connection', payload:{peerId:'A'}}` (mapeamento snake→camel).
3. Resolução de `execute` (ex.: `'ok'`) → servidor recebe JSON `{corr_id:'abc', result:'ok'}`.
4. `execute` rejeita → cliente ainda responde `{corr_id, result}` com indicação de erro **[shape do result de erro PENDE D3]**.
5. Token vazio → servidor fecha com 1008 → `connect()` rejeita.
6. **[PENDE D3]** Reconexão automática (política/limites a decidir).
7. **[PENDE D2]** `sendTelemetry` (sem sink no Hub hoje).
