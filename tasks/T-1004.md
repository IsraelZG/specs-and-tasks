# T-1004 · Interface MCP (Model Context Protocol) e Servidor Headroom

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v22+
- **Package Manager:** `pnpm`
- **Test Runner:** `vitest`

## 1. Objetivo
Implementar o servidor MCP no `nexus-backend` que expõe ferramentas nativas para agentes. Integrar internamente chamadas ao `headroom-ai` (Python CLI ou API Node) para garantir que qualquer resposta grande de contexto seja enviada na forma comprimida de tokens.

## 2. Contexto RAG (Spec-Driven Development)
- [ ] [Plano de Implementação Nexus](../plano-de-implementacao.md)

## 3. Escopo de Arquivos (Inputs e Outputs)
- **[CREATE]** `apps/nexus-backend/src/mcp/server.ts`
- **[CREATE]** `apps/nexus-backend/src/services/headroom.client.ts`

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [ ] **Framework:** Vitest
- [ ] **Ambiente do Teste:** Simular um cliente MCP e garantir que a ferramenta `nexus_read_context` retorna JSON comprimido em vez do texto bruto.

## 5. Instruções de Execução (Step-by-Step)
1. Instale o `@modelcontextprotocol/sdk`.
2. Defina e registre a tool `nexus_read_context(query)`.
3. Na execução da tool: Use o Roteador (T-1003), faça busca no TurboVec (T-1002), pegue o texto resultante, jogue no Headroom, e retorne o output do Headroom para o cliente MCP.
4. Registre também `nexus_retrieve_detail(id)` e `nexus_run_safe_script(scriptId)`.
