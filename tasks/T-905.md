# T-905 · Bateria de Testes Adversariais (Chaos/Fuzzer)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (pacotes core/protocol) e `playwright` (E2E/Frontend)

## 1. Objetivo
Criar o framework de fuzzing final (Chaos Monkey local) que injeta mensagens malformadas aleatórias e partições arbitrárias para quebrar a convergência.

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [ ] **Framework:** Playwright (Workers múltiplos)
- [ ] **Métricas:** 1 milhão de iterações sem o SQLite entrar em estado corrompido (sempre reverte ou dropa).

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** teste regras de UI aqui. O fuzzer injeta pacotes MessagePack inválidos diretamente no NetworkAdapterPort.
> - **NÃO** use Math.random solto, continue forçando a seed fixa para que um crash seja reproduzível 100% das vezes.

1. **[TDD]** Crie os testes garantindo que falhem.
2. Implemente a funcionalidade no pacote indicado.
3. Garanta que o linter aprove o código.
