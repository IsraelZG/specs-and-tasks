# T-905 · Bateria de Testes Adversariais (Chaos/Fuzzer)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro, SimNetwork)
- **Capacidade-alvo:** opus-spike

## 1. Objetivo
Implementar framework de testes adversariais (Chaos/Fuzzer) no pacote `@plataforma/testkit`: injeta mensagens malformadas, partições arbitrárias e falhas de rede para validar convergência sob estresse. Usa SimNetwork (T-006) + RBSR (T-302) + CPE (T-404) + Fork/Merge (T-601) + BLOBs (T-801)
*(derivado de `docs/visao-arquitetural.md` §10 — "12 cenários adversariais"; `docs/plano-de-implementacao.md` §2.5)*.

### Contratos
```ts
// --- packages/testkit/src/chaos/Fuzzer.ts ---
import { SimNetwork } from '../SimNetwork.js'; // T-006
import { SeededRandom } from '../random.js'; // T-003

export interface ChaosScenario {
  name: string;
  iterations: number;
  seed: string;
  actions: ChaosAction[];
}

export type ChaosAction =
  | { type: 'partition'; groups: PeerId[][] }
  | { type: 'heal' }
  | { type: 'dropMessages'; peerId: PeerId; rate: number }
  | { type: 'corruptMessage'; peerId: PeerId; rate: number }
  | { type: 'delayMessages'; peerId: PeerId; maxMs: number }
  | { type: 'restartPeer'; peerId: PeerId };

export interface ChaosResult {
  scenario: string;
  passed: boolean;
  iterations: number;
  failures: string[];
  convergenceTimeMs: number;
}

/** Executa cenário de caos e verifica convergência ao final. */
export function runChaosScenario(scenario: ChaosScenario, sim: SimNetwork): Promise<ChaosResult>;

/** 12 cenários canônicos (visao-arquitetural.md §10). */
export const ADVERSARIAL_SCENARIOS: ChaosScenario[];
```
- `runChaosScenario`: aplica ações aleatórias (seed fixa → reprodutível), depois chama `expectConverged` (T-007).
- 12 cenários: assinatura inválida, STALE_EPOCH, double-spend, UCAN expirado, partição + merge, etc.
- Métrica: 1M iterações sem corrupção de estado.

## 4. Estratégia de Testes
Casos de teste:
1. Cenário "assinatura inválida": nós corrompidos são rejeitados, convergência mantida.
2. Cenário "partição + merge": após heal, peers convergem.
3. Fuzzer com seed fixa → 1000 iterações determinísticas → resultado idêntico em qualquer máquina.
4. Nó corrompido não propaga para peers íntegros.

## 5. Instruções de Execução
1. Implemente `runChaosScenario` com SimNetwork multi-peer.
2. Defina `ADVERSARIAL_SCENARIOS`.
3. Escreva teste de fumaça com 1000 iterações.
4. Rode build + test.
