# T-1048 · Composition root + inversão de dependência no superapp

## 1. Objetivo
Adotar/formalizar **ports-and-adapters + composition root** no superapp para que a direção de camadas `@plataforma/protocol ← crypto ← core ← transport ← workers` (`visao-arquitetural.md §1`) seja **invariante verificável por ferramenta**: nenhum pacote importa um adapter concreto "pra cima", nenhum ciclo entre pacotes. O core conhece **só as portas** (`StoragePort`, `GraphStorePort`, `NetworkAdapters` — ADR-005); os adapters concretos são instanciados e injetados na **raiz de composição** (entry do app), nunca importados pelo core.

> **Sizing (Dimensionamento).** O escopo real (varrer todo o workspace, decidir onde fica a raiz de composição, corrigir N violações) é diffuso demais para uma task ≤ Sonnet. Esta é a **spike**: o entregável é o ADR canônico + o inventário de violações reais + a **decomposição** em sub-tasks de correção (uma por par de pacotes). As correções em si são as filhas.

## 2. Contexto RAG
- `docs/visao-arquitetural.md` §1 (as 5 camadas) e §3 (agnosticismo de engine, `GraphStorePort`).
- `docs/adr/adr-005-storage-engine-agnosticism.md` (portas já existentes).
- Gate de acoplamento do template de task (`generate-task.mjs` §7 DoD: "import respeita a direção … e NÃO fecha ciclo").
- `docs/conceitos/visao-arquitetural.md`.

## 3. Escopo (Inputs/Outputs)
- **[READ]** relatório de ciclos/imports-upward reais: rodar `npx madge --circular --extensions ts packages apps` e inspecionar imports que violam a direção de camadas.
- **[CREATE]** `docs/adr/adr-0xx-composition-root-superapp.md`: pattern canônico — **onde** os adapters concretos são instanciados/injetados; regra "core não importa transport/adapter concreto"; como novos pacotes se encaixam no DAG.
- **[UPDATE]** apenas as violações **triviais/óbvias** (mover instanciação de adapter para a raiz, trocar import upward por injeção via porta) que couberem no orçamento da spike.
- **[DECOMPOSE]** as violações não-triviais em sub-tasks filhas (`generate-task.mjs`), uma por par de pacotes, linkadas a esta.

## 4. Testes
- `npx madge --circular` do workspace: **zero** ciclos ao fim (ou os remanescentes rastreados em filhas linkadas).
- `pnpm -r build` limpo; suites dos pacotes tocados sem regressão.

## 5. DoD
Direção de camadas provada por ferramenta (inventário de ciclos + upward imports); ADR canônico da raiz de composição existe; violações **corrigidas** ou **decompostas** em sub-tasks linkadas com critério claro.

## 6. Feedback de Especificação
- **Não redesenhar** as camadas — a direção já é canônica (§1). Só remover violações.
- Se uma "violação" for design intencional (ex.: barrel de tipos compartilhados), documentar no ADR como exceção — não forçar refactor.
- Este é o pré-requisito conceitual de T-1049 (guardrails): sem saber os ciclos reais, habilitar `import/no-cycle` deixa o baseline vermelho.

## 7. Verificação (comandos exatos)
```bash
npx madge --circular --extensions ts packages apps   # inventário de ciclos
pnpm -r build                                         # limpo
pnpm -r test                                          # sem regressão nos pacotes tocados
```
