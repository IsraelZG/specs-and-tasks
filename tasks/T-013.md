# T-013 · Pipeline Turbo de tokens + cache (exceção Vite)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo (`pnpm build`, `pnpm test`, `pnpm lint` na raiz afetam todos os pacotes)
- **Test Runner:** `vitest`
- **Capacidade-alvo:** haiku

## 1. Objetivo
Integrar o build do design system ao Turborepo de forma correta e cacheável: os tokens (Style Dictionary) devem compilar **antes** de qualquer build que os consome, e os artefatos devem entrar no cache do Turbo. Formalizar a **exceção ao padrão `tsup`**: `packages/design-system` e `apps/design-system-showcase` usam **Vite** (precisam de Tailwind v4 + bundle de CSS/assets).

## 2. Contexto RAG (Spec-Driven Development)
- [ ] [plano-de-implementacao.md §1.1](../docs/plano-de-implementacao.md) — decisão "tsup para build"; esta tarefa documenta a exceção.
- [ ] [T-001 · Bootstrap](./T-001.md) — `turbo.json` base.
- [ ] [T-011 · Incorporar a lib](./T-011.md) — scripts `build:tokens`/`build:react`/`build:index`. **Pré-requisito.**

## 3. Escopo de Arquivos (Inputs e Outputs)
- **[READ]** `packages/design-system/package.json` (scripts existentes `build:tokens`/`build:react`/`build:index` — definidos em T-011)
- **[READ]** `turbo.json` (raiz) — baseline criado em T-001 (`tasks.build.dependsOn: ["^build"]`, `outputs: ["dist/**"]`)
- **[UPDATE]** `turbo.json` (raiz) — nova task `build:tokens` com `outputs: ["build/web/**"]`; atualizar `build.dependsOn` para `["^build", "build:tokens"]`; adicionar `"build/**"` aos `outputs` de `build`
- **[UPDATE]** `packages/design-system/package.json` — se necessário, expor `build:tokens` como script de entrada (já criado pela T-011)
- **[CREATE]** `docs/adr/adr-003-design-system-build.md` — ADR registrando a exceção Vite vs `tsup` para o Design System (necessita Tailwind v4 + bundle de CSS/assets)

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [x] **Framework:** validação de pipeline — rodar `pnpm build` na raiz e inspecionar o grafo do Turbo.
- [x] **Métricas/Cobertura:** binário — build a frio gera tokens antes do bundle; segundo build usa cache (`>>> FULL TURBO`).
- [x] **Ambiente do Teste:** Node puro, raiz.
- [x] **Fora de Escopo:** conteúdo dos tokens.

### Casos de teste (numerados):
1. `turbo.json` declara task `build:tokens` com `outputs: ["build/web/**"]`.
2. `turbo.json` declara `build.dependsOn: ["^build", "build:tokens"]` e `build.outputs` inclui `"build/**"`.
3. `pnpm build` a frio na raiz executa `build:tokens` **antes** de `build` (ordem visível no log do Turbo: `build:tokens` → `^build` → `build`).
4. Segundo `pnpm build` consecutivo sem mudança de source acerta cache do Turbo (`>>> FULL TURBO`).
5. `build/web/theme-light.css` e `build/web/theme-dark.css` são gerados.
6. ADR `docs/adr/adr-003-design-system-build.md` existe e documenta a justificativa Vite vs `tsup`.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** force os pacotes Vite a usar `tsup` — registre a exceção, não a "conserte".
> - **NÃO** deixe `build:tokens` fora do grafo de dependências (causaria bundle com tokens defasados).
> - **NÃO** marque outputs de token como não-cacheáveis sem motivo (perde o ganho do Turbo).

1. Em `turbo.json`, declarar `build:tokens` com seus `outputs` (ex.: `build/web/**`).
2. Ajustar `build` para `dependsOn: ["^build", "build:tokens"]`.
3. Rodar `pnpm build` a frio; confirmar ordem (tokens → bundle).
4. Rodar `pnpm build` de novo; confirmar cache hit.
5. Escrever o ADR `docs/adr/adr-003-design-system-build.md` registrando a justificativa da adoção do Vite para o Design System, contrastando com o padrão `tsup` das outras libs.
