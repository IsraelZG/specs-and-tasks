# T-801 · Storage Engine de BLOBs (Chunking)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar `Chunker` + `MerkleTree` no pacote `@plataforma/media`: fragmentação de arquivos em chunks de 256 KB, hashing SHA-256 por chunk, e Merkle Tree para integridade verificável
*(derivado de `docs/plano-de-implementacao.md` §4 T-801; T-106 — schema base)*.

### Contratos exatos
```ts
// --- packages/media/src/blob/Chunker.ts ---

/** Chunk de arquivo com hash e posição. */
export interface Chunk {
  index: number;
  offset: number;
  size: number;
  hash: Uint8Array;  // SHA-256
  data: Uint8Array;
}

/** Fragmenta arquivo em chunks de tamanho fixo. */
export function chunkFile(data: Uint8Array, chunkSize?: number): Chunk[]; // default 262144 (256KB)

/** Reconstitui arquivo original dos chunks. */
export function assembleFile(chunks: Chunk[]): Uint8Array;
```

```ts
// --- packages/media/src/blob/MerkleTree.ts ---

/** Nó da Merkle Tree. */
export interface MerkleNode {
  hash: Uint8Array;     // SHA-256(nó)
  left?: MerkleNode;
  right?: MerkleNode;
}

/** Raiz da Merkle Tree com a prova de um chunk. */
export interface MerkleProof {
  rootHash: Uint8Array;
  chunkIndex: number;
  proof: MerkleProofStep[];
}

export interface MerkleProofStep {
  position: 'left' | 'right';
  hash: Uint8Array;
}

/** Constrói Merkle Tree a partir dos hashes dos chunks. */
export function buildMerkleTree(chunks: Chunk[]): MerkleNode;

/** Gera prova de Merkle para um chunk específico. */
export function generateProof(root: MerkleNode, chunks: Chunk[], index: number): MerkleProof;

/** Verifica prova de Merkle. */
export function verifyProof(proof: MerkleProof, chunkHash: Uint8Array): boolean;
```

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/media test`.

Casos de teste:
1. `chunkFile` com 1 MB → 4 chunks de 256 KB; último chunk menor.
2. `assembleFile` recompõe arquivo idêntico ao original (SHA-256 confere).
3. Chunk adulterado → `assembleFile` produz hash diferente.
4. `buildMerkleTree` com 4 chunks → árvore balanceada, root hash calculado.
5. `generateProof` + `verifyProof` roundtrip para chunk[0].
6. `verifyProof` com chunk adulterado → false.
7. Arquivo de 5 MB → chunks + Merkle Tree em < 200ms.

## 5. Instruções de Execução
1. **[TDD]** Escreva testes com buffers de tamanhos variados.
2. Implemente `Chunker` com streams (iterador de chunks, sem carregar tudo em RAM).
3. Implemente `MerkleTree` com construção bottom-up.
4. Re-exporte. Rode build + test.
