# T-001 · Bootstrap do monorepo

## 1. Objetivo
Criar a fundação física do projeto: pnpm workspaces, turborepo, tsconfig base estrito, ESLint/Prettier, e Vitest configurado. Criar as pastas dos pacotes vazias exportando apenas a versão.

## 2. Contexto RAG (Spec-Driven Development)
- [visao-arquitetural.md](../docs/visao-arquitetural.md) (Seção de Arquitetura de Implementação)
- [plano-de-implementacao.md](../docs/plano-de-implementacao.md) (Seção 1.1 Monorepo)

## 3. Estratégia de Testes (Test-Driven Development)
- [ ] **Abordagem**: Testes estruturais via Vitest. O \`pnpm test\` deve rodar sem erros na raiz, executando os testes falsos/mínimos em cada pacote do monorepo.

## 4. Referências de Código
- Arquivos Alvo: `package.json` (raiz), `turbo.json`, `pnpm-workspace.yaml`, e as pastas `packages/*` e `apps/*`.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** use npm ou yarn. Use estritamente o \`pnpm\`.
> - **NÃO** instale bibliotecas não solicitadas (ex: lodash, react, etc). Instale APENAS as devDependencies estritamente necessárias para TypeScript, Vitest, Turborepo, ESLint e Prettier.
> - **NÃO** implemente nenhuma lógica real nos pacotes. Eles devem exportar apenas uma string ou constante simples (ex: \`export const VERSION = "0.0.1";\`).

1. **[TDD Estrutural]** Antes de configurar o Turbo, crie a árvore exata de pastas exigida na Seção 1.1 do plano (packages: `protocol`, `crypto`, `core`, `transport`, `media`, `workers`, `client-sdk`, `testkit`; apps: `system-peer`, `bancada`, `web`).
2. Em cada pacote, crie um `package.json` básico, um `src/index.ts` exportando uma constante vazia e um `tests/index.test.ts` do Vitest afirmando algo trivial (ex: `expect(true).toBe(true)`).
3. Adicione `pnpm-workspace.yaml` mapeando `packages/*` e `apps/*`.
4. Instale e configure o Turborepo (`turbo.json`) para orquestrar os pipelines de `build`, `lint` e `test`.
5. Configure um `tsconfig.base.json` na raiz com regras estritas (`strict: true`, `noImplicitAny: true`, `esModuleInterop: true`) para que todos os sub-pacotes o estendam.
6. Configure ESLint e Prettier na raiz.
7. Garanta que o comando `pnpm test` no diretório raiz chame o Turbo e rode todos os testes em paralelo com sucesso.
