# T-001 · Bootstrap do monorepo (Exemplo Gold Standard)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** \`pnpm\` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo (\`pnpm build\`, \`pnpm test\`, \`pnpm lint\` na raiz afetam todos os pacotes)
- **Test Runner:** \`vitest\` (pacotes core/protocol) e \`playwright\` (E2E/Frontend)
- **Capacidade-alvo:** haiku

## 1. Objetivo
Criar a fundação física do projeto (monorepo). Configurar \`pnpm workspaces\`, \`turborepo\`, \`tsconfig.base.json\`, linter, e instanciar os pacotes raiz (ainda vazios) exigidos pela arquitetura.

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [ ] **Framework:** \`vitest\` orquestrado pelo \`turborepo\`.
- [ ] **Métricas/Cobertura:** Apenas testes estruturais mockados nesta fase. Não há métrica de cobertura de linhas exigida na T-001. O critério de sucesso é binário: o CI de build passa ou não passa.
- [ ] **Ambiente do Teste:** Node puro, executado a partir do diretório raiz.
- [ ] **Fora de Escopo:** Testes lógicos reais.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** instale bibliotecas de features (\`react\`, \`express\`, \`sqlite\`). Instale APENAS devDependencies necessárias para fundação (\`turbo\`, \`vitest\`, \`typescript\`, \`eslint\`, \`typescript-eslint\`, \`prettier\`).
> - **NÃO** crie lógicas dentro dos \`index.ts\`. Exporte apenas \`export const VERSION = "0.0.1";\`.
> - **NÃO** edite arquivos fora do Escopo de Arquivos listado na Seção 3.
