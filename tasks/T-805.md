# T-805 · Pipeline de reidratação no browser

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `playwright` (E2E browser)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar pipeline de reidratação no browser: Service Worker intercepta `/blobs/{infohash}` → MessageChannel → Sync Worker → adapter → Crypto Worker (zero-copy transferables) → ReadableStream/SourceBuffer; tag_region buscada primeiro; backpressure 20/5 MB; chunks decifrados nunca expostos ao JS da página
*(derivado de `docs/plano-de-implementacao.md` §11 T-805)*.

### Contratos exatos (assinaturas TS fixadas)

```ts
// --- packages/workers/src/rehydration/pipeline.ts ---
import type { InfoHash } from '@plataforma/media'; // Derivado de T-801 §1 e T-804 §1

export interface RehydrationRequest {
  infoHash: InfoHash;
  range?: { start: number; end?: number };
  priority: 'streaming' | 'prefetch';
}

export interface RehydrationConfig {
  backpressureHighWaterMark: number;  // default 20 * 1024 * 1024 (20 MB) - Derivado de caderno-3-sdk/02 §8
  backpressureLowWaterMark: number;   // default 5 * 1024 * 1024 (5 MB) - Derivado de caderno-3-sdk/02 §8
  maxConcurrentChunks: number;        // default 4
}

export interface RehydrationPipeline {
  /** Inicia reidratação: intercepta /blobs/{infohash} no Service Worker fetch handler. */
  intercept(request: Request): Promise<Response>;

  /** Pipeline: Service Worker → MessageChannel → Sync Worker → Crypto Worker. */
  startPipeline(request: RehydrationRequest): Promise<ReadableStream<Uint8Array>>;

  /** Busca tag_region primeiro (metadados de decifração). Concatenação de 16B authTags por chunk (T-804 §1). */
  fetchTagRegion(infoHash: InfoHash): Promise<Uint8Array>;

  /** Aplica backpressure (pausa quando buffer > 20 MB, retoma quando < 5 MB). */
  applyBackpressure(bufferSize: number): boolean;
}

// Protocolo de Mensagens via MessageChannel (SW <-> Sync Worker)
export type SWMessage =
  | { type: 'REQUEST_CHUNK'; infoHash: string; index: number }
  | { type: 'PAUSE_STREAM'; infoHash: string }
  | { type: 'RESUME_STREAM'; infoHash: string };

export type SyncWorkerMessage =
  | { type: 'CHUNK_DATA'; infoHash: string; index: number; chunk: Uint8Array; authTag: Uint8Array; nonce: Uint8Array }
  | { type: 'ERROR'; infoHash: string; message: string };

// Protocolo do Crypto Worker (zero-copy transferables)
export interface CryptoDecryptionRequest {
  ciphertext: Uint8Array;
  key: Uint8Array;
  nonce: Uint8Array;
  authTag: Uint8Array;
}

export interface CryptoDecryptionResponse {
  plaintext: Uint8Array;
  error?: string;
}
```
- Service Worker intercepta `GET /blobs/{infohash}` → MessageChannel.
- Fluxo: Service Worker → MessageChannel → Sync Worker (seleciona adapter e busca chave AES no nó `ASSET:FILE` no DB SQLite/Automerge local por `infoHash`) → Crypto Worker (decifra) via zero-copy transferables.
- `tag_region` buscada primeiro — contém as tags de autenticação concatenadas para decifração (16 bytes por chunk, indexados em `offset = index * 16`, derivado de `T-804` §6 Decisão #2).
- Backpressure: pausa quando buffer > 20 MB, retoma quando < 5 MB (derivado de `caderno-3-sdk/02` §8 and `caderno-5-transport/01` §3.4).
- Chunks decifrados **nunca** expostos ao JS da página principal (isolamento via Worker).
- `<video>` toca MP4 cifrado via P2P/WebSeed.

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [x] **Framework:** Playwright (E2E browser testing).
- [x] **Ambiente do Teste:** `pnpm --filter @plataforma/bancada test:e2e` (runs in the browser).
- [x] **Fora de Escopo:** Testes unitários de Node puro (já cobertos por T-801/T-804).

Casos de teste — E2E Playwright (`apps/bancada/e2e/rehydration.test.ts`):
1. **Streaming E2E de Vídeo Cifrado:** Carrega uma página de teste na Bancada, inicia um player `<video src="/blobs/{infohash}?token=..." controls>`, e garante que o vídeo começa a reproduzir (`HTMLVideoElement.currentTime > 0` e evento `playing` disparado) consumindo dados decifrados em tempo real via Service Worker.
2. **Precedência de tagRegion:** Intercepta as requisições de rede no teste e valida que a requisição de download para a `tag_region` do arquivo (metadados) ocorre e é completada com sucesso antes que qualquer chunk de dados (peça do vídeo) seja solicitado ao Sync Worker / WebSeed.
3. **Backpressure de Alta (High Watermark):** Simula um player lento (ou pausado). Insere um atraso artificial no consumo do `ReadableStream` no Service Worker. Valida que o pipeline pausa o download de novos chunks do WebSeed/P2P quando a quantidade de bytes descriptografados em buffer no Service Worker excede `backpressureHighWaterMark` (20 MB).
4. **Resolução de Backpressure (Low Watermark):** Retoma o consumo do stream. Valida que o pipeline volta a solicitar novos chunks apenas após os bytes em buffer caírem abaixo de `backpressureLowWaterMark` (5 MB).
5. **Isolamento e Segurança:** Valida que nenhuma variável global ou objeto no escopo `window` do JavaScript da página principal contém trechos dos bytes descriptografados (garante isolamento estrito via Service Worker/Worker).
6. **Throttling e Resiliência:** Aplica limitação de banda via Playwright (`network.throttleNetwork`). Valida que o pipeline gerencia eventos de `PAUSE` e `RESUME` e conclui a reprodução sem quebras ou travamentos de stream.
7. **Fallback Swarm → WebSeed:** Configura o teste com zero seeders P2P ativos para o `infoHash`. Garante que o Service Worker executa o fallback transparente solicitando os chunks ao WebSeed via token, e o player reproduz o vídeo normalmente sem erros.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** exponha chunks decifrados ao JS da página principal. O pipeline deve rodar isolado no Service Worker/Crypto Worker.
> - **NÃO** pule a busca da tag_region antes dos chunks. Ela é necessária para obter as tags de integridade.
> - **NÃO** instale bibliotecas adicionais no Service Worker ou Workers — use as APIs nativas do browser (`ReadableStream`, `MessageChannel`, `WebCrypto`).

1. **[TDD]** Crie `apps/bancada/e2e/rehydration.test.ts` com os 7 casos de teste Playwright acima.
2. Implemente o Service Worker em `apps/web/public/sw.js` para interceptar requests de `/blobs/{infohash}`.
3. Implemente a comunicação por `MessageChannel` e o pipeline em `packages/workers/src/rehydration/pipeline.ts` de acordo com os contratos de interface.
4. Conecte o Service Worker ao Sync Worker e Crypto Worker usando transferência de buffers (zero-copy).
5. Implemente o controle de backpressure de 20MB/5MB na leitura do stream.
6. Re-exporte as interfaces e a classe principal em `packages/workers/src/index.ts`.
7. Execute o Gate de Evidência completo e anexe os resultados.
