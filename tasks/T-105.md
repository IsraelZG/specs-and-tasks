## 1. Objetivo
Implementar `DevicePeerId` e `PersonaPeerId` + parser minimalista de Multiaddr no pacote `@plataforma/protocol`
*(derivado de `docs/conceitos/peer-id.md` — DevicePeerId = blake2s256(DEVICE_PUB_KEY), PersonaPeerId = blake2s256(PERSONA_PUB_KEY); `docs/plano-de-implementacao.md` §4 T-105)*.

### Contratos exatos (assinaturas TS fixadas)

```ts
// --- packages/protocol/src/peerId.ts ---
import { Ed25519PublicKey } from '@plataforma/crypto'; // T-101

/** DevicePeerId = blake2s256(DEVICE_PUB_KEY). Identidade de transporte. 32 bytes. */
export type DevicePeerId = Uint8Array;

/** PersonaPeerId = blake2s256(PERSONA_PUB_KEY). Identidade de aplicação. 32 bytes. */
export type PersonaPeerId = Uint8Array;

/** Union type para funções genéricas. */
export type PeerId = DevicePeerId | PersonaPeerId;

/** Deriva DevicePeerId da chave pública do dispositivo. */
export function deriveDevicePeerId(devicePubKey: Ed25519PublicKey): DevicePeerId;

/** Deriva PersonaPeerId da chave pública da persona. */
export function derivePersonaPeerId(personaPubKey: Ed25519PublicKey): PersonaPeerId;

// === Multiaddr minimalista ===

/** Componente de um Multiaddr. Ex: { protocol: 'ip4', value: '127.0.0.1' }. */
export interface MultiaddrComponent {
  protocol: string;  // 'ip4' | 'dns4' | 'tcp' | 'ws' | 'wss' | 'p2p'
  value: string;
}

/** Multiaddr como string formatada. Ex: "/ip4/127.0.0.1/tcp/3000/ws/p2p/<peerId>". */
export type MultiaddrString = string;

/** Faz parse de string Multiaddr em componentes. */
export function parseMultiaddr(addr: MultiaddrString): MultiaddrComponent[];

/** Serializa componentes de volta para string Multiaddr. */
export function formatMultiaddr(components: MultiaddrComponent[]): MultiaddrString;

/** Codifica PeerId (32 bytes) como string base58btc para uso em Multiaddr. */
export function peerIdToString(peerId: PeerId): string;

/** Decodifica string base58btc → PeerId. */
export function stringToPeerId(str: string): PeerId;
```

### Protocolos suportados (derivado de plano-de-implementacao §1.2)
`ip4`, `dns4`, `tcp`, `ws`, `wss`, `p2p`. Sem dependência de `@multiformats/multiaddr`.

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/protocol test`.

Casos de teste:
1. `deriveDevicePeerId(keyA)` ≠ `derivePersonaPeerId(keyA)` (chaves diferentes para mesma pubkey).
2. `deriveDevicePeerId` determinístico: mesma pubkey → mesmo resultado.
3. `parseMultiaddr("/ip4/127.0.0.1/tcp/3000/ws")` → 4 componentes corretos.
4. `parseMultiaddr` + `formatMultiaddr` roundtrip.
5. `peerIdToString` + `stringToPeerId` roundtrip.
6. `parseMultiaddr("/p2p/<peerIdB58>")` → extrai peerId corretamente.
7. `parseMultiaddr` com protocolo desconhecido → erro (não suportado).
