## 1. Objetivo
Derivar `DevicePeerId` (identidade de transporte) e `PersonaPeerId` (identidade de aplicação)
via `blake2s256`; encoder/parser local de strings Multiaddr nos formatos restritos do projeto.

*(Derivado de: `docs/plano-de-implementacao.md` §4 T-105; `docs/caderno-2-protocol/02-cryptographic-lineage-and-auth.md` §1.4; `docs/caderno-5-transport/01-p2p-transport-and-reconciliation.md` §2.2; `docs/conceitos/peer-id.md`)*

### Contratos exatos (assinaturas TS fixadas)

```ts
// --- packages/protocol/src/peerId.ts ---

/** Derivado de caderno-2/02 §1.4 + caderno-5/01 §2.2 (RFC-005 §A.5). */
export type PeerId = string; // hex lowercase, 64 chars (32 bytes blake2s256)

/**
 * Identidade de transporte: derivada da chave Ed25519 estável e exclusiva do dispositivo.
 * DevicePeerId = blake2s256(DEVICE_PUB_KEY)
 * Derivado de: caderno-2/02 §1.4, caderno-5/01 §2.2
 */
export function deriveDevicePeerId(devicePublicKey: Uint8Array): PeerId;

/**
 * Identidade de aplicação: derivada da chave pública da persona.
 * PersonaPeerId = blake2s256(PROFILE:PERSONA_PUB_KEY)
 * Derivado de: caderno-2/02 §1.4, conceitos/peer-id.md
 */
export function derivePersonaPeerId(personaPublicKey: Uint8Array): PeerId;

// --- Multiaddr mínimo (parser local — NÃO instala @multiformats/multiaddr) ---

/**
 * Formatos suportados (derivado de plano §4 T-105):
 *   /dns4/<host>/tcp/<port>/wss
 *   /ip4/<addr>/tcp/<port>/wss
 *   /dns4/<host>/tcp/<port>/wss#<fragment>
 *   /ip4/<addr>/tcp/<port>/wss#<fragment>
 */
export interface ParsedMultiaddr {
  protocol: 'dns4' | 'ip4';
  host: string;       // hostname ou endereço IPv4
  port: number;       // 1–65535
  transport: 'wss';   // fixo
  fragment?: string;  // fragmento opcional após '#'
}

/**
 * Codifica uma ParsedMultiaddr em string canônica.
 * Derivado de: plano §4 T-105 "encode/parse de multiaddr básico"
 */
export function encodeMultiaddr(addr: ParsedMultiaddr): string;

/**
 * Faz parse de uma string Multiaddr para ParsedMultiaddr.
 * Retorna null se o formato não for suportado (não lança).
 * Derivado de: plano §4 T-105 "parsing round-trip"
 */
export function parseMultiaddr(raw: string): ParsedMultiaddr | null;
```

## 4. Estratégia de Testes Estrita (Test-Driven Development)

- **Framework:** Vitest (Node puro)
- **Ambiente:** `pnpm --filter @plataforma/protocol test`
- **Fora de Escopo:** Interações de UI, conexões de rede reais, WebRTC, Noise_XX

Casos de teste (numerados):

1. **Derivação determinística DevicePeerId:** mesma `devicePublicKey` (32 bytes) → mesmo `PeerId` em duas chamadas; resultado é string hex lowercase de 64 chars.
2. **Derivação determinística PersonaPeerId:** mesma `personaPublicKey` (32 bytes) → mesmo `PeerId`; resultado ≠ DevicePeerId (chaves diferentes).
3. **Distinção por tipo:** `deriveDevicePeerId(key)` ≠ `derivePersonaPeerId(key)` mesmo com a mesma chave de entrada (papéis ortogonais).
4. **Vetor blake2s:** `deriveDevicePeerId(Uint8Array(32).fill(0x01))` → valor hex esperado derivado de `blake2s256(bytes)` (calcular o valor esperado com `@noble/hashes/blake2s` como oráculo no teste).
5. **encode + parse round-trip dns4:** `parseMultiaddr(encodeMultiaddr({protocol:'dns4', host:'relay.example.com', port:443, transport:'wss'}))` → objeto original.
6. **encode + parse round-trip ip4:** `parseMultiaddr(encodeMultiaddr({protocol:'ip4', host:'192.168.1.1', port:8080, transport:'wss'}))` → objeto original.
7. **Fragment:** `parseMultiaddr('/dns4/relay.example.com/tcp/443/wss#abc123')` → `{...fragment:'abc123'}`.
8. **Formato inválido:** `parseMultiaddr('/dns4/host/tcp/443/ws')` → `null` (transport `ws` não suportado).
9. **Formato inválido:** `parseMultiaddr('')` → `null`; `parseMultiaddr('garbage')` → `null`.
10. **Porta fora de range:** `parseMultiaddr('/ip4/1.2.3.4/tcp/99999/wss')` → `null` (port > 65535).
