# T-110 · Key Vault v0

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo — pacote alvo: `packages/core/`
- **Test Runner:** `vitest` (Node puro)

## 1. Objetivo
Custódia temporária em RAM de chaves de época (AES-256-GCM) com TTL de **4 horas** via `ClockPort`.
API local `requestKey(scope)` para o Sync Worker + face de rede `requestEpochKey(ucan, scope, prova_de_delegação)`
servida via frames `KEY_REQUEST`/`KEY_RESPONSE` dentro do canal Noise (stub de validação UCAN nesta tarefa).

*(Derivado de: `docs/plano-de-implementacao.md` §4 T-110; `docs/caderno-2-protocol/02-cryptographic-lineage-and-auth.md` §2.2, §3.3.2; `docs/caderno-3-sdk/02-sync-worker-and-memory-lifecycle.md` §1.2, §1.5)*

### Contratos exatos (assinaturas TS fixadas)

```ts
// --- packages/core/src/keyVault.ts 
---

import type { ClockPort } from '@plataforma/protocol';

/** Entrada interna do vault — nunca serializada. Derivado de plano §4 T-110. */
interface VaultEntry {
  key: Uint8Array;     // 32 bytes — AES-256-GCM epoch key
  expiresAt: number;   // timestamp ms (via ClockPort.now())
}

/**
 * Resultado de requestKey / requestEpochKey.
 * Derivado de caderno-2/02 §3.3.2: retorna chave ou DENIED.
 */
export type KeyResult =
  | { ok: true; key: Uint8Array }
  | { ok: false; reason: 'not_found' | 'expired' | 'denied' };

/**
 * Payload de prova de delegação — stub para M1.
 * Validação real de UCAN/DELEGATED_TO implementada no M5.
 * Derivado de caderno-2/02 §3.3.2 e plano §4 T-110.
 */
export interface DelegationProof {
  ucan: string;             // token UCAN serializado (opaque nesta fase)
  delegatedTo: string;      // hex pubkey do dispositivo solicitante
}

/**
 * Key Vault in-memory.
 * Derivado de: caderno-3-sdk/02 §1.2 "Cofre de Chaves (Key Vault)"
 *              plano §4 T-110 "Custódia de chaves de época em RAM com TTL 4h"
 */
export class KeyVault {
  /** TTL padrão = 4 horas em ms. Derivado de caderno-2/02 §3.1 + plano §4 T-110. */
  static readonly DEFAULT_TTL_MS = 4 * 60 * 60 * 1000; // 14_400_000

  constructor(clock: ClockPort);

  /**
   * Armazena uma chave de época para um escopo.
   * ttlMs: padrão = DEFAULT_TTL_MS.
   * Derivado de: plano §4 T-110
   */
  store(scope: string, key: Uint8Array, ttlMs?: number): void;

  /**
   * API LOCAL — consumo do Sync Worker para decifrar payloads.
   * Nunca exposta a peers remotos.
   * Derivado de: caderno-3-sdk/02 §1.5 + caderno-2/02 §3.3.2
   */
  requestKey(scope: string): KeyResult;

  /**
   * API DE REDE — servida a peers remotos dentro do canal Noise
   * via frames KEY_REQUEST/KEY_RESPONSE.
   * Stub M1: valida apenas que delegationProof não é vazio;
   *          nega se scope não existe ou chave expirada.
   * Validação real UCAN/DELEGATED_TO: M5.
   * Derivado de: plano §4 T-110, caderno-3-sdk/02 §1.5, caderno-2/02 §3.3.2
   */
  requestEpochKey(
    scope: string,
    proof: DelegationProof
  ): KeyResult;

  /**
   * Remove entradas expiradas (chamado internamente pelo tick do ClockPort).
   * Derivado de: plano §4 T-110 "expiração de TTL limpa material"
   */
  purgeExpired(): number; // retorna quantidade de entradas removidas

  /**
   * Retorna o número de entradas ativas (não expiradas).
   * Usado em testes de não-vazamento por referência.
   */
  size(): number;
}
```

## 4. Estratégia de Testes Estrita (Test-Driven Development)

- **Framework:** Vitest (Node puro) + `VirtualClock` do `@plataforma/testkit` (dep de dev)
- **Ambiente:** `pnpm --filter @plataforma/core test`
- **Fora de Escopo:** Persistência em DB, socket real, Noise_XX, validação UCAN real

Casos de teste (numerados):

1. `store(scope, key)` + `requestKey(scope)` → `{ok:true, key}` com o mesmo `Uint8Array`.
2. `requestKey('inexistente')` → `{ok:false, reason:'not_found'}`.
3. **Expiração por TTL:** `store(scope, key, 1000)` → avança relógio 1001ms via `VirtualClock.advance(1001)` → `requestKey(scope)` → `{ok:false, reason:'expired'}`.
4. **purgeExpired:** após expiração de 2 entradas, `purgeExpired()` retorna `2`; `size()` diminui.
5. **TTL default = 4h:** store sem ttlMs; avança 4h-1ms → `ok:true`; avança +1ms → `expired`.
6. `requestEpochKey(scope, validProof)` com chave ativa → `{ok:true, key}`.
7. `requestEpochKey(scope, validProof)` com chave expirada → `{ok:false, reason:'expired'}`.
8. `requestEpochKey('inexistente', validProof)` → `{ok:false, reason:'not_found'}`.
9. **Stub de delegação:** `requestEpochKey(scope, {ucan:'', delegatedTo:''})` → `{ok:false, reason:'denied'}` (proof vazia = negada neste stub).
10. **Não-vazamento por referência:** armazena key; modifica o `Uint8Array` original externamente; `requestKey` ainda retorna os bytes originais (implementação deve copiar ou congelar).
11. **Vetor adversarial 7 (plano §2.5):** `requestKey` e `requestEpochKey` não expõem material de chave expirado — após expiração, a referência interna é nulificada ou substituída.

## 5. Instruções de Execução (Step-by-Step)

> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** persista as chaves de época na DB local neste momento. O Key Vault é estritamente *in-memory* (transitório).
> - **NÃO** implemente validação real de UCAN/cadeia de delegação nesta tarefa — é um **stub** (validação real no M5/T-505).
> - **NÃO** exponha `requestKey` a peers remotos. Apenas `requestEpochKey` tem face de rede.
> - **NÃO** use `Map` sem limpeza — chaves expiradas devem ser removidas via `purgeExpired`.

1. **[TDD]** Crie `packages/core/tests/keyVault.test.ts` com os 11 casos da Seção 4.
2. Implemente a classe `KeyVault` com o `Map<string, VaultEntry>` interno.
3. Implemente `store` (copia `key` com `Uint8Array.from(key)`, calcula `expiresAt = clock.now() + ttlMs`).
4. Implemente `requestKey` (verifica expiração via `clock.now()`, retorna `KeyResult`).
5. Implemente `requestEpochKey` (stub: nega se `proof.ucan === '' || proof.delegatedTo === ''`, depois delega para `requestKey`).
6. Implemente `purgeExpired` (itera o Map, remove expirados, retorna contador).
7. Injete `purgeExpired` no `ClockPort` como tick periódico (opcional nesta fase — pode ser chamado manualmente nos testes).
8. Re-exporte em `packages/core/src/index.ts`.
9. Rode build + test (Seção 7) e cole saída no handover.
