## 1. Objetivo
Implementar `KeyVault` — custódia temporária em RAM de chaves de época AES com TTL, no pacote `@plataforma/core`
*(derivado de `docs/visao-arquitetural.md` §7 — "Chaves de época (AES-256-GCM) são efêmeras (TTL 4 h), entregues sob demanda pelo Key Vault mediante UCAN válido"; `docs/plano-de-implementacao.md` §4 T-110)*.

### Contratos exatos (assinaturas TS fixadas)

```ts
// --- packages/core/src/keyVault.ts ---
import { ClockPort } from '@plataforma/protocol'; // T-004
import { AesKey } from '@plataforma/crypto';      // T-101

/** Entrada no Key Vault. */
export interface VaultEntry {
  key: AesKey;
  expiresAt: number; // timestamp ms (ClockPort.now() + TTL)
}

/** Opções de construção. */
export interface KeyVaultOptions {
  clock: ClockPort;
  defaultTTL?: number; // ms, default: 4 * 60 * 60 * 1000 (4h)
}

/** Cofre de chaves em RAM. Chaves expiradas são removidas sob consulta. */
export class KeyVault {
  constructor(opts: KeyVaultOptions);

  /** Armazena chave sob `keyId`. Sobrescreve se já existir. */
  set(keyId: string, key: AesKey, ttl?: number): void;

  /** Recupera chave. Retorna null se não encontrada ou expirada (remove expiradas). */
  get(keyId: string): AesKey | null;

  /** Verifica se chave existe e não expirou. */
  has(keyId: string): boolean;

  /** Remove chave explicitamente. */
  delete(keyId: string): void;

  /** Remove todas as chaves expiradas. Retorna contagem de removidas. */
  purge(): number;

  /** Número de entradas ativas no vault. */
  get size(): number;
}
```

### Invariantes (derivado de visao-arquitetural.md §7)
- Chaves **nunca** são persistidas em disco/DB — apenas RAM.
- TTL default = 4 horas (14.400.000 ms).
- `get()` remove entradas expiradas (lazy eviction).
- `ClockPort` é injetado para testabilidade (VirtualClock de T-003).

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/core test`. Usar `VirtualClock` de T-003.

Casos de teste:
1. `set` + `get` → retorna a chave; `has` → true.
2. `get` para keyId inexistente → null.
3. `set` sobrescreve chave existente.
4. `delete` remove; `get` → null; `has` → false.
5. TTL expirado: `clock.advance(ttl + 1)` → `get` retorna null e remove entrada.
6. `purge` remove entradas expiradas, preserva válidas.
7. `size` reflete entradas ativas (não conta expiradas após `get`).
8. `VirtualClock` não avançado → chave dentro do TTL continua válida.
