# T-507 · STALE_EPOCH no transporte

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar tratamento de `STALE_EPOCH` no transporte no pacote `@plataforma/transport`: wire frames carregam `identity_epoch_index`; divergência de época de identidade durante transferência interrompe RBSR com `STALE_EPOCH` e força catch-up
*(derivado de `docs/plano-de-implementacao.md` §8 T-507)*.

### Contratos exatos
```ts
// --- packages/transport/src/staleEpoch.ts ---
import { WireFrame } from '@plataforma/protocol'; // T-201b
import { RBSRSession } from '@plataforma/protocol'; // T-302a

export const STALE_EPOCH_ERROR = 'STALE_EPOCH' as const;

export interface StaleEpochEvent {
  localEpochIndex: number;
  remoteEpochIndex: number;
  peerId: string;
  frameType: number;
  interruptedSession: string | null; // RBSR session ID ou null
}

export interface StaleEpochHandler {
  /** Detecta STALE_EPOCH comparando identity_epoch_index do frame com local. */
  detect(frame: WireFrame, localEpochIndex: number): boolean;

  /** Interrompe RBSR session se STALE_EPOCH detectado. */
  interruptRBSR(session: RBSRSession, event: StaleEpochEvent): Promise<void>;

  /** Força catch-up de identidade (desvia para pipeline Onda 0 — T-203). */
  triggerCatchUp(event: StaleEpochEvent): Promise<void>;

  /** Callback quando STALE_EPOCH ocorre. */
  onStaleEpoch(handler: (event: StaleEpochEvent) => void): void;
}
```
- `STALE_EPOCH` refere-se exclusivamente à Época de Identidade (`identity_epoch_index`).
- Divergência de época de conteúdo é tratada pelo Key Vault na aplicação, nunca como erro de transporte.
- `detect`: compara `frame.identity_epoch_index !== localEpochIndex` → se divergente durante transferência, é `STALE_EPOCH`.
- `interruptRBSR`: para a sessão RBSR corrente, emite evento, e desvia para catch-up de identidade (Onda 0 via T-203).
- Vetor adversarial 2: `STALE_EPOCH` no meio de RBSR → interrupção + desvio para catch-up Onda 0.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest + SimNetwork (T-005) + VirtualClock (T-003).
- [x] **Ambiente:** `pnpm --filter @plataforma/transport test`.

Casos de teste:
1. `detect` com épocas iguais → false.
2. `detect` com épocas divergentes → true.
3. `interruptRBSR` para sessão ativa → estado da sessão alterado.
4. Vetor adversarial 2: rotação de época no meio de sync longo → `STALE_EPOCH` emitido.
5. Após `STALE_EPOCH`: `triggerCatchUp` desvia para pipeline de catch-up (Onda 0).
6. `STALE_EPOCH` não confunde época de identidade com época de conteúdo.
7. Frame com `identity_epoch_index` divergente e em handshake → tratado por T-203, não por T-507.
8. Múltiplos frames com épocas divergentes → apenas o primeiro gera evento.

## 5. Instruções de Execução

> - **NÃO** confunda época de identidade com época de conteúdo.
> - **NÃO** emita `STALE_EPOCH` durante o handshake — isso é responsabilidade do T-203.
