## 1. Objetivo
Implementar adapter WebTorrent no pacote `@plataforma/transport`: download por WebRTC + tracker WSS privado (T-408); entrega de chunks ao pipeline de decifração
*(derivado de `docs/plano-de-implementacao.md` §11 T-803)*.

### Contratos exatos
```ts
// --- packages/transport/src/media/webTorrentAdapter.ts 
---
import { InfoHash } from '@plataforma/protocol'; // T-801a

export interface TorrentPeerInfo {
  peerId: string;
  ip: string;
  port: number;
  progress: number;             // 0–1
  downloadSpeed: number;         // bytes/s
}

export interface ChunkDelivery {
  chunkIndex: number;
  data: Uint8Array;             // chunk cifrado
  fromPeerId: string;
  receivedAt: number;
}

export interface WebTorrentConfig {
  trackerWsUrl: string;         // URL do tracker WSS (T-408)
  maxPeers: number;             // default 50
  uploadLimitBps?: number;
  downloadLimitBps?: number;
}

export class WebTorrentAdapter {
  constructor(config: WebTorrentConfig);

  /** Inicia download de um blob por InfoHash. */
  startDownload(infoHash: InfoHash): Promise<void>;

  /** Inicia seeding de um blob. */
  startSeed(infoHash: InfoHash, data: Uint8Array[]): Promise<void>;

  /** Recebe chunks para pipeline de decifração. */
  onChunk(handler: (delivery: ChunkDelivery) => void): void;

  /** Lista peers conectados no swarm. */
  listPeers(infoHash: InfoHash): Promise<TorrentPeerInfo[]>;

  /** Pausa download. */
  pause(infoHash: InfoHash): void;

  /** Resume download. */
  resume(infoHash: InfoHash): void;

  /** Para download/seed e libera recursos. */
  stop(infoHash: InfoHash): void;
}
```
- `startDownload`: conecta ao tracker WSS (T-408) → obtém lista de peers → estabelece DataChannel (WebRTC) → baixa chunks.
- `startSeed`: registra no tracker como seeder → serve chunks para peers conectados.
- `onChunk`: entrega chunks ao pipeline de decifração — adapter não decifra.
- Dois browsers seedando/baixando um blob cifrado de 50 MB no Anel 3.

## 4. Estratégia de Testes
- [x] **Framework:** Playwright (2+ browser contexts).
- [x] **Ambiente:** `pnpm --filter bancada test:e2e` (Anel 3).

Casos de teste (Playwright):
1. Dois browsers: A faz seed, B faz download → B recebe todos os chunks.
2. `listPeers` mostra seeders e leechers corretamente.
3. `pause` pausa download → chunks param de chegar.
4. `resume` retoma download após pausa.
5. `stop` libera recursos e fecha conexões.
6. Chunks entregues cifrados (não-decifrados pelo adapter).
