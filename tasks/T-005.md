## 1. Objetivo
Criar o `SimNetwork` v1 no pacote `@plataforma/testkit`. É uma implementação `in-memory` do contrato `NetworkAdapterPort` que conecta múltiplos peers (instâncias) dentro do mesmo processo Node.js para testes de integração ultra-rápidos.
*(derivado de `docs/plano-de-implementacao.md` §2.1 Anel 1 — Simulação Determinística; contrato `NetworkAdapterPort` em T-004 §1)*.

### Contratos exatos (assinaturas TS fixadas)

```ts
// --- packages/testkit/src/SimNetwork.ts ---
import { NetworkAdapterPort, PeerId, WireData, MessageHandler } from '@plataforma/protocol';
import { VirtualClock } from './clock';

/** Hub de rede simulada in-memory para N peers. */
export class SimNetwork {
  /** @param clock Relógio virtual para latência. Se omitido, entrega é síncrona/assíncrona imediata. */
  constructor(clock?: VirtualClock);

  /** Cria um adapter que implementa NetworkAdapterPort para o peer informado. */
  createAdapter(peerId: PeerId): NetworkAdapterPort;

  /** Particiona a rede: mensagens que cruzam fronteiras de grupos são descartadas. */
  partition(...groups: PeerId[][]): void;

  /** Restaura conectividade total entre todos os peers. */
  heal(): void;
}
```

Cada adapter retornado por `createAdapter` implementa `NetworkAdapterPort`:
- `connect(to)`: registra intenção de conexão (no-op imediato em v1; mantido por contrato).
- `listen()`: no-op (adapters são automaticamente registrados no hub ao serem criados).
- `send(to, data)`: entrega `data` no `onMessage` do adapter de `to`. Se `clock` foi provido no `SimNetwork`, agenda via `clock.setTimeout(callback, 0)` — entrega assíncrona com latência zero fixa.
- `onMessage(handler)`: registra callback para mensagens recebidas.
- `close()`: remove o adapter do hub; `send` para peer fechado é no-op silencioso.

Casos de teste (numerados):
1. `createAdapter` retorna objeto que implementa `NetworkAdapterPort` (todos os 5 métodos presentes).
2. 2 peers: A envia `send('B', data)` → B recebe via `onMessage` com o mesmo `data` e `peerId`='A'.
3. 3 peers em cadeia: A→B, B→C, C→A — todos recebem exatamente 1 mensagem cada.
4. `partition(['A'], ['B'])`: A→B é descartado (B não recebe). A→A (self-send) funciona.
5. `partition(['A','B'], ['C'])`: A→B funciona, A→C é descartado, C→B é descartado.
6. `heal()` após partição: conectividade total restaurada, A→B funciona novamente.
7. `close()` em um adapter: `send` para peer fechado é no-op (não dispara erro nem callback).
8. `VirtualClock` integrado: `send` agenda via `clock.setTimeout`; `clock.advance(0)` dispara a entrega.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** use WebSockets reais, TCP, ou EventEmitter global nativo que quebre se instanciado em múltiplos testes paralelos. Toda a topologia deve ficar dentro do objeto `SimNetwork`.
> - **NÃO** escreva lógicas complexas de degradação agora (perda de pacotes, jitter, NAT). Isso é escopo da T-006. Aqui assumimos entrega confiável apenas limitada pelas partições manuais.
> - **NÃO** implemente `connect()` com handshake — no-op em v1.
> - **NÃO** use `VirtualClock.advance()` com valores >0 — latência zero fixa em v1.

1. **[TDD]** Crie `packages/testkit/tests/SimNetwork.test.ts` com casos 1–8 da Seção 4. Falham (classe não existe).
2. Implemente `SimNetwork` em `packages/testkit/src/SimNetwork.ts`: hub com `Map<PeerId, Adapter>`, adapter interno que implementa `NetworkAdapterPort`.
3. `createAdapter(peerId)`: instancia adapter, registra no `Map`, retorna.
4. `send(to, data)`: se particionado (peer origem e destino em grupos diferentes) → descarta. Senão: se `clock` existe → `clock.setTimeout(() => target.onMessage(peerId, data), 0)`; senão → `Promise.resolve().then(() => target.onMessage(...))` (microtask).
5. `partition(...groups)`: armazena regra. Mensagem é bloqueada se `senderGroup !== receiverGroup`.
6. `heal()`: limpa regra de partição.
7. `close()`: remove adapter do `Map`.
