## 1. Objetivo
Implementar `SimNetwork` v1 no pacote `@plataforma/testkit` — hub in-memory multi-peer que implementa `NetworkAdapterPort` por peer, com API de partição/heal para testes de topologia
*(derivado de `docs/plano-de-implementacao.md` §2.1 Anel 1 — Simulação Determinística; `T-004` §1 — `NetworkAdapterPort`)*.

### Contratos exatos
```ts
// --- packages/testkit/src/SimNetwork.ts ---
import { NetworkAdapterPort, PeerId, WireData, MessageHandler } from '@plataforma/protocol'; // T-004

/** Hub central que simula rede in-memory entre N peers. */
export class SimNetwork {
  constructor();

  /** Cria adapter para `peerId`. Registra no hub. */
  createAdapter(peerId: PeerId): NetworkAdapterPort;

  /** Particiona a rede em grupos. Tráfego entre grupos diferentes é descartado. */
  partition(groups: PeerId[][]): void;

  /** Restaura conectividade total entre todos os peers. */
  heal(): void;

  /** Latência artificial em ms para todas as mensagens (default: 0). */
  setLatency(ms: number): void;

  /** Número de mensagens em trânsito (fila de latência). */
  get pendingCount(): number;

  /** Força entrega de todas as mensagens pendentes. */
  flush(): Promise<void>;
}
```
- `createAdapter`: cada peer ganha um adapter com `send()` → roteado pelo hub para o destino se não houver partição entre eles.
- `partition([["A","B"],["C"]])`: mensagens A↔B ok, C isolado.
- Entrega é assíncrona (microtask) para simular rede real, mesmo com latência 0.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/testkit test`.

Casos de teste:
1. 2 peers: `A.send(B, data)` → `B.onMessage` recebe os dados.
2. 3 peers: `A → B` e `A → C` entregues corretamente.
3. `partition([["A"],["B"]])`: `A.send(B)` → nunca chega em B.
4. `heal()` após partição → `A.send(B)` volta a funcionar.
5. `setLatency(50)` + `flush()` → mensagens entregues após flush.
6. `createAdapter` retorna objeto que implementa `NetworkAdapterPort` (type-check).
7. `connect`/`listen` no adapter são no-op (SimNetwork não precisa de conexão explícita).
