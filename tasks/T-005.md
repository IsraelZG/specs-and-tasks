# T-005 · SimNetwork v1

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (pacotes core) / `playwright` (E2E/Frontend)

## 1. Objetivo
Criar o \`SimNetwork\` v1 no pacote \`testkit\`. É uma implementação \`in-memory\` do contrato \`NetworkAdapterPort\` que conectará múltiplos peers (instâncias) dentro do mesmo processo Node.js para testes de integração ultra-rápidos.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** use WebSockets reais, TCP, ou EventEmitter global nativo que quebre se instanciado em múltiplos testes paralelos. Toda a topologia deve ficar dentro do objeto \`SimNetwork\`.
> - **NÃO** escreva lógicas complexas de degradação agora (perda de pacotes, jitter, NAT). Isso é escopo da T-006. Aqui assumimos entrega confiável apenas limitada pelas partições manuais.

1. **[TDD]** No diretório `tests/` do pacote `testkit`, escreva os testes do cenário de 3 peers trocando mensagens com partição e cura.
2. Implemente a classe `SimNetwork`. Ela atua como um hub central onde cada `NetworkAdapterPort` gerado (via um método tipo `createAdapter(peerId)`) se registra.
3. Quando o adapter do peer `A` chamar `send('B', data)`, o \`SimNetwork\` deve usar `VirtualClock` (se necessário para latência fixa) ou simplesmente entregar os dados síncrona/assincronamente no `onMessage` do adapter de `B`.
4. Implemente a API de API de controle do hub: `net.partition(grupos)`, que define regras de drop para qualquer tráfego que cruze as fronteiras dos grupos definidos, e `net.heal()`, que restaura conectividade total.
5. Garanta que a asserção no teste passe perfeitamente.
