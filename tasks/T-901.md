# T-901 · PWA produto v0 (apps/web) — prova de consumo do SDK

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+ / browser (PWA)
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `playwright` (E2E/PWA) + `vitest` (unit onde couber)
- **Capacidade-alvo:** opus-spike *(ver Seção 6 — app inteiro, superfície do client-sdk não fixada; candidato a épico/subdivisão antes de virar `ready`)*

## 1. Objetivo
Construir um shell PWA instalável e offline-first em `apps/web` que **consome o `client-sdk`**
(não a Bancada) para provar que o SDK é utilizável por um produto externo: login nos 3 modelos de
identidade, lista/edição de um tipo de `CONTENT` colaborativo, e indicador de conectividade/topologia
*(derivado de `docs/plano-de-implementacao.md` §12 — "T-901 · PWA produto v0")*.

## 2. Contexto RAG
- [plano-de-implementacao.md](../docs/plano-de-implementacao.md) (§12 · T-901; §2.3 — Playwright multi-context)
- App de referência de consumo do SDK: `apps/bancada` (T-008) e reset/superfície de controle (T-009)
- `client-sdk` (pacote consumido) — **superfície a fixar (ver Seção 6)**

## 3. Escopo de Arquivos
- **[CREATE]** `apps/web/**` (Vite + React, PWA: manifest + service worker)
- **[CREATE]** `apps/web/tests/*.spec.ts` (Playwright)
- **[READ]** `apps/bancada/**` (padrões de consumo do client-sdk já estabelecidos em T-008/T-009)

## 4. Estratégia de Testes Estrita
- [x] **Framework:** Playwright (browser real). Unit/JSDOM **não** basta para esta UI.
- [x] **Cenário E2E:** abrir → instalar PWA → login → editar um `CONTENT` → ficar offline
      (`context.setOffline`) → editar → voltar online → confirmar sincronização.
- [x] **Métrica de aceite:** Lighthouse PWA ≥ 90; funciona offline após o primeiro load.
- [ ] **Fora de escopo:** definir (ver Seção 6 — quais features de produto entram no v0).

## 5. Instruções de Execução
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** reimplemente lógica de protocolo na app — consuma só a API pública do `client-sdk`.
> - **NÃO** copie a Bancada inteira; a Bancada é ferramenta de teste, `apps/web` é produto.

1. Scaffold Vite+React PWA com manifest + service worker.
2. Religar login (3 modelos) sobre o client-sdk.
3. Tela de lista/edição de um `CONTENT` colaborativo + indicador de topologia.
4. Escrever o E2E Playwright multi-context. Rodar build + e2e.

## 6. Feedback de Especificação
> **DECISÕES EM ABERTO — escaladas, não inventadas (bloqueiam `draft → ready`):**
> - **Superfície do client-sdk:** o plano não fixa a API pública consumida (métodos de login,
>   subscribe a `CONTENT`, status de conexão). Precisa apontar a(s) task(s) que definem o
>   `client-sdk` e citar suas assinaturas — sem isso, contratos seriam inventados.
> - **"Um tipo de CONTENT colaborativo":** qual SPEC concreta (doc? nota?) é o v0?
> - **Dimensionamento:** um PWA completo excede Sonnet. Decidir: subdividir em épico
>   (scaffold+PWA / login / editor+sync / E2E) ou tratar como `opus-spike` com PoC.
