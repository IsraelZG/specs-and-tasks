# T-004 · Portas fundamentais

## 1. Objetivo
Definir as abstrações centrais de injeção de dependência no pacote `@plataforma/protocol`: \`NetworkAdapterPort\`, \`StoragePort\`, \`KeyStorePort\`, \`ClockPort\`, \`RandomPort\` e \`LoggerPort\`.

## 2. Contexto RAG (Spec-Driven Development)
- [plano-de-implementacao.md](../docs/plano-de-implementacao.md) (Seção 0.1, Item 4: "Portas antes de adapters")

## 3. Estratégia de Testes (Test-Driven Development)
- [ ] **Abordagem**: Verificação via Type-Check (`tsc --noEmit`). Testes Vitest importando as interfaces para simular stubs que as implementem, garantindo que o compilador aprove.

## 4. Referências de Código
- Arquivos Alvo: `packages/protocol/src/ports.ts` ou arquivos separados em `packages/protocol/src/ports/`.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** implemente as lógicas reais de rede, storage, ou banco de dados ainda. O pacote \`protocol\` é PURO e isomórfico. Ele só define os contratos (Types/Interfaces).
> - **NÃO** importe bibliotecas nativas como \`fs\`, \`window\`, ou \`crypto\` neste pacote. 
> - **NÃO** crie tipos globais vagos (`any` ou `unknown` onde tipos literais podem ser usados).

1. **[TDD]** No diretório `tests/` do pacote `protocol`, crie um arquivo mock \`ports.test.ts\`. Escreva classes Stub vazias (ex: \`class DummyClock implements ClockPort\`) e ateste que elas tipam corretamente. Elas darão erro inicial pois as portas não existem.
2. Crie as definições das interfaces em `packages/protocol/src/ports/`. 
   - \`NetworkAdapterPort\`: Métodos \`connect()\`, \`listen()\`, \`send(peerId, data)\`, \`onMessage(callback)\`, \`close()\`.
   - \`StoragePort\`: Métodos genéricos de \`exec()\`, \`transaction()\`, \`migrate()\`.
   - \`ClockPort\`: Retorna o tempo atual em milissegundos.
   - \`RandomPort\`: Retorna bytes aleatórios gerados.
   - \`LoggerPort\`: Logs de debug/info.
3. Exporte todas as interfaces no `src/index.ts` do `protocol`.
4. Documente as interfaces com JSDoc explicando as premissas (ex: "ClockPort deve ser monotônico").
5. Rode `pnpm test` e o linter para validar.
