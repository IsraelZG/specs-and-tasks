# T-1031 · Skills e docs: alinhar ao novo ciclo de vida MGTIA

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Repo:** Docs (controle). Edição de markdown de skills + docs; sem build/test de pacote.
- **Capacidade-alvo:** sonnet *(muitos arquivos, mudanças cirúrgicas e consistentes — precisa cuidado, não é mecânico puro)*
- **NÃO renomeia skills** (decisão do humano 2026-07-01: rename fica para outra task).

## 1. Objetivo
Alinhar as skills e a documentação ao ciclo de vida novo (T-1028/017/018): verbos de endurecimento pelo serviço, `in_review`+`claim`, `draft:<sub>`, auto-promote e auto-close-pai automáticos. As skills param de editar `spec_status`/`status` no frontmatter à mão e passam a chamar os verbos do serviço; os docs canônicos descrevem o novo fluxo.

## 3. Escopo de Arquivos (Inputs e Outputs)
Skills (`.claude/skills/<nome>/SKILL.md`):
- **[UPDATE]** `endurecer-task` — trocar edição de `spec_status:` no frontmatter por chamadas ao serviço: `triage` / `harden` / `block_decision` / `decompose` (via `manage-task.mjs <verbo> <ID> <EU> "msg"`). Remover carimbo manual de `hardened_at` (derivado do Log agora).
- **[UPDATE]** `endurecer-fila` — idem, em lote.
- **[UPDATE]** `arquiteto-decisoes` — usar `decide` (draft:pending_decision → draft:hardened) via serviço em vez de reescrever `spec_status`.
- **[UPDATE]** `arquiteto-promover` — passa a ser **safety-net**: a promoção agora é automática (T-1029). A skill só varre e chama `promote` no que sobrou em `draft:hardened` com deps done (raro). Documentar que é rede de segurança, não passo obrigatório.
- **[UPDATE]** `qa-review` — **primeiro passo vira `claim`** (`review → in_review`) antes de auditar, para travar a task; se `claim` falhar porque já está `in_review`, outro reviewer pegou → a skill PARA (não rouba a revisão).
- **[UPDATE]** `integrar-task` — remover o passo manual de "encerra pais decomposed" (agora é `parentAutoClose` automático no serviço, T-1029); manter o resto. Notar que auto-promote de dependentes também é automático.
- **[UPDATE]** `vincular-rag` — se escrever `spec_status`, remover (deps continuam sendo inferidas normalmente).
- **[UPDATE]** `executar-task`, `rework-task` — conferir referências a `status`/`spec_status`; ajustar prosa se citarem o fluxo antigo (ex.: "task em draft" → "draft:*").

Docs:
- **[UPDATE]** `docs/conceitos/mgtia-workflow.md` — novos status (`draft:<sub>`, `in_review`), verbos de endurecimento, auto-promote, auto-close-pai, lock de revisão. Substituir a descrição do eixo `spec_status` pela do `draft:<sub>`.
- **[UPDATE]** `CLAUDE.md` — bloco MGTIA: ciclo de vida atualizado (`draft:placeholder→…→ready→in_progress→review→in_review→done|rework`), remover a prosa do eixo `spec_status` separado, citar os verbos novos e os automatismos. Manter a regra de identidade e a de paralelismo (intactas).
