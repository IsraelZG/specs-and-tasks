# T-008 · App Bancada (PWA-ready)

## 1. Objetivo
Inicializar a "Bancada", que será a UI puramente de testes para o sistema P2P. É um PWA em Vite + React que servirá como cockpit de diagnóstico, exibindo abas para Identidade, Rede, Sync, etc.

## 2. Contexto RAG (Spec-Driven Development)
- [plano-de-implementacao.md](../docs/plano-de-implementacao.md) (Seção 3, T-008)

## 3. Estratégia de Testes (Test-Driven Development)
- [ ] **Abordagem (Para Agentes Cegos)**: Como esta é uma tarefa visual, o teste obrigatório será via \`@testing-library/react\` renderizando no JSDOM para certificar que o shell base, a navegação de abas e o registro do Service Worker estão montados sem erros de react/javascript. Não adicione testes de Playwright profundos no M0 se não houver backend para testar contra. 

## 4. Referências de Código
- Arquivos Alvo: `apps/bancada/index.html`, `apps/bancada/src/App.tsx`, `apps/bancada/vite.config.ts`.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** importe nenhuma biblioteca de componentes pesados (MUI, Ant Design). Use TailwindCSS purista ou CSS vanilla para a bancada (é uma ferramenta interna de diagnóstico).
> - **NÃO** crie lógicas de banco de dados (TinyBase/SQLite) na Bancada agora. Isso entra a partir da T-111 e M2.
> - **NÃO** ignore o manifesto do PWA. A instalação offline-first precisa do Service Worker e do \`vite-plugin-pwa\` configurados.

1. **[TDD Estrutural]** Adicione o pacote \`@testing-library/react\`. Crie o teste \`App.test.tsx\` verificando a renderização dos botões (abas): "Identidade", "Rede", "Sync", "Auth", "Dados", "Cenários".
2. No \`apps/bancada\`, configure o Vite e o plugin de PWA (\`vite-plugin-pwa\`). Crie um \`manifest.json\` válido com cores simples e instale a estratégia `generateSW`.
3. Implemente a UI: um layout simples tipo shell, com um sidebar/header para navegação entre as abas descritas acima. As abas podem renderizar conteúdos "Em Construção".
4. Verifique que o Vite Build passa com sucesso e os assets da PWA são gerados sem erro.
