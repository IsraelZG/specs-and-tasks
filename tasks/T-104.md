# T-104 · BIP39 + Derivação de Chaves + Desbloqueio

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo — pacote alvo: `packages/crypto/`
- **Test Runner:** `vitest` (Node puro, isomórfico)

## 1. Objetivo
Mnemônico de 12/24 palavras → chave mestra Ed25519; chave do dispositivo via PBKDF2 da senha.
A senha é **fator de desbloqueio local**, nunca identidade de rede.

*(Derivado de: `docs/plano-de-implementacao.md` §4 T-104; `docs/caderno-2-protocol/02-cryptographic-lineage-and-auth.md` §4.1.1 e §4.3)*

### Contratos exatos (assinaturas TS fixadas)

```ts
// --- packages/crypto/src/bip39.ts ---

/**
 * Gera um mnemônico BIP39 aleatório.
 * strength: 128 → 12 palavras | 256 → 24 palavras
 * Derivado de: plano §4 T-104; caderno-2/02 §4.3
 */
export function generateMnemonic(strength: 128 | 256): string;

/**
 * Valida se um mnemônico é válido (checksum BIP39).
 * Derivado de: plano §4 T-104 — "mesmo mnemônico ⇒ mesmas chaves"
 */
export function validateMnemonic(mnemonic: string): boolean;

/**
 * Converte mnemônico em seed de 64 bytes (BIP39 PBKDF2-HMAC-SHA512, 2048 rounds).
 * passphrase: "" por default (BIP39 standard).
 * Derivado de: plano §4 T-104
 */
export function mnemonicToSeed(mnemonic: string, passphrase?: string): Promise<Uint8Array>;

/**
 * Deriva o par de chaves Ed25519 mestre a partir do seed BIP39 (64 bytes → 32 bytes via HKDF-SHA256).
 * info: "plataforma-master-key"
 * Derivado de: caderno-2/02 §3.1 "Chave Mestra (Ed25519)"
 */
export function deriveMasterKeyPair(seed: Uint8Array): Promise<{ privateKey: Uint8Array; publicKey: Uint8Array }>;

/**
 * Deriva a chave de desbloqueio do dispositivo via PBKDF2-SHA256.
 * Parâmetros (FIXADOS — ver Seção 6): 210 000 iterações, salt = UTF8("plataforma-device-unlock:" + deviceId), keylen = 32 bytes.
 * Derivado de: caderno-2/02 §4.1.1 "Senha não é a identidade"
 */
export function deriveDeviceUnlockKey(
  password: string,
  deviceId: string
): Promise<Uint8Array>;

/**
 * Verifica se uma senha está correta ao tentar decifrar o material cifrado.
 * Nunca vaza timing grosseiro (usa comparação de tag AES-GCM).
 * Derivado de: plano §4 T-104 — "senha errada falha sem vazar timing grosseiro"
 */
export function verifyPassword(
  password: string,
  deviceId: string,
  encryptedMaterial: Uint8Array
): Promise<boolean>;
```

## 4. Estratégia de Testes Estrita (Test-Driven Development)

- **Framework:** Vitest (Node puro)
- **Ambiente:** `pnpm --filter @plataforma/crypto test`
- **Fora de Escopo:** UI, rede, storage persistente

Casos de teste (numerados):

1. `generateMnemonic(128)` → string com 12 palavras; `validateMnemonic(resultado)` → `true`.
2. `generateMnemonic(256)` → string com 24 palavras; `validateMnemonic(resultado)` → `true`.
3. `validateMnemonic("palavra inválida")` → `false`.
4. **Determinismo:** mesmo mnemônico → mesmo seed (64 bytes) → mesmo par de chaves Ed25519 (`privateKey`, `publicKey`) em duas chamadas independentes.
5. **Vetor conhecido BIP39:** mnemônico de test vector oficial BIP39 (`"abandon abandon abandon ... about"`, 12 palavras, passphrase `""`) → seed hex conhecido (derivado de https://iancoleman.io/bip39/).
6. `deriveDeviceUnlockKey("senha-certa", "device-abc")` → 32 bytes; chamadas repetidas produzem o mesmo resultado (determinismo por salt fixo).
7. `verifyPassword("senha-certa", "device-abc", encryptedMaterial)` → `true`.
8. `verifyPassword("senha-errada", "device-abc", encryptedMaterial)` → `false` (sem timing leak: diferença < 5ms em 100 tentativas).
9. **Isolamento:** `deriveMasterKeyPair` chamado com seed diferente → par de chaves diferente.
10. Fuzz: 50 mnemônicos gerados — todos validam e produzem seeds e pares de chaves distintos.

## 6. Feedback de Especificação (Spec Feedback Loop)

> **✅ OPEN DECISION #1 RESOLVIDA (2026-06-21 — arquiteto) — Parâmetros PBKDF2 de `deriveDeviceUnlockKey`:**
> O plano (§4 T-104) especificava "PBKDF2 da senha" sem fixar iterações nem formato de salt; o
> caderno-2/02 §4.1.1 confirma o uso mas não enumera parâmetros.
> **Decisão: Opção (a)** — `deriveDeviceUnlockKey` usa **210 000 iterações PBKDF2-SHA256**, salt
> **determinístico** `UTF8("plataforma-device-unlock:" + deviceId)`, keylen = 32 bytes. Já refletido
> nas assinaturas da Seção 1 e nos casos de teste.
>
> **Por que não (b)/(c):** (b) 600k + salt aleatório de 16 bytes exigiria persistir o salt junto ao
> material cifrado (mais estado e um contrato de storage que esta task não tem) — o salt determinístico
> por `deviceId` dispensa isso sem reduzir a segurança no modelo de ameaça local (a senha é fator de
> desbloqueio local, não identidade de rede). (c) Argon2id não está disponível no WebCrypto nativo e o
> pacote é isomórfico (Seção 0) — adicionaria dependência não-nativa fora do escopo.
> **Não há mais decisão em aberto nesta task.**
