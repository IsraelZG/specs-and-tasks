# T-009a · ControlPort (interface + implementação core)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)

## 1. Objetivo
Definir interface `ControlPort` no pacote `@plataforma/core` e implementar `InMemoryControlPort`: superfície de controle para comandos (drop_connection, partition, sync_now) e eventos de telemetria (peer_connected, sync_complete), sem acoplamento à UI
*(extraído de T-009 §1)*.

### Contratos exatos
```ts
// --- packages/core/src/ports/ControlPort.ts ---
import { PeerId } from '@plataforma/protocol'; // T-004

export interface ControlCommand {
  corrId: string;
  cmd: string;
  payload: unknown;
}

export interface ControlEvent {
  type: string;
  peerId?: PeerId;
  data: unknown;
  timestamp: number;
}

export type ControlEventHandler = (event: ControlEvent) => void;

export interface ControlPort {
  execute(command: ControlCommand): Promise<unknown>;
  onEvent(handler: ControlEventHandler): void;
  dropConnection(peerId: PeerId): Promise<void>;
  setPartition(groups: PeerId[][]): Promise<void>;
  healPartition(): Promise<void>;
}

export class InMemoryControlPort implements ControlPort {
  constructor();
  execute(command: ControlCommand): Promise<unknown>;
  onEvent(handler: ControlEventHandler): void;
  dropConnection(peerId: PeerId): Promise<void>;
  setPartition(groups: PeerId[][]): Promise<void>;
  healPartition(): Promise<void>;
  /** Emite evento para listeners registrados. */
  emit(event: ControlEvent): void;
}
```

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).

Casos de teste:
1. `execute({cmd:'drop_connection', payload:{peerId:'A'}})` → `dropConnection('A')` chamado.
2. `onEvent` registra listener; `emit` dispara callback.
3. `setPartition`/`healPartition` alternam estado.
4. `execute` com comando desconhecido → erro controlado.
5. `corrId` preservado na resposta.
