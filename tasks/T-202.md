## 1. Objetivo
Implementar handshake Noise_XX (3 round-trips) no pacote `@plataforma/crypto`, trocando `DevicePeerId`, `identity_epoch_index` e nonce assinado, sobre `NetworkAdapterPort`
*(derivado de `docs/conceitos/noise-xx.md` — "autenticação mútua entre peers, 3 round-trips"; `docs/conceitos/peer-id.md` — DevicePeerId = blake2s256(DEVICE_PUB_KEY); `docs/plano-de-implementacao.md` §4 T-202)*.

### Contratos exatos
```ts
// --- packages/crypto/src/noise.ts ---
import { NetworkAdapterPort, PeerId } from '@plataforma/protocol'; // T-004
import { Ed25519PublicKey, Ed25519PrivateKey } from './wrappers.js'; // T-101

/** Estado do handshake Noise_XX. */
export enum NoiseState {
  HANDSHAKE = 'handshake',
  CONNECTED = 'connected',
  FAILED   = 'failed',
}

/** Resultado do handshake bem-sucedido. */
export interface NoiseResult {
  remoteDevicePeerId: PeerId;
  remoteIdentityEpochIndex: number;
  encryptKey: Uint8Array;   // chave de sessão (AES-256-GCM)
  decryptKey: Uint8Array;   // chave de sessão (AES-256-GCM)
}

/** Inicia handshake Noise_XX como iniciador.
 *  deviceKeyPair: chave Ed25519 do dispositivo local.
 *  transport: NetworkAdapterPort já conectado (WebSocket, WebRTC, etc).
 */
export function noiseHandshakeInitiator(
  deviceKeyPair: { publicKey: Ed25519PublicKey; privateKey: Ed25519PrivateKey },
  identityEpochIndex: number,
  transport: NetworkAdapterPort,
  remotePeerId: PeerId,
): Promise<NoiseResult>;

/** Inicia handshake como respondedor. */
export function noiseHandshakeResponder(
  deviceKeyPair: { publicKey: Ed25519PublicKey; privateKey: Ed25519PrivateKey },
  identityEpochIndex: number,
  transport: NetworkAdapterPort,
): Promise<NoiseResult>;

/** Cifra mensagem com chave de sessão pós-handshake. */
export function noiseEncrypt(key: Uint8Array, plaintext: Uint8Array): Promise<Uint8Array>;

/** Decifra mensagem com chave de sessão. */
export function noiseDecrypt(key: Uint8Array, ciphertext: Uint8Array): Promise<Uint8Array>;
```
- Handshake troca: `DevicePeerId` + `identity_epoch_index` + nonce assinado com chave privada Ed25519.
- Fluxo de 3 RT: (1) init → ephemeral key, (2) resp → ephemeral + static key, (3) init → static key.
- Usa `@noble/ciphers` para AES-GCM ou `noise-c.wasm` (isomórfico).
- **Observação:** chave do dispositivo, NÃO chave de persona (T-105).

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/crypto test`. Usar `SimNetwork` (T-005) ou mock de `NetworkAdapterPort`.

Casos de teste:
1. Handshake iniciador ↔ respondedor completo: ambos recebem `NoiseResult`, estado `CONNECTED`.
2. `encrypt`/`decrypt` com chaves de sessão → roundtrip.
3. `decrypt` com chave errada → erro.
4. Respondedor com `deviceKeyPair` errado → handshake `FAILED`.
5. `identity_epoch_index` divergente → handshake ok mas resultado com índice remoto diferente (catch-up implícito).
6. Chave de sessão derivada é diferente do par (cada lado tem `encryptKey`/`decryptKey` trocados).
