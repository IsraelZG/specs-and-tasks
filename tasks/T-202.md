## 1. Objetivo e Contratos
Implementar o handshake Noise_XX de autenticação mútua (3 round-trips) sobre qualquer `NetworkAdapterPort`, trocando `DevicePeerId`, `identity_epoch_index` e uma **assinatura Ed25519 do handshake hash** (channel binding — ver [ADR-noise-xx](../docs/adr/ADR-noise-xx.md)).
*(derivado de `docs/conceitos/noise-xx.md`; `docs/caderno-5-transport/01-p2p-transport-and-reconciliation.md` §Handshake; `docs/plano-de-implementacao.md` §5 T-202)*.

### Contratos exatos (assinaturas TS fixadas)

```ts
// --- packages/transport/src/noiseHandshake.ts ---
import { NetworkAdapterPort, PeerId } from '@plataforma/protocol';
import type { Ed25519PublicKey, Ed25519PrivateKey } from '@plataforma/crypto';

/** Par de chaves Ed25519 do device. Definido LOCALMENTE: o `@plataforma/crypto` NÃO exporta
 *  `Ed25519Keypair`, e o campo é `privateKey` (não `secretKey`). Decisão D4 do ADR-noise-xx. */
export interface Ed25519Keypair {
  readonly publicKey: Ed25519PublicKey;
  readonly privateKey: Ed25519PrivateKey;
}

/** Resultado do handshake Noise_XX. */
export interface NoiseHandshakeResult {
  /** DevicePeerId do peer remoto, extraído do handshake. */
  readonly remoteDeviceId: PeerId;
  /** identity_epoch_index do peer remoto. */
  readonly remoteEpochIndex: number;
  /** Canais de comunicação pós-handshake (encriptados). */
  readonly send: (data: Uint8Array) => Promise<void>;
  readonly receive: () => AsyncIterable<Uint8Array>;
  /** Flag: true se épocas divergirem (desvio para Catch-up de Identidades). */
  readonly epochMismatch: boolean;
}

/** Erro lançado quando o handshake falha criptograficamente. */
export class NoiseHandshakeError extends Error {
  constructor(
    message: string,
    public readonly reason: 'invalid_signature' | 'wrong_key' | 'protocol_error' | 'timeout',
  );
}

/**
 * Executa o handshake Noise_XX (iniciador) sobre um adapter já conectado.
 *
 * @param adapter  NetworkAdapterPort já com `connect()`/`listen()` resolvido.
 * @param localKey Par de chaves Ed25519 do dispositivo local.
 * @param localEpochIndex  Índice da Época de Identidade vigente no dispositivo local.
 * @param timeoutMs  Timeout do handshake em ms (default 10_000).
 * @param expectedRemoteDevicePub  Ed25519 public key esperada do peer remoto (opcional).
 *   Quando fornecida, o handshake rejeita com `reason: 'wrong_key'` se a chave
 *   recebida não bater byte-a-byte (pinning). Omitir = TOFU (primeiro contato).
 * @returns Resultado com DevicePeerId remoto e canais encriptados.
 * @throws NoiseHandshakeError se assinatura inválida, chave errada ou timeout.
 *
 * Derivado de docs/conceitos/noise-xx.md §Fluxo: 3 RTs, troca DevicePeerId +
 * identity_epoch_index + nonce assinado Ed25519.
 */
export async function initiateNoiseXX(
  adapter: NetworkAdapterPort,
  localKey: Ed25519Keypair,
  localEpochIndex: number,
  timeoutMs?: number,
  expectedRemoteDevicePub?: Ed25519PublicKey,
): Promise<NoiseHandshakeResult>;

/**
 * Executa o handshake Noise_XX (respondedor) sobre um adapter já conectado.
 * Mesma semântica de `initiateNoiseXX`, invertendo os papéis de iniciador/respondedor.
 *
 * @param expectedRemoteDevicePub  Ed25519 public key esperada do peer remoto (opcional).
 *   Quando fornecida, rejeita com `reason: 'wrong_key'` se a chave não bater.
 */
export async function respondNoiseXX(
  adapter: NetworkAdapterPort,
  localKey: Ed25519Keypair,
  localEpochIndex: number,
  timeoutMs?: number,
  expectedRemoteDevicePub?: Ed25519PublicKey,
): Promise<NoiseHandshakeResult>;
```

> **Biblioteca (decisão D2 do [ADR-noise-xx](../docs/adr/ADR-noise-xx.md)):** montar o **Noise_XX
> mínimo sobre `@noble/*`** — `@noble/curves` (X25519), `@noble/ciphers` (ChaCha20-Poly1305),
> `@noble/hashes` (SHA-256/BLAKE2s). `@noise-crypto/noise` **não existe no npm**; `noise-protocol` tem
> deps nativas (quebra em Win ARM64/browser); `noise-c.wasm` está estagnado. O spike T-200 (PoC verde)
> provou a viabilidade. **Mitigações:** SÓ o padrão XX (não o framework inteiro), validar contra os
> **test vectors oficiais do Noise** (caso 9), manter pequeno (~200–300 LOC).

> **Binding Ed25519↔X25519 (decisão D3 do ADR):** Noise_XX roda com chave estática **X25519 efêmera
> por sessão**; a identidade **Ed25519** do device liga-se ao canal **assinando o handshake hash `h`
> do Noise** — é isso que "nonce assinado com Ed25519" significa (sem timestamp, sem nonce ad-hoc). A
> contraparte verifica a assinatura contra a Ed25519 pubkey do `DevicePeerId` esperado. (X25519 static
> persistente por device é otimização futura — exigiria provisionamento em T-104; v0 usa efêmero.)

> **DevicePeerId:** `blake2s256(DEVICE_PUB_KEY)`. Derivado de T-105 §1 + `docs/conceitos/peer-id.md`.
> A chave do dispositivo (Device Key) é a chave Ed25519 estável gerada no provisionamento
> (T-104), **nunca** a chave de persona.

> **identity_epoch_index:** índice da Época de Identidade (RFC-005 §A.1). Derivado de
> `docs/conceitos/epoca-de-identidade.md` + `docs/conceitos/noise-xx.md`
> §Comportamento em caso de divergência de época. Divergência → `epochMismatch: true`
> (conexão mantida, desvio para Catch-up de Identidades/Onda 0; NÃO descarta a conexão).

Casos de teste (numerados):
1. **Handshake feliz (2 peers):** iniciador e respondedor com chaves Ed25519 válidas e mesmo `identity_epoch_index` → `NoiseHandshakeResult` com `remoteDeviceId` correto, `epochMismatch: false`, canais `send`/`receive` funcionais.
2. **Mensagem pós-handshake:** após handshake feliz, A envia `Uint8Array([1,2,3])` via `send` → B recebe `Uint8Array([1,2,3])` via `receive`.
3. **Mensagem no sentido inverso:** B envia → A recebe (canais bidirecionais independentes).
4. **Chave errada (pinning):** iniciador passa `expectedRemoteDevicePub` diferente da chave real do respondedor → `NoiseHandshakeError` com `reason: 'wrong_key'`. Verificado com comparação em tempo constante após binding Ed25519.
5. **Assinatura inválida:** nonce adulterado no trânsito → `NoiseHandshakeError` com `reason: 'invalid_signature'`.
6. **Timeout:** iniciador envia RT1, respondedor nunca responde → após `timeoutMs`, `NoiseHandshakeError` com `reason: 'timeout'`.
7. **Divergência de época:** iniciador com `epochIndex=5`, respondedor com `epochIndex=3` → handshake conclui com `epochMismatch: true` (conexão NÃO descartada — `docs/conceitos/noise-xx.md` §Comportamento).
8. **Handshake sobre SimNetwork:** usar `SimNetwork` (T-005) como adapter → handshake funciona deterministicamente sem rede real.
9. **Test vectors oficiais do Noise_XX (anti-bug do hand-roll):** rodar o handshake contra ≥1 vetor oficial do Noise (chaves/efêmeros fixos → transcript/mensagens esperados byte-a-byte). > **Decisões originais (mantidas para histórico — RESOLVIDAS acima pelo ADR-noise-xx):**
