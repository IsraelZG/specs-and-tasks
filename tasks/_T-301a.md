## 1. Objetivo
Implementar `RBSRBTree` no pacote `@plataforma/protocol`: árvore binária balanceada (AVL) em RAM indexada por ULID, com fingerprint individual (SHA-256) e XOR agregado mantido incrementalmente no insert/delete
*(extraído de T-301 §1)*.

### Contratos exatos
```ts
// --- packages/protocol/src/rbsr/BTree.ts ---
import { ULID } from '@plataforma/core'; // T-102

export type Fingerprint = Uint8Array; // 32 bytes = SHA-256

export interface RBSRNode {
  id: ULID;
  fingerprint: Fingerprint;
  rangeXor: Fingerprint; // XOR agregado da sub-árvore
  left?: RBSRNode;
  right?: RBSRNode;
}

export class RBSRBTree {
  constructor();
  insert(id: ULID, fingerprint: Fingerprint): void;
  delete(id: ULID): void;
  get(id: ULID): Fingerprint | undefined;
  get rootXor(): Fingerprint; // XOR de toda a árvore
  get size(): number;
}

export function xorFingerprints(a: Fingerprint, b: Fingerprint): Fingerprint;
```

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).

Casos de teste:
1. `insert` + `get` roundtrip.
2. `rootXor` = XOR de todos os fingerprints inseridos.
3. `rootXor` se atualiza corretamente após `delete`.
4. `delete` de id inexistente → no-op (não lança).
5. 1000 nós: `insert` escala O(log n), `rootXor` correto.
6. `xorFingerprints` idempotente: a XOR a XOR b = b.
