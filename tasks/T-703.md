## 1. Objetivo
Implementar estratégias de merge por classe no pacote `@plataforma/core`: preferências LWW por HLC; rascunhos via Automerge; cache de prefetch union-based *(derivado de `docs/plano-de-implementacao.md` §10 T-703)*.

### Contratos exatos (assinaturas TS fixadas)
```ts
// --- packages/core/src/sync/mergeStrategies.ts ---
import { EntityId, ULID } from '../ulid.js'; // derivado de packages/core/src/ulid.ts §Contrato
import { HLCTimestamp } from '../hlc.js'; // derivado de packages/core/src/hlc.ts §HLCTimestamp

export enum MergeClass {
  PREFERENCE = 'preference',         // LWW por HLC
  DRAFT = 'draft',                   // Automerge (CRDT)
  PREFETCH_CACHE = 'prefetch_cache', // union-based
}

export interface MergeInput {
  localValue: unknown;
  remoteValue: unknown;
  localHlc: HLCTimestamp;
  remoteHlc: HLCTimestamp;
  mergeClass: MergeClass | string;
}

export interface MergeResult {
  resolved: unknown;
  strategy: MergeClass | string;
  conflict: boolean;
  conflictResolution: 'lww' | 'automerge' | 'union' | 'none';
}

export interface MergeStrategies {
  /** LWW: valor com maior HLC vence. */
  lwwMerge(input: MergeInput): MergeResult;

  /** Automerge: merge CRDT para rascunhos. */
  automergeMerge(input: MergeInput): MergeResult;

  /** Union: união de caches de prefetch (sem sobrescrita). */
  unionMerge(input: MergeInput): MergeResult;

  /** Seleciona estratégia baseada na classe declarada. */
  merge(input: MergeInput): MergeResult;

  /** Registra classe customizada (extensível). */
  registerClass(mergeClass: MergeClass | string, strategy: (input: MergeInput) => MergeResult): void;
}
```
- `lwwMerge`: `localHlc > remoteHlc` ? local : remote (Last Writer Wins). Se os HLCs forem iguais, desempata deterministicamente por comparação lexicográfica das representações JSON stringified dos valores (`JSON.stringify(local) >= JSON.stringify(remote) ? local : remote`), garantindo convergência estrita e eliminando forks em caso de conflito de concorrência com o mesmo HLC logical clock.
- `automergeMerge`: se local/remoto forem `Uint8Array`s de documentos Automerge serializados, carrega, mescla e salva de volta via `@automerge/automerge` (CRDT sem perda). Se forem documentos/objetos já carregados, mescla-os.
- `unionMerge`: se local/remoto forem arrays, retorna união de conjuntos sem duplicação (ordem determinística).
- `merge`: despacha dinamicamente conforme `mergeClass`. Se for uma classe customizada registrada via `registerClass`, executa a estratégia registrada; caso contrário lança erro de classe não suportada.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/core test`.

Casos de teste:
1. `lwwMerge`: remote HLC maior (LWW) → remote vence.
2. `lwwMerge`: local HLC maior (LWW) → local vence.
3. `lwwMerge`: HLCs idênticos → desempate determinístico estável (ex: JSON stringify do valor local >= remoto).
4. `automergeMerge`: dois rascunhos (como `Uint8Array` válidos de Automerge) com campos diferentes mesclados → merge sem perda, com valores de ambos preservados.
5. `automergeMerge`: convergência determinística (mesmo input de binários Automerge em ordens diferentes gera o mesmo output binário).
6. `unionMerge`: dois arrays de strings com elementos duplicados → retorna array único contendo a união deduplicada.
7. `unionMerge`: local e remoto não são arrays → trata-os como arrays de 1 elemento e faz a união deduplicada.
8. `merge`: seleciona `lwwMerge` para `MergeClass.PREFERENCE`.
9. `merge`: seleciona `automergeMerge` para `MergeClass.DRAFT`.
10. `merge`: seleciona `unionMerge` para `MergeClass.PREFETCH_CACHE`.
11. `registerClass`: registra classe customizada e valida que o dispatcher `merge` a chama corretamente.
