# T-402 · Adapter WebRTC DataChannel

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo
- **Test Runner:** `playwright` (E2E browser) — WebRTC não roda em Node puro
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar `WebRTCAdapter` no pacote `@plataforma/transport`, implementando `NetworkAdapterPort` sobre WebRTC DataChannel, para sincronização RBSR direta peer-to-peer sem passar pelo Peer do Sistema
*(derivado de T-204 — modelo de adapter; T-302 — RBSR exchange; `docs/plano-de-implementacao.md` §4 T-402)*.

### Contratos exatos
```ts
// --- packages/transport/src/webrtc/WebRtcAdapter.ts ---
import { NetworkAdapterPort, PeerId, WireData, MessageHandler } from '@plataforma/protocol'; // T-004

export interface WebRtcConfig {
  iceServers?: RTCIceServer[];  // STUN/TURN
  signalTransport: NetworkAdapterPort; // canal de sinalização (WebSocket) para trocar SDP/ICE
}

export class WebRTCAdapter implements NetworkAdapterPort {
  constructor(config: WebRtcConfig);
  connect(peerId: PeerId): Promise<void>;
  listen(): Promise<void>;
  send(peerId: PeerId, data: WireData): Promise<void>;
  onMessage(handler: MessageHandler): void;
  close(): Promise<void>;
}
```
- `connect`: cria `RTCPeerConnection`, gera oferta SDP, troca via `signalTransport`, estabelece DataChannel.
- `send`: envia `WireData` pelo DataChannel (binary).
- `onMessage`: recebe mensagens do DataChannel e invoca handler.
- Sinalização via WebSocket adapter (T-204) — mesmo `NetworkAdapterPort`, multiplexado.

## 4. Estratégia de Testes
- [x] **Framework:** Playwright (2 browser contexts).
- [x] **Ambiente:** `pnpm --filter bancada test:e2e`.

Casos de teste:
1. 2 peers estabelecem DataChannel via oferta/resposta SDP.
2. `send`/`onMessage` roundtrip via DataChannel.
3. `close` encerra RTCPeerConnection.
4. WebRTC + RBSR: 2 peers reconciliam 10 nós divergentes via DataChannel.

2. Use `signalTransport` (WebSocket) para trocar SDP/ICE candidates.
3. Escreva teste E2E Playwright com 2 contexts compartilhando `signalTransport`.
4. Rode build + test.
