## 1. Objetivo
Deixar `pnpm --filter @plataforma/bancada build` **verde**. Hoje o `vite build` do `bancada` falha ao empacotar o WASM do Automerge:
```
[vite-plugin-pwa:build] Could not load .../@automerge/automerge/.../automerge_wasm_bg.wasm:
"ESM integration proposal for Wasm" is not supported currently.
Use vite-plugin-wasm or `.wasm?init` / `.wasm?url`.
```
**Cadeia de import (por que o WASM entra no bundle):** `apps/bancada` importa `@plataforma/transport`, que usa `@automerge/automerge` em `automergeShell.ts`/`portNetworkAdapter.ts`. Ao empacotar o `transport`, o Vite/Rollup encontra o import ESM do `.wasm` e não sabe carregá-lo sem plugin.

Esta é a **3ª dívida basal** desmascarada na série C-31/C-32 (era ocultada pela falha anterior do `transport`). Não é regressão da C-32 — só ficou visível quando o build passou do `transport`.

### Pegadinhas conhecidas
- **Abordagem recomendada (revisada 2026-07-17):** adicionar **só** `vite-plugin-wasm` e definir `build.target: 'esnext'`. O init do Automerge usa top-level await, mas com `target: 'esnext'` o próprio Vite/Rollup emite TLA nativo — **não** é preciso `vite-plugin-top-level-await`. Ordem no array: `wasm()` **antes** do `VitePWA`. Ex.:
>  ```ts
>  import wasm from "vite-plugin-wasm";
>  // plugins: [react(), wasm(), VitePWA({...})]
>  // build: { target: 'esnext', rollupOptions: { ... } }
>  ```
> - **Por que NÃO `vite-plugin-top-level-await`:** aquele plugin reescreve TLA via `@swc/core` (compilador Rust nativo) — o pnpm 11 bloqueia o postinstall dele sem `allowBuilds['@swc/core']` no `pnpm-workspace.yaml`, arquivo **fora do escopo** desta task. `target: 'esnext'` entrega o mesmo resultado sem dep nativa nenhuma. (A rodada anterior parou exatamente nesse ponto — ver §8.)
> - Fixe versão compatível com `vite ^6`: `vite-plugin-wasm ^3.6`.
> - Se após o plugin o erro persistir só no **PWA/generateSW** (workbox tentando o `.wasm`), aí sim ajuste o `workbox.globIgnores` para excluir `**/*.wasm` do precache — mas só se o erro mudar para essa fase.
> - `bancada` é ARM64 Windows nativo; `pnpm install` da nova dep ~poucos s (ver memória de ambiente). Não rodar install gigante no meio da tool.

1. **Reverter o resíduo da rodada anterior:** remover `vite-plugin-top-level-await` do `apps/bancada/package.json` (a rodada 1 o adicionou; deixar só `vite-plugin-wasm`).
2. Confirmar a fase exata do erro: `pnpm --filter @plataforma/bancada build` e ler se é bundling (Rollup) ou generateSW (PWA).
3. `pnpm --filter @plataforma/bancada add -D vite-plugin-wasm` (só ele). Deve instalar sem `ERR_PNPM_IGNORED_BUILDS`.
4. Editar `apps/bancada/vite.config.ts`: importar e inserir `wasm()` antes do `VitePWA`; adicionar `build.target: 'esnext'`.
5. `pnpm --filter @plataforma/bancada build` → verde. Rodar `test`. Colar saída na Seção 8.
