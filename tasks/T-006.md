## 1. Objetivo
Evoluir `SimNetwork` para v2 no pacote `@plataforma/testkit`: degradação determinística (drop%, jitter, latência, limite de banda) + perfis de NAT (`open`, `cone`, `symmetric`), usando `VirtualClock` e `SeededRandom`

### Contratos exatos
```ts
// --- packages/testkit/src/SimNetwork.ts (extensão) ---
export enum NatType { Open = 'open', Cone = 'cone', Symmetric = 'symmetric' }

export interface LinkConfig {
  latencyMs?: number;     // default 0
  jitterMs?: number;      // ±jitter uniforme
  dropRate?: number;      // 0.0–1.0
  bandwidthBps?: number;  // bytes/s; 0 = ilimitado
}

export interface PeerConfig {
  natType?: NatType;      // default Open
  link?: LinkConfig;
}

// Adicionar a SimNetwork existente:
export class SimNetwork {
  // ... (métodos v1 mantidos)
  setPeerConfig(peerId: PeerId, config: PeerConfig): void;

  /** NAT: Cone → recebe de qualquer destino conhecido.
   *  Symmetric → só recebe de destinos que já enviou antes (mapeamento por par ip:port). */
}
```
- `setPeerConfig`: configura degradação por peer.
- `send()`: aplica drop%, latência+jitter, e checa NAT antes de entregar.
- `VirtualClock.advance()` usado para simular passagem de tempo; sem `setTimeout` real.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/testkit test`.

Casos de teste:
1. `dropRate=1.0` → 100% das mensagens perdidas (zero entregas).
2. `latencyMs=100, clock.advance(50)` → mensagem não entregue; `clock.advance(60)` → entregue.
3. `jitterMs=10, seed fixa` → latências variam deterministicamente dentro de [latency±10].
4. NAT `Symmetric`: A→B ok (B já conhece A), C→B bloqueado (C nunca recebeu de B).
5. NAT `Cone`: A→B ok, C→B ok (qualquer destino conhecido).
6. `bandwidthBps=100`: mensagem de 200 bytes → entrega em 2 segundos virtuais.
7. v1 API (`partition`/`heal`) não quebrada (testes existentes passam).
