### Critérios de aceite — Tier A (Button, Checkbox, Input, Switch, Tabs)
1. **Smoke:** renderiza sem erro com props mínimas obrigatórias.
2. **Interação real:** a ação primária (clique / toggle / digitação / troca de aba) dispara o callback público esperado (`onClick`/`onCheckedChange`/`onChange`/`onValueChange`, conforme a metadata) com o valor correto.
3. **Estado `disabled`:** quando o componente suporta `disabled`, a interação **não** dispara o callback. (`Tabs` por aba: aba `disabled` não ativa.)
4. **A11y:** role ARIA resolvível via `getByRole` e nome acessível — `button`, `checkbox`, `textbox`, `switch`, `tab`/`tablist`/`tabpanel` respectivamente.
5. **Ref forwarding:** se o componente usa `React.forwardRef`, o ref chega ao nó DOM esperado.

### Critérios de aceite — Tier B (Modal)
1. **Fechado:** com `open={false}` (ou estado inicial fechado), o conteúdo **não** está no DOM.
2. **Abrir:** abrir via prop/interação renderiza `role="dialog"` (ou `alertdialog`) com nome acessível (`aria-labelledby`/`aria-label`).
3. **Fechar:** o mecanismo documentado de fechar (botão close / Escape / clique no overlay, conforme a metadata) dispara o callback (`onOpenChange`/`onClose`).
4. **Ref forwarding:** se `React.forwardRef`, ref chega ao nó esperado.

### Pegadinhas conhecidas
- **Modal:** usa portal — o conteúdo renderiza fora do container; use `screen.getByRole('dialog')`, não `container.querySelector`. Fechar por Escape: `userEvent.keyboard('{Escape}')`.
- **Switch / Checkbox:** testar `aria-checked` e o callback `onCheckedChange` (não `onChange` cru) conforme a metadata real.
- **Tabs:** roles `tablist`/`tab`/`tabpanel`; ativar aba via clique dispara `onValueChange`; aba `disabled` não ativa.
- **Input:** `userEvent.type` dispara `onChange`; verificar `disabled`/`readOnly` na metadata antes de assumir.
- **Button:** se o componente suporta `asChild`/polimorfismo (`Slot`), o role pode mudar — leia a metadata antes de assumir `role="button"`.
