## 1. Objetivo
Implementar `rotate` e `revoke` no `EpochRotation`: gera nova chave AES-256, registra no KeyVault com TTL, governado por UCAN. Revoke remove chave do vault imediatamente
*(extraído de T-505 §1; T-501 — UCAN; T-110 — KeyVault)*.

### Contratos exatos
```ts
// --- packages/core/src/vault/EpochRotation.ts ---
import { KeyVault } from '../keyVault.js'; // T-110 (packages/core/src/keyVault.ts)
import { SignedUcan } from '../auth/ucan.js'; // T-501 (packages/core/src/auth/ucan.ts)
import { AesKey } from '@plataforma/crypto'; // T-101 (packages/crypto/src/wrappers.ts)

export interface EpochConfig {
  epochIndex: number;
  key: AesKey;
  issuedAt: number;
  expiresAt: number;
  issuerUcan: SignedUcan;
}

export class EpochRotation {
  constructor(vault: KeyVault, ttlMs?: number);

  /** Gera nova época. UCAN deve autorizar a ability 'ROTATE_EPOCH' para o resource 'epoch:*'. */
  rotate(authorization: SignedUcan): Promise<EpochConfig>;

  /** Remove época do vault (revogação imediata). */
  revokeEpoch(index: number): void;

  get currentIndex(): number;
}
```
- `rotate`: gera `AesKey` (32 bytes aleatórios via WebCrypto `crypto.getRandomValues`), registra no vault com TTL (default 4 horas/14.400.000 ms se `ttlMs` não for passado no construtor), incrementa índice (iniciando em 0 para a primeira rotação).
- Forward secrecy: histórico NÃO re-cifrado. Novos nós usam chave da nova época.
- UCAN gate: A validação do UCAN é feita usando `validateUcan(authorization)`. Além disso, `authorization.payload.capabilities` deve conter uma capability com `resource: 'epoch:*'` (ou correspondente) e `ability: 'ROTATE_EPOCH'`.
