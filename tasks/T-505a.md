## 1. Objetivo
Implementar `rotate` e `revoke` no `EpochRotation`: gera nova chave AES-256, registra no KeyVault com TTL, governado por UCAN. Revoke remove chave do vault imediatamente
*(extraído de T-505 §1; T-501b — UCAN; T-110 — KeyVault)*.

### Contratos exatos
```ts
// --- packages/core/src/vault/EpochRotation.ts ---
import { KeyVault } from '../keyVault.js'; // T-110
import { SignedUcan } from '../auth/ucan-codec.js'; // T-501a
import { AesKey } from '@plataforma/crypto'; // T-101b

export interface EpochConfig {
  epochIndex: number;
  key: AesKey;
  issuedAt: number;
  expiresAt: number;
  issuerUcan: SignedUcan;
}

export class EpochRotation {
  constructor(vault: KeyVault, ttlMs?: number);

  /** Gera nova época. UCAN deve autorizar 'ROTATE_EPOCH'. */
  rotate(authorization: SignedUcan): Promise<EpochConfig>;

  /** Remove época do vault (revogação imediata). */
  revokeEpoch(index: number): void;

  get currentIndex(): number;
}
```
- `rotate`: gera `AesKey` (WebCrypto), registra no vault com TTL, incrementa índice.
- Forward secrecy: histórico NÃO re-cifrado. Novos nós usam chave da nova época.
- UCAN gate: `authorization.att` deve conter `{with:'epoch:*', can:'ROTATE_EPOCH'}`.
