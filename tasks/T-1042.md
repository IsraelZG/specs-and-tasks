# T-1042 · Migrar o adapter SQLite atual para GraphStorePort

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)

## 1. Objetivo
Migrar a implementação atual de armazenamento SQLite (tanto Node quanto WASM/OPFS de browser) para implementar e expor a interface `GraphStorePort` criada em T-1041, confinando o dialeto SQL para dentro dos adapters.

- **Metodologia:** Atualizar os testes de conformidade de persistência SQLite em `@plataforma/core` para exercitar diretamente a nova porta `GraphStorePort` com inserção de nodes, edges, busca e range scans.

## 5. Instruções de Execução (Step-by-Step)
1. Modifique `SqliteStorage` em `packages/core/src/sqliteStorage.ts` para que implemente `GraphStorePort` em vez de `StoragePort` (no que diz respeito às tabelas de nodes/edges).
2. Modifique `SqliteWasmStorage` em `packages/core/src/sqliteWasmStorage.ts` de forma idêntica.
3. Transponha as chamadas e consultas SQL cruas internas de escrita/leitura de nós e arestas para métodos internos de tradução privada nas classes. O SQL de nodes/edges deve ficar isolado dentro desses arquivos.
4. Garanta que a transação (`transaction`) retorne `GraphStoreTx` implementando o isolamento de transações.

**[M1] Contrato de `putNode` CONTRADIZ T-1041 (e ADR-005) — MAJOR**

- T-1041 (`ports.ts`): `/** Insere um nó no grafo. Falha se id já existe (imutável). */`
- ADR-005 (citado em ambas as specs): "grafo append-only, **imutável** e replicado".

- Viola: ADR-005, T-1041, princípio de append-only.
- Ação: trocar `INSERT OR REPLACE` por `INSERT` (ou `INSERT OR IGNORE` se quiser tolerância). Atualizar o jsdoc em `ports.ts:64` para alinhar com T-1041: "Falha se id já existe (imutável)."
