# T-1049 · Guardrails anti-ciclo do monorepo (itens 1, 2, 4)

## 1. Objetivo
Instalar as **travas baratas** que impedem a reintrodução de dependências cíclicas em **todo o workspace** (estaleiro + superapp — é um único `pnpm` workspace, então instala-se uma vez e cobre os dois):

1. **`import/no-cycle`** (ESLint) na config compartilhada → entra no Gate de lint (já obrigatório desde 2026-07-06).
2. **`madge --circular`** como script de CI que **falha** em qualquer ciclo.
4. **Higiene de tsconfig** por pacote (`exclude: ["dist","node_modules"]`, `rootDir: "src"`, `outDir: "dist"`) → mata `TS5055` na raiz (o `.d.ts` emitido nunca volta como input).

> (Item 3 — project references + `tsc -b` — é a task irmã T-1050.)

## 2. Contexto RAG
- Handoff TS5055 do EST-33 (causa-raiz: ciclo + `outDir: dist` compartilhado).
- CLAUDE.md — Gate de Evidência (lint no gate desde 2026-07-06, após 3 reworks por regressão de lint).
- Gate de acoplamento do template (`generate-task.mjs` §7 DoD).

## 3. Escopo (Inputs/Outputs)
- **[UPDATE]** config ESLint compartilhada (preset da raiz / `packages/config-eslint`): habilitar `import/no-cycle`, com `import/resolver` ciente do workspace TS (`typescript` resolver).
- **[CREATE]** script raiz `"check:cycles": "madge --circular --extensions ts packages apps"` no `package.json`; plugar no pipeline de CI (e opcionalmente no `turbo` lint).
- **[UPDATE]** tsconfig de cada pacote onde faltar: `exclude` `dist`/`node_modules`, `rootDir`/`outDir` explícitos. **Idempotente** — só onde ausente.
- **[ADD]** devDependency `madge`.

## 4. Testes
- Introduzir um ciclo local de teste dispara falha em `pnpm lint` **e** em `pnpm check:cycles`; removê-lo volta a verde.
- `pnpm -r build` continua limpo (nenhum TS5055 possível por config).

## 5. DoD
`import/no-cycle` ativo e **verde** no workspace; `check:cycles` roda no CI; nenhum pacote pode emitir `.d.ts` como input.

## 6. Feedback de Especificação
- **Ordem crítica:** só habilitar `import/no-cycle` **depois** que EST-34 e as correções de T-1048 removeram os ciclos existentes — senão o baseline de lint fica vermelho.
- Se sobrar ciclo não resolvido, é **BLOCKER** → escalar (pause/decisão), **nunca** desabilitar a regra por pacote para "passar".

## 7. Verificação (comandos exatos)
```bash
pnpm lint            # inclui import/no-cycle — zero erros
pnpm check:cycles    # madge — zero ciclos
pnpm -r build        # limpo
```
