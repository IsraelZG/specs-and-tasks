## 1. Objetivo

modo headless por papel/tarefa (em git worktrees da T-1012), avançando dev→QA. NÃO

uma lista fixa de perguntas + (opcional) uma PoC mínima de 1 task.

## 3. Escopo de Arquivos (Inputs e Outputs)
- **[CREATE]** `docs/adr/ADR-piloto-automatico.md` — responder EXPLICITAMENTE:
  1. O OpenCode tem modo **headless/não-interativo scriptável** (CLI/SDK)? Como invocar?
  2. Como **selecionar o modelo por execução** (Claude arquiteto / DeepSeek V4 dev / Gemini-Antigravity QA)?
  3. Os provedores estão **disponíveis/contratados** no ambiente do usuário? (chaves/limites)
  4. É possível rodar uma execução **com cwd num worktree** específico (`task/<ID>`)?
  5. Como o agente **reporta de volta** (chama `nexus_transition_task` via MCP? exit code?)?
  6. Recomendação: **seguir / ajustar / bloquear**, com o menor próximo passo.
- **[CREATE]** (opcional) `apps/nexus-backend/src/services/orchestrator.spike.ts` — PoC
  atrás de flag (`NEXUS_ORCHESTRATOR=1`) que executa 1 task: worktree → dev → `finish`
  → QA → `approve`. Sem segredos no código (usar env).
