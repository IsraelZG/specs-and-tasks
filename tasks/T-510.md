## 1. Objetivo
Implementar Shamir Secret Sharing 2-de-3 no pacote `@plataforma/crypto`: split/reconstrução sobre GF(2^8); shard dispositivo + shard provedor (protegido por hash de senha) + shard canal externo
*(derivado de `docs/plano-de-implementacao.md` §8 T-510)*.

### Contratos exatos
```ts
// --- packages/crypto/src/shamir.ts ---

export const SHAMIR_MIN_SHARES = 2;
export const SHAMIR_TOTAL_SHARES = 3;

export interface ShamirShard {
  index: number;                 // 1, 2, ou 3
  value: Uint8Array;             // 32 bytes sobre GF(2^8)
  label: 'device' | 'provider' | 'external';
  protected?: boolean;           // true se shard é protegido por hash de senha
}

export interface ShamirResult {
  shards: ShamirShard[];
  checksum: Uint8Array;          // blake2s256(secret) para verificação
}

/** Divide segredo (32 bytes) em 3 shards (threshold=2). */
export function splitSecret(secret: Uint8Array, seed?: Uint8Array): ShamirResult;

/** Reconstrói segredo a partir de 2+ shards. */
export function reconstructSecret(shards: ShamirShard[]): Uint8Array;

/** Verifica se reconstrução é correta. */
export function verifyReconstruction(reconstructed: Uint8Array, checksum: Uint8Array): boolean;

/** Protege shard com hash de senha (PBKDF2). */
export function protectShard(shard: ShamirShard, password: string): ShamirShard;

/** Desprotege shard com senha. */
export function unprotectShard(shard: ShamirShard, password: string): ShamirShard;
```
- GF(2^8) com polinômio irredutível padrão.
- 3 shards: device (local), provider (protegido por hash de senha), external (canal out-of-band).
- Threshold = 2: qualquer 2 shards reconstroem o segredo.
- `protectShard`: aplica XOR com HKDF derivado de PBKDF2(password) sobre o shard value.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/crypto test`.

Casos de teste:
1. `splitSecret` gera 3 shards distintos.
2. `reconstructSecret` com shards 1+2 → segredo original.
3. `reconstructSecret` com shards 1+3 → segredo original.
4. `reconstructSecret` com shards 2+3 → segredo original.
5. `reconstructSecret` com apenas 1 shard → erro.
6. `verifyReconstruction` com reconstrução correta → true.
7. `verifyReconstruction` com reconstrução errada → false.
8. `protectShard` + `unprotectShard` roundtrip com senha correta.
9. `unprotectShard` com senha errada → shard incorreto (não reconstrói).
10. Vetores de teste fixos: mesmo secret + seed → mesmos shards (determinístico).
