# T-018a · Testes Tier A batch 1 — 8 componentes interativos

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` + `@testing-library/react` (JSDOM) + `@testing-library/user-event`
- **Capacidade-alvo:** haiku

## 1. Objetivo
Escrever testes unitários para **8 componentes interativos** do `@plataforma/design-system`, cobrindo os critérios do Tier A (smoke, interação real, disabled, a11y role, ref forwarding).
*(Derivado de T-018 — subdivisão para execução por modelos haiku)*.

### Componentes cobertos por esta task
1. `Accordion`
2. `Breadcrumb`
3. `Calendar`
4. `Carousel`
5. `Collapsible`
6. `Combobox`
7. `FormField`
8. `InputOTP`

### Critérios de aceite (Tier A — interação)
1. **Smoke:** renderiza sem erro com props mínimas obrigatórias.
2. **Interação real:** a ação primária (clique, digitação, seleção) dispara o callback público esperado (`onChange`/`onValueChange`/`onSelect`, conforme a metadata) com o valor correto.
3. **Estado `disabled`** (quando a metadata listar essa prop): interação não dispara o callback.
4. **A11y:** role ARIA esperado resolvível via `getByRole`; nome acessível presente.
5. **Ref forwarding:** se o componente usa `React.forwardRef`, ref chega ao nó DOM esperado (verificar código-fonte antes de escrever — nem todo componente usa).

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - NÃO copie o mesmo `it()` para todos sem ler o componente real.
> - NÃO corrija bugs de produto — documente na Seção 6.
> - NÃO crie `Sidebar.test.tsx` (T-017).
> - NÃO use `findByRole` sem `await` — a maioria dos componentes deste tier não tem animação assíncrona.

### Pegadinhas conhecidas
- `Checkbox.test.tsx`: padrão para testar `indeterminate` (propriedade DOM, não atributo HTML) e callback ref + `useRef`.
- `Modal.test.tsx`: `flushEnterAnimation` para componentes com listener de teclado pós-animação. `Collapsible` e `Combobox` podem precisar disso se usarem portal + animação de entrada.
- Componentes com portal renderizam fora do container de `render()` — use `screen.getByRole` (busca `document.body` inteiro).

1. Para cada componente: leia `<Nome>.tsx` + `<Nome>.metadata.ts`.
2. Escreva `tests/components/<Nome>.test.tsx` cobrindo os 5 critérios do Tier A.
3. Rode `pnpm --filter @plataforma/design-system test` após cada 2-3 componentes.
4. Ao final dos 8, rode build + lint (Seção 7).
