## 1. Objetivo
Implementar `tryPromote` no `ConnectionPromotionEngine`: tenta migrar conexão relay → WebRTC direta com base no perfil NAT do peer. Usa SimNetwork (T-006b) para testes determinísticos
*(extraído de T-404 §1)*.

### Contratos exatos
```ts
// --- packages/transport/src/promotion/Engine.ts 
---
import { PeerId } from '@plataforma/protocol'; // T-004
import { NatType } from '@plataforma/testkit'; // T-006b

export enum PromotionState { RELAY_ONLY, PROMOTING, DIRECT, FAILED }

export class ConnectionPromotionEngine {
  constructor(maxAttempts?: number);

  /** Tenta promover conexão relay → direta. Retorna novo estado. */
  tryPromote(peerId: PeerId, natType: NatType): Promise<PromotionState>;
}
```
- NAT `Symmetric` → `FAILED` na primeira tentativa.
- NAT `Cone` → `DIRECT` após handshake WebRTC bem-sucedido.
- Make-before-break: relay só fecha após WebRTC confirmado.
- `maxAttempts` (default 3): após N falhas → `FAILED` permanente.
