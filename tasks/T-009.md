## 1. Objetivo
Implementar `resetLocalState()` no pacote `@plataforma/client-sdk` que apaga **todo** o estado local do peer no browser (OPFS, IndexedDB, CacheStorage, localStorage, Service Worker), expor o hook `window.__bancada.reset()` e adicionar o botão **"Reset deste peer"** na Bancada.
*(Derivado de `docs/plano-de-implementacao.md` §3 E0.3 T-009 e §2.4 tabela "Browser (peer)")*

### Contratos exatos (assinaturas TS fixadas)

```ts
// --- packages/client-sdk/src/reset.ts 
---

/**
 * Apaga todo o estado local do peer no browser.
 * Ordem obrigatória (derivada de §2.4):
 *   1. Fecha workers (Sync/Crypto/Index) se ativos.
 *   2. Apaga OPFS: navigator.storage.getDirectory() recursivo.
 *   3. Apaga IndexedDB: IDBFactory.databases() + deleteDatabase().
 *   4. Apaga CacheStorage: caches.keys() + caches.delete().
 *   5. Limpa localStorage / sessionStorage.
 *   6. Desregistra Service Worker: navigator.serviceWorker.getRegistrations() → unregister().
 *   7. Retorna { cleared: string[] } com lista dos recursos apagados.
 * Não lança em ambiente que não suporte alguma API — apenas omite da lista.
 */
export async function resetLocalState(): Promise<{ cleared: string[] }>;

// --- apps/bancada/src/hooks/useBancadaReset.ts ---

/**
 * Hook React que expõe resetLocalState() + injeta window.__bancada.reset.
 * Deve ser montado no shell principal da Bancada (App.tsx).
 */
export function useBancadaReset(): {
  isResetting: boolean;
  reset: () => Promise<void>;
};

// --- apps/bancada/src/global.d.ts (augmentação de Window) ---
interface Window {
  __bancada: {
    reset(): Promise<void>;
  };
}
```

**Unit (Vitest — mock das Web APIs):**
1. `resetLocalState()` chama `navigator.storage.getDirectory()` e apaga recursivamente arquivos OPFS mock.
2. `resetLocalState()` chama `indexedDB.databases()` e `indexedDB.deleteDatabase()` para cada banco retornado.

7. API ausente (ex: `navigator.serviceWorker` undefined) → não lança, omite da lista `cleared`.
