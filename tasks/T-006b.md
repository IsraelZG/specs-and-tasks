# T-006b · SimNetwork v2: perfis NAT (open/cone/symmetric)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** haiku

## 1. Objetivo
Adicionar perfis de NAT (`open`, `cone`, `symmetric`) ao `SimNetwork`, controlando quais peers podem se comunicar diretamente
*(extraído de T-006 §1)*.

### Contratos exatos
```ts
// --- packages/testkit/src/SimNetwork.ts (extensão) ---
export enum NatType { Open = 'open', Cone = 'cone', Symmetric = 'symmetric' }

export interface PeerConfig {
  natType?: NatType;
  link?: LinkConfig; // T-006a
}

export class SimNetwork {
  setPeerConfig(peerId: PeerId, config: PeerConfig): void;
}
```
- `Open`: qualquer peer pode conectar.
- `Cone`: peer recebe de qualquer destino que já contatou.
- `Symmetric`: peer só recebe de pares (ip:port) que ele próprio já enviou antes.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).

Casos de teste:
1. NAT `Symmetric`: A→B ok (B conhece A), C→B bloqueado.
2. NAT `Cone`: A→B ok, C→B ok.
3. NAT `Open`: qualquer comunicação permitida.
4. `setPeerConfig` altera NAT em runtime.

## 5. Instruções de Execução
1. Escreva testes com 3 peers + perfis variados.
2. Implemente lógica de NAT: tracking de pares conhecidos por peer.
3. Rode build + test.
