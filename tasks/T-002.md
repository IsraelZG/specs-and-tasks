# T-002 · Pipeline de CI (GitHub Actions)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo (criado em T-001)
- **Test Runner:** `vitest` (unitário), `playwright` (E2E, job noturno fora de escopo)
- **Capacidade-alvo:** haiku

## 1. Objetivo
Criar o pipeline de Integração Contínua (CI) usando GitHub Actions. Validar build, lint e testes unitários de todos os pacotes via Turborepo, com cache de pnpm store e `.turbo`.

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [x] **Framework:** O próprio CI é o "teste" (GitHub Actions runner valida o workflow).
- [x] **Ambiente do Teste:** ubuntu-latest (GitHub-hosted runner).
- [x] **Fora de Escopo:** Deploy, release, E2E noturno (Anel 3 do plano).

Casos de verificação (o CI falha se):
1. `pnpm install --frozen-lockfile` falha (lockfile desatualizado ou pacote ausente).
2. `pnpm lint` falha em qualquer pacote.
3. `pnpm build` (tsc via turbo) falha em qualquer pacote.
4. `pnpm test` (vitest via turbo) falha em qualquer pacote.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** crie jobs separados por pacote. Use `turbo` para orquestrar paralelismo.
> - **NÃO** crie deployments, releases, ou jobs noturnos.
> - **NÃO** use `pnpm install` sem `--frozen-lockfile` (CI deve rejeitar lockfile divergente).

1. Crie `.github/workflows/ci.yml` com triggers: `push` em `master` + `pull_request`.
2. Job `ci`: `runs-on: ubuntu-latest`, steps: checkout → setup-node v20 → `pnpm/action-setup` → cache pnpm store + `.turbo` → `pnpm install --frozen-lockfile` → `pnpm lint` → `pnpm build` → `pnpm test`.
3. Abra PR com o workflow; o próprio CI valida a si mesmo no push do PR.
