# T-002 · Pipeline de CI (GitHub Actions)

## 1. Objetivo
Criar o pipeline de Integração Contínua (CI) usando GitHub Actions. Ele deve validar o build, rodar o linter, os testes unitários e de integração, garantindo o cacheamento do Turborepo e do pnpm.

## 2. Contexto RAG (Spec-Driven Development)
- [plano-de-implementacao.md](../docs/plano-de-implementacao.md) (Seção 3, T-002)

## 3. Estratégia de Testes (Test-Driven Development)
- [ ] **Abordagem**: O próprio CI é a validação. A criação de um Pull Request falso deve disparar a Action, que deve ficar verde e completar rapidamente devido ao cache.

## 4. Referências de Código
- Arquivos Alvo: `.github/workflows/ci.yml`

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** crie jobs complexos separados para cada pacote. Use o Turborepo (`pnpm build`, `pnpm test`) para orquestrar paralelismo nativamente.
> - **NÃO** instale dependências globalmente no runner (exceto o próprio pnpm se necessário via action `pnpm/action-setup`).
> - **NÃO** crie deployments ainda. Apenas CI.

1. **[TDD Estrutural]** Escreva o `.github/workflows/ci.yml` disparando em `push` na branch `main` e em `pull_request`.
2. Configure o checkout e o setup do Node (versão 20+ recomendada).
3. Utilize `pnpm/action-setup` para instalar o pnpm.
4. Restaure o cache do Turborepo (pasta `.turbo`) e do pnpm store para otimizar os tempos de build.
5. Execute os comandos `pnpm install --frozen-lockfile` (ou equivalente estrito do pnpm).
6. Execute `pnpm run lint`, `pnpm run build` e `pnpm run test`.
7. O job noturno E2E (M0 - Anel 3) não precisa ser criado agora, apenas o fluxo de CI principal.
