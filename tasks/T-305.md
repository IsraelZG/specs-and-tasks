## 1. Objetivo
Implementar sync RBSR escopado por UCAN no pacote `@plataforma/protocol`: a B-Tree de fingerprints é filtrada pelo `edge_filter` do UCAN, impedindo vazamento de subgrafos não autorizados
*(derivado de `docs/visao-arquitetural.md` §8 — "sync dirigido por UCAN: peer sem permissão não recebe nem fingerprints do subgrafo"; T-302 — RBSR exchange; T-501 — UCAN)*.

### Contratos
```ts
// --- packages/protocol/src/auth/ucanScope.ts 
---
import { SignedUcan, UcanCapability } from '@plataforma/core'; // T-501
import { RBSRBTree, Fingerprint } from '../rbsr/BTree.js'; // T-301

/** Filtra B-Tree de RBSR pelas capacidades do UCAN.
 *  Apenas nós cujo entity_id está no escopo do UCAN são incluídos no fingerprint. */
export function scopeRBSRTree(tree: RBSRBTree, ucan: SignedUcan): RBSRBTree;

/** Verifica se UCAN autoriza acesso a um entity_id específico. */
export function canAccess(ucan: SignedUcan, entityId: string, action: string): boolean;
```
- `scopeRBSRTree`: reconstrói B-Tree apenas com nós dentro dos `with` do UCAN.
- Peer sem UCAN válido → recebe B-Tree vazia (rootXor = 0).
