# T-006a · SimNetwork v2: degradação (drop/latência/jitter/bandwidth)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Estender `SimNetwork` (T-005) com parâmetros de degradação determinística por peer: drop rate, latência, jitter e limite de banda. Usa `VirtualClock` e `SeededRandom` de T-003 para reprodutibilidade
*(extraído de T-006 §1)*.

### Contratos exatos
```ts
// --- packages/testkit/src/SimNetwork.ts (extensão) ---
export interface LinkConfig {
  latencyMs?: number;     // default 0
  jitterMs?: number;      // ±jitter uniforme
  dropRate?: number;      // 0.0–1.0
  bandwidthBps?: number;  // 0 = ilimitado
}

export class SimNetwork {
  // ... API v1 mantida
  setPeerConfig(peerId: PeerId, config: { link?: LinkConfig }): void;
}
```
- `send()`: aplica drop (SeededRandom), enfileira com latência+jitter (VirtualClock), aplica bandwidth throttling.
- Entrega é assíncrona via VirtualClock — sem `setTimeout` real.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).

Casos de teste:
1. `dropRate=1.0` → 100% mensagens perdidas.
2. `latencyMs=100, clock.advance(50)` → não entregue; `clock.advance(60)` → entregue.
3. `jitterMs=10, seed fixa` → latência varia deterministicamente.
4. `bandwidthBps=100` → mensagem 200B demora 2s virtuais.
5. v1 API (`partition`/`heal`) não quebrada.

## 5. Instruções de Execução
1. Escreva testes com seeds fixas + VirtualClock.
2. Implemente fila de eventos baseada em clock virtual.
3. Garanta que testes v1 originais seguem passando.
4. Rode build + test.
