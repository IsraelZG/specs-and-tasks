## 1. Objetivo
Estender `SimNetwork` (T-005) com parâmetros de degradação determinística por peer: drop rate, latência, jitter e limite de banda. Usa `VirtualClock` e `SeededRandom` de T-003 para reprodutibilidade.
*(extraído de T-006 §1)*.

### Contratos exatos
```ts
// --- packages/testkit/src/SimNetwork.ts (extensão) 
---
import { PeerId, RandomPort, ClockPort } from '@plataforma/protocol';

export interface LinkConfig {
  latencyMs?: number;     // default 0
  jitterMs?: number;      // ±jitter uniforme
  dropRate?: number;      // 0.0–1.0 (probabilidade de drop)
  bandwidthBps?: number;  // Bytes por segundo (0 = ilimitado)
}

export interface PeerConfig {
  link?: LinkConfig;
}

export class SimNetwork {
  // Construtor estendido para opcionalmente receber random port
  constructor(clock?: VirtualClock, random?: RandomPort);

  // ... API v1 mantida
  setPeerConfig(peerId: PeerId, config: PeerConfig): void;
}
```
- **Fórmulas de Degradação:**
  - **Drop:** Se `SeededRandom.nextDouble() < dropRate`, o pacote é descartado imediatamente (no-op).
  - **Jitter e Latência:** Atraso base é `latencyMs`. O jitter uniforme adiciona um valor no intervalo `[-jitterMs, +jitterMs]`, calculado como `(random.nextDouble() * 2 - 1) * jitterMs`. O atraso da latência é `latencyMs + jitterMsOffset`. Se for `< 0`, assume-se `0`.
  - **Bandwidth Throttling:** O atraso de transmissão de dados é `data.length / bandwidthBps` (em segundos). Em milissegundos: `(data.length / bandwidthBps) * 1000`. Esse atraso de transmissão é somado à latência total.
  - A entrega final é programada via `this.clock.setTimeout(deliverMsg, totalDelayMs)`.
- `send()`: aplica drop (SeededRandom), enfileira com latência+jitter (VirtualClock), aplica bandwidth throttling.
- Entrega é assíncrona via VirtualClock — sem `setTimeout` real.
