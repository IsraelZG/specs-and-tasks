# T-1015 · Seletor estrutural + export (sobre a interface Compressor)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`

## 1. Objetivo
Resolver o **problema 1** (contexto grande demais) substituindo o
`generate_project_summary.js` por um **export inteligente**: (1) **seleciona** os
documentos relevantes pelo grafo do wiki (frontmatter `tags` + `[[wikilinks]]`),
(2) **monta** o pacote, (3) **comprime** via a interface `Compressor` (T-1018), e
(4) devolve o artefato compacto **+ os trechos originais** para recuperação local
(reversibilidade fica AQUI, não no compressor — ver ADR-headroom).

## 3. Escopo de Arquivos (Inputs e Outputs) — assinaturas FIXAS
- **[CREATE]** `apps/nexus-backend/src/services/wikilink.ts`
  - `export function parseWikilink(raw: string): { slug: string; anchor?: string }`
  - `export function extractWikilinks(content: string): string[]` (slugs únicos do texto)
- **[CREATE]** `apps/nexus-backend/src/services/selector.ts`
  ```ts
  export interface SelectInput { slugs?: string[]; tags?: string[] }
  export interface SelectedDoc { slug: string; path: string; content: string; frontmatter: Record<string, unknown> }
  export interface SelectOptions { rootDir?: string; depth?: number } // depth default 1
  export function selectScope(input: SelectInput, opts?: SelectOptions): Promise<SelectedDoc[]>
  ```
  Regras: docs ficam em `<rootDir>/docs` (default rootDir = `NEXUS_ROOT_DIR` ou raiz do
  repo). Semente = docs cujo `slug` ∈ `slugs` OU cujo frontmatter `tags` intersecta `tags`.
  Expande seguindo `[[wikilinks]]` até `depth` níveis. Dedup por slug; ordem estável
  (alfabética por slug). Slug de um arquivo = nome sem `.md`.
- **[CREATE]** `apps/nexus-backend/src/services/export.service.ts`
  ```ts
  import type { Compressor } from './compressor.js'
  export interface ExportSection { slug: string; original: string }
  export interface ExportResult {
    artifact: string                 // texto comprimido (montado + Compressor)
    sections: ExportSection[]        // originais, p/ recuperação local
    stats: { docs: number; originalChars: number; compressedChars: number; ratio: number; engine: string }
  }
  export interface ExportOptions extends SelectOptions { compressor?: Compressor }
  export function buildExport(input: SelectInput, opts?: ExportOptions): Promise<ExportResult>
  ```
  Pipeline: `selectScope` → montar string (cada doc precedido de `\n\n## [[slug]]\n`) →
  `compressor.compress(montado)` (default `getCompressor()`) → preencher `ExportResult`.
- **[UPDATE]** `apps/nexus-backend/src/index.ts` — `POST /api/export`, body
  `{ slugs?: string[], tags?: string[], depth?: number }` → 200 com `ExportResult`.
- **[UPDATE]** `apps/nexus-backend/src/mcp/server.ts` — tool `nexus_build_export`
  (mesmo input; usar o mesmo `buildExport`). Registrar em `TASK_TOOL_DEFS`? Não — criar
  defs próprias ou um array `EXPORT_TOOL_DEFS`; manter o padrão de delegação.
- **[CREATE]** `apps/nexus-backend/src/tests/selector.test.ts` e `export.service.test.ts`.
