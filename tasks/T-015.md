# T-015 · Quality gate do design system no CI

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo (cache de pnpm/turbo no CI)
- **Test Runner:** `vitest` + `playwright`

## 1. Objetivo
Garantir, no CI (GitHub Actions da T-002), que o design system não quebre: validação de metadados, typecheck, build de tokens + Tailwind v4 e um smoke test básico de Playwright para os componentes entram no pipeline, orquestrados pelo Turbo, sem jobs separados por pacote.

## 2. Contexto RAG (Spec-Driven Development)
- [ ] [T-002 · CI](./T-002.md) — pipeline GitHub Actions, cache pnpm/turbo. **Pré-requisito.**
- [ ] [T-011 · Incorporar a lib](./T-011.md) — scripts `validate`/`typecheck`/`build`. **Pré-requisito.**
- [ ] [T-013 · Pipeline Turbo de tokens](./T-013.md) — ordem tokens→bundle (recomendado concluir antes).

## 3. Escopo de Arquivos (Inputs e Outputs)
- **[READ]** `.github/workflows/*.yml` (criado na T-002)
- **[READ]** `packages/design-system/package.json` (scripts disponíveis)
- **[UPDATE]** o workflow para incluir `pnpm build` + `pnpm test` cobrindo o novo pacote (via Turbo, sem job dedicado)
- **[UPDATE]** se necessário, `turbo.json` para expor `validate`/`typecheck` como tasks pipeline

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [ ] **Framework:** o próprio pipeline — `validate-metadata` + `tsc` + build Tailwind/Vite; teste básico de Smoke no Playwright (garantir que o showcase sobe e renderiza sem erros JS críticos).
- [ ] **Métricas/Cobertura:** binário — PR fica verde com o pacote incluído; PR que quebra metadados/typecheck fica vermelho.
- [ ] **Ambiente do Teste:** runner do GitHub Actions.
- [ ] **Fora de Escopo:** suíte visual completa por componente (incremental).

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** crie job separado por pacote — use `pnpm build`/`pnpm test` via Turbo (paralelismo nativo), como manda a T-002.
> - **NÃO** instale Tailwind/Style Dictionary globalmente no runner — vêm como devDeps do workspace.
> - **NÃO** tente cobrir todos os componentes visualmente. Configure apenas o smoke test básico (app sobe e não dá crash) como bloqueante.

1. Confirmar que o pacote entra em `pnpm install --frozen-lockfile` na raiz.
2. Garantir que `pnpm build` e `pnpm test` (Turbo) cobrem `@plataforma/design-system` (validate + typecheck + build).
3. Validar cache de Turbo/pnpm no runner.
4. Abrir PR de prova e confirmar pipeline verde (incluindo o Playwright smoke test no runner); simular quebra de metadados e confirmar pipeline vermelho.
