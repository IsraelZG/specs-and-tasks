## 1. Objetivo
Implementar `insertNode` + `hashNode` no pacote `@plataforma/core`: valida invariantes de linhagem (HLC filho > HLC pai, parentHash confere, entity_id herda) antes de persistir, detecta e rejeita ciclos, atualiza `entity_heads` via trigger
*(extraído de T-108 §1; `docs/conceitos/hlc.md` — invariante de pai; `docs/conceitos/entity-id.md` — herança)*.

### Contratos exatos
```ts
// --- packages/core/src/lineage.ts ---
import { SignedNode } from './signature.js'; // T-107
import { ULID, EntityId } from './ulid.js'; // T-102
import { StoragePort } from '@plataforma/protocol'; // T-004

export type NodeHash = Uint8Array; // SHA-256 do canonicalize

export function hashNode(node: SignedNode): NodeHash;

export async function insertNode(
  storage: StoragePort,
  node: SignedNode,
  parentId?: ULID,
): Promise<void>;
```
Validações:
1. Se `parentId`: carrega nó pai, confere `hashNode(pai)`
2. `HLC(filho) > HLC(pai)` (invariante — `docs/conceitos/hlc.md`)
3. `entity_id` do filho = `entity_id` do pai
4. Se nó-raiz (sem pai): `entity_id = id`
5. Detecta ciclos (walk reverso até raiz, rejeita se encontrar `node.id`)
6. Persiste nó + aresta MUTATES; trigger `trg_nodes_insert_entity_head` atualiza head

## 4. Estratégia de Testes
- [x] **Framework:** Vitest + `better-sqlite3` `:memory:`.

Casos de teste:
1. `insertNode(nó-raiz)`: persistido, `entity_id = id`.
2. `insertNode(filho, parentId=raiz)`: filho tem `entity_id` herdado do pai.
3. `insertNode` com `HLC(filho) ≤ HLC(pai)` → erro.
4. `insertNode` com `parentHash` errado → erro.
5. Ciclo: `A → B → tentar A` → erro.
6. `hashNode` determinístico: mesma entrada → mesmo hash.
