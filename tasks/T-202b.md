# T-202b · Noise_XX: handshake responder + encrypt/decrypt

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar `noiseHandshakeResponder` + `noiseEncrypt`/`noiseDecrypt` no pacote `@plataforma/transport`: completa o handshake Noise_XX como respondedor e provê cifra/decifra com chaves de sessão pós-handshake
*(extraído de T-202 §1)*.

### Contratos exatos
```ts
// --- packages/transport/src/noise.ts (extensão) ---

/** Handshake como respondedor. */
export function noiseHandshakeResponder(
  deviceKeyPair: { publicKey: Ed25519PublicKey; privateKey: Ed25519PrivateKey },
  identityEpochIndex: number,
  transport: NetworkAdapterPort,
): Promise<NoiseResult>;

/** Cifra mensagem com chave de sessão (AES-256-GCM via Noise). */
export function noiseEncrypt(key: Uint8Array, plaintext: Uint8Array): Promise<Uint8Array>;

/** Decifra mensagem. */
export function noiseDecrypt(key: Uint8Array, ciphertext: Uint8Array): Promise<Uint8Array>;
```

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).

Casos de teste:
1. Iniciador (T-202a) ↔ Respondedor → ambos recebem `NoiseResult` com estado `CONNECTED`.
2. `noiseEncrypt(keyA, msg)` → `noiseDecrypt(keyB, ciphertext)` roundtrip (chaves trocadas).
3. `noiseDecrypt` com chave errada → erro.
4. Respondedor com `deviceKeyPair` errado → handshake `FAILED`.
5. `identity_epoch_index` divergente → handshake ok, índices diferentes (catch-up).

3. Implemente `noiseEncrypt`/`noiseDecrypt` (AES-256-GCM sobre chaves de sessão).
4. Rode build + test.
