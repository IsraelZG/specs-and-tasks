# T-311 · Bancada: aba Sync

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (pacotes core/protocol) e `playwright` (E2E/Frontend)

## 1. Objetivo
Exibir o progresso do RBSR, ranges pendentes e botões para forçar divergência e testar a UI. Como parte da Fase B, os dados são consumidos unicamente pela API de `/control` e telemetria WebSocket do Hub, sem lógica nativa de sincronização rodando na PWA.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** misture lógica de protocolo dentro da View React. Consuma as respostas JSON da API `/control` do Hub.

1. Instancie o `SyncTab.tsx` usando Tailwind.
2. Mostre os `Root Fingerprints` baseados na telemetria empurrada via WebSocket.
3. Despache `POST /control/sync/diverge` ao clicar no botão de forçar partição.
