# T-311 · Bancada: aba Sync

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (React Testing Library) / `playwright`
- **Capacidade-alvo:** haiku

## 1. Objetivo
Implementar aba "Sync" no app Bancada: exibe progresso do RBSR (root fingerprints, ranges pendentes, nós importados), com botões para forçar partição e testar convergência
*(derivado de T-008 — Bancada; T-302 — RBSR exchange)*.

### Contratos
```tsx
// --- apps/bancada/src/tabs/SyncTab.tsx ---
import { RBSRSession } from '@plataforma/protocol'; // T-302

export interface SyncTabProps {
  session?: RBSRSession;
}

export function SyncTab({ session }: SyncTabProps): JSX.Element;
```
- Exibe: root fingerprint local vs remoto, ranges pendentes, nós importados na última rodada.
- Botão "Forçar Partição" → dispara `ControlPort.setPartition`.
- Botão "Forçar Sync" → dispara reconciliação manual.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest + React Testing Library (JSDOM).

Casos de teste:
1. Sem sessão → mostra "Aguardando conexão".
2. Com sessão IDLE → mostra root fingerprints.
3. Com ranges pendentes → lista ranges.
4. Clique "Forçar Sync" → callback disparado.

## 5. Instruções de Execução
1. Escreva testes com stub de `RBSRSession`.
2. Implemente `SyncTab`.
3. Integre ao shell de abas da Bancada.
4. Rode build + test.
