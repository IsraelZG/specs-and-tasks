# T-802 · Manifesto, renditions e fontes

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar manifesto, renditions e fontes no pacote `@plataforma/core`: nó `CONTENT` por rendition com schema do caderno-3/05 §4.3; arestas `RELATES:MEDIA:RENDITION` e `SERVES` (fontes duráveis); `MUTATES` proibido entre renditions (lint de grafo)
*(derivado de `docs/plano-de-implementacao.md` §11 T-802)*.

### Contratos exatos
```ts
// --- packages/core/src/media/manifesto.ts ---
import { EntityId } from '@plataforma/protocol'; // T-102
import { StoragePort } from '@plataforma/protocol'; // T-004

export interface RenditionSchema {
  mediaType: string;            // MIME type (ex: "video/mp4")
  width?: number;
  height?: number;
  bitrate?: number;             // bps
  durationMs?: number;
  language?: string;
  codec?: string;
}

export interface MediaManifest {
  manifestId: EntityId;         // nó CONTENT do manifesto
  originalFilename: string;
  totalSize: number;            // bytes totais (todas as renditions)
  infoHash: string;             // InfoHash do arquivo original
  renditions: RenditionEntry[];
}

export interface RenditionEntry {
  renditionId: EntityId;        // nó CONTENT da rendition
  schema: RenditionSchema;
  infoHash: string;             // InfoHash da rendition cifrada
  size: number;                 // bytes
}

export interface FonteEntry {
  fonteId: EntityId;            // nó CONTENT da fonte
  infoHash: string;
  fonteType: 'webtorrent' | 'webseed' | 'ipfs' | 'cdn';
  url?: string;
  priority: number;             // 0=mais alta
}

export interface MediaManifestService {
  /** Cria manifesto com renditions. */
  createManifest(filename: string, renditions: RenditionSchema[]): Promise<MediaManifest>;

  /** Adiciona rendition ao manifesto. */
  addRendition(manifestId: EntityId, rendition: RenditionSchema): Promise<RenditionEntry>;

  /** Adiciona fonte durável via aresta SERVES. */
  addFonte(renditionId: EntityId, fonte: FonteEntry): Promise<void>;

  /** Remove fonte via lápide. */
  removeFonte(fonteId: EntityId): Promise<void>;

  /** Lint de grafo: MUTATES é proibido entre renditions. */
  validateRenditionGraph(manifestId: EntityId): Promise<{ valid: boolean; violations: string[] }>;
}
```
- `MUTATES` **proibido** entre renditions — renderizações são imutáveis (lint de grafo).
- Fontes adicionadas/removidas por arestas `SERVES` e lápides, nunca por mutação.
- Schema do `caderno-3/05 §4.3`: campos obrigatórios (mediaType, size, infoHash) + opcionais (width, height, bitrate, etc.).
- Cada rendition é um nó `CONTENT` separado, ligado ao manifesto via `RELATES:MEDIA:RENDITION`.
