# T-1008 · Automação via MCP Safe Runner (Pre-approved Scripts)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v22+
- **Package Manager:** `pnpm`
- **Test Runner:** `vitest`

## 1. Objetivo
Criar um motor de scripts executáveis no backend Node, onde um arquivo JSON armazena "scripts permitidos" (ex: "rodar linter", "build project"). O MCP Gateway exporá a ferramenta `nexus_run_safe_script` para que agentes rodem esses comandos sem precisar pedir "ask_permission" de console, permitindo total autonomia num escopo seguro.

## 2. Contexto RAG (Spec-Driven Development)
- [ ] [Plano de Implementação Nexus](../plano-de-implementacao.md)

## 3. Escopo de Arquivos (Inputs e Outputs)
- **[CREATE]** `apps/nexus-backend/src/services/safe_runner.ts`
- **[UPDATE]** `apps/nexus-backend/src/mcp/server.ts`
- **[CREATE]** `apps/nexus-backend/config/safe-scripts.json`

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [ ] **Framework:** Vitest
- [ ] **Métricas/Cobertura:** Testar rejeição de comandos não cadastrados. Evitar "command injection".

## 5. Instruções de Execução (Step-by-Step)
1. Crie o arquivo de mapeamento `safe-scripts.json`.
2. Implemente o parser no backend que valida a string que o agente enviar via MCP.
3. Use a API `child_process` do Node.js de forma isolada para retornar o output do comando.
