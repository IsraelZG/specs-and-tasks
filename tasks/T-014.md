# T-014 · Migrar hook de metadados + skill para .claude da raiz

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo
- **Test Runner:** smoke manual do hook (PowerShell)

## 1. Objetivo
Integrar a automação de autoria de UI que vem com o design system ao monorepo: o hook `PostToolUse` (que valida metadados e reconstrói o índice ao editar `*.metadata.ts`) e o skill `design-system-component-author`. Ajustar os paths para o novo local (`packages/design-system`) sem colidir com o `.claude` e o `CLAUDE.md` da wiki.

## 2. Contexto RAG (Spec-Driven Development)
- [ ] [T-011 · Incorporar a lib](./T-011.md) — traz `scripts/validate-metadata.mjs` e `build-component-index.mjs`. **Pré-requisito.**
- [ ] Fonte: `c:\Dev2026\Design System\.claude\` (settings.json + skills/) — registrado em `design-system-stub.txt`.

## 3. Escopo de Arquivos (Inputs e Outputs)
- **[READ]** `c:\Dev2026\Design System\.claude\settings.json` (hook original com `Push-Location 'design-system'`)
- **[READ]** `c:\Dev2026\Design System\.claude\skills\design-system-component-author\SKILL.md`
- **[UPDATE]** `.claude/settings.json` (raiz) — mesclar o `PostToolUse` ajustando `Push-Location 'packages/design-system'`
- **[CREATE]** `.claude/skills/design-system-component-author/SKILL.md` (migração do skill)
- **[KEEP]** `packages/design-system/CLAUDE.md` e `AGENTS.md` permanecem **aninhados** (não mover para a raiz)

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [ ] **Framework:** smoke manual — editar um `*.metadata.ts` e confirmar que `validate` + `build:index` disparam.
- [ ] **Métricas/Cobertura:** binário — o hook roda no path certo e regenera `components.index.json`.
- [ ] **Ambiente do Teste:** PowerShell (o hook é `shell: powershell`).
- [ ] **Fora de Escopo:** lógica de validação de metadados (já testada pelos scripts do pacote).

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** sobrescreva hooks já existentes no `.claude/settings.json` da raiz — **mescle** no array `PostToolUse`.
> - **NÃO** mova o `CLAUDE.md`/`AGENTS.md` do pacote para a raiz (são contratos de UI escopados ao diretório).
> - **NÃO** deixe o `Push-Location` apontando para `design-system` (path antigo).

1. Ler o hook original e adaptar o comando: `Push-Location 'packages/design-system'`.
2. Mesclar o objeto de hook no `.claude/settings.json` da raiz (preservando hooks existentes).
3. Copiar o skill para `.claude/skills/design-system-component-author/`.
4. Smoke: editar um `*.metadata.ts` em `packages/design-system` e observar o statusMessage + `components.index.json` atualizado.
