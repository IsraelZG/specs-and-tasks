**Solução proposta (não inventar, escolher uma das 3):**
- **Opção A — Tornar o regex path-agnostic:** `/(?:^|\/)packages\/design-system\/.*\.metadata\.ts$/`
  — casa com qualquer path que **contenha** `packages/design-system/...metadata.ts`.
  Mantém a defesa em profundidade (outros pacotes `.metadata.ts` ainda não casam).
  **Recomendação do auditor** — corrige com 1 caractere de mudança.
- **Opção B — Usar `Resolve-Path` no PowerShell para normalizar:** converter o path
  absoluto para relativo antes do regex (`Resolve-Path $f -Relative`). Mais robusto
  contra symlinks, working directorys custom, mas mais código PowerShell.
- **Opção C — Reescrever em Node:** trocar PowerShell por hook Node que faz
  `path.relative(process.cwd(), filePath)` antes do regex. Mais limpo, portável
  cross-platform, mas reescrita substancial.
