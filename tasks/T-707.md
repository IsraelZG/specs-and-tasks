## 1. Objetivo
Implementar `device_state.db` no pacote `@plataforma/core`: banco SQLite isolado para a categoria Local+Persistente: rascunhos, cache de prefetch, preferências, histórico de peers (consumido pelo T-406); roteamento via matriz IoC (T-310) cai aqui automaticamente
*(derivado de `docs/plano-de-implementacao.md` §10 T-707 — realocado de T-701)*.

### Contratos exatos
```ts
// --- packages/core/src/deviceState/deviceStateDb.ts ---
import { StoragePort, ClockPort } from '@plataforma/protocol'; // T-004 — injetados no construtor (Decisão §6 #2)

/** Minimalista, casa 1:1 com a tabela `peer_history` de T-701a (Decisão §6 #1). */
export interface PeerHistoryEntry {
  peerId: string;
  lastMultiaddr: string | null;
  lastSeenAt: number;
  successCount: number;
  failureCount: number;
}

export interface DeviceStateSchema {
  tables: {
    drafts: string;              // rascunhos (Automerge doc state)
    prefetchCache: string;       // cache de prefetch (union-based)
    preferences: string;         // preferências do usuário (LWW)
    peerHistory: string;         // histórico de peers (T-406)
    blindArchives: string;       // archives cegos (T-313)
  };
}

export interface DeviceStateDb {
  /** Construtor recebe StoragePort e ClockPort explicitamente (Decisão §6 #2) — não instancia StoragePort por conta própria. */
  new (storage: StoragePort, clock: ClockPort): DeviceStateDb;

  /** Inicializa banco isolado com schema (storagePath saiu daqui — vem do StoragePort injetado). */
  init(): Promise<void>;

  /** Armazena rascunho. */
  saveDraft(docId: string, state: Uint8Array): Promise<void>;

  /** Carrega rascunho. */
  loadDraft(docId: string): Promise<Uint8Array | null>;

  /** Atualiza cache de prefetch (merge union). Valor via codecEncode/MessagePack (Decisão §6 #3). */
  upsertPrefetch(key: string, value: Uint8Array): Promise<void>;

  /** Consulta cache de prefetch. */
  getPrefetch(key: string): Promise<Uint8Array | null>;

  /** Salva preferência (LWW). Valor tipado como string (Decisão §6 #3, sem JSON.stringify genérico). */
  setPreference(key: string, value: string, hlc: number): Promise<void>;

  /** Carrega preferência. */
  getPreference(key: string): Promise<string | null>;

  /** Salva entrada de histórico de peers. */
  savePeerHistory(entry: PeerHistoryEntry): Promise<void>;

  /** Carrega histórico de peers. */
  getPeerHistory(): Promise<PeerHistoryEntry[]>;

  /** Fecha banco. */
  close(): Promise<void>;

  /** Reseta banco (apaga todos os dados). */
  reset(): Promise<void>;
}
```
- Banco isolado: separado do grafo principal (`nodes`/`edges`).
- Local+Persistente: dados que não são replicados via RBSR.
- Roteamento: `TransportMatrix` (T-310) roteia `device_state_db` para este banco automaticamente.
- Categorias: rascunhos (Automerge estado), prefetch cache (union-based), preferências (LWW), peer history (T-406), blind archives (T-313).

## 6. Feedback de Especificação

### Decisões de Arquiteto (2026-07-01, resolvidas via `/arquiteto-decisoes`)

**Decisão #1 — `PeerHistoryEntry`: DECIDIDO opção (B).** Simplifica para a `PeerHistoryEntry`
minimalista já implementada em `T-701a` (peerId, lastMultiaddr, lastSeenAt, successCount,
failureCount). Lógica complexa de latência/probes fica em RAM no `@plataforma/transport`, não
persistida. Evita retrabalho na tabela SQLite já pronta.

**Decisão #2 — injeção de storage: DECIDIDO opção (A).** Construtor do `DeviceStateDb` recebe
`StoragePort` e `ClockPort`; `storagePath` é removido de `init()`. Mesmo padrão de I/O explícito
aplicado em T-504/T-705 neste lote de decisões — sem instanciar `StoragePort` por conta própria.

**Decisão #3 — serialização de `unknown`: DECIDIDO opção (B).** `upsertPrefetch`/`setPreference`
tipam concretamente: `Uint8Array` (rascunhos/prefetch, via `codecEncode`/MessagePack) e `string`
(preferências) — sem `unknown` nem `JSON.stringify` genérico. Alinhado com `T-701.md`.
