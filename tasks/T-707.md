## 1. Objetivo
Implementar `device_state.db` no pacote `@plataforma/core`: banco SQLite isolado para a categoria Local+Persistente: rascunhos, cache de prefetch, preferências, histórico de peers (consumido pelo T-406); roteamento via matriz IoC (T-310) cai aqui automaticamente
*(derivado de `docs/plano-de-implementacao.md` §10 T-707 — realocado de T-701)*.

### Contratos exatos
```ts
// --- packages/core/src/deviceState/deviceStateDb.ts ---
import { StoragePort } from '@plataforma/protocol'; // T-004
import { PeerHistoryEntry } from './types.js'; // Definido localmente ou importado após decisão

export interface DeviceStateSchema {
  tables: {
    drafts: string;              // rascunhos (Automerge doc state)
    prefetchCache: string;       // cache de prefetch (union-based)
    preferences: string;         // preferências do usuário (LWW)
    peerHistory: string;         // histórico de peers (T-406)
    blindArchives: string;       // archives cegos (T-313)
  };
}

export interface DeviceStateDb {
  /** Inicializa banco isolado com schema. */
  init(storagePath?: string): Promise<void>;

  /** Armazena rascunho. */
  saveDraft(docId: string, state: Uint8Array): Promise<void>;

  /** Carrega rascunho. */
  loadDraft(docId: string): Promise<Uint8Array | null>;

  /** Atualiza cache de prefetch (merge union). */
  upsertPrefetch(key: string, value: unknown): Promise<void>;

  /** Consulta cache de prefetch. */
  getPrefetch(key: string): Promise<unknown>;

  /** Salva preferência (LWW). */
  setPreference(key: string, value: unknown, hlc: number): Promise<void>;

  /** Carrega preferência. */
  getPreference(key: string): Promise<unknown>;

  /** Salva entrada de histórico de peers. */
  savePeerHistory(entry: PeerHistoryEntry): Promise<void>;

  /** Carrega histórico de peers. */
  getPeerHistory(): Promise<PeerHistoryEntry[]>;

  /** Fecha banco. */
  close(): Promise<void>;

  /** Reseta banco (apaga todos os dados). */
  reset(): Promise<void>;
}
```
- Banco isolado: separado do grafo principal (`nodes`/`edges`).
- Local+Persistente: dados que não são replicados via RBSR.
- Roteamento: `TransportMatrix` (T-310) roteia `device_state_db` para este banco automaticamente.
- Categorias: rascunhos (Automerge estado), prefetch cache (union-based), preferências (LWW), peer history (T-406), blind archives (T-313).

## 6. Feedback de Especificação

> [!WARNING]
> **Decisões abertas para o Arquiteto:**
>
> 1. **Definição de `PeerHistoryEntry` e Mapeamento DDL**:
>    - Em `T-406.md`, `PeerHistoryEntry` possui campos como `multiaddrs: Multiaddr[]`, `lastLatencyMs: number`, `connectionCount: number`, `lastTransport`.
>    - Em `T-701a` DDL, a tabela `peer_history` contém apenas `peer_id`, `last_multiaddr`, `last_seen_at`, `success_count`, `failure_count`.
>    - **Opções**:
>      - (A) Ajustar a tabela SQLite em `T-701a` para conter colunas adequadas para latência, conexões, transporte, ou persistir como payload serializado.
>      - (B) Simplificar a interface de `T-707` para usar a `PeerHistoryEntry` minimalista do `T-701.md` (peerId, lastMultiaddr, lastSeenAt, successCount, failureCount) e delegar a lógica complexa de latência/probes ao próprio `@plataforma/transport` em RAM.
>
> 2. **Injeção de `StoragePort` vs `storagePath` no `init()`**:
>    - O contrato especifica `init(storagePath?: string)`. No entanto, as diretrizes de isolamento exigem que toda persistência ocorra via o adaptador isomórfico `StoragePort` injetado (como feito em T-106 e T-701).
>    - **Opções**:
>      - (A) Mudar o construtor do `DeviceStateDb` para receber `StoragePort` e `ClockPort`, removendo `storagePath` do `init()`.
>      - (B) Permitir que o banco instancie seu próprio `StoragePort` caso `storagePath` seja passado, quebrando o isolamento de adaptadores.
>
> 3. **Políticas de Serialização de `unknown`**:
>    - `upsertPrefetch(key, value: unknown)` e `setPreference(key, value: unknown)` aceitam valores genéricos.
>    - **Opções**:
>      - (A) Serializar usando `JSON.stringify`/`JSON.parse` para texto e `codecEncode`/`codecDecode` (MessagePack) para blobs.
>      - (B) Alterar os tipos no contrato para `Uint8Array` (rascunhos/prefetch) e `string` (preferências), simplificando a camada do banco de dados (conforme `T-701.md`).
