## 1. Objetivo
Criar o DDL v1 do SQLite (tabelas `nodes` e `edges` + índices) e o array `MIGRATIONS` no pacote `@plataforma/core`

### Contratos exatos
```ts

  version: number;
  sql: string;
}

/** Schema v1. Adições futuras ganham version > 1. */
export const MIGRATIONS: Migration[] = [
  { version: 1, sql: `...DDL abaixo...` },
];
```

DDL (SQL fixo):
```sql
CREATE TABLE IF NOT EXISTS nodes (
  id TEXT NOT NULL PRIMARY KEY, node_type TEXT NOT NULL,
  payload BLOB NOT NULL, hlc TEXT NOT NULL,
  public_key BLOB NOT NULL, signature BLOB NOT NULL,
  created_at INTEGER NOT NULL DEFAULT (unixepoch())
);
CREATE TABLE IF NOT EXISTS edges (
  id TEXT NOT NULL PRIMARY KEY, edge_type TEXT NOT NULL,
  source_id TEXT NOT NULL REFERENCES nodes(id),
  target_id TEXT NOT NULL REFERENCES nodes(id),
  payload BLOB, hlc TEXT NOT NULL,
  public_key BLOB NOT NULL, signature BLOB NOT NULL,
  created_at INTEGER NOT NULL DEFAULT (unixepoch())
);
CREATE INDEX IF NOT EXISTS idx_nodes_type ON nodes(node_type);
CREATE INDEX IF NOT EXISTS idx_edges_source ON edges(source_id);
CREATE INDEX IF NOT EXISTS idx_edges_target ON edges(target_id);
CREATE INDEX IF NOT EXISTS idx_edges_type ON edges(edge_type);
```

## 4. Estratégia de Testes
- [x] **Framework:** Vitest + `better-sqlite3` `:memory:`.

Casos de teste:
1. Rodar DDL contra SQLite `:memory:` → tabelas `nodes` e `edges` criadas.
2. INSERT em `nodes` com campos obrigatórios → OK.
3. INSERT em `edges` com FK para `nodes` existente → OK.
4. INSERT em `edges` com FK inexistente → erro de constraint.
5. Índices (`idx_nodes_type`, `idx_edges_source`, `idx_edges_target`, `idx_edges_type`) existem.
6. Rodar DDL 2× → idempotente (IF NOT EXISTS).
