# T-018 · Runner de cenários (pnpm scenario)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar CLI `pnpm scenario <nome>` no pacote `@plataforma/testkit`: carrega cenários JSON reproduzíveis (seeds, peers, ações, asserções) e os executa deterministicamente via SimNetwork + VirtualClock, com relatório de convergência
*(derivado de `docs/plano-de-implementacao.md` M0/E0.3 — T-018; realocado de T-011 por colisão — `tasks/_correlacao-plano.md` §1)*.

### Contratos exatos
```ts
// --- tools/scenarios/types.ts ---
export interface ScenarioStep {
  action: 'createPeer' | 'connect' | 'sendMessage' | 'partition' | 'heal' | 'advanceClock';
  params: Record<string, unknown>;
}

export interface Scenario {
  name: string;
  seed: string;
  peers: number;
  steps: ScenarioStep[];
  assertions: Array<{ type: 'converged' | 'state'; params: Record<string, unknown> }>;
}

export interface ScenarioResult {
  name: string;
  passed: boolean;
  durationMs: number;
  steps: number;
  failures: string[];
}
```

```bash
# CLI
pnpm scenario genesis-corporativa    # roda cenário específico
pnpm scenario --all                  # roda todos os cenários
pnpm scenario --list                 # lista cenários disponíveis
```

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).

Casos de teste:
1. Cenário com 2 peers convergindo → `passed: true`.
2. Cenário com partição + heal → convergência após heal.
3. Seed fixa → resultado idêntico em qualquer máquina.
4. Cenário com falha esperada → `passed: false`, `failures` contém descrição.
5. `--list` lista todos os cenários disponíveis.

## 5. Instruções de Execução
1. Crie `tools/scenarios/types.ts`.
2. Implemente runner CLI.
3. Crie cenário `genesis-corporativa.json`.
4. Adicione script no `package.json`.
5. Rode `pnpm scenario genesis-corporativa`.
