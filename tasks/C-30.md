### Casos de teste (Vitest, node, sem browser)

1. **Anti-fake corrigido (m1) — `provider-remote-smoke.test.ts:107-116`.**
   O teste 4 atual usa `try { await probeProvider(...) } catch (e) { expect(...) }` — se `probeProvider` **não** lançar, o `expect` nunca roda e o teste passa silenciosamente.
   - **Correção:** adicionar `await expect(probeProvider(...)).rejects.toMatchObject({ code: "UPSTREAM_ERROR" })` **antes** do bloco `try/catch`.
   - **Verificação:** `pnpm --filter @plataforma/estaleiro test:integration` → o teste 4 agora FALHA se `probeProvider` resolver em vez de rejeitar.

2. **Dead code removido (m2) — `provider-remote-smoke.test.ts:39`.**
   Variável `content` (`parsed.messages?.[...]?.content ?? "MARKER_REMOTE"`) é computada mas nunca usada — o corpo da resposta é fixo `"MARKER_REMOTE"` na linha 49.
   - **Correção:** remover a linha `const content = ...`.
   - **Verificação:** lint do `@plataforma/estaleiro-core` permanece limpo (a variável morta não gerava warning de lint, mas o código fica mais limpo).

3. **Path validation no fake server (m3) — `provider-remote-smoke.test.ts:28-55` (`handleRequest`).**
   `handleRequest` não valida `url.pathname` — qualquer rota responde 200 com o corpo OpenAI. Se `@ai-sdk/openai` mudar a concatenação de path, o teste passa pelo motivo errado.
   - **Correção:** adicionar após o parse do URL (antes do check de `error`/`delay`):
     ```typescript
     if (url.pathname !== "/v1/chat/completions") {
       res.writeHead(404);
       res.end();
       return;
     }
     ```
   - **Verificação:** testes 1-3 continuam passando (`pnpm --filter @plataforma/estaleiro test:integration`). Nenhum novo teste necessário — o path fix garante que os testes existentes só passam se o adapter enviar para o path correto.

4. **Auditoria de bump major (i1) — `@ai-sdk/openai` `^1.3.24` → `^2.0.0`.**
   - Grep no monorepo (`packages/*/src`, `apps/*/src`, `apps/*/core/src`) por `createOpenAI` para confirmar que só `provider-probe.ts` o consome *(derivado de EST-43a §8 Reviewer 1, i1 — "grep mostrou apenas provider-probe.ts em apps/estaleiro/core")*.
   - Ler release notes de `@ai-sdk/openai` v2.0.0 (npm: https://www.npmjs.com/package/@ai-sdk/openai ou GitHub: https://github.com/vercel/ai).
   - Rodar `pnpm audit` e registrar o resultado (disposition).
