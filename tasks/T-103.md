### Contratos exatos (assinaturas TS fixadas)

```ts
// --- packages/core/src/hlc.ts ---
import { ClockPort } from '@plataforma/protocol'; // T-004

/** HLC empacotado como inteiro 48+16 bits: (pt << 16) | c.
 *  Derivado de docs/conceitos/hlc.md. */
export type HLCTimestamp = bigint;

/** Componentes desempacotados do HLC. */
export interface HLCParts {
  pt: number; // wall-clock em ms (48 bits efetivos)
  c:  number; // contador lógico (16 bits, 0–65535)
}

/** Limite de drift: se pt_remoto > wall_clock_local + MAX_DRIFT_MS, rejeita.
 *  Derivado de docs/conceitos/hlc.md §Contrato. */
export const MAX_DRIFT_MS = 300_000; // 5 minutos

/** Relógio Lógico Híbrido. Estado mutável: mantém (pt, c) corrente.
 *  Invariantes documentados em docs/conceitos/hlc.md. */
export class HybridLogicalClock {
  constructor(clock: ClockPort);

  /** Retorna HLC corrente (sem avançar). */
  now(): HLCTimestamp;

  /** Evento local: avança pt para max(clock.now(), pt) e incrementa c (ou zera se pt avançou). */
  tick(): HLCTimestamp;

  /** Evento de recebimento: incorpora HLC remoto. Aplica regras de monotonicidade e drift. */
  receive(remote: HLCTimestamp): HLCTimestamp;

  /** Desempacota HLC em (pt, c). */
  static unpack(hlc: HLCTimestamp): HLCParts;

  /** Empacota (pt, c) em HLCTimestamp. */
  static pack(pt: number, c: number): HLCTimestamp;

  /** Compara dois HLCs. Retorna <0, 0, >0. */
  static compare(a: HLCTimestamp, b: HLCTimestamp): number;
}
```

### Regras de drift (derivado de hlc.md §Contrato)
- Se `pt_remoto > wall_clock_local + MAX_DRIFT_MS` → HLC remoto é rejeitado (nó em quarentena). O relógio local **não** incorpora o valor.
- `tick()`: `pt = max(clock.now(), pt)`; se pt mudou → `c = 0`; senão → `c++`. Se `c` atingir 65535, avance `pt` em 1ms e resete `c = 0`.
- `receive(remote)`: `pt = max(clock.now(), pt, remote_pt)`; `c = (pt === pt_anterior && pt === remote_pt) ? max(c, remote_c) + 1 : (pt === remote_pt ? remote_c + 1 : 0)`. Se `c` atingir 65535, avance `pt` em 1ms e resete `c = 0`.

- [x] **Framework:** Vitest (Node puro).

- [x] **Fora de Escopo:** Validação de transporte (T-201), assinatura (T-107).

Casos de teste (numerados):
1. `tick()` com clock fixo: primeiro tick → pt=clock, c=0; segundo tick → pt igual, c=1.
2. `tick()` após `clock.advance(100)`: pt avança, c zera.
3. `receive(hlc_remoto)` com `hlc_remoto < hlc_local`: HLC local não regride (monotonicidade).
4. `receive(hlc_remoto)` com `hlc_remoto > hlc_local`: pt avança, c=remote_c+1 (evento de recebimento).
5. `receive(hlc_remoto)` com `hlc_remoto.pt` além de `MAX_DRIFT_MS`: rejeitado (erro).
6. `HLC.compare(a, b)` ordenação correta para (pt_a < pt_b), (pt_a == pt_b, c_a < c_b), (iguais).
7. `HLC.pack` + `HLC.unpack` roundtrip.
8. Sequência de 1000 `tick()` com clock avançando → pt monotônico, c cicla 0→65535.
9. Dois peers com mesmo clock: `peerA.tick()` e `peerB.receive(peerA.now())` → `peerA.now() < peerB.now()` (causal).
10. `tick()` após 65536 chamadas sem avanço de pt → c satura e lança erro (overflow de 16 bits).
