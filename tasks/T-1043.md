# T-1043 · Reescrever os 8 sitios de SQL do core para GraphStorePort

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm` (NÃO USE npm ou yarn)
- **Monorepo:** Turborepo (`pnpm build`, `pnpm test`, `pnpm lint` na raiz afetam todos os pacotes)
- **Test Runner:** `vitest` (pacotes core/protocol)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Reescrever as 86 ocorrências de consultas SQL espalhadas nos ~8 arquivos do core para utilizarem a nova interface `GraphStorePort`, removendo qualquer dependência direta de SQL no núcleo do protocolo e do core.

> [!WARNING]
> **Tarefa de Alto Risco.** Esta reescrita é mecanicamente ampla e afeta diretamente a lógica de sincronização, consistência e convergência do grafo. Riscos de regressão no sync devem ser mitigados utilizando rigorosamente a suíte de convergência do `@plataforma/testkit` como rede de proteção de qualidade (DoD).

## 5. Instructions of Execution (Step-by-Step)
1. Identifique e mapeie cada um dos ~86 pontos de consulta SQL aos arquivos `nodes` e `edges`.
2. Substitua as chamadas do tipo `storage.exec("SELECT ...")` por métodos da `GraphStorePort` (`getNode`, `getEdges`, `rangeScan`, etc.).
3. Rode repetidamente os testes de unidade de cada módulo (`pnpm --filter @plataforma/core test`) após migrar cada arquivo individualmente para isolar regressões rapidamente.
4. Execute a suíte de simulação e convergência de rede de `@plataforma/testkit` para assegurar que o sync e replicação P2P continuam convergindo perfeitamente sob partições e perdas.
