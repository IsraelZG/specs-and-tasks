# T-705 · Cerimônia QR + SAS — pareamento de dispositivo

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro) + `playwright` (E2E)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar cerimônia de pareamento QR + SAS no pacote `@plataforma/core`: par efêmero de pareamento (TTL 5 min) + QR (multiaddr transiente, chave efêmera, nonce); canal Noise_XX de cerimônia; confirmação SAS derivada do transcript; emissão de `ASSET:PERMISSION` + `DELEGATED_TO` → chave definitiva do novo dispositivo + bump da Época de Identidade
*(derivado de `docs/plano-de-implementacao.md` §10 T-705 — "RFC-005 §A.5")*.

### Contratos exatos
```ts
// --- packages/core/src/pairing/pairingCeremony.ts ---
import { PeerId } from '@plataforma/protocol'; // T-105a
import { Ed25519PrivateKey, Ed25519PublicKey } from '@plataforma/crypto'; // T-101a

export interface PairingSession {
  sessionId: string;             // ULID da sessão
  ephemeralKeyPair: { priv: Ed25519PrivateKey; pub: Ed25519PublicKey };
  nonce: Uint8Array;             // 16 bytes
  multiaddr: string;             // multiaddr transiente
  createdAt: number;
  expiresAt: number;             // TTL 5 min
}

export interface QrPayload {
  multiaddr: string;
  ephemeralPubKey: Ed25519PublicKey;
  nonce: Uint8Array;
  checksum: Uint8Array;          // blake2s256 dos campos acima
}

export interface SasConfirmation {
  displayCode: string;           // 6 dígitos para confirmação humana
  transcript: Uint8Array;        // transcript do Noise_XX (derivação do SAS)
  verified: boolean;             // se usuário confirmou
}

export interface PairingResult {
  success: boolean;
  newDevicePeerId?: PeerId;
  permissionEntityId?: string;   // ASSET:PERMISSION emitido
  delegatedToEntityId?: string;  // DELEGATED_TO
  identityEpochBumped: boolean;
  reason?: string;
}

export interface PairingCeremony {
  /** Inicia sessão de pareamento (Lado A: device existente). */
  initiatePairing(deviceKey: Ed25519PrivateKey): Promise<PairingSession>;

  /** Gera QR code a partir da sessão. */
  generateQR(session: PairingSession): Promise<QrPayload>;

  /** Lado B: lê QR e inicia Noise_XX de cerimônia. */
  scanQR(qr: QrPayload, newDeviceKey: Ed25519PrivateKey): Promise<PairingSession>;

  /** Deriva SAS do transcript Noise_XX para confirmação humana. */
  deriveSAS(session: PairingSession): Promise<SasConfirmation>;

  /** Confirma SAS (usuário comparou os 6 dígitos). */
  confirmSAS(session: PairingSession, confirmed: boolean): Promise<PairingResult>;

  /** Finaliza: emite ASSET:PERMISSION + DELEGATED_TO + bump de época. */
  finalizePairing(session: PairingSession): Promise<PairingResult>;
}
```
- Par efêmero: TTL 5 minutos; chaves efêmeras destruídas após pareamento.
- QR: multiaddr transiente, chave efêmera pública, nonce, checksum.
- SAS: 6 dígitos derivados do transcript Noise_XX — confirmação humana contra MITM.
- Chave mestra **nunca** atravessa o canal.
- Novo dispositivo obtém chaves de época via `requestEpochKey` (T-110).
- Revogação do dispositivo (lápide + bump) nega `requestEpochKey` subsequente.

## 6. Feedback de Especificação

> [!WARNING]
> **Decisões abertas para o Arquiteto:**
>
> 1. **Acesso ao Handshake Hash `h` do Noise**:
>    - A função `deriveSAS` precisa do handshake transcript hash `h` (ou `handshakeHash`). No entanto, o `initiateNoiseXX` e `respondNoiseXX` no `@plataforma/transport` retornam apenas `NoiseHandshakeResult`, que oculta o transcript `h` interno do handshake.
>    - **Opções**:
>      - (A) Modificar a interface `NoiseHandshakeResult` no `@plataforma/transport` para expor `handshakeHash: Uint8Array`.
>      - (B) Reescrever a orquestração do handshake em `pairingCeremony.ts` instanciando diretamente a classe de baixo nível `NoiseXX` para expor o hash.
>
> 2. **Injeção de Rede e Persistência**:
>    - A interface `PairingCeremony` não recebe `NetworkAdapterPort` (para o handshake no Lado B), nem `StoragePort` (para salvar o novo `ASSET:PERMISSION` no Lado A).
>    - **Opções**:
>      - (A) Passar `StoragePort` e `NetworkAdapterPort` (ou um construtor/classe que os gerencie) para a classe da cerimônia.
>      - (B) Fazer as funções de cerimônia retornarem estruturas puras de dados a serem enviadas/salvas pela camada superior.
>
> 3. **Algoritmo de Derivação do SAS**:
>    - A especificação exige "6 dígitos para confirmação humana" derivados do transcript. Qual a fórmula/KDF exata para obter esses 6 dígitos do hash `h`?
>    - **Opções**:
>      - (A) `SAS = (HKDF-SHA256(h, salt="", info="SAS") % 1000000).toString().padStart(6, '0')`.
>      - (B) Converter os primeiros 3 bytes de `h` em número e aplicar o mod.
