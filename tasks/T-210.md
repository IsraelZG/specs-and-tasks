# T-210 · Convite `ASSET:INVITE` — emissão e cerimônia

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar emissão e cerimônia de consumo do `ASSET:INVITE` no pacote `@plataforma/core`: asset consumível com expiry/single-use, payload com multiaddr hints, invite_code, inviter_peer_id e assinatura, com cerimônia completa de validação
*(derivado de `docs/plano-de-implementacao.md` §5 T-210)*.

### Contratos exatos
```ts
// --- packages/core/src/invite.ts ---
import { PeerId, StoragePort } from '@plataforma/protocol';
import { Ed25519PrivateKey, Ed25519PublicKey } from '@plataforma/crypto';

export interface InvitePayload {
  multiaddrHints: string[];      // multiaddr hints do inviter
  inviteCode: string;            // código único do convite
  inviterPeerId: PeerId;
  expiry: number;                // timestamp Unix ms
}

export interface SignedInvite {
  payload: InvitePayload;
  signature: Uint8Array;         // Ed25519 do inviter sobre payload canônico
}

export interface InviteCeremonyResult {
  success: boolean;
  profileEntityId?: string;      // PROFILE:AUTHENTICATION criado
  vouchesEdgeId?: string;        // aresta VOUCHES_FOR
  inviteTombstoneId?: string;    // lápide do convite consumido
}

/** Emite convite assinado. */
export function issueInvite(
  inviterKey: Ed25519PrivateKey,
  inviterPeerId: PeerId,
  multiaddrHints: string[],
  expiryMs?: number, // default 3600000 (1h)
): Promise<SignedInvite>;

/** Valida convite (assinatura + expiry + não-gasto). */
export function validateInvite(
  storage: StoragePort,
  invite: SignedInvite,
  inviterPubKey: Ed25519PublicKey,
): Promise<boolean>;

/** Cerimônia completa de consumo do convite. */
export function consumeInvite(
  storage: StoragePort,
  invite: SignedInvite,
  inviteePeerId: PeerId,
  inviteePubKey: Ed25519PublicKey,
): Promise<InviteCeremonyResult>;
```
- `validateInvite`: verifica assinatura Ed25519, expiry não ultrapassado, e que o invite_code não consta na tabela `tombstones` como consumido.
- `consumeInvite`: executa validação → Noise (conexão implícita) → cria `PROFILE:AUTHENTICATION` + aresta `VOUCHES_FOR` → lápide do convite (single-use).
- Vetor adversarial 9: convite reutilizado/expirado → cerimônia de consumo falha.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest + SQLite `:memory:`.
- [x] **Ambiente:** `pnpm --filter @plataforma/core test`.

Casos de teste:
1. `issueInvite` + `validateInvite` roundtrip: convite válido → true.
2. `validateInvite` com assinatura adulterada → false.
3. `validateInvite` com expiry ultrapassado → false.
4. `consumeInvite` fluxo feliz: cria `PROFILE:AUTHENTICATION` + `VOUCHES_FOR` + lápide.
5. `consumeInvite` segunda vez com mesmo invite → falha (single-use, vetor 9).
6. `consumeInvite` com invite expirado → falha (vetor 9).
7. `issueInvite` com expiry customizado → `expiry` reflete corretamente.
