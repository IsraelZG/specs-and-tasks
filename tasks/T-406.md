## 1. Objetivo
Implementar descoberta morna no pacote `@plataforma/transport`: `device_state.db` guarda histórico de peers (multiaddrs, últimas latências); no resume, tenta peers conhecidos em paralelo ao fallback WS; presença efêmera só em cache local, nunca no grafo
*(derivado de `docs/plano-de-implementacao.md` §7 T-406)*.

### Contratos exatos
```ts
// --- packages/transport/src/discovery/graphRouting.ts ---
import { PeerId, StoragePort } from '@plataforma/protocol';

export interface PeerHistoryEntry {
  peerId: PeerId;
  multiaddrs: string[];        // multiaddr string representation (T-105)
  lastLatencyMs: number;       // última latência medida
  lastSeenAt: number;          // timestamp do último contato
  connectionCount: number;     // quantas vezes conectou
  lastTransport: 'ws' | 'webrtc' | 'relay';
}

export interface GraphRoutingConfig {
  maxHistoryEntries?: number;  // default 200
  pruneAfterMs?: number;       // default 7 * 24 * 3600 * 1000 (7 dias)
  parallelProbes?: number;     // default 3 peers em paralelo
  probeTimeoutMs?: number;     // default 5000 (5s por probe)
}

export interface DiscoveryResult {
  connectedPeers: PeerId[];
  failedPeers: PeerId[];
  transportUsed: Map<PeerId, 'ws' | 'webrtc' | 'relay'>;
}

export class GraphRoutingDiscovery {
  constructor(storage: StoragePort, config?: GraphRoutingConfig);

  /** Registra peer encontrado/conectado no histórico. */
  recordPeer(entry: PeerHistoryEntry): Promise<void>;

  /** Tenta reconectar a peers conhecidos. */
  discover(): Promise<DiscoveryResult>;

  /** Ordena peers por prioridade (latência × recência). */
  rankPeers(): Promise<PeerHistoryEntry[]>;

  /** Remove entradas antigas além do pruneAfterMs. */
  prune(): Promise<number>;

  /** Retorna histórico de peers. */
  getHistory(): Promise<PeerHistoryEntry[]>;
}
```
- `recordPeer`: armazena no `device_state.db` (T-707) — cache local, nunca no grafo.
- `discover`: tenta `parallelProbes` peers em paralelo; usa multiaddrs históricos; se nenhum conhecido disponível, fallback WS.
- `rankPeers`: ordena por score = `1/latencyMs * log(connectionCount + 1) * recency`.
- Presença efêmera: histórico é volátil, não-replicado, local-only.
