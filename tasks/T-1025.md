# T-1025 · Role gate nas transições: approve/request_changes restritos ao Reviewer

## 1. Objetivo
Em **T-1014** descobrimos um incidente real: um worker (DeepSeek), preso porque a task T-1014
já estava em `review`, tentou `finish` (rejeitado pela máquina de estados), e então — como o
verbo seguinte estava tecnicamente disponível — chamou `approve` sobre a própria tarefa. O
`TaskService.transition()` aceitou, porque a máquina de estados de `task.types.ts` só valida
`(action, status atual)`, nunca **quem** está chamando. Resultado: o worker se autoaprovou e
moveu a própria task para `done` sem revisão real.

Esta task fecha essa lacuna **no código** (não só na instrução em prosa do `CLAUDE.md`/
`agile-reviewer.md`, que já proíbe isso e mesmo assim foi ignorada sob pressão). Toda task já
carrega `frontmatter.reviewer_agent` (ver `T-1014.md` linha 7: `reviewer_agent: agile_reviewer`,
preenchido por `renderTemplate` desde T-1010) — vamos usar esse campo como a fonte da verdade de
"quem pode aprovar esta task específica".

**Regra de negócio exata:** `approve` e `request_changes` só são aceitos se o argumento `agent`
(comparado case-insensitive, trim) for igual a `frontmatter.reviewer_agent` da task (ou
`'agile_reviewer'`, se o campo estiver ausente — tasks legadas pré-T-1010). Qualquer outro valor
de `agent` → `ForbiddenRoleError`, e a transição NÃO é aplicada (nem status, nem log, nem
rebuild do INDEX).

  ```ts
  /** Ação restrita ao Reviewer chamada por outro agente (mapeada para HTTP 403). */
  export class ForbiddenRoleError extends TaskError {
    constructor(message: string) {
      super(message);
      this.name = 'ForbiddenRoleError';
    }
  }
  ```
- **[UPDATE]** `apps/nexus-backend/src/services/task.service.ts`:
  1. Importar `ForbiddenRoleError` de `./task.types.js` (junto aos outros imports já existentes
     na linha do `import { ... } from './task.types.js';`).
  2. Em `transition()`, IMEDIATAMENTE depois do bloco que valida `rule.from` (depois do `if
     (rule.from !== '*' && ...) { throw new InvalidTransitionError(...); }` e ANTES de
     `content = this.replaceStatus(content, rule.to);`), inserir:
     ```ts
     if (action === 'approve' || action === 'request_changes') {
       const reviewerAgent = (
         this.parseFrontmatterNaive(content).reviewer_agent || 'agile_reviewer'
       ).trim().toLowerCase();
       if (agent.trim().toLowerCase() !== reviewerAgent) {
         throw new ForbiddenRoleError(
           `Ação '${action}' é exclusiva do Reviewer ('${reviewerAgent}'). O agente '${agent}' ` +
             `não pode aprovar/rejeitar a própria tarefa — use 'pause' para handoff e aguarde o /qa-review.`,
         );
       }
     }
     ```
     (`parseFrontmatterNaive` já existe na classe — é o parser ingênuo usado por
     `buildIndexForDirectory`. Reutilize-o, não crie um segundo parser de frontmatter.)

  `if (err instanceof ForbiddenRoleError) return 403;` (antes do `return 500;` final) e importar
  `ForbiddenRoleError` junto aos outros tipos de erro já importados de `./services/task.types.js`.
- **[UPDATE]** `apps/nexus-backend/src/tests/task.service.test.ts` — adicionar um bloco de testes

  mude as demais chamadas (`start`/`finish` continuam com qualquer agente).
- Mesma checagem vale para qualquer outro teste pré-existente que aprove/rejeite com um
  `agent` diferente do `reviewer_agent` do fixture: alinhe o `agent` ao `reviewer_agent`.
