# P-01 · Gate canonico (pnpm gate) + artefato de evidencia por tree-sha

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+ · **Package Manager:** `pnpm`
- **DOIS REPOS:** o **script `gate`** vive no **superapp** (`C:\Dev2026\superapp`); a **validação no `finish`** vive no **controle** (`C:\Dev2026\Docs\tools\scripts\manage-task.mjs`). Task de **tooling** (ver `/executar-task` Caminho A-tooling): sem worktree nova; o superapp é editado direto e o Docs enfileira.
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Colapsar as 4–5 chamadas de gate (build de deps + build + test + lint) numa **única** — `pnpm gate <pkg>` — que constrói o grafo via turbo, roda test+lint do alvo, e **grava a evidência num artefato commitável amarrado ao `tree-sha`** do HEAD. Isso mata: (a) o P-012 (turbo já resolve deps — ver RAG); (b) ~100 s/task de overhead de harness (§2a da análise); (c) a classe "finish sem commit real" e o slip de transcrição de evidência (worker cola no bloco errado).

## 4. Estratégia de Testes Estrita
- [x] **Framework:** verificação funcional (script de controle, sem build tsc próprio). Node puro.
- [x] **Casos:** (1) `pnpm gate @plataforma/core` numa worktree fria (sem `dist/` de deps) termina verde sem pré-build manual — prova que o turbo resolve o grafo; (2) o artefato `.gate/<tree>.json` é criado com `treeSha == git rev-parse HEAD^{tree}`; (3) `manage-task.mjs finish` **falha** se o artefato ausente ou com `treeSha` ≠ HEAD atual; **passa** se casar.
- [x] **Fora de Escopo:** paralelizar fases; cache remoto do turbo.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** reimplementar ordenação topológica — delegue ao `turbo run build --filter=<pkg>`.
> - **NÃO** gitignorar `.gate/` — o artefato é a evidência versionada.
> - **NÃO** tornar o `finish` dependente de rede ou do turbo — ele só **lê e compara** o artefato local.

Contrato do `gate.mjs`:
- Input: `<pkg>` (ex.: `@plataforma/core`). Fases: `turbo run build --filter=<pkg>` → `pnpm --filter <pkg> test` → `pnpm --filter <pkg> lint`. Para na 1ª falha real, identificando a fase.
- Output em stdout: resumo pronto para colar na §8 (fase · exitCode · wallMs · placar de teste).
- Artefato `C:\Dev2026\superapp\.gate\<treeSha>.json`: `{ pkg, treeSha, headSha, phases:[{name,cmd,startedAt,wallMs,exitCode,tail}], allGreen, generatedAt }`. `treeSha = git rev-parse HEAD^{tree}`.
- Emite também um evento por fase no formato JSONL de telemetria (P-02 definirá o `lib`; por ora, escrever em `tasks/.telemetry/<ID>.jsonl` se a env `MGTIA_TASK` estiver setada — degrada silenciosamente se ausente).

No `manage-task.mjs finish`: se existe `.gate/<HEAD^{tree}>.json` com `allGreen:true` → ok; senão, **abortar** com mensagem clara ("gate ausente ou stale para o tree atual — rode `pnpm gate <pkg>` e commite"). Só valida quando a task tem worktree de código (tooling-do-controle isenta, como hoje).
