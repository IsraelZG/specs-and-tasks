# T-104b · BIP39: Derivação seed → chave mestra Ed25519 + chave dispositivo

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** haiku

## 1. Objetivo
Implementar `mnemonicToSeed` (PBKDF2), `seedToMasterKey` (deriva chave Ed25519), e `deriveDeviceKey` (PBKDF2 da senha do usuário) no pacote `@plataforma/crypto`
*(extraído de T-104 §1; depende de T-104a para mnemônico e T-101a para Ed25519)*.

### Contratos exatos
```ts
// --- packages/crypto/src/derivation.ts ---
import { Ed25519PrivateKey, Ed25519PublicKey } from './ed25519.js'; // T-101a

/** Deriva seed (64 bytes) do mnemônico + passphrase via PBKDF2 (2048 iterações, HMAC-SHA512). */
export function mnemonicToSeed(mnemonic: string, passphrase?: string): Promise<Uint8Array>;

/** Deriva chave mestra Ed25519 a partir do seed (primeiros 32 bytes). */
export function seedToMasterKey(seed: Uint8Array): Promise<{ publicKey: Ed25519PublicKey; privateKey: Ed25519PrivateKey }>;

/** Deriva chave do dispositivo via PBKDF2(senha, salt, 100k iterações, 32 bytes). */
export function deriveDeviceKey(password: string, salt: Uint8Array): Promise<Uint8Array>;
```

Casos de teste:
1. `mnemonicToSeed` com vetor TREZOR conhecido → seed correto.
2. Mesmo mnemônico + passphrase → mesmo seed (determinismo).
3. Mnemônicos diferentes → seeds diferentes.

5. `deriveDeviceKey` com mesma senha+salt → mesma chave.
6. `deriveDeviceKey` com senhas diferentes → chaves diferentes.
7. `deriveDeviceKey` com 100k iterações → < 2s.
