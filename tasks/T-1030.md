```js
function nextAction(task) {
  const { status, depsOk } = task;
  if (status === 'in_progress')            return 'busy';
  if (status === 'in_review')              return 'busy';   // ← claimed: orquestrador NÃO despacha
  if (status === 'review')                 return 'review';
  if (status === 'rework')                 return 'rework';
  if (status === 'ready')                  return 'work';
  if (status === 'draft:pending_decision') return 'decide';
  if (status === 'draft:decomposed')       return null;     // container — nunca despacha
  if (status === 'draft:hardened')         return depsOk ? 'promote' : null;
  if (status === 'draft:triaged')          return depsOk ? 'harden' : null;
  if (status === 'draft:placeholder' || status === 'draft') return 'triage';
  return null;
}

```

```
  para cada arquivo em tasks/ e meta-tasks/ (exceto _*, INDEX.md, LEDGER.md):

                { ts, id, from: statusAntigo, to: novo, action: 'migrate', agent: 'system' }

    regenera INDEX ao final (via TaskService.rebuildIndexes ou rebuild-index.mjs)
  --apply, passo final: para cada task agora em `draft:hardened`, se deps todas done,
       chama TaskService.transition(id, 'promote', 'system', 'migração: deps done') → ready
```

### Pegadinhas conhecidas
- **Gate anti-drift:** o serviço compara o topo do `.nexus/transitions.jsonl` daquele id com o `status:` do arquivo. Por isso a migração PRECISA semear a entrada `migrate` no ledger — senão a primeira transição pós-migração de cada task quebra com `StatusDriftError`.
- **CRLF:** os arquivos podem ter `\r\n` (Windows). Normalize na leitura (`.replace(/\r\n/g,'\n')`) mas preserve o EOL ao reescrever, ou o diff fica gigante.
- **`ledger.mjs` coletor:** remova a leitura de `spec_status` (§196) e o campo `specStatus` do objeto — mas confira se algo mais no arquivo usa (o markdown-mode e o `--json`).
- **203 arquivos:** o script é I/O pesado — rode local, não via tool que trava (ver PITFALLS/ memória do usuário sobre comandos longos). Se demorar, o worker roda no terminal e cola a saída.

1. Escreva `migrate-status-axis.mjs` com `--dry-run`/`--apply` e o mapa.
2. Reescreva `nextAction()` e o coletor em `ledger.mjs`.
3. Adapte `hardening.mjs` e `generate-task.mjs`.
4. `--dry-run` → confira 131/13/43/11/5 → `--apply` → `--dry-run` de novo (idempotência).
5. Rode `ledger.mjs --json --idle` e `hardening.mjs` para confirmar consumidores ok. Cole tudo na Seção 8.
