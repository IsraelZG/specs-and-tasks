## 1. Objetivo
Implementar `ULID` (Universally Unique Lexicographically Sortable Identifier) e o tipo `EntityId` no pacote `@plataforma/core`, consumindo `ClockPort` e `RandomPort` para determinismo nos testes
*(derivado de `docs/conceitos/entity-id.md` — entity_id é o ULID do nó-raiz da linhagem; `docs/plano-de-implementacao.md` §4 T-102)*.

### Contratos exatos (assinaturas TS fixadas)

```ts
// --- packages/core/src/ulid.ts ---
import { ClockPort, RandomPort } from '@plataforma/protocol'; // T-004

/** ULID como string Base32 Crockford (26 caracteres). Especificação: github.com/ulid/spec. */
export type ULID = string;

/** EntityId = ULID do nó-raiz da linhagem. Sempre igual ao id do primeiro nó da entidade.
 *  Derivado de docs/conceitos/entity-id.md. */
export type EntityId = ULID;

/** Conjunto de portas injetáveis para ULIDFactory. */
export interface ULIDFactoryOptions {
  clock: ClockPort;
  random: RandomPort;
}

/** Fábrica de ULIDs determinística (via portas injetadas). */
export class ULIDFactory {
  constructor(opts: ULIDFactoryOptions);

  /** Gera novo ULID. Timestamp de clock.now() (48 bits) + random.bytes(10) (80 bits). */
  generate(): ULID;

  /** Decodifica ULID string → { timestamp: number, random: Uint8Array }. */
  static decode(ulid: ULID): { timestamp: number; random: Uint8Array };

  /** Valida se string é ULID sintaticamente válido (26 chars Crockford). */
  static isValid(candidate: string): candidate is ULID;
}

/** Codificador/decodificador Base32 Crockford (alfabeto: 0123456789ABCDEFGHJKMNPQRSTVWXYZ). */
export const CrockfordBase32: {
  encode(bytes: Uint8Array): string;
  decode(str: string): Uint8Array;
};
```

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente do Teste:** `pnpm --filter @plataforma/core test`.
- [x] **Fora de Escopo:** Integração com SQLite.

Casos de teste (numerados):
1. `ULIDFactory.generate()` com `VirtualClock(0)` + `SeededRandom('a')` → ULID começa com `0000000000` (timestamp 0).
2. Dois ULIDs gerados em sequência com mesmo clock e random → lexicograficamente crescentes (timestamp igual, random diferente).
3. `ULID.decode(ulid)` retorna timestamp e random corretos (roundtrip).
4. `ULID.isValid('01ARZ3NDEKTSV4RRFFQ69G5FAV')` → true.
5. `ULID.isValid('invalid')` → false (tamanho errado).
6. `ULID.isValid('01ARZ3NDEKTSV4RRFFQ69G5FAU')` → false (caractere 'U' não pertence ao alfabeto Crockford).
7. `CrockfordBase32.encode` + `decode` roundtrip com 16 bytes aleatórios.
8. ULIDs gerados com seeds diferentes → diferentes.
9. ULIDs gerados com mesma seed e clock fixo → idênticos (determinismo).
10. Tipo `EntityId` é assignable de `ULID` e vice-versa (type-check).
