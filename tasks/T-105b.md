# T-105b · Multiaddr: parser + formatador minimalista

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** haiku

## 1. Objetivo
Implementar parser/formatador minimalista de Multiaddr no pacote `@plataforma/protocol`, suportando os protocolos `ip4`, `dns4`, `tcp`, `ws`, `wss`, `p2p`. Sem dependência de `@multiformats/multiaddr`
*(extraído de T-105 §1)*.

### Contratos exatos
```ts
// --- packages/protocol/src/multiaddr.ts ---
export interface MultiaddrComponent {
  protocol: string; // 'ip4' | 'dns4' | 'tcp' | 'ws' | 'wss' | 'p2p'
  value: string;
}

export type MultiaddrString = string;

/** Parse: "/ip4/127.0.0.1/tcp/3000/ws/p2p/<peerIdB58>" → componentes. */
export function parseMultiaddr(addr: MultiaddrString): MultiaddrComponent[];

/** Serializa componentes → string Multiaddr. */
export function formatMultiaddr(components: MultiaddrComponent[]): MultiaddrString;
```

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).

Casos de teste:
1. `parseMultiaddr("/ip4/127.0.0.1/tcp/3000/ws")` → 4 componentes corretos.
2. `parseMultiaddr` + `formatMultiaddr` roundtrip.
3. `parseMultiaddr("/dns4/example.com/wss/p2p/abc123")` → 3 componentes.
4. `parseMultiaddr` com protocolo desconhecido → erro.
5. `formatMultiaddr` com array vazio → string vazia.
