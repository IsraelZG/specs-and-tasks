# T-502 · `ASSET:PERMISSION`/`ASSET:ROLE` físicos

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** sonnet

## 1. Objetivo
Implementar `ASSET:PERMISSION` e `ASSET:ROLE` como nós físicos no grafo no pacote `@plataforma/core`: nós + arestas `AGGREGATES`/`REQUIRES` para `entity_id`; resolução de direitos somente pelo DAG físico (nunca payload de SPECIFICATION); invariante de traversal profundo (depth>1 ⇒ edge_filter obrigatório com pares `aresta→tipo_alvo`)
*(derivado de `docs/plano-de-implementacao.md` §8 T-502)*.

### Contratos exatos
```ts
// --- packages/core/src/auth/permissions.ts ---
import { EntityId, ULID } from '@plataforma/protocol'; // T-102
import { StoragePort } from '@plataforma/protocol'; // T-004

export interface PermissionNode {
  entityId: EntityId;
  type: 'ASSET:PERMISSION' | 'ASSET:ROLE';
  target: string;              // recurso alvo (entity_id ou wildcard)
  action: string;              // ação permitida (ex: "read", "write", "delegate")
  restrictions?: string[];     // restrições adicionais
}

export interface RoleAggregation {
  roleId: EntityId;
  aggregatedPermissions: EntityId[];
}

export interface EdgeFilter {
  edgeType: string;            // tipo de aresta (ex: "MUTATES")
  targetType: string;          // tipo de nó alvo (ex: "CONTENT")
}

export interface ResolutionResult {
  granted: boolean;
  matchedPermissions: EntityId[];
  traversalDepth: number;
}

/** Emite nó de permissão/role no grafo. */
export function issuePermission(
  storage: StoragePort,
  permission: PermissionNode,
  issuerPubKey: Uint8Array,
): Promise<EntityId>;

/** Agrega permissões em um role via aresta AGGREGATES. */
export function aggregateRole(
  storage: StoragePort,
  roleId: EntityId,
  permissionIds: EntityId[],
): Promise<void>;

/** Define requisito entre roles via aresta REQUIRES. */
export function requireRole(
  storage: StoragePort,
  parentRoleId: EntityId,
  requiredRoleId: EntityId,
): Promise<void>;

/** Resolve direitos percorrendo o DAG físico. */
export function resolvePermission(
  storage: StoragePort,
  entityId: EntityId,
  action: string,
  edgeFilter?: EdgeFilter,
): Promise<ResolutionResult>;

/** Valida invariante: depth>1 sem edge_filter → erro. */
export function validateDeepTraversal(
  storage: StoragePort,
  entityId: EntityId,
  maxDepth: number,
  edgeFilter?: EdgeFilter,
): Promise<boolean>;
```
- Resolução de direitos: somente pelo DAG físico (nodes + edges), nunca do payload de `SPECIFICATION`.
- `validateDeepTraversal`: se depth > 1 e não tem `edgeFilter`, rejeita com erro de invariante.
- `edge_filter` contém pares `{edgeType → targetType}` que delimitam a traversal.
