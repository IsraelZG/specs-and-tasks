## 0. Ambiente de Execução Obrigatório

## 1. Objetivo
Implementar wrappers isomórficos (Node/Browser) para Ed25519 — `generateKeyPair`, `sign`, `verify` — no pacote `@plataforma/crypto`, usando `@noble/curves/ed25519`
*(derivado de `docs/visao-arquitetural.md` §1 camada 2; escopo extraído de T-101 §1)*.

### Contratos exatos
```ts
// --- packages/crypto/src/ed25519.ts ---

export type Ed25519PrivateKey = Uint8Array; // 32 bytes
export type Ed25519PublicKey = Uint8Array;  // 32 bytes
export type Ed25519Signature = Uint8Array;  // 64 bytes

/** Gera par de chaves Ed25519. Backend: @noble/curves/ed25519. */
export function ed25519GenerateKeyPair(): Promise<{ publicKey: Ed25519PublicKey; privateKey: Ed25519PrivateKey }>;

/** Assina `message` com chave privada. */
export function ed25519Sign(privateKey: Ed25519PrivateKey, message: Uint8Array): Promise<Ed25519Signature>;

/** Verifica assinatura. */
export function ed25519Verify(publicKey: Ed25519PublicKey, message: Uint8Array, signature: Ed25519Signature): Promise<boolean>;
```

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/crypto test`.

Casos de teste (numerados):
1. `ed25519GenerateKeyPair` gera chaves de 32 bytes; `publicKey !== privateKey`.
2. `ed25519Sign` + `ed25519Verify` roundtrip com vetor RFC 8032 §7.1 (TEST 2).
3. `ed25519Verify` retorna `false` com assinatura corrompida (1 bit trocado).
4. `ed25519Verify` retorna `false` com chave pública errada.
5. `ed25519Sign` com mensagem vazia → assinatura válida (64 bytes).
