# T-801a · Storage BLOBs: Chunker (fragmentação + reassembly)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)
- **Capacidade-alvo:** haiku

## 1. Objetivo
Implementar `Chunker` no pacote `@plataforma/media`: fragmenta arquivos em chunks de 256 KB com SHA-256 por chunk, e reassembly determinístico
*(extraído de T-801 §1)*.

### Contratos exatos
```ts
// --- packages/media/src/blob/Chunker.ts ---

export interface Chunk {
  index: number;
  offset: number;
  size: number;
  hash: Uint8Array; // SHA-256
  data: Uint8Array;
}

/** Fragmenta arquivo em chunks de chunkSize (default 256KB). */
export function chunkFile(data: Uint8Array, chunkSize?: number): Chunk[];

/** Reconstitui arquivo original dos chunks (valida ordem por index). */
export function assembleFile(chunks: Chunk[]): Uint8Array;
```

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).

Casos de teste:
1. `chunkFile` com 1 MB → 4 chunks de 256 KB; último chunk de 256 KB.
2. `chunkFile` com 100 bytes → 1 chunk.
3. `assembleFile` reconstitui arquivo idêntico (SHA-256 confere).
4. Chunk com `data` adulterado → reassembly produz hash diferente.
5. Chunks fora de ordem → `assembleFile` reordena por `index` e reconstitui correto.

## 5. Instruções de Execução
1. Escreva testes com buffers de tamanhos variados.
2. Implemente `chunkFile` (slice + SHA-256 por chunk).
3. Implemente `assembleFile` (sort por index + concat).
4. Re-exporte. Rode build + test.
