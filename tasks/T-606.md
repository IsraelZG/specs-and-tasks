## 1. Objetivo
Implementar congelamento escopado por linhagem no pacote `@plataforma/core`: ausência do validador/quórum da linhagem congela apenas aquele ativo como read-only; resto da rede (leitura, gossip, RBSR, comutativos) segue normalmente; UI sinaliza o freeze
*(derivado de `docs/plano-de-implementacao.md` §9 T-606)*.

### Contratos exatos
```ts
// --- packages/core/src/intent/freezeScope.ts ---
import { EntityId } from '@plataforma/protocol'; // T-102
import { StoragePort } from '@plataforma/protocol'; // T-004

export enum FreezeState {
  ACTIVE = 'active',
  FROZEN_PENDING = 'pending',    // aguardando quórum/validador
  FROZEN = 'frozen',             // read-only
  THAWED = 'thawed',             // validador retornou
}

export interface FreezeScope {
  entityId: EntityId;            // linhagem congelada
  state: FreezeState;
  frozenAt: number;
  reason: string;                // ex: "validator_unavailable", "quorum_lost"
  validatorEntityId: EntityId;   // validador faltante
  pendingIntents: EntityId[];    // intents na fila aguardando thaw
}

export interface FreezeConfig {
  validatorTimeoutMs?: number;   // default 30000 (30s sem resposta)
  quorumSize?: number;           // default 1 (K=1)
}

export class FreezeManager {
  constructor(storage: StoragePort, config?: FreezeConfig);

  /** Verifica disponibilidade do validador/quórum. */
  checkValidator(scopeEntityId: EntityId): Promise<boolean>;

  /** Congela linhagem se validador ausente. */
  freezeIfNeeded(scopeEntityId: EntityId, reason: string): Promise<FreezeScope | null>;

  /** Descongela quando validador retorna. */
  thaw(scopeEntityId: EntityId): Promise<FreezeScope>;

  /** Drena fila de intents pendentes após thaw. */
  drainPendingIntents(scopeEntityId: EntityId): Promise<number>;

  /** Verifica se operação é permitida no estado atual. */
  isOperationAllowed(scopeEntityId: EntityId, operation: 'read' | 'write' | 'debit' | 'credit'): boolean;
}
```
- `isOperationAllowed`: no estado `FROZEN`, apenas leitura e créditos são permitidos; débitos bloqueados.
- Congelamento escopado: apenas a linhagem com validador ausente é afetada.
- Resto da rede (RBSR, gossip, leitura, comutativos) segue normalmente.
- UI deve sinalizar claramente quais linhagens estão congeladas.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest + SQLite `:memory:`.
- [x] **Ambiente:** `pnpm --filter @plataforma/core test`.

Casos de teste:
1. `checkValidator` com validador presente → true.
2. `checkValidator` com validador ausente (timeout) → false.
3. `freezeIfNeeded` com validador ausente → `FreezeState.FROZEN`.
4. `freezeIfNeeded` com validador presente → null (não congela).
5. `isOperationAllowed` no estado `FROZEN`: read→true, credit→true, debit→false.
6. `isOperationAllowed` no estado `ACTIVE`: todas operações permitidas.
7. `thaw` descongela → `drainPendingIntents` processa fila.
8. Cenário: derrubar validador no SimNetwork → débito pende, créditos e RBSR fluem.
