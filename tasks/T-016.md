## 1. Objetivo

### Contratos exatos (assinaturas TS fixadas)

```ts
// --- packages/design-system/src/lib/themeEngine.ts (novas exports) 
---

/** Flat override map: "theme.intent.primary.fill" → "#7c3aed".
 *  Derivado de caderno-3-sdk/09 §5 "Formato de Declaração". */
export type ThemeOverrideMap = Record<string, string>;

/** Nível de escopo para compileScopedOverrides.
 *  Derivado de caderno-3-sdk/09 §1: data-ds-module | data-ds-page. */
export type ScopedLevel = 'module' | 'page';

/** Nome do atributo DOM por nível.
 *  Derivado de caderno-3-sdk/09 §1 invariante DOM. */
export const SCOPE_SELECTORS: Record<ScopedLevel, string>;

/**
 * Compila overrides em CSS escopado para injeção via <style>.
 * - Chaves prefixadas "theme." → --ds-theme-*
 * - Demais chaves → --ds-component-*
 * - Referências {theme.xxx} → var(--ds-theme-xxx)
 * - Valores literais → mantidos como literais
 * Derivado de caderno-3-sdk/09 §4 "Bloco global <style>" + §5 "Formato".
 */
export function compileScopedOverrides(
  overrides: ThemeOverrideMap,
  scope: ScopedLevel,
  scopeId: string
): string;

/**
 * Converte overrides para objeto de estilo inline React.
 * { "button.primary.bg": "#333" } → { "--ds-component-button-primary-bg": "#333" }
 * Derivado de caderno-3-sdk/09 §4 "Componente/Instância — inline".
 */
export function overridesToStyleObject(
  overrides: ThemeOverrideMap
): Record<string, string>;

// --- packages/design-system/src/lib/themeOverrideKeys.ts (CREATE) ---

/** Conjunto canônico de chaves de override válidas.
 *  Derivado de caderno-3-sdk/09 §5 "dicionário canônico". */
export const VALID_OVERRIDE_KEYS: ReadonlySet<string>;

/** Valida se uma chave pertence ao dicionário canônico.
 *  Derivado de caderno-3-sdk/09 §5 "chave não documentada gera erro". */
export function isValidOverrideKey(key: string): boolean;

// --- packages/design-system/src/lib/ThemeScope.tsx (CREATE, opcional) ---

/** Wrapper React que aplica atributo data-ds-module ou data-ds-page.
 *  Derivado de caderno-3-sdk/09 §1. */
export function ThemeScope(props: {
  level: ScopedLevel;
  scopeId: string;
  children: React.ReactNode;
}): JSX.Element;
```

### Regras de conversão de chave → variável CSS (derivado de caderno-3-sdk/09 §5)
- Prefixo `theme.` → `--ds-theme-` (ex: `theme.intent.primary.fill` → `--ds-theme-intent-primary-fill`)
- Sem prefixo → `--ds-component-` (ex: `card.radius` → `--ds-component-card-radius`)
- Referências `{theme.xxx}` → `var(--ds-theme-xxx)`; `{component.xxx}` → `var(--ds-component-xxx)`
- Separação de tokens por `.` → kebab-case via `toKebabCase` existente

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [x] **Framework:** `vitest` (Node puro, sem JSDOM para os testes unitários de compilação de CSS)
- [x] **Ambiente do Teste:** `pnpm --filter @plataforma/design-system test`

Casos de teste (numerados) — todos derivados de caderno-3-sdk/09:

1. **Scoped CSS module:** `compileScopedOverrides({"theme.intent.primary.fill": "#333"}, "module", "chat")` → CSS contém `[data-ds-module="chat"]` e `--ds-theme-intent-primary-fill: #333`.
2. **Scoped CSS page:** `compileScopedOverrides({"card.radius": "8px"}, "page", "settings")` → CSS contém `[data-ds-page="settings"]` e `--ds-component-card-radius: 8px`.
3. **Reference resolution:** `compileScopedOverrides({"button.primary.bg": "{theme.intent.primary.fill}"}, "module", "x")` → CSS contém `var(--ds-theme-intent-primary-fill)` não valor estático.
4. **Inline style object:** `overridesToStyleObject({"button.primary.bg": "#333"})` → `{"--ds-component-button-primary-bg": "#333"}`.
5. **Kebab conversion:** `overridesToStyleObject({"theme.intent.primary.fillHover": "red"})` → chave contém `--ds-theme-intent-primary-fill-hover`.
6. **Valid key:** `isValidOverrideKey("theme.intent.primary.fill")` → `true`.
7. **Invalid key:** `isValidOverrideKey("garbage.x")` → `false`.
8. **Empty overrides:** `compileScopedOverrides({}, "module", "x")` → string CSS válida (bloco vazio).
9. **Multiple overrides:** `compileScopedOverrides` com 3+ entradas → todas aparecem no bloco CSS.
10. **No XSS:** valor contendo `</style><script>` → escapa/valida adequadamente.
