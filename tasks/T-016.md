# T-016 · Implementar RFC de Tema Multi-Nível no Design System

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest`

## 1. Objetivo
Implementar a [RFC de Temas Multi-Nível](../docs/caderno-3-sdk/09-hierarchical-theme-customization.md) no código fonte da biblioteca `@plataforma/design-system`. Isso exige alterar a compilação de tokens (`outputReferences: true`) e estender o motor de temas (`themeEngine.ts`) para suportar a resolução achatada em runtime, gerando CSS escopado e inlines baseados em hierarquia DOM.

- [ ] [T-011](./T-011.md) e [T-013](./T-013.md) — Configurações base. **Pré-requisitos.**

## 3. Escopo de Arquivos (Inputs e Outputs)
- **[UPDATE]** `packages/design-system/style-dictionary.config.js` — Habilitar `outputReferences: true` nos builds correspondentes.
- **[UPDATE]** `packages/design-system/src/lib/themeEngine.ts` — Adaptar lógicas de compilação de CSS escopado (blocos `<style>`) e objetos de estilo inline, além de suportar a resolução de `var()` na emissão em vez de valores estáticos.
- **[CREATE]** `packages/design-system/src/lib/themeOverrideKeys.ts` — Utilitário ou tipagem derivado dos tokens base validando chaves de override (se necessário).
- **[CREATE]** `packages/design-system/src/lib/ThemeScope.tsx` — (Opcional, conforme a RFC) Helper React para criar os wrappers `[data-ds-module]`, etc.

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [ ] **Framework:** `vitest` unitário para o `themeEngine.ts`.
- [ ] **Métricas/Cobertura:**
  - Garantir que um objeto de override em formato achatado vira CSS válido com as custom properties com prefixo `--ds-`.
  - Garantir que a compilação do style-dictionary não quebra e que referências entre temas usem `var()`.
- [ ] **Ambiente do Teste:** Node/JSDOM via Vitest no pacote.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - **NÃO** misture a lógica visual dos componentes existentes. O objetivo é a engine de tema subjacente.
> - **NÃO** instancie React Components complexos de teste. Foque nos utilitários expostos.

1. Configurar `outputReferences: true` no style dictionary e recompilar os tokens. Verificar o CSS de saída.
2. Atualizar o `themeEngine.ts` para prover métodos de `compileScopedOverrides` (gera bloco de `<style>`) e `overridesToStyleObject` (gera inlines).
3. Escrever e rodar `vitest` unitário para as conversões.
4. (Opcional) Testar manualmente no `apps/design-system-showcase` (já migrado via T-012) se os escopos funcionam.
