## 1. Objetivo
Implementar predicado de bloqueio na liberação de chaves no pacote `@plataforma/core`: Key Vault consulta arestas `BLOCKS` do autor antes de entregar chave; latência de bloqueio = TTL da chave em RAM; bloqueio social (filtro de leitura) para escopo público
*(derivado de `docs/plano-de-implementacao.md` §8 T-506)*.

### Contratos exatos
```ts
// --- packages/core/src/auth/blockPredicate.ts ---
import { EntityId } from '@plataforma/protocol'; // T-102
import { StoragePort } from '@plataforma/protocol'; // T-004
import { Ed25519PublicKey } from '@plataforma/crypto'; // T-101a

export interface BlockPredicate {
  authorEntityId: EntityId;
  targetPeerId: EntityId;
  blockReason: string;
  blockedAt: number;
  ttlMs: number;                 // latência de bloqueio = TTL em RAM
}

export interface KeyReleaseRequest {
  authorPubKey: Ed25519PublicKey;
  requesterEntityId: EntityId;
  scopeEntityId: EntityId;
  epochIndex: number;
}

export interface KeyReleaseResult {
  allowed: boolean;
  reason?: string;
  blockExpiresAt?: number;       // quando o bloqueio expira
}

/** Verifica se o autor bloqueou o requisitante. */
export function checkBlockPredicate(
  storage: StoragePort,
  request: KeyReleaseRequest,
): Promise<KeyReleaseResult>;

/** Cria aresta BLOCKS do autor para o peer bloqueado. */
export function setBlock(
  storage: StoragePort,
  authorEntityId: EntityId,
  targetPeerId: EntityId,
  reason: string,
  authorPubKey: Ed25519PublicKey,
): Promise<EntityId>;

/** Remove aresta BLOCKS. */
export function removeBlock(
  storage: StoragePort,
  authorEntityId: EntityId,
  targetPeerId: EntityId,
): Promise<void>;

/** Bloqueio social: filtra leitura em escopo público. */
export function socialBlockFilter(
  storage: StoragePort,
  readerEntityId: EntityId,
  scopeEntityId: EntityId,
): Promise<{ visible: EntityId[]; blocked: EntityId[] }>;
```
- `checkBlockPredicate`: consulta arestas `BLOCKS` do autor; se requester está bloqueado, nega liberação de chave.
- Latência de bloqueio = TTL da chave em RAM: chave em cache não é removida, mas novas requisições são negadas.
- Rotação de época **não** é disparada por bloqueio (separação de concerns).
- `socialBlockFilter`: para escopo público, filtra entidades cujo autor bloqueou o leitor.
