# T-602 · Ciclo de commit Automerge

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)

## 1. Objetivo
Implementar ciclo de commit Automerge: `pending_changes` local; ephemeral Changes entre co-editores (canal T-403); gatilhos de commit por inatividade/limiar; consolidação `Automerge.save()` → nó-versão assinado + `MUTATES` + `AUTHORED` com sumário; limpeza de staging.
*(Normativo: `docs/plano-de-implementacao.md` §9 T-602, linha 295. **Pré-requisito documental: ADR-001** `docs/adr/adr-001-automerge-unico.md` — existe. Aceite: dois editores concorrentes convergem; tabela `nodes` recebe só commits, nunca micro-changes.)*

> **⚠️ CONFRONTAÇÃO (endurecimento 2026-07-08, claude-opus).** Três divergências reais contra o
> código/arquitetura atuais (ver Seção 6, D1–D4):
> 1. **Violação de camada:** o contrato coloca `commitCycle.ts` em `@plataforma/core` mas importa
>    `AutomergeDocId` de `@plataforma/transport`. `visao-arquitetural.md §1` fixa a ordem
>    `protocol ← crypto ← core ← transport` — **core NÃO depende de transport** (confirmado:
>    `core/package.json` não lista `@plataforma/transport`). Esse import **inverte a seta / fecha
>    ciclo** (mesma classe do bug de T-1033).
> 2. **Automerge não integrado:** `Automerge.save()` está na spec, mas **nenhum `package.json` tem
>    dependência `automerge`/`@automerge/automerge`** (grep vazio) e **T-403 está `ready` mas NÃO
>    implementado** — `packages/transport/src/automerge/` não existe. A "casca" que forneceria o
>    doc Automerge vivo (p/ chamar `.save()`) não resolve no estado atual.
> 3. **Sem helper de aresta AUTHORED nem fonte de chave de assinatura** — ver D3/D4.

### Contratos exatos (parcialmente contingentes — ver D1–D4)
```ts
// --- packages/core/src/sync/commitCycle.ts  (⚠️ localização do pacote PENDE D1) ---
import type { EntityId, ULID, StoragePort } from '@plataforma/protocol'; // T-004/T-102 (derivado — protocol é camada base)
// ⚠️ AutomergeDocId de @plataforma/transport VIOLA a direção de camadas (D1) — NÃO importar em core
import type { Ed25519PrivateKey } from '@plataforma/crypto'; // T-101a (derivado)

export interface PendingChange {
  changeId: string;
  docId: string;
  authorId: EntityId;
  payload: Uint8Array;          // Automerge change bytes
  timestamp: number;
}

export interface CommitTrigger {
  inactivityMs: number;          // default 3000 (3s de inatividade)
  changeThreshold: number;       // default 50 changes pendentes
  maxPendingMs: number;          // default 30000 (30s máximo pendente)
}

export interface CommitResult {
  nodeId: ULID;                  // nó-versão assinado
  authoredEdgeId: string;        // aresta AUTHORED com sumário
  mutatesEdgeId: string;         // aresta MUTATES
  changesConsolidated: number;   // quantas changes foram consolidadas
}

export interface CommitCycle {
  /** Adiciona change ao buffer de staging. */
  stageChange(change: PendingChange): void;

  /** Recebe change efêmera de co-editor (canal T-403). */
  receiveRemoteChange(change: PendingChange): void;

  /** Verifica gatilhos de commit e executa se disparado. */
  checkTriggers(): Promise<CommitResult | null>;

  /** Força commit independente dos gatilhos. */
  forceCommit(): Promise<CommitResult>;

  /** Limpa buffer de staging após commit. */
  clearStaging(docId: string): void;

  /** Registra callback de commit configurável por SPEC. */
  configureTriggers(triggers: Partial<CommitTrigger>): void;
}
```
- `pending_changes`: buffer local de changes não-commitadas.
- Ephemeral Changes: recebidas via canal do T-403 (AutomergeShell) entre co-editores.
- Gatilhos: inatividade (3s sem mudança), limiar (50 changes), timeout máximo (30s).
- Commit: `Automerge.save()` → nó-versão assinado + aresta `MUTATES` + aresta `AUTHORED` com sumário.
- Tabela `nodes` recebe só commits, nunca micro-changes.
