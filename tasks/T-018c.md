## 1. Objetivo
Escrever testes unitários para **8 componentes de overlay/disclosure** do `@plataforma/design-system`, cobrindo os critérios do Tier B (smoke fechado, abrir com role, fechar via interação, ref).

### Critérios de aceite (Tier B — overlay/disclosure)
1. **Smoke fechado:** `queryByRole` retorna `null` quando fechado (padrão `Modal.test.tsx`).
2. **Abrir overlay:** via prop controlada `open`/`isOpen` ou interação, conforme a API. Conteúdo renderiza com o role esperado (`dialog`, `menu`, `tooltip`, etc.) e atributo `aria-*` correspondente.
3. **Fechar:** via interação documentada (clique fora, Escape, clique no trigger) dispara callback de fechamento.
4. **Ref forwarding:** se `React.forwardRef`, ref chega ao nó DOM esperado.

### Pegadinhas conhecidas
- **Portal:** todos os componentes deste tier usam portal (`createPortal`). Conteúdo do overlay renderiza **fora** do container de `render()`. Use `screen.getByRole` (busca `document.body` inteiro), não `container.querySelector`.
- **Animação de entrada:** `Modal.test.tsx` usa `flushEnterAnimation` para esperar listener de teclado pós-animação. `AlertDialog`, `Sheet`, `Popover` e `Command` podem precisar do mesmo padrão — verifique se há `useEffect` com `requestAnimationFrame` no código antes de copiar.
- **Tooltip/HoverCard:** o trigger é hover, não clique — use `userEvent.hover(target)` e `userEvent.unhover(target)`, não `userEvent.click`.
- **DropdownMenu/Menubar:** API composta (Trigger + Content + Item). Testar separadamente: trigger abre menu, item dispara callback.

1. Para cada componente: leia `<Nome>.tsx` + `<Nome>.metadata.ts`.
