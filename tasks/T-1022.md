# T-1022 · Runner: AgentAdapter (interface + CommandAdapter genérico)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Test Runner:** `vitest`
- **Capacidade-alvo:** sonnet *(spawn de subprocesso com timeout/captura; assinaturas fixas)*

## 1. Objetivo
Abstração que o runner usa para disparar um agente (worker/reviewer/architect) como
subprocesso headless, com `cwd` no worktree, env injetado e timeout. Entrega a **interface**
`AgentAdapter` + uma implementação **genérica** `CommandAdapter` que monta o comando a partir
de um **template configurável por papel** (placeholders), NÃO hardcoded. A recipe específica
do OpenCode/Claude é só config — sua validação empírica é da spike T-1017.

## 3. Escopo de Arquivos (Inputs e Outputs) — assinaturas FIXAS
- **[CREATE]** `apps/nexus-backend/src/runner/agent-adapter.ts`
  ```ts
  export type AgentRole = 'worker' | 'reviewer' | 'architect';

  export interface AgentRunOptions {
    role: AgentRole;
    taskId: string;
    cwd: string;                          // worktree da task
    env?: Record<string, string>;          // mesclado SOBRE process.env
    timeoutMs?: number;                    // default 1_800_000 (30 min)
  }
  export interface AgentRunResult {
    exit: number | null;   // exit code (null se morto por timeout)
    timedOut: boolean;
    tail: string;          // últimas 40 linhas de stdout+stderr
  }
  export interface AgentAdapter {
    run(opts: AgentRunOptions): Promise<AgentRunResult>;
  }

  export interface CommandAdapterConfig {
    /** Template de comando por papel, com placeholders {taskId} {role} {cwd}. */
    templates: Record<AgentRole, string>;
  }

  /**
   * Substitui os placeholders no template do papel e executa via
   * `child_process.spawn(cmd, { cwd, env: {...process.env, ...opts.env}, shell: true })`.
   * Captura stdout+stderr (guarda só as últimas 40 linhas). Mata o processo em timeout
   * (`timedOut:true`, `exit:null`). NUNCA rejeita por exit!=0 — devolve o exit no resultado.
   */
  export class CommandAdapter implements AgentAdapter {
    constructor(config: CommandAdapterConfig);
    run(opts: AgentRunOptions): Promise<AgentRunResult>;
  }

  /** Lê os templates de env: NEXUS_AGENT_CMD_WORKER / _REVIEWER / _ARCHITECT.
   *  Papel sem template definido → lança Error claro ao tentar `run` aquele papel. */
  export function commandAdapterFromEnv(): CommandAdapter;
  ```

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [ ] **Framework:** Vitest (Node puro). Templates de teste usam `node -e` (portável).
- [ ] **Cobertura:**
  - substituição de placeholders: template `node -e "console.log('{role}:{taskId}')"`
    → `tail` contém `worker:T-1`; `exit:0`, `timedOut:false`.
  - exit != 0: template `node -e "process.exit(3)"` → `exit:3`, sem lançar.
  - timeout: template que dorme (`node -e "setTimeout(()=>{},10000)"`) com `timeoutMs:200`
    → `timedOut:true`, `exit:null`.
  - `cwd`/`env`: template `node -e "console.log(process.cwd(), process.env.FOO)"` com
    `cwd=<tmpdir>` e `env:{FOO:'bar'}` → tail contém o tmpdir e `bar`.
  - `commandAdapterFromEnv`: papel sem env definido → `run` daquele papel lança Error claro.
- [ ] **Ambiente:** Node puro. Restaurar `process.env` alterado no `afterEach`.
- [ ] **Fora de Escopo:** invocar OpenCode/Claude de verdade (é a PoC da T-1017).

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - NÃO hardcodar comandos de OpenCode/Claude — só o `CommandAdapter` genérico + templates de config.
> - NÃO acoplar segredos no código — chaves de API entram via `opts.env`/`process.env`.
> - NÃO transicionar tasks aqui — o adapter só roda o processo e devolve o resultado.

### Pegadinhas conhecidas
- **`spawn` com `shell:true`** é necessário (template é uma linha de shell). Como o template é
  config confiável (não input do usuário), o uso de shell é aceitável — documente.
- **Timeout precisa MATAR o processo** (`child.kill()`); só setar uma flag não basta. Limpe o
  timer no `close`. Resultado de timeout: `timedOut:true`, `exit:null` (não lance).
- **Merge de env:** `{ ...process.env, ...opts.env }` — não substitua process.env inteiro.
- **Buffer de saída:** acumule e guarde só as últimas 40 linhas (não estoure memória em logs longos).
