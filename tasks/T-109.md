# T-109 · Projeção entity_heads + tombstones

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)

## 1. Objetivo
Implementar trigger SQLite `trg_nodes_insert_entity_head` (head por maior HLC) + sistema de lápides (`active=0`) + projeção `active_edges`. Deleção lógica via lápide: registros permanecem no grafo mas somem das consultas. Head correto sob mutações concorrentes
*(derivado de `docs/plano-de-implementacao.md` M1 — T-109)*.

### Contratos exatos
```ts
// --- packages/core/src/schema.ts (migration v2) ---

// Migration v2: entity_heads + trigger
export const MIGRATIONS: Migration[] = [
  { version: 1, ... },
  { version: 2, sql: `
    CREATE TABLE IF NOT EXISTS entity_heads (
      entity_id TEXT NOT NULL PRIMARY KEY,
      head_id   TEXT NOT NULL REFERENCES nodes(id),
      head_hlc  INTEGER NOT NULL
    );
    CREATE TRIGGER IF NOT EXISTS trg_nodes_insert_entity_head
    AFTER INSERT ON nodes
    BEGIN
      INSERT INTO entity_heads (entity_id, head_id, head_hlc)
      VALUES (NEW.entity_id, NEW.id, NEW.hlc)
      ON CONFLICT(entity_id) DO UPDATE
      SET head_id = NEW.id, head_hlc = NEW.hlc
      WHERE NEW.hlc > head_hlc;
    END;

    -- Tombstones
    ALTER TABLE nodes ADD COLUMN active INTEGER NOT NULL DEFAULT 1;
    ALTER TABLE edges ADD COLUMN active INTEGER NOT NULL DEFAULT 1;
    CREATE VIEW IF NOT EXISTS active_edges AS
      SELECT * FROM edges WHERE active = 1;
  `},
];
```

Casos de teste:
1. Insert 3 nós mesma entity → `entity_heads` aponta pro maior HLC.
2. Insert nó com HLC menor → head não atualiza.
3. `tombstoneNode` → `active=0`; SELECT na view `active_edges` exclui.
4. Tombstone preserva registro original (não DELETE).
5. Trigger não quebra com insert em batch (3 nós mesma entity, HLC variado).
