# T-108b · Linhagem Layer 2: navegação + fork detection

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`
- **Monorepo:** Turborepo
- **Test Runner:** `vitest` (Node puro)

## 1. Objetivo
Implementar funções de consulta e navegação da linhagem: `getLineage`, `getHead`, `validateChain`, `detectFork` no pacote `@plataforma/core`
*(extraído de T-108 §1)*.

### Contratos exatos
```ts
// --- packages/core/src/lineage.ts (extensão) ---
import { SignedNode } from './signature.js'; // T-107
import { StoragePort } from '@plataforma/protocol'; // T-004
import { ULID } from './ulid.js'; // T-102

export async function getLineage(storage: StoragePort, entityId: ULID): Promise<SignedNode[]>;
export async function getHead(storage: StoragePort, entityId: ULID): Promise<SignedNode | null>;
export async function validateChain(storage: StoragePort, entityId: ULID): Promise<boolean>;
export async function detectFork(storage: StoragePort, sourceId: ULID): Promise<boolean>;
```
- `getLineage`: do head à raiz (ordem reversa de HLC), seguindo arestas MUTATES.
- `getHead`: `SELECT * FROM entity_heads WHERE entity_id = ?`.
- `validateChain`: percorre linhagem verificando `hashNode` de cada pai.
- `detectFork`: conta arestas MUTATES com mesmo `source_id` ≥ 2.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest + SQLite `:memory:`.

Casos de teste:
1. `getHead` com linhagem de 3 nós → retorna nó com maior HLC.
2. `getLineage` → [head, ..., raiz] em ordem reversa.
3. `validateChain` com cadeia íntegra → true.
4. `validateChain` com nó adulterado → false.
5. `detectFork`: 1 aresta → false; 2 arestas → true.
6. Entity inexistente → `getHead` retorna null.
