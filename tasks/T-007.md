## 1. Objetivo
Implementar helper `expectConverged` no pacote `@plataforma/testkit`: verifica se múltiplos peers chegaram a estado idêntico (nós + arestas), com diff descritivo em caso de divergência
*(derivado de T-005 — SimNetwork para testes multi-peer; `docs/plano-de-implementacao.md` §2.1)*.

### Contratos exatos
```ts
// --- packages/testkit/src/assertions.ts ---
import { StoragePort, PeerId } from '@plataforma/protocol'; // T-004

export interface PeerState {
  peerId: PeerId;
  storage: StoragePort;
}

/** Verifica que todos os peers têm os mesmos nós e arestas. Lança AssertionError com diff se divergirem. */
export async function expectConverged(peers: PeerState[]): Promise<void>;

/** Dump de tabela como array de registros para diff. */
export async function dumpNodes(storage: StoragePort): Promise<Record<string, unknown>[]>;
export async function dumpEdges(storage: StoragePort): Promise<Record<string, unknown>[]>;
```
- `expectConverged`: compara `dumpNodes` + `dumpEdges` de todos os peers via `deepStrictEqual`.
- Em caso de divergência, imprime diff no console (ids ausentes em cada peer) e lança `AssertionError`.

## 4. Estratégia de Testes
- [x] **Framework:** Vitest (Node puro).
- [x] **Ambiente:** `pnpm --filter @plataforma/testkit test`.

Casos de teste:
1. 2 peers com storage idêntico → `expectConverged` não lança.
2. 2 peers com 1 nó divergente → lança AssertionError com diff.
3. 2 peers com 1 aresta divergente → lança AssertionError.
4. 3 peers, 2 idênticos e 1 divergente → lança indicando qual peer divergiu.
5. `dumpNodes` sobre storage vazio → array vazio.
