## 1. Objetivo
Escrever testes unitários para **7 componentes display/estruturais** do `@plataforma/design-system`, cobrindo os critérios do Tier C (smoke, a11y role/atributo, ref se aplicável).

5. `Separator`

### Critérios de aceite (Tier C — display/estrutural)
1. **Smoke:** renderiza sem erro com props mínimas obrigatórias.
2. **A11y:** role/atributo esperado presente (ex.: `Progress` com `role="progressbar"` e `aria-valuenow`, `Separator` com `role="separator"`, `NavItem` com `role="link"` ou `button` — verificar metadata de cada um).
3. **Ref forwarding:** somente se `React.forwardRef` — maioria deste tier é função simples sem ref.
4. **Não é necessário** teste de interação — são componentes sem estado interativo próprio.

### Pegadinhas conhecidas
- **Progress:** verificar `role="progressbar"`, `aria-valuenow`, `aria-valuemin`, `aria-valuemax`. Se houver variante `indeterminate`, testar ausência de `aria-valuenow`.
- **Separator:** verificar `role="separator"` e orientação (`aria-orientation="horizontal"` default).
- **Skeleton:** sem role específico — testar classes de animação e variantes (text, circle, rect).
- **Toast:** pode usar portal + timer. Verificar se o componente expõe uma API imperativa (tipo `toast()`) — se sim, testar chamada da função + aparecimento/desaparecimento.
- **ScrollArea:** verificar se children renderiza dentro do viewport; testar ref forwarding se houver.
- **NavItem:** pode ter estado `active` + variante `collapsed` (Sidebar). Verificar role (`link` ou `button`) conforme prop `as`.

1. Para cada componente: leia `<Nome>.tsx` + `<Nome>.metadata.ts`.
2. Escreva `tests/components/<Nome>.test.tsx` cobrindo os critérios do Tier C.
3. Rode `pnpm --filter @plataforma/design-system test` após cada 2-3 componentes.
4. Ao final dos 7, rode build + lint (Seção 7).
