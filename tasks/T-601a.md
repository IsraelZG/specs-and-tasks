# T-601a · Fork/Merge: detectForks (SQL)

## 0. Ambiente de Execução Obrigatório
- **Runtime:** Node.js v20+
- **Package Manager:** `pnpm`

## 1. Objetivo
Implementar `detectForks` no pacote `@plataforma/core`: query SQL que detecta forks estruturais (≥2 arestas MUTATES ativas com mesmo `source_id`) e retorna os nós divergentes
*(extraído de T-601 §1; depende de T-106a para schema, T-108 para linhagem)*.

### Contratos exatos
```ts
// --- packages/core/src/dag/detect.ts ---
import { StoragePort } from '@plataforma/protocol'; // T-004
import { SignedNode } from '../signature.js'; // T-107
import { ULID } from '../ulid.js'; // T-102

/** Detecta forks: >=2 arestas MUTATES ativas com mesmo source_id. */
export function detectForks(storage: StoragePort, sourceId: ULID): Promise<SignedNode[]>;

/** Lista todos os source_ids com fork ativo no grafo. */
export function listForkedSources(storage: StoragePort): Promise<ULID[]>;
```
Query: `SELECT target_id FROM edges WHERE edge_type='MUTATES' AND source_id=?` → se ≥2 resultados, é fork.

Casos de teste:
1. 1 aresta MUTATES → `detectForks` retorna array vazio.
2. 2 arestas MUTATES do mesmo source → retorna 2 SignedNodes.
3. 3 arestas → detecta todas.
4. `listForkedSources` retorna apenas sources com fork.
5. Grafo sem forks → `listForkedSources` retorna array vazio.
