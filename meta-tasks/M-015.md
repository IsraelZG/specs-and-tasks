# M-015 · Gerar arquivos de Task (T-XXX) para o monorepo Nexus

## 0. Ambiente de Execução Obrigatório
- **Capacidade-alvo:** haiku | sonnet *(Tarefa repetitiva de CLI).*

## 1. Objetivo
Ler o backlog recém-criado em `docs/backlog-modulos.md` e gerar os mais de 90 arquivos de task (`T-XXX.md`) na pasta `/tasks` do projeto, usando a CLI oficial `generate-task.mjs`.

## 2. Contexto RAG (Spec-Driven Development)
- `docs/backlog-modulos.md`
- `tools/scripts/generate-task.mjs`

## 3. Escopo de Arquivos (Inputs e Outputs)
- **[READ]** `docs/backlog-modulos.md`
- **[CREATE]** Dezenas de arquivos `tasks/T-XXX.md` via script CLI.

## 4. Estratégia de Testes Estrita (Test-Driven Development)
- [x] **Framework:** N/A (Scripts)
- [x] **Métricas:** O número de tarefas no dashboard `/tasks/INDEX.md` aumentará de ~60 para ~150.

## 5. Instruções de Execução (Step-by-Step)
> **⚠️ REGRAS DO QUE NÃO FAZER:**
> - Não crie arquivos manualmente! Chame `node tools/scripts/generate-task.mjs` para cada task listada.

1. Abra e leia atentamente o `docs/backlog-modulos.md`.
2. Para cada subitem da seção "Tarefas P0 e P1" e "Spikes de Arquitetura", extraia o ID (ex: `T-DS-01`), o Título (ex: `importar pacote de tokens`), a Complexidade (estime 1 a 5 baseado no DoD), e o Agente Alvo (ex: `devops_agent`, `frontend_agent`, etc).
3. Execute no terminal:
   `node tools/scripts/generate-task.mjs <ID> "<Título>" <Complexidade> <TargetAgent>`
   Ex: `node tools/scripts/generate-task.mjs T-DS-01 "Importar pacote de tokens" 3 devops_agent`
4. Após criar as ~96 tarefas P0/P1 e os Spikes, verifique se `/tasks/INDEX.md` foi atualizado.
