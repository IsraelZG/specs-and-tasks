# Triagem — rfc-013 (ERP Operacional e CRM)

**Achados:** 12 · **Vereditos:** INCORPORAR 3 · JA-COBERTO 3 · UI->INVENTARIO 6 · REJEITAR 0 · REVISAR-HUMANO 0

> **REVISAR-HUMANO:** nenhum.

> **Nota de auditoria (L-04, médio):** o review §2 levanta "CRM e LGPD" como se fosse uma lacuna de protocolo. **Não é.** O mecanismo (crypto-shredding via rotação de épocas, `ASSET:CONSENT`, retenção legal prevalecente, anonimização da linhagem) já é **canônico** em [[03-legal-and-compliance-framework]] §2–3 (caderno-1-vision) — ver linha `013-04` (JA-COBERTO). A lacuna **real** é que o CRM da rfc-013 (contatos/leads como `PROFILE`, A.5) **não costura** seus dados pessoais a essas primitivas (consentimento/expurgo/retenção). Isso é INCORPORAR como **uma frase de costura** apontando o dado pessoal do CRM para `ASSET:CONSENT` + expurgo por rotação de época — não invenção de mecânica nova — ver linha `013-05`.
