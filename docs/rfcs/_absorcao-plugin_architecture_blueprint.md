## Notas de execução para o incorporador

- **AB-03, AB-04, AB-05, AB-08** (as 4 subtasks `haiku` de archetype) devem, juntas, formar uma nova seção consistente em `docs/caderno-3-sdk/12-plugins-e-computacao.md` (sugestão: `§8 — Archetypes de Referência` ou subseções dentro de `§3`). Recomenda-se que o incorporador trate essas 4 como uma única unidade de edição sequencial no mesmo arquivo para evitar conflitos de numeração de seção, mesmo despachadas como subtasks separadas.
- Nenhuma das subtasks `haiku` introduz categoria de capacidade nova (`compute`/`connector`/`infra`/`ui` já são exaustivas — §3 do caderno). Todas mapeiam para categorias existentes.
- **AB-06 (descartada)** não deve ser passada ao incorporador em hipótese alguma — inclui-la aqui apenas para rastreabilidade do manifesto.
- Após resolução das 4 subtasks `revisar-humano` (AB-01, AB-02, AB-07, AB-09) e da subtask de escopo (AB-10), reavaliar se `docs/rfcs/_status.md` deve registrar esta RFC como absorvida — só apagar `rfc- plugin_architecture_blueprint - draft.md` quando todas as subtasks (exceto a descartada) estiverem `[x]`.
